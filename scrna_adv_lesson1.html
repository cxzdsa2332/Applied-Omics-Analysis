39 专题概述

第6章的标准流程默认输入来自单一样本。然而真实研究几乎总是多样本的：多个供体、多个批次、多种处理条件。一旦合并多样本数据，技术批次效应便与生物学信号混杂——直接做 PCA 与聚类，往往得到”按样本分开”而非”按细胞类型分开”的结果。本专题首先讲解跨样本整合（Integration）如何在保留真实生物学差异的前提下消除批次效应，比较 Seurat 锚点、Harmony 与 scVI 三类方法；随后转向多样本研究的两个核心统计问题：差异丰度（Differential Abundance，哪些细胞类型的比例变了）与差异表达（Differential Expression，某细胞类型内部哪些基因变了）。贯穿全篇的一条统计原则是：以”细胞”为重复单位会严重低估方差、放大假阳性，正确的重复单位是”生物学样本”。

学习目标：理解批次效应与生物学信号的混杂机制；掌握 Harmony 与 Seurat RPCA 整合的实操及质量评估方法；理解拟bulk（pseudobulk）差异表达为何优于逐细胞检验；能运行 muscat 拟bulk流程与 miloR 邻域差异丰度检验，并正确解读结果。

40 跨样本整合与批次校正

40.1 批次效应的本质

批次效应（Batch Effect）指由非生物学因素引入的系统性技术变异，其来源包括供体个体差异、样本采集与处理时间、文库制备批次、测序批次以及 10x 化学版本差异。在 scRNA-seq 中，批次效应的典型表现是：同一种细胞类型（如 CD14+ 单核细胞）来自不同样本时，在 PCA/UMAP 空间中分散为按样本聚集的多个团块，而非汇聚为一个统一的细胞类型簇。

批次效应的危险在于它与生物学信号混杂（confounded）。如果实验设计中每种处理条件只用一个批次处理，则批次与条件完全混淆，任何观测到的差异都无法归因于生物学还是技术。因此整合分析的前提是合理的实验设计——理想情况下每个批次应包含各种条件的样本（平衡设计）。

整合不能挽救糟糕的实验设计： 若所有对照样本在第一天测序、所有处理样本在第二天测序，则”条件”与”测序日”完全混淆。此时任何整合算法在消除批次效应的同时，都会一并抹去真实的生物学差异，或将技术差异误判为生物学差异。整合算法只能校正与生物学分组正交的批次变异，无法分离完全混杂的两个因素。

40.2 整合方法谱系

跨样本整合方法可大致分为三代，复杂度与计算代价递增：

  • 锚点法（Anchor-based，Seurat CCA/RPCA）：通过典型相关分析（CCA）或反向 PCA（RPCA）在样本对之间寻找”互最近邻”（Mutual Nearest Neighbors, MNN）作为锚点，再据此计算校正向量将一个数据集对齐到另一个。RPCA 比 CCA 更快、更保守（不易过度整合），是大数据集的推荐选择。
  • Harmony：在 PCA 嵌入空间中迭代执行”软聚类 + 批次特异性线性校正”两步，直至批次在每个聚类内均匀混合（Korsunsky et al., 2019）。Harmony 不修改表达矩阵，只输出校正后的降维坐标，计算极快，适合数十万细胞规模的数据。
  • scVI（single-cell Variational Inference）：基于变分自编码器的深度生成模型，以负二项分布建模原始计数，将批次作为协变量编码进神经网络，输出批次校正后的隐变量空间与归一化表达（Lopez et al., 2018）。scVI 对复杂非线性批次效应的建模能力最强，但需要 GPU 与 Python 环境。

40.3 Harmony 整合实操

下面使用 IFN-β 刺激的 PBMC 数据集（ifnb，Kang et al., 2018）演示整合。该数据集包含未处理（CTRL）和 IFN-β 刺激（STIM）两种条件，两者间存在强烈的干扰素响应转录变化——若不整合，细胞会首先按条件分开，掩盖细胞类型结构。

运行环境说明： 本节代码依赖 R 包 Seurat（v5）、SeuratData、harmony。ifnb 数据集约 150 MB，首次需 SeuratData::InstallData("ifnb") 下载。完整流程在普通笔记本上约需数分钟。

library(Seurat)
library(SeuratData)
library(harmony)
library(patchwork)

# Load the two-condition PBMC dataset (CTRL vs IFN-beta STIM)
ifnb <- LoadData("ifnb")
ifnb <- UpdateSeuratObject(ifnb)

# Standard preprocessing on the merged object (no correction yet)
ifnb <- NormalizeData(ifnb)
ifnb <- FindVariableFeatures(ifnb, nfeatures = 2000)
ifnb <- ScaleData(ifnb)
ifnb <- RunPCA(ifnb, npcs = 30)

# Uncorrected UMAP: cells separate by condition (stim), not by cell type
ifnb <- RunUMAP(ifnb, dims = 1:30, reduction.name = "umap.uncorr")
#| label: fig-integration-before-after
#| fig-cap: "Harmony 整合前后的 UMAP 对比（ifnb 数据集，按处理条件着色）。左图为未校正的 PCA-UMAP，CTRL 与 STIM 细胞按条件完全分离，相同细胞类型被批次效应拆分；右图为 Harmony 校正后的嵌入，两种条件的同类细胞充分混合，UMAP 结构反映细胞类型而非处理批次。"
library(Seurat)
library(harmony)

# Run Harmony on the PCA embedding, treating 'stim' (condition/batch) as the
# variable to integrate over. Harmony returns a corrected low-dimensional space.
ifnb <- RunHarmony(ifnb, group.by.vars = "stim", dims.use = 1:30)
ifnb <- RunUMAP(ifnb, dims = 1:30, reduction = "harmony",
                reduction.name = "umap.harmony")

p_before <- DimPlot(ifnb, reduction = "umap.uncorr", group.by = "stim") +
    ggplot2::ggtitle("Before integration (PCA)")
p_after  <- DimPlot(ifnb, reduction = "umap.harmony", group.by = "stim") +
    ggplot2::ggtitle("After integration (Harmony)")
p_before + p_after

Seurat v5 也提供统一的 IntegrateLayers() 接口，可在不离开 Seurat 生态的情况下切换不同整合算法（包括 Harmony 与 RPCA）：

# Seurat v5 unified integration interface (alternative to RunHarmony)
# Split the RNA assay into condition-specific layers, then integrate.
ifnb[["RNA"]] <- split(ifnb[["RNA"]], f = ifnb$stim)
ifnb <- NormalizeData(ifnb)
ifnb <- FindVariableFeatures(ifnb)
ifnb <- ScaleData(ifnb)
ifnb <- RunPCA(ifnb, npcs = 30)

# method = RPCAIntegration is faster and more conservative than CCAIntegration
ifnb <- IntegrateLayers(
    object         = ifnb,
    method         = RPCAIntegration,
    orig.reduction = "pca",
    new.reduction  = "integrated.rpca"
)
ifnb[["RNA"]] <- JoinLayers(ifnb[["RNA"]])   # re-merge layers for downstream DE

40.4 整合质量评估

整合的目标是在”混合批次”与”保留生物学结构”之间取得平衡。两类指标分别量化这两个目标：

  • 批次混合度：iLISI（integration LISI）或 kBET 衡量每个细胞的局部邻域中是否包含多个批次。混合越充分，分数越高，说明批次效应去除越彻底。
  • 生物学保真度：cLISI（cell-type LISI）衡量局部邻域是否仍以单一细胞类型为主。理想整合应使 cLISI 接近 1（邻域内细胞类型纯净）。

二者构成此消彼长的权衡：过度整合（over-correction）会人为合并不同的细胞类型，表现为 iLISI 升高但 cLISI 恶化。

判断是否过度整合的实操方法： 整合后检查已知标志基因的表达是否仍在预期的细胞类型中特异分布。例如，若整合后 CD3D（T 细胞）与 LYZ（单核细胞）的表达边界变得模糊，或两个本应分开的细胞类型簇被强行合并，则提示过度整合。对生物学差异较大的样本，优先选用更保守的 RPCA 而非 CCA。

41 多样本差异表达：拟bulk方法

41.1 伪重复问题

获得整合后的细胞图谱并完成注释后，一个常见目标是检验某细胞类型在两种条件间的差异表达基因。直觉上似乎可以直接用 FindMarkers() 的 Wilcoxon 检验，把”细胞”当作样本逐个比较。这是单细胞差异分析中最常见的统计错误。

问题在于伪重复（pseudo-replication）：同一供体的数千个细胞并非相互独立的观测，它们共享该供体的遗传背景、处理批次与技术状态。把细胞当作独立重复，等于把样本量从”几个供体”虚增到”几千个细胞”，导致方差被严重低估、P 值极度偏小。多项基准研究表明，逐细胞检验会产生大量假阳性，其”显著基因”很多只反映个别供体的特异性，而非可重复的条件效应（Squair et al., 2021）。

黄金法则： 多样本差异表达的统计重复单位是生物学样本（供体），而不是细胞。任何把成千上万个细胞当作独立重复的检验，都会因伪重复而产生不可重复的假阳性。正确的做法是先将细胞聚合为拟bulk，再以样本为单位做检验。

41.2 拟bulk聚合的逻辑

拟bulk（Pseudobulk）方法的思想是：对每个样本、每种细胞类型，将其所有细胞的原始计数逐基因相加，得到一个”伪批量”表达谱。这样，一个含 8 供体 × 2 条件的实验，对某细胞类型就得到 16 个拟bulk样本——恰好回到了经典 bulk RNA-seq 的统计框架，可直接套用 DESeq2/edgeR 的负二项模型（参见第4章）。

聚合（求和而非求平均）保留了计数性质，使 DESeq2 的离散度估计与方差稳定化变换仍然适用。聚合后样本间的变异自然反映了真实的生物学重复变异，从而给出校准良好的 P 值。

41.3 muscat 拟bulk差异分析实操

下面使用 Kang 数据集的 16 样本版本（muscData::Kang18_8vs8，含 sample_id、group_id、cluster_id 三个关键元数据列）演示 muscat 拟bulk流程（Crowell et al., 2020）。

运行环境说明： 本节依赖 Bioconductor 包 muscat、muscData、SingleCellExperiment、edgeR。Kang18_8vs8 是一个已注释的 SingleCellExperiment 对象，可直接用于拟bulk聚合，无需重新整合与注释。

library(muscat)
library(muscData)
library(SingleCellExperiment)

# Load the 16-sample (8 donors x CTRL/STIM) annotated SCE
sce <- Kang18_8vs8()

# muscat requires three metadata columns: sample / group / cluster (cell type)
sce <- prepSCE(
    sce,
    kid = "cell",        # cluster_id  -> cell type label
    gid = "stim",        # group_id    -> condition (ctrl vs stim)
    sid = "ind",         # sample_id   -> donor identity (the replicate unit)
    drop = TRUE
)

# Step 1: aggregate counts to pseudobulk (sum over cells per sample x cell type)
pb <- aggregateData(sce, assay = "counts",
                    fun = "sum", by = c("cluster_id", "sample_id"))

# Step 2: pseudobulk DE per cell type, using edgeR's negative-binomial model
res <- pbDS(pb, method = "edgeR", verbose = FALSE)

# Inspect top condition-responsive genes in CD14+ Monocytes
tbl_mono <- res$table[[1]][["CD14+ Monocytes"]]
tbl_mono <- tbl_mono[order(tbl_mono$p_adj.loc), ]
head(tbl_mono[, c("gene", "logFC", "p_val", "p_adj.loc")], 10)

聚合矩阵也可手动导出后交由 DESeq2 处理，二者等价；Seurat 用户则可用 AggregateExpression() 得到拟bulk矩阵再构建 DESeqDataSet：

# Seurat-native pseudobulk: sum counts per donor x condition x cell type
# (assumes the integrated object carries 'donor', 'stim' and 'celltype' metadata)
pb_seurat <- AggregateExpression(
    ifnb_annotated,
    assays    = "RNA",
    group.by  = c("celltype", "donor", "stim"),
    return.seurat = FALSE
)$RNA
# pb_seurat is now a genes x (celltype_donor_stim) count matrix -> feed to DESeq2

细胞数过少的细胞类型不可靠： 当某细胞类型在某些样本中细胞数很少（如 < 10 个）时，其拟bulk计数受少数细胞主导，离散度估计不稳定。实践中应过滤掉在多数样本中细胞数过低的细胞类型，或在结果解读时标注这些类型的置信度较低。muscat 的 pbDS() 可结合 filterSCE() 预先剔除此类细胞类型。

42 差异丰度分析

42.1 组成数据的挑战

差异表达回答”基因变了多少”，差异丰度（Differential Abundance, DA）则回答”细胞类型的比例变了多少”——例如 IFN-β 刺激后，某个激活态细胞群是否扩张。最朴素的做法是统计每个样本中各细胞类型的比例，再做组间 t 检验。但这种做法有两个缺陷：

第一，依赖离散聚类。预先定义的细胞类型聚类可能掩盖聚类内部的连续状态变化——真正发生扩张的或许只是某聚类中的一个亚状态，而非整个聚类。

第二，组成数据的约束。各细胞类型比例之和恒为 1，因此一个类型的真实扩张会机械地压低其他所有类型的比例，产生虚假的”减少”信号。这种”此消彼长”是组成数据（compositional data）分析的核心难题。

42.2 miloR 邻域差异丰度检验

miloR 通过两项设计绕开了上述问题（Dann et al., 2022）。其一，它不依赖离散聚类，而是在 KNN 图上定义大量重叠的邻域（neighborhood，每个邻域是一个代表细胞及其近邻），在邻域这一精细尺度上检验丰度变化，从而捕捉连续状态的扩张/收缩。其二，它用负二项广义线性模型（NB-GLM）对每个邻域的细胞计数建模，以样本为重复单位，并通过加权 SpatialFDR 校正邻域间的重叠与多重检验。

{{Wiki:Compositional_data}}

library(miloR)
library(SingleCellExperiment)
library(scater)

# Build a Milo object from an integrated, dimensionally-reduced SCE
# (reducedDim 'PCA' must already exist on the object)
milo <- Milo(sce)
milo <- buildGraph(milo, k = 30, d = 30, reduced.dim = "PCA")

# Define overlapping neighborhoods by sampling representative cells
milo <- makeNhoods(milo, prop = 0.1, k = 30, d = 30, refined = TRUE)

# Count cells from each sample within each neighborhood
milo <- countCells(milo, meta.data = as.data.frame(colData(milo)),
                   sample = "sample_id")

# Design matrix: one row per sample, condition as the tested covariate
design_df <- data.frame(colData(milo))[, c("sample_id", "group_id")]
design_df <- unique(design_df)
rownames(design_df) <- design_df$sample_id

# NB-GLM test for differential abundance across condition, per neighborhood
milo <- calcNhoodDistance(milo, d = 30, reduced.dim = "PCA")
da_results <- testNhoods(milo, design = ~ group_id, design.df = design_df)

# Neighborhoods with SpatialFDR < 0.1 are significantly DA;
# logFC sign indicates expansion (+) or depletion (-) in the test condition
table(da_results$SpatialFDR < 0.1)
#| label: fig-milo-da

除 miloR 外，scCODA 提供了另一种思路：它用贝叶斯狄利克雷-多项式模型直接建模细胞类型的组成结构，并通过指定一个”参考细胞类型”来锚定组成约束，从而给出统计严谨的组成差异推断。miloR 适合在连续状态流形上做精细的 DA 检验，scCODA 则适合在已定义的离散细胞类型层面做组成检验，二者互补。

差异表达与差异丰度的互补解读： 一个条件效应可能表现为差异丰度（某细胞类型扩张）、差异表达（某细胞类型内部基因上调），或两者兼有。例如 IFN-β 刺激既使干扰素响应基因（如 ISG15、IFI6）在各细胞类型中普遍上调（DE），也可能改变某些激活态细胞群的比例（DA）。完整的多样本分析应同时报告这两个层面，才能准确刻画扰动的生物学后果。

43 方法总结与衔接说明

本专题确立了多样本单细胞分析的统计基石：先用 Harmony/RPCA/scVI 整合消除批次效应得到统一的细胞状态空间，再以生物学样本为重复单位，通过拟bulk方法做差异表达、通过 miloR/scCODA 做差异丰度检验。整合后的统一图谱是后续所有高级分析的共同输入——专题 7.2 将在此基础上重建细胞分化的拟时序轨迹，专题 7.3 推断细胞通讯与调控网络，模块 C 则以注释后的图谱作为空间解卷积的参考。掌握”细胞不是重复、样本才是重复”这一原则，是避免单细胞研究中不可重复结论的关键。

  2. Seurat CCA、Seurat RPCA、Harmony 与 scVI 四种整合方法在核心思想与计算代价上有何区别？在数十万细胞规模、批次较多的数据上，应优先考虑哪种？

  3. 整合质量评估中的 iLISI（或 kBET）与 cLISI 分别衡量什么？为什么二者构成此消彼长的权衡？什么是过度整合（over-correction）？

  4. 为什么在多样本差异表达分析中，直接用 FindMarkers() 的逐细胞 Wilcoxon 检验是错误的？请用”伪重复”（pseudo-replication）这一概念解释。

  5. 拟bulk（pseudobulk）聚合时为什么要对计数求和而非求平均？聚合后为何可以套用 DESeq2/edgeR 的负二项模型？

  6. 一个实验所有对照样本在第一天测序、所有处理样本在第二天测序。整合算法能否挽救这一设计？为什么？

  7. 差异丰度分析中，“组成数据约束”（细胞类型比例之和为 1）会带来什么问题？miloR 通过哪两项设计绕开了这一问题以及”依赖离散聚类”的局限？

  8. 某研究者报告 IFN-β 刺激后 CD14+ 单核细胞中有 3000 个差异表达基因（逐细胞 Wilcoxon，P < 0.05），而用拟bulk + DESeq2 仅得到 200 个（FDR < 0.05）。请解释这一巨大差异的统计学原因，以及应以哪个结果为准。

参考答案 (Reference Answers)

  1. 批次效应是由非生物学因素（供体差异、处理时间、文库制备批次、测序批次、化学版本等）引入的系统性技术变异。当不同样本的同类细胞被这些技术因素系统性地偏移时，技术变异的幅度可能超过细胞类型间的生物学差异，于是 PCA 的主要变异方向（前几个主成分）捕捉的是”样本/批次”而非”细胞类型”，导致聚类首先按样本分开。

  2. Seurat CCA 用典型相关分析寻找 MNN 锚点，最慢、最易过度整合；RPCA 用反向 PCA，更快更保守；Harmony 在 PCA 空间迭代执行软聚类与批次校正，只输出校正后的降维坐标，计算极快；scVI 用变分自编码器以负二项分布建模计数、将批次作为协变量，建模能力最强但需 GPU。数十万细胞、批次较多时应优先 Harmony（速度）或 RPCA（保守、稳健），scVI 适合更复杂的批次结构但成本更高。

  3. iLISI/kBET 衡量批次混合度（局部邻域中是否包含多个批次，越高表示批次去除越彻底）；cLISI 衡量生物学保真度（局部邻域是否仍以单一细胞类型为主，理想接近 1）。二者权衡：强行提高批次混合度可能把不同细胞类型也混到一起，使 cLISI 恶化。过度整合即批次混合度升高但生物学结构被破坏（不同细胞类型被人为合并）的现象。

  4. 逐细胞 Wilcoxon 检验把同一供体的成千上万个细胞当作独立重复，但这些细胞共享供体的遗传背景、处理批次与技术状态，并非独立观测，这就是伪重复。其后果是样本量被虚增、方差被严重低估、P 值极度偏小，产生大量不可重复的假阳性——许多”显著基因”只反映个别供体的特异性而非可重复的条件效应。

  5. 求和保留了原始计数的整数性质与测序深度信息，使负二项分布的均值-方差关系成立，从而可直接套用 DESeq2/edgeR 的离散度估计与方差稳定化变换；求平均会破坏计数性质并丢失深度信息。聚合后每个样本×细胞类型成为一个拟bulk样本，样本间变异反映真实生物学重复变异，统计框架等价于经典 bulk RNA-seq。

  6. 不能。此时”处理条件”与”测序日”完全混淆（confounded），二者正交分量为零。整合算法只能校正与生物学分组正交的批次变异；当批次与条件完全混杂时，算法在去除批次效应的同时会一并抹去真实生物学差异，或把技术差异误判为生物学差异。补救只能靠重新设计实验（每批次包含各条件的平衡设计）。

  7. 组成数据约束指各细胞类型比例之和恒为 1，因此一个类型的真实扩张会机械地压低其他类型的比例，制造虚假的”减少”信号。miloR 的两项设计：（1）不依赖离散聚类，而在 KNN 图上定义大量重叠的邻域，在精细尺度检验丰度变化，捕捉连续状态的扩张/收缩；（2）用负二项 GLM 以样本为重复单位对邻域计数建模，并用加权 SpatialFDR 校正邻域重叠与多重检验，从而给出统计严谨的 DA 推断。

  8. 逐细胞 Wilcoxon 因伪重复严重低估方差，把几千个非独立细胞当作独立重复，导致 P 值系统性偏小、假阳性大量涌现，故得到虚高的 3000 个”显著”基因；拟bulk + DESeq2 以供体为重复单位，方差估计反映真实生物学重复变异，给出校准良好的结果。应以拟bulk + DESeq2 的 200 个基因为准——它们才是在供体层面可重复的条件效应。
