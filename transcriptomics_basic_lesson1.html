21  测序原理、文库特性与原始数据质量控制

Published

May 31, 2026

21.1 学习目标

完成本专题后，学生应能够：

  1. 区分 poly-A 富集与 rRNA 去除两种 RNA-seq 文库策略的适用场景与覆盖度差异；
  2. 解释链特异性文库的 dUTP 构建原理，并使用 infer_experiment.py 确认链方向；
  3. 解读 FastQC 报告中的六个核心质控模块，识别 RNA-seq 特有的异常模式；
  4. 编写 fastp 修剪命令并正确设置 Adapter 和质量过滤参数；
  5. 使用 MultiQC 生成多样本质控汇总报告并识别批次异常样本。

21.2 从 Microarray 到 RNA-seq：范式转变

在 RNA-seq 出现之前，转录组研究的主流工具是微阵列（Microarray）。微阵列依赖于已知序列的探针杂交，只能检测预先设计好的转录本，动态范围有限（约 2 个数量级），且在低表达基因和高同源性基因上信噪比差。2008 年，多个研究组几乎同时发表了使用高通量测序直接测量 RNA 分子的方法，即 RNA-seq。这一范式转变带来了质的飞跃：RNA-seq 不依赖先验序列知识，能够发现新的转录本、检测可变剪接事件，动态范围扩展至 4–5 个数量级，并且理论上可以达到单碱基分辨率。

历史注记： RNA-seq 的早期代表性论文包括 Mortazavi et al.（2008，Nature Methods）对小鼠肌肉组织的 mRNA 测序，以及 Wang et al.（2008，Nature）对人类转录组可变剪接的系统分析。这两篇论文奠定了 RNA-seq 分析的基本框架，至今仍是该领域引用最高的文献之一。

尽管技术进步迅速，RNA-seq 的核心分析流程在过去十余年间已趋于成熟和标准化。理解这一流程的每个步骤背后的原理，不仅有助于正确执行分析，更重要的是能在遇到异常数据时准确定位问题来源。

21.3 RNA-seq 文库构建：两条技术路线

21.3.1 poly-A 富集（mRNA-seq）

poly-A 富集文库利用 oligo-dT 磁珠捕获含有 poly-A 尾的成熟 mRNA。由于真核细胞总 RNA 中约 80% 是核糖体 RNA（rRNA），直接建库会导致测序资源被 rRNA 大量占用。poly-A 富集有效去除 rRNA，使 mRNA 分子富集至总 reads 的 90% 以上。

优势：操作简便，成本低，信噪比高。

局限性：

  • 只能捕获含 poly-A 尾的 RNA，遗漏组蛋白 mRNA（无 poly-A 尾）和大多数非编码 RNA；
  • oligo-dT 对 mRNA 3’ 端的结合亲和力高于 5’ 端，导致转录本的 3’ 端覆盖度偏高，称为3’ 端偏差（3’ bias）；
  • RNA 质量要求高：降解的 RNA（RIN < 7）会加剧 3’ 端偏差，导致 5’ 端几乎无覆盖。

21.3.2 rRNA 去除（Total RNA-seq / Ribo-Zero）

rRNA 去除文库通过与 rRNA 互补的探针捕获并去除 rRNA，保留其余全部 RNA 分子，包括 pre-mRNA、lncRNA、circRNA、组蛋白 mRNA 等。

优势：覆盖全转录组，适合研究非编码 RNA；对 RNA 质量要求相对宽松；覆盖度更均匀。

局限性：成本较高；低表达 mRNA 的检出灵敏度略低于 poly-A 富集方法（因为非 mRNA 分子占用了部分测序资源）。

选择建议： 若研究目标明确为蛋白质编码基因的差异表达，且 RNA 质量良好（RIN ≥ 8），优先选择 poly-A 富集；若需要研究非编码 RNA、RNA 降解样本，或希望覆盖更完整的转录组，选择 rRNA 去除方案。本章使用的 pasilla 数据集（SRR1039508–SRR1039517）为 poly-A 富集文库。

21.3.3 链特异性文库

常规 RNA-seq 文库在逆转录后会丢失链方向信息——测序 read 无法判断来自正义链（sense strand）还是反义链（antisense strand）。对于有大量正反链重叠基因或需要区分正反义转录的研究，链方向信息至关重要。

目前最主流的链特异性文库构建方法是 dUTP 法（TruSeq Stranded 采用此方案），流程如下：

  1. mRNA 随机打断；
  2. 逆转录合成第一链 cDNA（方向为 3’→5’，与 mRNA 反向互补）；
  3. 以第一链为模板，合成含 dUTP 而非 dTTP 的第二链 cDNA；
  4. 用尿嘧啶-DNA 糖基化酶（UDG）降解含 dUTP 的第二链；
  5. 剩余第一链加接头，测序。

由于测序的 read 来自第一链（反义链），read 2（R2）与原始 mRNA 方向相同，read 1（R1）与 mRNA 反向互补，即所谓的 RF 链方向（Illumina TruSeq Stranded 标准）。

21.4 链方向的生物信息学确认

在开始比对前，必须确认文库的链特异性方向，因为后续定量步骤（featureCounts 的 -s 参数、Salmon 的 --libType 参数）需要明确指定。错误的链方向设置会导致约 50% 的 reads 被丢弃或错误计数。

推荐工具：RSeQC 的 infer_experiment.py

该工具从比对的 BAM 文件中随机抽取读段，与参考注释中基因的转录方向进行对比，统计 read 方向与基因注释一致和不一致的比例：

# Install RSeQC
pip install RSeQC

# Download reference BED file (gene annotation in BED12 format)
# Convert GTF to BED12 using gtfToGenePred + genePredToBed
gtfToGenePred Homo_sapiens.GRCh38.109.gtf stdout | \
  genePredToBed stdin annotation.bed

# Infer experiment strand
infer_experiment.py -r annotation.bed -i sample_sorted.bam -s 200000

典型输出解读：

This is PairEnd Data
Fraction of reads failed to determine: 0.0250
Fraction of reads explained by "1++,1--,2+-,2-+": 0.0368  # RF: antisense
Fraction of reads explained by "1+-,1-+,2++,2--": 0.9382  # FR: sense

上述结果表明该样本为 FR-stranded（约 94% reads 与 mRNA 同向），即 read 1 与 mRNA 同向，对应 featureCounts 的 -s 1 参数。

常见错误： 不同测序平台和文库制备试剂盒对应不同的链方向编码。Illumina TruSeq Stranded mRNA 通常为 RF（-s 2），而某些文库为 FR（-s 1）。务必对每批数据独立确认，不要依赖试剂盒说明书假设。

21.5 原始数据质量控制

21.5.1 FastQC：单样本质量评估

FastQC 是 RNA-seq 质控的标准起点，对 FASTQ 文件执行 10 余项统计检测。以下是 RNA-seq 数据最关键的六个模块：

1. Per Base Sequence Quality

展示每个循环（cycle）位置上碱基质量分数的箱形图分布。RNA-seq 与 DNA-seq 的正常模式相同：read 末端（3’ 端）质量通常低于 5’ 端，这是信号强度随测序循环递减的结果。

  • 绿色区域（Q ≥ 28）：合格；
  • 黄色区域（20 ≤ Q < 28）：边界；
  • 红色区域（Q < 20）：不合格，提示需要截短。

2. Per Tile Sequence Quality（仅 Illumina）

显示流动槽（Flow Cell）上各 Tile 位置的质量。异常的 Tile 图案（蓝色代表高质量，橙/红代表低质量）提示测序过程中的物理问题（气泡、污染、激光问题），通常无法通过过滤解决。

3. Per Sequence GC Content

正常 RNA-seq 数据应呈现与物种基因组 GC 含量一致的单峰正态分布。RNA-seq 特有的异常模式：

  • 双峰或宽分布：提示 rRNA 污染（rRNA 的 GC 含量与 mRNA 显著不同）或样本混合；
  • 窄高峰偏移：提示 Adapter 二聚体（Adapter dimer）污染，通常位于 GC 约 50% 的位置。

4. Sequence Duplication Levels

显示各重复度级别的 reads 比例。RNA-seq 数据天然存在较高的重复率，因为高表达基因会产生大量相同序列的 cDNA，这与 DNA 测序中重复暗示 PCR 扩增偏差的情形不同。因此，RNA-seq 不建议对重复 reads 进行常规去除（除非有 UMI 标记）。FastQC 的 Duplicate Levels 模块对 RNA-seq 通常显示 WARN 或 FAIL，可忽略。

5. Overrepresented Sequences

列出出现频率超过 0.1% 的序列及其可能来源。需要关注的异常：

  • 未被去除的 Illumina Universal Adapter 序列（AGATCGGAAGAGC...）；
  • poly-A 或 poly-T 序列（提示 Read-through 发生）；
  • 特定基因的 3’ 端序列（严重 3’ 偏差时高表达基因的 3’ 末端大量重复）。

6. Adapter Content

直接检测各循环位置上 Adapter 序列的比例。若在 read 末端 20–30 bp 处出现 Adapter 信号，说明 Insert Size 小于 read 长度，发生了 Read-through，需要进行 Adapter 修剪。

21.5.2 fastp：Adapter 修剪与质量过滤

fastp 是目前速度最快、功能最全面的 FASTQ 预处理工具，集 Adapter 检测、质量过滤、低复杂度过滤于一体，并输出内置 HTML 质控报告。

# fastp for paired-end RNA-seq (TruSeq Stranded mRNA)
fastp \
  --in1 SRR1039508_1.fastq.gz \
  --in2 SRR1039508_2.fastq.gz \
  --out1 SRR1039508_1_trimmed.fastq.gz \
  --out2 SRR1039508_2_trimmed.fastq.gz \
  --json SRR1039508_fastp.json \
  --html SRR1039508_fastp.html \
  --detect_adapter_for_pe \
  --trim_poly_x \
  --qualified_quality_phred 20 \
  --length_required 36 \
  --thread 8

# Key parameters explained:
# --detect_adapter_for_pe  : auto-detect adapter sequences for PE data
# --trim_poly_x            : trim poly-A/T/G/C tails (RNA-seq specific)
# --qualified_quality_phred: minimum base quality threshold (Q20)
# --length_required        : discard reads shorter than 36 bp after trimming

参数选择逻辑：

  • --detect_adapter_for_pe：自动检测时 fastp 会从数据中推断接头序列，适合不确定接头类型的情况；若已知接头（如 Illumina TruSeq：AGATCGGAAGAGCACACGTCTGAAC），可用 --adapter_sequence 明确指定，速度更快且更可靠；
  • --trim_poly_x：去除 read 末端的 poly-A/T 尾，这在 RNA-seq 中尤为重要；
  • --qualified_quality_phred 20：去除质量低于 Q20 的碱基，对应错误率 1%；
  • --length_required 36：修剪后过短的 reads 对比对没有贡献，反而增加多比对（multimapping）风险。

黄金法则：先质控，后比对。 始终在 fastp 修剪后再次运行 FastQC，确认质控效果。FastQC-fastp-FastQC 的三步流程是 RNA-seq 质控的标准操作规程（SOP）。

21.6 转录组层面质控：超越碱基质量

FastQC 只反映读段本身的质量，而转录组层面的质控需要将 reads 与参考基因组比对后才能评估。以下指标在 RNA-seq 中尤为重要：

21.6.1 RSeQC：转录本覆盖均匀性

# Read distribution across genomic features
read_distribution.py -r annotation.bed -i sample_sorted.bam

# Expected output (poly-A enriched library):
# Total Reads                   20,547,893
# Total Tags                    21,034,510
# CDS_Exons                    17,234,102  (81.9%)  <-- should be majority
# 5'UTR_Exons                   1,024,391   (4.9%)
# 3'UTR_Exons                   1,887,234   (9.0%)
# Introns                         789,342   (3.8%)
# Intergenic                      99,441    (0.5%)

# 3' bias assessment
geneBody_coverage.py -r annotation.bed \
  -i sample1.bam,sample2.bam,sample3.bam \
  -o coverage_output

geneBody_coverage.py 将转录本从 5’ 端（0%）到 3’ 端（100%）归一化，绘制覆盖度曲线。正常 poly-A 富集样本呈现轻微的 3’ 端升高；若 5’ 端覆盖度极低（RNA 降解），说明样本质量差，下游分析需谨慎。

21.6.2 MultiQC：多样本汇总

MultiQC 自动扫描当前目录及子目录中的 FastQC、STAR、featureCounts、Salmon 等工具的输出文件，生成多样本汇总报告：

# Run MultiQC on all output files in current directory
multiqc . --outdir multiqc_report --filename multiqc_report

# Expected output: multiqc_report.html

MultiQC 使用建议： 在比对和定量完成后，再次在包含所有输出目录的父目录中运行 MultiQC，生成从质控到定量全流程的综合报告，便于发现样本间的系统性差异（如某几个样本比对率显著偏低、特定样本的 GC 分布偏移等）。

21.7 实践：pasilla 数据集质控流程

以下代码演示对 pasilla 数据集中一个样本（SRR1039508）的完整质控流程。实际分析中应对所有 10 个样本循环执行。

# === Environment setup ===
# Tools required: sra-tools, fastqc, fastp, multiqc, rseqc
# Activate conda environment with all tools installed
conda activate rnaseq-env

# === Step 1: Download pasilla dataset ===
# Use ENA FTP for faster download (preferred over NCBI SRA)
# pasilla dataset: 8 samples (4 treated + 4 untreated)
# SRR1039508 = untreated sample 1 (GSM1275862)

mkdir -p data/raw data/trimmed data/qc/fastqc data/qc/fastp

# Download via ENA FTP
wget -P data/raw \
  ftp://ftp.sra.ebi.ac.uk/vol1/fastq/SRR103/008/SRR1039508/SRR1039508_1.fastq.gz \
  ftp://ftp.sra.ebi.ac.uk/vol1/fastq/SRR103/008/SRR1039508/SRR1039508_2.fastq.gz

# === Step 2: Pre-trimming QC ===
fastqc \
  data/raw/SRR1039508_1.fastq.gz \
  data/raw/SRR1039508_2.fastq.gz \
  --outdir data/qc/fastqc \
  --threads 8

# === Step 3: Adapter trimming with fastp ===
fastp \
  --in1  data/raw/SRR1039508_1.fastq.gz \
  --in2  data/raw/SRR1039508_2.fastq.gz \
  --out1 data/trimmed/SRR1039508_1.fastq.gz \
  --out2 data/trimmed/SRR1039508_2.fastq.gz \
  --json data/qc/fastp/SRR1039508_fastp.json \
  --html data/qc/fastp/SRR1039508_fastp.html \
  --detect_adapter_for_pe \
  --trim_poly_x \
  --qualified_quality_phred 20 \
  --length_required 36 \
  --thread 8

# === Step 4: Post-trimming QC ===
fastqc \
  data/trimmed/SRR1039508_1.fastq.gz \
  data/trimmed/SRR1039508_2.fastq.gz \
  --outdir data/qc/fastqc/post_trim \
  --threads 8

# === Step 5: Generate MultiQC report ===
multiqc data/qc/ \
  --outdir data/qc/multiqc \
  --filename pasilla_qc_report

完成后，打开 data/qc/multiqc/pasilla_qc_report.html，重点检查以下几点：

  1. Adapter Content：修剪后所有样本的接头含量应降至接近 0%；
  2. Sequence Length Distribution：正常修剪后应在 36–150 bp 范围内分布，peak 在 150 bp；
  3. Per Sequence GC Content：10 个样本的 GC 分布应高度一致，偏离的样本提示潜在污染或批次问题。

21.8 RNA 完整性与样本质量评级

在送测之前，RNA 样本的完整性（Integrity）是决定数据质量的关键先决条件。降解的 RNA 在 poly-A 富集建库时会产生严重的 3’ 端偏差，甚至导致整个实验失败。

21.8.1 RNA 完整性评分（RIN）

RNA 完整性数字（RNA Integrity Number, RIN）是由 Agilent Bioanalyzer 或 Fragment Analyzer 根据 28S/18S rRNA 峰比值与降解特征自动计算的 1–10 评分：

RIN 值 完整性描述 适用场景
9–10 完整，无降解 所有建库方法
7–8 轻度降解 poly-A 富集可接受
5–6 中度降解 推荐 rRNA 去除；总 RNA 分析
< 5 严重降解 仅适合 3’ 端定向建库或放弃

RIN 的局限性： RIN 基于 rRNA 完整性推断 mRNA 完整性，但两者降解速率不同。植物样本（叶绿体 rRNA 比例高）和 FFPE（福尔马林固定石蜡包埋）样本的 RIN 解读需要额外谨慎。FFPE 样本通常 RIN < 3，但用 DV200（> 200 nt 片段的比例）评估更适合。

21.8.2 Phred 质量分数的数学基础

Illumina 测序仪为每个碱基分配一个 Phred 质量分数 \(Q\)，定义为：

\[Q = -10 \log_{10} P_{\text{error}}\]

其中 \(P_{\text{error}}\) 是该碱基被错误识别的概率。常用对应关系：

Q 值 错误率 碱基准确率
Q10 10% 90%
Q20 1% 99%
Q30 0.1% 99.9%
Q40 0.01% 99.99%

在 FASTQ 文件中，每个碱基的 Q 值用单个 ASCII 字符编码（Phred+33 编码）：实际字符 = chr(Q + 33)。例如，I 对应 ASCII 73，Q = 73 - 33 = 40。

21.9 批次效应的早期检测

批次效应（Batch Effect）是 RNA-seq 实验中最常见的技术干扰来源之一，指由于样本在不同时间、不同操作人员、不同试剂批次或不同测序机器上处理，引入的与生物学处理无关的系统性变异。批次效应如果未被识别和校正，会产生大量假阳性差异基因，严重时可完全掩盖真实的生物学信号。

历史案例： 2010 年 Nature Methods 上发表了一篇影响深远的文章，展示了两篇顶级期刊（Nature、Science）的 TCGA 癌症基因组数据中存在显著的批次效应，部分”发现”的分子亚型实际上是不同测序批次的人工产物。这一事件推动了批次效应校正成为 RNA-seq 分析的标准步骤。

21.9.1 批次效应的来源

  • 文库制备批次：不同日期制备的文库，试剂状态和操作细节差异；
  • 测序批次：不同 Flow Cell、不同 Lane 之间的系统误差；
  • RNA 提取批次：提取试剂盒批次、RIN 差异；
  • 实验操作人员：不同人员的操作差异；
  • 样本保存方式：新鲜组织 vs 冻存 vs FFPE。

21.9.2 PCA 检测批次效应

主成分分析（PCA）是检测批次效应最直观的方法。在 PCA 图中，用不同颜色标记处理组和批次：若 PC1/PC2 的分离方向与处理组一致，说明生物学信号主导；若分离方向与批次一致，说明批次效应超过了生物学效应。

library(DESeq2)
library(ggplot2)

# Assume dds has been created and vst applied
vsd <- vst(dds, blind = TRUE)

# Add batch information to colData
# (in real experiments, record batch during sample collection)
colData(dds)$batch <- factor(c("batch1", "batch1", "batch2",
                               "batch1", "batch2", "batch2"))

# PCA colored by condition AND batch
pca_data <- plotPCA(vsd, intgroup = c("condition", "batch"),
                    returnData = TRUE)
percent_var <- round(100 * attr(pca_data, "percentVar"))

ggplot(pca_data, aes(x = PC1, y = PC2,
                     color = condition, shape = batch)) +
  geom_point(size = 4) +
  xlab(paste0("PC1: ", percent_var[1], "% variance")) +
  ylab(paste0("PC2: ", percent_var[2], "% variance")) +
  labs(title = "PCA: check for batch effects",
       subtitle = "Shape = batch; Color = condition") +
  theme_classic()

# If batch separates samples more than condition,
# batch correction is needed before DEA (see Lesson 3)

21.9.3 RLE 图：检测系统性偏移

相对对数表达量图（Relative Log Expression, RLE）是另一种有效的批次效应检测工具。对每个样本，计算每个基因的表达量与所有样本中该基因中位数的差值（对数尺度），然后绘制箱形图。正常情况下，所有样本的箱体中位线应接近 0；若某些样本整体偏移，说明存在系统性偏差。

library(EDASeq)

# Compute RLE plot from raw counts
counts_matrix <- counts(dds)
rle_data <- log2(counts_matrix + 1) -
  rowMedians(log2(counts_matrix + 1))

rle_df <- as.data.frame(rle_data)
rle_long <- tidyr::pivot_longer(rle_df, everything(),
                                names_to = "sample",
                                values_to = "rle")

ggplot(rle_long, aes(x = sample, y = rle, fill = sample)) +
  geom_boxplot(outlier.size = 0.3) +
  geom_hline(yintercept = 0, linetype = "dashed",
             color = "firebrick") +
  labs(title = "RLE Plot: check for systematic expression shifts",
       y = "Relative Log Expression",
       x = "Sample") +
  theme_classic() +
  theme(axis.text.x = element_text(angle = 45, hjust = 1),
        legend.position = "none")

实践规范： 批次效应的检测应在 DEA 之前完成。若发现批次效应，有两种处理策略：（1）将批次作为协变量加入线性模型（推荐，保留批次内信息）；（2）使用 ComBat/limma::removeBatchEffect 从表达矩阵中直接去除批次效应（适合可视化，但不推荐直接用于 DEA 输入）。详见 Lesson 3 批次效应校正部分。

21.10 关于 pasilla 数据集

pasilla（西班牙语”葡萄干辣椒”）数据集来自 Brooks et al.（2011）对果蝇 RBP 蛋白 pasilla 基因的 RNA-seq 研究，但本教材使用的 SRR1039508–SRR1039517 是 DESeq2 Bioconductor 包教程中广泛使用的人类 A549 细胞系数据集（地塞米松 dex 处理 vs 对照），技术社区习惯沿用”pasilla”这一称呼。

SRR1039509 untreated single
SRR1039512 untreated paired
SRR1039513 untreated single

  1. poly-A 富集文库和 rRNA 去除文库在非编码 RNA 检测能力上有何本质区别？请举例说明各自适合的研究场景。

  2. 在 RNA-seq 链特异性检测中，infer_experiment.py 输出如下结果："1++,1--,2+-,2-+"：0.89，"1+-,1-+,2++,2--"：0.05。这说明该样本的链方向是什么？featureCounts 应使用 -s 参数的哪个值？

  3. 为什么 RNA-seq 数据的 FastQC Sequence Duplication Levels 通常会显示 WARN 或 FAIL，但这并不意味着数据质量有问题？

  4. fastp 的 --trim_poly_x 参数在 RNA-seq 分析中为何特别重要？如果不修剪 poly-A 尾，会对下游比对步骤产生什么影响？

  5. 在 MultiQC 报告中，你发现 10 个样本中有 2 个样本的比对率比其他样本低 15%，且 GC 分布出现双峰。可能的原因是什么？你会如何进一步排查？

参考答案 (Reference Answers)

  1. poly-A 富集文库仅捕获含 poly-A 尾的成熟 mRNA，无法检测组蛋白 mRNA、lncRNA、circRNA 等非编码 RNA，适合研究蛋白质编码基因差异表达；rRNA 去除文库保留全部非 rRNA 分子，适合研究 lncRNA、circRNA 等非编码 RNA，以及使用降解 RNA 的样本（如 FFPE 组织）。

  2. 该样本为 RF-stranded（read 1 与 mRNA 反向互补，read 2 与 mRNA 同向），即 Illumina TruSeq Stranded 标准方向。featureCounts 应使用 -s 2（RF 方向）。

  3. RNA-seq 中高表达基因会产生大量具有相同序列的 cDNA 片段，这是生物学现象而非 PCR 扩增偏差。FastQC 无法区分生物学重复与 PCR 重复，因此对 RNA-seq 数据的重复率警告通常可以忽略。只有在有 UMI 标记的文库中，才能真正区分并去除 PCR 重复。

  4. poly-A 富集步骤有时会捕获 poly-A 本身延伸到 read 中（特别是 Insert Size 较短时），这些 poly-A 序列在基因组比对时无法有效映射，会显著降低比对率并产生软截断（soft-clipping）。修剪 poly-A 尾可提高比对率 1–5%，特别是在短 Insert Size 的样本中效果更明显。

  5. 可能原因：（1）这 2 个样本存在 rRNA 或 DNA 污染（GC 双峰）；（2）文库构建失败导致 Adapter 二聚体过多；（3）物种或组织来源不同（如样本混淆）；（4）RNA 降解严重导致短片段偏多。排查步骤：检查 FastQC 的 Overrepresented Sequences 模块确认污染类型；查看 RSeQC read_distribution 结果中各基因组区域的 reads 比例；若 rRNA 比例高，考虑重新建库。
