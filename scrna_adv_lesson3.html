40  细胞通讯分析与基因调控网络推断

Published

June 22, 2026

41 专题概述

差异表达与拟时序揭示了基因表达”变了什么”，但要理解”为什么变”，需要刻画两类关系：细胞之间如何通过分子信号相互协调，以及细胞内部哪些转录因子驱动了状态转变。本专题合并讲解这两个互补的关系推断问题。细胞通讯分析（Cell-Cell Communication）从配体-受体的表达模式推断细胞类型之间的信号传递；基因调控网络推断（Gene Regulatory Network, GRN）从共表达与转录因子结合 motif 的证据推断基因之间的调控关系。CellChat 与 SCENIC 分别是这两类分析的代表性工具，二者结合可以重建”外部信号输入 → 细胞内转录因子响应 → 下游靶基因表达”这一完整调控链条。

学习目标：理解细胞通讯推断的配体-受体逻辑与 CellChat 的质量作用模型；能运行 CellChat 并解读信号通路网络与中心性指标；理解 SCENIC 三步框架（共表达 → motif 富集 → 活性打分）与 Regulon 概念；能运行 pySCENIC 并识别细胞类型特异的关键转录因子。

42 细胞通讯分析

42.1 从表达到通讯：推断逻辑

多细胞生物的协调依赖细胞间持续的信号交流。scRNA-seq 提供了一种计算推断细胞通讯的途径，其基本逻辑是：若细胞类型 A 高表达某配体（Ligand），而细胞类型 B 高表达该配体对应的受体（Receptor），则推断 A → B 之间存在该信号通路的潜在通讯。这一推断建立在一个简化假设上——配体与受体的 mRNA 表达水平能够代表蛋白质水平的信号交互能力。

{{Wiki:Cell_signaling}}

需要清醒认识这一假设的局限：mRNA 表达不等于蛋白质丰度，更不等于实际发生的物理结合；通讯推断给出的是”潜在可能”而非”已证实的相互作用”，所有计算预测都需要实验验证（如共培养、邻近标记、配体阻断）。

42.2 CellChat 的通讯概率模型

CellChat 是主流细胞通讯工具之一，其核心改进在于使用了人工整理的、包含多亚单位复合体的配体-受体数据库 CellChatDB，并用质量作用定律（mass action law）量化通讯强度（Jin et al., 2021）。对一对配体-受体，从细胞类型 \(i\) 到细胞类型 \(j\) 的通讯概率 \(P_{i,j}\) 由配体在 \(i\) 中的平均表达 \(L_i\) 与受体在 \(j\) 中的平均表达 \(R_j\) 共同决定，并通过 Hill 函数引入非线性饱和效应。

CellChat 的两项关键设计：其一，对由多个亚单位组成的受体复合体，取各亚单位表达的几何均值，从而要求所有必需亚单位都表达才认定受体有功能；其二，通过对细胞类型标签的置换检验（permutation test）评估通讯概率的统计显著性，控制假阳性。

42.3 CellChat 实操

下面在 PBMC 3k 注释数据上演示 CellChat 流程。

运行环境说明： 本节依赖 R 包 CellChat（GitHub 安装：devtools::install_github("jinworks/CellChat")）及其依赖 NMF、ComplexHeatmap。输入为第6章产出的、带 celltype 注释的标准化 Seurat 对象 pbmc。

library(CellChat)
library(Seurat)

# Build a CellChat object from normalized data + cell-type labels
data_input <- GetAssayData(pbmc, assay = "RNA", slot = "data")  # log-normalized
meta <- data.frame(labels = Idents(pbmc), row.names = colnames(pbmc))
cellchat <- createCellChat(object = data_input, meta = meta, group.by = "labels")

# Use the human ligand-receptor database (secreted signaling subset shown here)
cellchat@DB <- subsetDB(CellChatDB.human, search = "Secreted Signaling")

# Preprocess: identify over-expressed ligands/receptors and project onto PPI
cellchat <- subsetData(cellchat)
cellchat <- identifyOverExpressedGenes(cellchat)
cellchat <- identifyOverExpressedInteractions(cellchat)
# Core inference: communication probability per L-R pair, then aggregate to pathways
cellchat <- computeCommunProb(cellchat)            # mass-action + permutation test
cellchat <- filterCommunication(cellchat, min.cells = 10)
cellchat <- computeCommunProbPathway(cellchat)     # aggregate L-R pairs to pathways
cellchat <- aggregateNet(cellchat)
#| label: fig-cellchat-chord

42.4 通讯网络的解读：中心性与通讯模式

得到通讯网络后，CellChat 提供网络分析工具揭示每个细胞类型在信号网络中扮演的角色。中心性（centrality）分析量化细胞类型作为信号发送者（outdegree，配体输出强度）还是接收者（indegree，受体输入强度）的倾向：

# Compute network centrality to classify signaling roles per cell type
cellchat <- netAnalysis_computeCentrality(cellchat, slot.name = "netP")

# Heatmap of outgoing (sender) vs incoming (receiver) signaling strength
netAnalysis_signalingRole_heatmap(cellchat, pattern = "outgoing")
netAnalysis_signalingRole_heatmap(cellchat, pattern = "incoming")

更进一步，identifyCommunicationPatterns() 用非负矩阵分解（NMF）将复杂的通讯网络分解为若干”通讯模式”，揭示哪些细胞类型协同使用同一组信号通路，从而把数十条通路压缩为少数可解释的功能模块。

CellChat 与 CellPhoneDB 的方法学差异： 两者都依赖人工整理的配体-受体数据库，但统计策略不同。CellPhoneDB 直接对配体-受体表达做置换检验、在 L-R 对层面报告显著性；CellChat 额外引入了通路层面的抽象与质量作用模型。二者结果不完全一致是正常的，不应将任一工具的输出当作金标准。对关键通讯结论，建议用两种工具交叉验证，并最终通过湿实验确证。

43 基因调控网络推断：SCENIC

43.1 从共表达到调控：SCENIC 的三步框架

差异表达告诉我们哪些基因变了，却不告诉我们谁在指挥这些变化。基因调控网络推断的目标是识别转录因子（Transcription Factor, TF）及其靶基因构成的调控关系。SCENIC（Single-Cell rEgulatory Network Inference and Clustering）是单细胞 GRN 推断的标杆工具，其精妙之处在于用三步逐级提纯，把海量的统计共表达关系收敛为有生物学依据的调控模块（Aibar et al., 2017）：

  1. 共表达网络构建（GENIE3 / GRNBoost2）：对每个基因，用树集成模型从所有 TF 的表达预测其表达，以变量重要性作为 TF-靶基因候选关系的权重（原理参见专题 5.6 的 GENIE3）。这一步只用表达数据，会产生大量假阳性——共表达不等于调控。
  2. motif 富集过滤（RcisTarget）：对每个 TF 的候选靶基因集，检验这些靶基因的启动子/调控区是否显著富集该 TF 的结合 motif。只有同时满足”共表达”与”motif 证据”的 TF-靶基因关系才被保留，构成一个 Regulon（调控子）——即一个 TF 及其经 motif 验证的直接靶基因集合。
  3. 调控子活性打分（AUCell）：对每个细胞，用 AUCell 计算每个 Regulon 的活性得分——本质是该 Regulon 的靶基因在该细胞表达排名中的富集程度（曲线下面积 AUC）。由此得到”细胞 × Regulon 活性”矩阵，可像表达矩阵一样用于聚类与可视化。

{{Wiki:Gene_regulatory_network}}

SCENIC 的核心价值在于第二步：通过引入 motif 这一独立的、机制性的证据，它从纯统计的共表达关系中筛出了更可能是真实直接调控的关系，大幅降低了假阳性。

43.2 Regulon 与 AUCell 的意义

Regulon 活性比单个 TF 的表达量更能反映其调控功能。原因有二：其一，TF 常受翻译后调控，其 mRNA 水平未必反映蛋白活性；其二，AUCell 基于一组靶基因的协同富集，对单个基因的测量噪声（dropout）更鲁棒。因此，“某细胞中某 Regulon 高活性”比”某细胞高表达某 TF”是更可靠的调控状态读数。

43.3 pySCENIC 实操

由于 SCENIC 的共表达步骤计算量大，大规模数据推荐使用 Python 实现 pySCENIC，它用更快的 GRNBoost2（基于梯度提升树）替代 GENIE3，并与 Scanpy/AnnData 生态兼容（Van de Sande et al., 2020）。

运行环境说明： 本节代码使用 Python 的 pySCENIC（pip install pyscenic）。运行需要三类辅助数据库文件：TF 列表、cisTarget motif 排名数据库（.feather）与 motif 注释表（.tbl），均可从 SCENIC 官方资源下载（物种特异）。共表达步骤对数千细胞仍可能耗时数十分钟，属重计算任务，课堂演示建议预先缓存中间结果。

import loompy as lp
import pandas as pd
from pyscenic.utils import load_tf_names

# Step 1: GRNBoost2 co-expression network (TF -> target importance)
# Run via the CLI for parallelism; shown here as the canonical command:
#   pyscenic grn expr_mat.loom allTFs_hg38.txt -o adj.tsv --num_workers 4

# Step 2: prune co-expression modules by cisTarget motif enrichment -> Regulons
#   pyscenic ctx adj.tsv \
#       hg38__refseq-r80__500bp_up_and_100bp_down_tss.mc9nr.feather \
#       --annotations_fname motifs-v9-nr.hgnc.tbl \
#       --expression_mtx_fname expr_mat.loom \
#       --output reg.csv --num_workers 4

# Step 3: AUCell scores each cell for each Regulon's activity
#   pyscenic aucell expr_mat.loom reg.csv --output pyscenic_output.loom --num_workers 4
import scanpy as sc
import numpy as np

# Load the AUCell Regulon-activity matrix (cells x regulons) for downstream analysis
lf = lp.connect("pyscenic_output.loom", mode="r", validate=False)
auc_mtx = pd.DataFrame(lf.ca.RegulonsAUC, index=lf.ca.CellID)
lf.close()

# Cell-type-averaged Regulon activity reveals lineage-defining TFs
# (e.g., high EOMES/TBX21 regulon activity in T/NK cells)
auc_mtx["cell_type"] = adata.obs["cell_type"].values
regulon_by_celltype = auc_mtx.groupby("cell_type").mean()
print(regulon_by_celltype.iloc[:, :5])
# Heatmap of Regulon activity across cell types highlights master regulators
sc.pl.matrixplot(
    adata,
    var_names=adata.var_names,    # replace with top regulons of interest
    groupby="cell_type",
    title="Regulon activity by cell type",
    save="regulon_heatmap.png",
)

上述热图的典型形态是：每种细胞类型由一组特异的高活性 Regulon 定义（呈对角线状的活性分块），这些 TF 即该细胞类型的”主调控因子”（master regulator）。

通讯与调控的整合解读： CellChat 给出”细胞收到了什么外部信号输入”，SCENIC 给出”细胞内哪些转录因子被激活、调控了哪些靶基因”。将二者串联，可以构建机制性假设：例如，若 CellChat 显示某细胞群强烈接收 IFN 信号，而 SCENIC 显示该群 STAT1/IRF 家族 Regulon 高活性，则二者共同支持”IFN 信号经 JAK-STAT 通路激活 IRF 转录程序”的调控链条。这类跨层次整合是高质量单细胞机制研究的标志。

44 方法总结与衔接说明

本专题从两个互补的视角刻画了细胞的调控逻辑：CellChat 推断细胞间的配体-受体通讯，SCENIC 推断细胞内的转录因子调控网络。两类方法都建立在专题 7.1 整合、注释的图谱之上，且都属于计算预测——其结论是待验证的假设，而非已证实的事实。下一专题（7.4）将转向单细胞多组学，通过在同一细胞中直接测量蛋白质（CITE-seq）或染色质可及性（10x Multiome），为这些调控推断提供正交的实验证据。

45 课后自测 (Post-Lesson Quiz)

  1. 细胞通讯分析的基本推断逻辑是什么？这一推断建立在哪个关于 mRNA 表达的简化假设上？该假设有何局限？

  2. CellChat 在处理多亚单位受体复合体时为何取各亚单位表达的几何均值？这一处理保证了什么生物学约束？

  4. SCENIC 三步框架分别是什么？每一步的输入与输出是什么？为什么说第二步（motif 富集）是 SCENIC 区别于纯共表达方法的关键？

  5. 什么是 Regulon（调控子）？为什么 Regulon 的 AUCell 活性比单个转录因子的 mRNA 表达量更能反映其调控功能？

  6. pySCENIC 相比原始 R 版 SCENIC 在共表达步骤上做了什么改进？为什么大规模数据推荐使用 pySCENIC？

  7. 为什么说 CellChat 与 SCENIC 的所有输出都属于”计算预测”而非”已证实的相互作用”？应如何对待关键结论？

  8. 某细胞群在 CellChat 中强烈接收 IFN 信号，同时在 SCENIC 中 STAT1/IRF 家族 Regulon 高活性。请说明如何整合这两个结果构建一个机制性假设，并指出验证该假设的实验方向。

参考答案 (Reference Answers)

  1. 基本逻辑：若细胞类型 A 高表达某配体、细胞类型 B 高表达对应受体，则推断 A→B 存在该通路的潜在通讯。它假设配体/受体的 mRNA 表达水平能代表蛋白质水平的信号交互能力。局限是 mRNA 不等于蛋白质丰度，更不等于实际发生的物理结合，因此推断给出的是”潜在可能”而非”已证实的相互作用”，需实验验证。

  2. 因为功能性受体复合体需要所有必需亚单位同时表达才能工作。取几何均值使得只要任一必需亚单位表达很低，整体受体信号就被拉低，从而强制要求所有亚单位共表达，避免把缺失关键亚单位的”半个受体”误判为有功能的受体。

  3. outdegree 衡量细胞类型作为配体输出源的信号发送强度，indegree 衡量其作为受体输入端的信号接收强度。若某细胞类型 outdegree 高，则它主要是信号发送者（如分泌配体的细胞）；若 indegree 高，则主要是接收者（如高表达受体响应信号的细胞）。

  4. 第一步共表达网络（GENIE3/GRNBoost2）：输入表达矩阵，输出 TF-靶基因候选权重；第二步 motif 富集（RcisTarget）：输入候选模块，输出经 motif 验证的 Regulon；第三步 AUCell：输入 Regulon 与表达矩阵，输出细胞×Regulon 活性矩阵。第二步关键是因为它引入了独立的、机制性的 motif 证据，从纯统计共表达关系中筛出更可能是真实直接调控的关系，大幅降低假阳性。

  5. Regulon 是一个转录因子及其经 motif 验证的直接靶基因集合。AUCell 活性更能反映调控功能的原因：（1）TF 常受翻译后调控，mRNA 水平未必反映蛋白活性；（2）AUCell 基于一组靶基因的协同富集，对单个基因的 dropout 噪声更鲁棒。因此”Regulon 高活性”比”TF 高表达”是更可靠的调控状态读数。

  6. pySCENIC 用更快的 GRNBoost2（基于梯度提升树）替代 GENIE3（随机森林）进行共表达推断，并与 Scanpy/AnnData 生态兼容。大规模数据推荐它是因为共表达步骤对每个基因都要训练树模型、计算量极大，GRNBoost2 在保持精度的同时显著提升了计算效率与可扩展性。

  7. 因为两者都仅基于表达数据与人工整理的数据库做统计推断，没有直接测量蛋白结合或调控的物理事件；共表达、motif 富集、配体-受体表达匹配都只是间接证据。对关键结论应：用不同工具交叉验证，并通过湿实验（如共培养、邻近标记、配体阻断、ChIP/报告基因实验）确证，避免把预测当作定论。

  8. 机制性假设：IFN 信号经细胞表面受体激活 JAK-STAT 通路，使 STAT1 磷酸化入核，进而激活 IRF 家族转录程序，驱动干扰素刺激基因（ISGs）表达。整合依据是 CellChat 提供”该群接收 IFN 信号输入”、SCENIC 提供”该群 STAT1/IRF Regulon 高活性”，二者在信号-转录两个层次相互印证。验证方向：用 IFN 中和抗体或 JAK 抑制剂阻断信号，观察 IRF Regulon 活性与 ISG 表达是否下降；或对 STAT1 做 ChIP-seq 验证其确实结合预测靶基因启动子。
