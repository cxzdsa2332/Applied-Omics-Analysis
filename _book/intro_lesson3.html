Published

April 9, 2026

5.1 Linux 系统简介

Linux 是一种开源的类 Unix 操作系统内核，由 Linus Torvalds 于 1991 年首次发布。它以其稳定性、安全性和灵活性著称，在服务器、嵌入式系统和桌面环境中广泛应用。在生物信息学领域，Linux 是处理大规模数据分析的首选平台。

5.1.1 Linux 发行版概述

Linux 内核本身并不构成完整的操作系统。为了提供用户友好的体验，开发者基于 Linux 内核创建了各种发行版（Distribution）。这些发行版通常包括： - Linux 内核 - GNU 工具链 - 包管理系统 - 桌面环境（可选） - 预装软件

发行版主要分为两大阵营： - RedHat 系列：企业级，注重稳定性，包管理系统使用 RPM - Debian 系列：社区驱动，注重自由软件，包管理系统使用 DEB

5.1.2 常见 Linux 发行版介绍

5.1.2.1 RedHat 系列

  • Red Hat Enterprise Linux (RHEL)：商业发行版，专为企业环境设计，提供长期支持和商业服务。广泛用于生产服务器。
    Red Hat 是第一家成功将 Linux 商业化的公司，成立于 1993 年，其吉祥物是一只红帽子的浣熊。
  • CentOS：RHEL 的社区克隆版，完全兼容 RHEL，但免费。常用于学术和科研环境的高性能计算集群。
  • Fedora：RedHat 的上游发行版，包含最新软件和技术，常用于开发和测试环境。
    Fedora 以美国帽子品牌命名，象征着自由和创新，其版本号常与科幻电影《星际迷航》中的企业号相关联。

5.1.2.2 Debian 系列

  • Debian：最古老的 Linux 发行版之一，以稳定性著称。遵循严格的自由软件准则，包管理系统优秀。
    Debian 以创始人 Ian Murdock 的女友 Debra Lynn 的名字命名，是第一个完全由志愿者维护的发行版。

  • Ubuntu：基于 Debian 的用户友好发行版，最流行的桌面 Linux 系统之一。Ubuntu Server 版本常用于云计算和生物信息学服务器。

  • Linux Mint：基于 Ubuntu 的发行版，提供更传统的桌面体验。

5.1.2.3 其他常见发行版

  • SUSE Linux Enterprise (SLES)：企业级发行版，与 RedHat 竞争，提供强大的管理工具。

  • openSUSE：SUSE 的社区版，注重易用性和最新软件。

  • Arch Linux：滚动更新发行版，注重简洁和最新软件，适合高级用户。

  • Gentoo：源代码编译发行版，提供高度定制化，但安装复杂。

5.1.3 在生物信息学中的应用

不同发行版在生物信息学社区中的应用各有优势： - CentOS/RHEL：学术 HPC 集群的首选，许多生物信息学软件在这些系统上经过充分测试。 - Ubuntu：生物信息学家入门和开发环境的热门选择，易于安装和配置 (个人配置首选)。 - Debian：稳定可靠，适合长期运行的服务器。 - Arch Linux：前沿用户，用于测试最新工具和库。

选择发行版时，应考虑： - 硬件兼容性：确保发行版支持你的硬件 - 软件生态：检查所需生物信息学工具的可用性 - 社区支持：活跃的社区能提供及时帮助 - 维护成本：商业支持 vs 社区支持

在本课程中，我们将重点使用类 CentOS 系统进行演示，但所学知识适用于所有 Linux 发行版。

5.2 告别鼠标，掌握算力的绝对控制权

对于绝大多数人来说，计算机等同于“视窗、图标和鼠标点击”（GUI，图形用户界面）。但在生命科学的大数据时代，一个标准的全基因组测序原始文件（FASTQ）动辄 50GB 甚至 100GB。如果你尝试用 Windows 的记事本或 Excel 去双击打开它，结果只有一个：电脑死机。

在生物信息学和极客的世界里，真正的计算发生在没有屏幕、没有鼠标的服务器机房里。本章通过学习纯文本的命令行界面（CLI, Command Line Interface）直接与计算机的对话。

5.3 为什么生物信息学离不开 Linux？

一般对 Windows 或 macOS 很熟悉，但想要处理高通量测序数据，Linux 操作系统是绕不过去的门槛。

  • 极致的算力释放：Windows 为了维持华丽的图形界面，在后台消耗了大量的 CPU 和内存。而运行在高性能计算集群（HPC）上的 Linux 服务器通常只有黑乎乎的终端，它将 \(99.9\%\) 的硬件资源毫无保留地倾注于数据计算。
  • 无可比拟的稳定性：Linux 系统设计哲学强调“分时、多用户与高并发”，一台配置得当的 Linux 服务器可以连续运行数年而不必重启，完美契合动辄需要计算几十个小时的基因组组装任务。
  • 开源生态与生信工具链：全球 \(99\%\) 以上的权威生物信息学软件（如 BWA, SAMtools, GATK）都是基于 Linux/Unix 环境利用 C/C++ 或 Python 开发的开源软件。许多软件压根就没有 Windows 版本。

5.4 核心架构：Linux 文件系统层级树

在 Windows 中，我们习惯了 C盘、D盘 的概念。在 Linux 中，没有盘符，只有一棵倒立的树。 Linux 的核心哲学之一是：“一切皆文件（Everything is a file）”。

5.4.1 两个最重要的符号：/ 与 ~

  • 根目录（/）：这是整个文件系统树的起点。所有的文件、外接硬盘、光驱，甚至内存，全部都挂载在这个根目录下的不同分支里。
  • 用户家目录（~）：在服务器中，你只是众多用户中的一个。系统会为你分配一个专属的地盘，通常位于 /home/你的用户名/。为了方便，Linux 用波浪号 ~ 来指代你自己的家目录。

5.4.2 定位逻辑：绝对路径 vs 相对路径

这是初学者最容易死机的概念： * 绝对路径（Absolute Path）：永远以根目录 / 开头。它就像地理上的经纬度绝对坐标，无论你当前在系统的哪个角落，顺着这个路径一定能找到目标。 * 例如：/home/user/data/sample.fastq * 相对路径（Relative Path）：不以 / 开头。它是基于你当前所在位置的路线指引。这就像问路时别人告诉你“前面路口左转”。 * . 代表当前目录。 * .. 代表上一级（父）目录。 * 例如：如果你现在在 /home/user/ 下，你想访问同样的文件，相对路径就是 data/sample.fastq 或 ./data/sample.fastq。

Note互动学习：终端与文件系统模拟器

为了帮助大家建立对命令行和文件目录树的直觉，请在下方的模拟终端中尝试以下命令： 1. 输入 pwd 查看你出生在哪儿。 2. 输入 ls 看看周围有什么。 3. 输入 mkdir data 创建一个数据文件夹。 4. 输入 cd data 走进去。 5. 输入 cd .. 退回上一级。

5.5 基本 bash 操作命令讲解

在掌握了文件系统的基础概念后，让我们学习一些最常用的 bash 命令。这些命令是生物信息学工作中不可或缺的工具，可以帮助你高效地管理文件、运行程序和处理数据。

5.5.1 文件和目录操作

5.5.1.1 ls - 列出目录内容

ls 命令用于查看当前目录下的文件和子目录。

ls                    # 列出当前目录的文件和文件夹
ls -l                 # 详细列表，包括权限、大小、修改时间等
ls -a                 # 显示所有文件，包括隐藏文件（以 . 开头的文件）
ls -lh                # 人类可读的文件大小（KB, MB 等）
ls /home/user/        # 列出指定目录的内容

5.5.1.2 cd - 改变当前目录

cd 命令用于切换到不同的目录。

cd /path/to/directory  # 切换到绝对路径的目录
cd relative/path       # 切换到相对路径的目录
cd ..                  # 切换到上级目录
cd ~                   # 切换到用户家目录
cd -                   # 切换到上一个目录

5.5.1.3 pwd - 显示当前工作目录

pwd 显示你当前所在的完整路径。

pwd  # 输出类似 /home/user/projects

5.5.1.4 mkdir - 创建目录

mkdir 用于创建新的目录。

mkdir new_folder           # 创建单个目录
mkdir -p path/to/nested    # 创建嵌套目录（如果父目录不存在也会创建）
mkdir data results logs    # 一次创建多个目录

5.5.1.5 touch - 创建空文件或更新文件时间戳

touch 主要用于创建空文件，或更新现有文件的访问和修改时间。

touch new_file.txt        # 创建一个空文件
touch file1.txt file2.txt # 创建多个空文件

5.5.2 文件操作

5.5.2.1 cp - 复制文件或目录

cp 用于复制文件和目录。

cp source.txt destination.txt        # 复制文件
cp -r source_dir destination_dir     # 递归复制目录
cp file1.txt file2.txt backup/       # 复制多个文件到目录
cp -i source.txt destination.txt     # 复制前询问是否覆盖

5.5.2.2 mv - 移动或重命名文件

mv 用于移动文件或重命名。

mv old_name.txt new_name.txt    # 重命名文件
mv file.txt /path/to/directory/ # 移动文件到新目录
mv *.txt archive/               # 移动所有 .txt 文件到 archive 目录

5.5.2.3 rm - 删除文件或目录

rm 用于删除文件和目录。注意：删除的文件通常无法恢复！

rm file.txt              # 删除文件
rm -r directory/         # 递归删除目录及其内容
rm -i file.txt           # 删除前询问确认
rm -rf directory/        # 强制删除目录，不询问（危险操作）

5.5.3 其他常用命令

5.5.3.1 echo - 输出文本

echo 用于在终端显示文本或变量内容。

echo "Hello World"       # 输出字符串
echo $HOME               # 输出环境变量的值
echo "Current date: $(date)"  # 使用命令替换

5.5.3.2 cat - 查看文件内容

cat 用于显示文件内容。

cat file.txt             # 显示整个文件内容
cat file1.txt file2.txt  # 显示多个文件内容
cat -n file.txt          # 显示文件内容并编号行

5.5.3.3 head 和 tail - 查看文件开头或结尾

head 显示文件的前几行，tail 显示文件的后几行。

head file.txt            # 显示前 10 行
head -20 file.txt        # 显示前 20 行
tail file.txt            # 显示后 10 行
tail -50 file.txt        # 显示后 50 行
tail -f log.txt          # 实时监控文件末尾（用于日志文件）

5.5.3.4 grep - 搜索文本

grep 用于在文件中搜索特定的文本模式。

grep "pattern" file.txt              # 在文件中搜索 pattern
grep -i "pattern" file.txt           # 忽略大小写搜索
grep -r "pattern" /path/to/directory # 在目录中递归搜索
grep -n "pattern" file.txt           # 显示匹配行的行号

这些基本命令构成了 bash 操作的基础。在生物信息学实践中，你会经常使用这些命令来管理数据文件、运行分析脚本和监控计算任务。建议你在终端中多加练习，熟悉这些命令的用法和选项。

5.6 服务器调度系统简介

在生物信息学领域，许多计算任务（如基因组组装、RNA-seq 分析、变异检测）需要大量的计算资源。这些任务往往运行在高性能计算集群（HPC, High Performance Computing）上，而不是单台个人电脑。为了高效管理和分配这些资源，HPC 系统通常配备了作业调度系统（Job Scheduler）。

5.6.1 为什么需要作业调度系统？

想象一下，一个生物信息学实验室有数百名研究人员同时提交计算任务。如果没有调度系统： - 所有任务同时运行会导致系统崩溃 - 资源分配不均，一些任务占用过多 CPU/内存 - 无法优先处理紧急任务 - 难以监控任务状态和资源使用

调度系统解决了这些问题： - 队列管理：任务按顺序排队等待执行 - 资源分配：根据任务需求分配 CPU 核心、内存、GPU 等 - 优先级控制：重要任务可以优先执行 - 负载均衡：避免单点过载 - 监控和报告：实时查看任务状态和系统负载

5.6.2 常见的作业调度系统

在学术和科研环境中，最常用的调度系统包括：

  1. Slurm：最流行，开源，广泛用于学术 HPC 集群
  2. PBS/Torque：历史悠久，许多老系统仍在使用
  3. LSF (Load Sharing Facility)：商业系统，功能强大但收费
  4. SGE (Sun Grid Engine)：开源，但逐渐被 Slurm 取代

在本课程中，我们重点介绍 Slurm，因为它是当前最主流的调度系统。

5.6.3 Slurm 基本概念

  • 作业 (Job)：用户提交的计算任务
  • 队列 (Queue/Partition)：任务等待执行的队列，不同队列可能有不同的资源限制
  • 节点 (Node)：集群中的计算服务器
  • 核心 (Core)：CPU 核心数量
  • 任务 ID (Job ID)：每个提交的作业都会获得一个唯一 ID

5.6.4 基本 Slurm 命令

5.6.4.1 sbatch - 提交批处理作业

使用 sbatch 提交脚本文件执行作业。

sbatch my_script.sh          # 提交作业脚本
sbatch -p gpu_partition script.sh  # 指定队列提交
sbatch -N 2 -n 16 script.sh  # 请求 2 个节点，16 个 CPU 核心

作业脚本示例（my_script.sh）：

#!/bin/bash
#SBATCH --job-name=my_analysis    # 作业名称
#SBATCH --output=output.log       # 输出文件
#SBATCH --error=error.log         # 错误文件
#SBATCH --time=24:00:00           # 最大运行时间
#SBATCH --mem=8GB                 # 内存需求
#SBATCH --cpus-per-task=4         # CPU 核心数

# 你的计算命令
echo "开始分析..."
bwa mem reference.fa reads.fastq > alignment.sam
echo "分析完成"

5.6.4.2 squeue - 查看作业队列

查看当前队列中的作业状态。

squeue                    # 查看所有用户的作业
squeue -u username        # 查看特定用户的作业
squeue -p partition_name  # 查看特定队列的作业

常见状态： - PD: Pending（等待中） - R: Running（运行中） - CG: Completing（完成中） - CD: Completed（已完成）

5.6.4.3 scancel - 取消作业

取消正在运行或等待的作业。

scancel 12345             # 取消作业 ID 12345
scancel -u username       # 取消用户的所有作业
scancel --name=job_name   # 取消指定名称的作业

5.6.4.4 sinfo - 查看集群状态

查看集群中节点和队列的信息。

sinfo                     # 查看所有队列和节点状态
sinfo -p partition_name   # 查看特定队列的状态

5.6.4.5 sacct - 查看作业历史

查看已完成作业的详细信息。

sacct                    # 查看最近作业的历史
sacct -j 12345           # 查看特定作业的详细信息
sacct --starttime=2024-01-01  # 查看指定时间后的作业

5.6.5 作业脚本编写要点

一个好的 Slurm 作业脚本应该包含： 1. Shebang：#!/bin/bash 2. 资源请求：使用 #SBATCH 指令指定 CPU、内存、时间等 3. 环境设置：加载必要的模块（module load） 4. 工作目录：切换到合适的工作目录 5. 命令执行：实际的计算命令 6. 输出重定向：虽然 #SBATCH --output 已指定，但仍可使用 > 重定向

示例完整脚本：

#!/bin/bash
#SBATCH --job-name=rnaseq_analysis
#SBATCH --output=rnaseq_%j.out    # %j 会替换为作业 ID
#SBATCH --error=rnaseq_%j.err
#SBATCH --time=48:00:00
#SBATCH --mem=32GB
#SBATCH --cpus-per-task=8
#SBATCH --partition=compute

# 加载软件模块
module load bwa/0.7.17
module load samtools/1.9

# 切换到工作目录
cd /path/to/your/data

# 执行分析
echo "开始 RNA-seq 比对..."
bwa mem reference.fa reads_1.fastq reads_2.fastq | samtools view -bS - > alignment.bam
samtools sort alignment.bam -o sorted_alignment.bam
samtools index sorted_alignment.bam

echo "分析完成！"

5.6.6 最佳实践

  • 合理估算资源：不要请求过多资源（浪费），也不要请求过少（失败）
  • 测试小任务：先提交小测试作业验证脚本正确性
  • 监控作业：定期检查 squeue 和日志文件
  • 清理文件：作业完成后及时清理临时文件
  • 使用数组作业：对于多个相似任务，使用作业数组提高效率

掌握作业调度系统是生物信息学实践的关键技能。它不仅帮助你高效利用计算资源，还能确保你的分析任务在大型集群上顺利执行。在后续课程中，我们会通过实际案例深入学习如何编写和优化 Slurm 作业脚本。
