11 Adapter 污染：从文库构建到污染机制

11.1 Adapter 序列的来源

在第1节中，我们介绍了文库构建时需要在 DNA 片段两端添加 P5/P7 接头（Adapter）。理解 adapter 污染的机制，需要回到文库构建的本质：测序仪读取的实际上是”Adapter + Insert + Adapter”这一完整分子。

{{Wiki:Illumina_dye_sequencing}}

标准 Illumina TruSeq Adapter 序列（Read 1 端）：

5'- AGATCGGAAGAGCACACGTCTGAACTCCAGTCAC -3'

这一序列在绝大多数 Illumina 文库中保持一致，因此成为 FastQC “Adapter Content” 模块的默认检测对象。

11.2 Insert Size 与 Read-Through 现象

Adapter 污染的出现具有明确的几何条件：当 insert size（DNA 片段长度）小于 read 长度时，测序仪在读完 insert 后会继续读入对侧的 adapter 序列。

正常情况（insert > read 长度）：
  [--Adapter P5--][--------Insert (300 bp)--------][--Adapter P7--]
  |<----Read 1 (150 bp)---->|

Read-through 情况（insert < read 长度）：
  [--Adapter P5--][--Insert (80 bp)--][--Adapter P7--]
  |<----Read 1 (150 bp)---->|
                              ^^^^ 这70 bp读到了Adapter P7

为什么短 insert 文库更易污染？

对于小 RNA（miRNA，约 22 nt）或 ATAC-seq（nucleosome-free region，约 150 bp）文库，insert size 远小于 read 长度，几乎所有 reads 都会发生 read-through。因此，这类实验数据在比对前必须进行 adapter trimming，否则短小的真实序列会被 adapter 序列淹没，导致比对完全失败。

11.3 Adapter 污染对下游分析的影响

如果不去除 adapter，将直接产生以下问题：

影响 原因 后果
比对失败 Adapter 序列在参考基因组中不存在 比对率下降 5–40%
假阳性变异 Adapter 序列在比对末端引入错配 SNP/Indel 假阳性增加
长度偏差 Adapter 使 reads 人工变长 Insert size 估算失准
拼接错误 Adapter 干扰 de novo 组装的 k-mer 计数 基因组组装破碎化

12 数据预处理：工具与实战

12.1 工具选型

目前主流的 trimming 工具有三种，各有侧重：

工具 发布年份 语言 特点
Trimmomatic 2014 Java 参数最灵活，管道式处理，文档完善
Cutadapt 2011 Python/C 最早的专用工具，精确控制每步操作
fastp 2018 C++ 速度最快（多线程），自动检测 adapter，内置 QC 报告

实际工作的工具选择建议：

新项目推荐使用 fastp，原因：①无需手动指定 adapter 序列（自动检测）；②单命令完成 QC + trimming + 报告生成；③速度比 Trimmomatic 快 3–5 倍；④输出 JSON 格式报告，便于 MultiQC 集成。Trimmomatic 仍在许多既有流程中广泛使用，理解其参数有助于阅读他人代码。

12.2 fastp 实战：处理真实 E. coli 数据

延续第1节的数据集（SRR2584863），对下载的 50k reads 子集进行预处理。

# Install fastp via conda
conda install -c bioconda fastp

# Run fastp on PE data
fastp \
    --in1  SRR2584863_1_50k.fastq.gz \
    --in2  SRR2584863_2_50k.fastq.gz \
    --out1 SRR2584863_1_clean.fastq.gz \
    --out2 SRR2584863_2_clean.fastq.gz \
    --html fastp_report.html \
    --json fastp_report.json \
    --thread 4 \
    --qualified_quality_phred 20 \
    --length_required 50 \
    --detect_adapter_for_pe \
    --correction \
    --overrepresentation_analysis

关键参数解析：

参数 含义 推荐值
--qualified_quality_phred 20 低于 Q20 的碱基替换为 N 20（宽松）或 30（严格）
--length_required 50 短于 50 bp 的 reads 丢弃 WGS: 50；miRNA: 15
--detect_adapter_for_pe 自动检测 PE adapter 推荐始终开启
--correction 对 R1/R2 重叠区域进行碱基校正 PE 数据推荐开启
--overrepresentation_analysis 检测过度富集序列 推荐开启

fastp 真实输出摘要：

Read1 before filtering:
  total reads: 50000
  total bases: 7550000
  Q20 bases: 7395121(97.95%)
  Q30 bases: 6989842(92.58%)

Read1 after filtering:
  total reads: 49312
  total bases: 7426836
  Q20 bases: 7366204(99.18%)
  Q30 bases: 6985761(94.05%)

Filtering result:
  reads passed filter: 98624 (98.62%)
  reads failed due to low quality: 916
  reads failed due to too short: 460
  reads with adapter trimmed: 2847 (5.69%)
  bases trimmed due to adapters: 38924

解读 fastp 报告

本数据集（E. coli WGS）中 adapter 污染率约 5.69%，属于轻度污染。这源于文库构建时少量 insert size 较短的片段（低于 150 bp）发生了 read-through。过滤后 Q30 比例从 92.58% 提升至 94.05%，表明 trimming 显著改善了数据质量。

12.3 Trimmomatic 参数深解

尽管 fastp 已是新项目首选，理解 Trimmomatic 的管道逻辑对阅读既有流程代码不可或缺。

# Install Trimmomatic
conda install -c bioconda trimmomatic

# PE mode: order of operations matters
trimmomatic PE \
    SRR2584863_1_50k.fastq.gz \
    SRR2584863_2_50k.fastq.gz \
    R1_paired.fastq.gz R1_unpaired.fastq.gz \
    R2_paired.fastq.gz R2_unpaired.fastq.gz \
    ILLUMINACLIP:TruSeq3-PE-2.fa:2:30:10:8:true \
    LEADING:3 \
    TRAILING:3 \
    SLIDINGWINDOW:4:15 \
    MINLEN:50

Trimmomatic 按照从左到右的顺序依次执行每个操作：

原始 Read → ILLUMINACLIP → LEADING → TRAILING → SLIDINGWINDOW → MINLEN → 输出

ILLUMINACLIP 参数格式： adapters.fa:<seed_mismatches>:<palindrome_clip>:<simple_clip>

ILLUMINACLIP 子参数 含义
2 (seed_mismatches) Adapter 种子序列允许的最大错配数
30 (palindrome_clip) PE palindrome 模式的匹配分阈值
10 (simple_clip) 简单模式的匹配分阈值
8 (min_adapter_length) 触发截除所需的最短 adapter 匹配长度

SLIDINGWINDOW:4:15 的含义：以 4 bp 为窗口滑动，若窗口内平均质量低于 Q15，则从该位置截断至末端。

Trimmomatic 输出的 “unpaired” 文件处理：

PE 模式下，若 R1 通过过滤而 R2 被丢弃（或反之），该 R1 会进入”unpaired”文件而非被直接丢弃。这些 unpaired reads 可以用于 SE 比对，但大多数 PE 流程会选择忽略它们以维持 read pair 配对关系的严格性。

12.4 预处理前后质量对比（Python 可视化）

import gzip
import numpy as np
import matplotlib.pyplot as plt
from collections import defaultdict

def per_position_quality(filepath, max_reads=10000):
    """Compute per-position mean quality from a gzipped FASTQ file."""
    pos_q = defaultdict(list)
    count = 0
    opener = gzip.open if filepath.endswith(".gz") else open
    with opener(filepath, "rt") as f:
        while True:
            header = f.readline()
            if not header:
                break
            _ = f.readline()
            __ = f.readline()
            qual = f.readline().strip()
            for pos, c in enumerate(qual):
                pos_q[pos].append(ord(c) - 33)
            count += 1
            if count >= max_reads:
                break
    positions = sorted(pos_q)
    means = [np.mean(pos_q[p]) for p in positions]
    return positions, means

files = {
    "Before (R1 raw)":   "SRR2584863_1_50k.fastq.gz",
    "After  (R1 clean)": "SRR2584863_1_clean.fastq.gz",
}

fig, ax = plt.subplots(figsize=(10, 5))
colors = ["#d62728", "#2ca02c"]
for (label, path), color in zip(files.items(), colors):
    positions, means = per_position_quality(path)
    ax.plot(positions, means, label=label, color=color, linewidth=1.5)

ax.axhline(30, color="gray", linestyle="--", linewidth=1, label="Q30 threshold")
ax.set_xlabel("Position in Read (bp)")
ax.set_ylabel("Mean Phred Quality Score")
ax.set_title("Per-Position Quality: Before vs After fastp Trimming")
ax.set_ylim(20, 42)
ax.legend()
ax.grid(True, alpha=0.3)
plt.tight_layout()
plt.savefig("trimming_comparison.png", dpi=150)
plt.show()

13 参考基因组：选择、结构与获取

13.1 什么是参考基因组

参考基因组（Reference Genome） 是一个由人工拼接、经过反复验证的代表性基因组序列，以 FASTA 格式存储。它并非真实个体的基因组，而是由多个个体的测序数据拼接而成的”共识序列”，是变异检测的坐标系基础。

{{Wiki:Reference_genome}}

参考基因组的哲学困境

参考基因组本质上是对”典型人类基因组”的近似。人类群体中存在大量结构变异（SV），某些基因座（如 HLA 区域、着丝粒）在不同个体间差异极大，任何单一参考序列都无法完美代表所有人群。这一问题正推动图泛基因组（Pangenome）研究的发展，但目前 hg38 仍是临床和科研的实用标准。

13.2 人类参考基因组的演变

{{Wiki:GRCh38}}

版本 别名 发布年份 主要改进 现状
GRCh36 hg18 2006 第一个完整草图 已废弃
GRCh37 hg19 2009 修复大量 assembly gap 仍有大量既有数据
GRCh38 hg38 2013 增加 ALT contigs、decoy 序列、修复着丝粒附近区域 新项目标准
T2T-CHM13 CHM13v2.0 2022 首次真正填补着丝粒和端粒缝隙，完全无 gap 科研前沿，临床未普及

hg19 与 hg38 坐标不可混用！

同一基因在 hg19 与 hg38 中的染色体坐标往往相差数千至数百万碱基。若将用 hg38 比对的 BAM 文件配合 hg19 的 VCF 或注释文件使用，会产生完全错误的结果。参考基因组版本必须在项目内保持一致，包括：FASTQ 比对所用参考、dbSNP 数据库版本、基因注释 GTF/GFF、GATK Bundle 文件，全部必须匹配。

13.3 GRCh38 参考基因组文件的结构

从 NCBI 或 Ensembl 下载的 hg38 参考基因组 FASTA 文件包含多种类型的序列，理解其命名规则对正确配置分析流程至关重要：

染色体命名 类型 说明
chr1 ~ chr22, chrX, chrY, chrM 主染色体 核心比对目标
chrUn_* 未定位序列 无法确定染色体归属的 contigs
chr*_random 随机序列 染色体已知但位置未确定
chr*_alt ALT contigs 高变异区域的替代单倍型序列
chrEBV 解链病毒 用于捕获 EBV 污染
hs37d5 / decoy Decoy 序列 陷阱序列，捕获无法比对至主染色体的 reads

Decoy 序列的作用 尤为重要：若不包含 decoy 序列，本应比对至非主染色体区域的 reads 会被强制比对至主染色体的低复杂度区域，产生大量假阳性变异。GATK 最佳实践（Best Practices）明确要求使用带 decoy 序列的 hg38（GRCh38_full_analysis_set_plus_decoy_hla.fa）。

13.4 实践：下载 E. coli 参考基因组

本节实践继续使用 E. coli K-12 MG1655。其参考基因组（RefSeq: NC_000913.3）经过完全封闭（fully closed），无 gap，是教学的理想材料。

# Download E. coli K-12 MG1655 reference genome from NCBI

     -O ecoli_K12.fna.gz

#for mac users, use curl with -L (follow redirects) and -C - (resume download)
curl -L -C - "https://ftp.ncbi.nlm.nih.gov/genomes/all/GCF/000/005/845/GCF_000005845.2_ASM584v2/GCF_000005845.2_ASM584v2_genomic.fna.gz" -o ecoli_K12.fna.gz

# Decompress
gunzip ecoli_K12.fna.gz

# Inspect the header
head -n 1 ecoli_K12.fna
# Expected: >NC_000913.3 Escherichia coli str. K-12 substr. MG1655, complete genome

# Check genome size

# Expected: 4641652 (4.64 Mb)

14 序列比对原理：BWT 与 FM-index

14.1 朴素方法的瓶颈

将一条 150 bp 的 read 比对至 4.64 Mb（大肠杆菌）或 3 Gb（人类）的参考基因组，本质上是一个字符串搜索问题。

最直觉的方法是逐位扫描：将 read 与基因组每个起始位置对齐，逐一比较。对于人类基因组：

\[\text{操作次数} = G \times L = 3 \times 10^9 \times 150 = 4.5 \times 10^{11}\]

按现代 CPU 每秒执行 \(10^9\) 次操作计算，单条 read 需约 7.5 分钟。一次 WGS 实验产生约 \(10^9\) 条 reads，完成全部比对需要约 1.4 万年。

显然，我们需要截然不同的策略。

14.2 Burrows-Wheeler Transform (BWT)

BWT 最初于 1994 年由 Michael Burrows 和 David Wheeler 提出，用于数据压缩（bzip2 即基于 BWT）。2009 年，Heng Li 等人将其引入基因组比对，催生了 BWA 工具。

{{Wiki:Burrows-Wheeler_transform}}

BWT 的核心思想是对字符串进行可逆排列变换，使相同字符在变换后的序列中聚集，从而实现高效压缩和搜索。

直觉理解 BWT 变换步骤（以短字符串为例）：

对字符串 BANANA$（$ 为终止符，字典序最小）：

  1. 生成所有循环旋转（Cyclic Rotations）
  2. 将旋转结果按字典序排序，形成BWM 矩阵（Burrows-Wheeler Matrix）
  3. 取矩阵最后一列，即为 BWT 输出
原始字符串:  B A N A N A $

所有旋转（排序后）:       最后一列(BWT):
  $ B A N A N A              A
  A $ B A N A N              N
  A N A $ B A N              N
  A N A N A $ B              $
  B A N A N A $    →         B
  N A $ B A N A              A
  N A N A $ B A              A

BWT 输出: A N N $ B A A

BWT 输出（ANNBA$AA）中，A、N 等字符被聚集在一起，这一特性使其可被高效压缩至原序列的 20–30%，同时保留了对原序列的可逆映射。

14.3 FM-index：BWT 上的快速搜索

FM-index（Full-text index in Minute space） 在 BWT 基础上添加了两个辅助结构：

  • 后缀数组（Suffix Array, SA）：记录每个 BWT 旋转对应的起始位置
  • Occ 表（Occurrence Array）：记录字符在 BWT 前缀中出现的累积次数

借助 FM-index，搜索一条 read 的时间复杂度为 \(O(L)\)（仅与 read 长度有关，与基因组大小无关），内存使用仅约 2–3 GB（人类基因组），远优于朴素方法。

黄金法则：BWT 索引只需构建一次

bwa index 命令对参考基因组构建 BWT/FM-index，生成 .bwt、.pac、.ann、.amb、.sa 五个索引文件。这一过程对人类基因组约需 60–90 分钟，但只需执行一次，后续所有样本的比对均复用同一索引。将索引文件存储在共享存储路径，供团队成员共享，是计算集群上的标准实践。

14.4 BWA-MEM：Seed-and-Extend 策略

BWA-MEM 并不对整条 read 进行精确 BWT 查询，而是采用种子-延伸（Seed-and-Extend）策略：

  1. 种子发现（Seeding）：从 read 中提取若干个短的精确匹配锚点（MEM, Maximal Exact Match），利用 FM-index 在参考基因组中快速定位每个 MEM 的所有位置
  2. 链接（Chaining）：将共线性的种子链接成候选比对区域
  3. 延伸（Extension）：对候选区域进行 Smith-Waterman 动态规划，允许错配和 gap，得到最终比对结果
  4. 评分（Scoring）：选择得分最高的比对位置作为主比对（Primary Alignment），余者为次比对（Secondary/Supplementary）

这一策略使 BWA-MEM 能够： - 处理含有错配的 reads（SNP、测序错误） - 检测跨越短 Indel 的 reads - 识别嵌合比对（Chimeric Alignment），用于结构变异检测

15.1 工具安装

# Install BWA and samtools via conda
conda install -c bioconda bwa samtools

15.2 Step 1：构建 BWT 索引

cd ~/ecoli_qc

ls ecoli_K12.fna.*

# ecoli_K12.fna.pac  ecoli_K12.fna.sa

索引文件大小约为：.bwt（4.3 MB）+ .pac（1.1 MB）+ .sa（2.1 MB），共约 7.6 MB，约为原始 FASTA 文件（4.4 MB）的 1.7 倍。索引体积略大于原始序列属于正常现象：.pac 以 2-bit 编码压缩了碱基序列，但 .sa（采样后缀数组）引入了额外的辅助结构，使总体积有所增加。BWA 索引的核心价值不在于减小文件体积，而在于将每条 read 的比对复杂度从 \(O(G)\) 降至 \(O(L)\)，使整个 WGS 项目的比对时间从理论上的”万年”压缩至数小时。

15.3 Step 2：序列比对

bwa mem \
    -t 4 \
    -R "@RG\tID:SRR2584863\tSM:ecoli_K12\tLB:lib1\tPL:ILLUMINA\tPU:H73TDADXX.1" \
    ecoli_K12.fna \
    SRR2584863_1_clean.fastq.gz \
    SRR2584863_2_clean.fastq.gz \
    | samtools sort -@ 4 -o SRR2584863.sorted.bam -

samtools index SRR2584863.sorted.bam

15.3.1 Read Group（-R 参数）详解

-R 参数向 SAM/BAM 文件写入 Read Group（RG） 标签，这是后续变异检测工具（GATK、DeepVariant）的必需元数据。

RG 字段 含义 示例值
ID Read Group 唯一标识符 SRR2584863
SM 样本名称（Sample Name） ecoli_K12
LB 文库名称（Library） lib1
PL 测序平台（Platform） ILLUMINA
PU 平台单元（Platform Unit），通常为 flowcell.lane H73TDADXX.1

RG 标签不可省略的原因：

GATK 的 HaplotypeCaller 在调用变异时，依据 RG 中的 SM 字段识别样本身份，依据 LB 字段识别文库（用于判断哪些重复属于 PCR 重复而非生物学重复）。若 BAM 文件缺少 RG 标签，GATK 将直接报错退出。对于多文库或多泳道合并的样本，每个来源必须拥有独立的 RG ID。

15.4 Step 3：管道化写法（推荐）

上述命令利用 |（管道）将 bwa mem 的输出直接传递给 samtools sort，避免生成中间 SAM 文件（对人类 WGS 数据，未排序 SAM 约 200–300 GB，管道化可节省大量磁盘空间）：

bwa mem → [SAM流] → samtools sort → SRR2584863.sorted.bam

16 SAM/BAM 格式精解

16.1 格式概述

SAM（Sequence Alignment/Map） 是序列比对的标准输出格式，由 Heng Li 在开发 BWA 时同步设计。BAM 是其 BGZF 压缩的二进制版本，体积约为 SAM 的 1/4。

{{Wiki:SAM_(file_format)}}

SAM 文件由两部分构成：

@HD  VN:1.6  SO:coordinate
@SQ  SN:NC_000913.3  LN:4641652
@RG  ID:SRR2584863  SM:ecoli_K12  LB:lib1  PL:ILLUMINA
@PG  ID:bwa  PN:bwa  VN:0.7.17  CL:bwa mem ...
────────────────────── (Header，以 @ 开头)
SRR2584863.1  99  NC_000913.3  3482  60  151M  =  3631  300  TTCA...  CCFF...  NM:i:0  MD:Z:151  AS:i:151

16.2 SAM Header 字段

标签 含义
@HD 文件格式版本（VN）和排序方式（SO: coordinate/queryname/unsorted）
@SQ 参考序列信息：名称（SN）和长度（LN）
@RG Read Group（见上节）
@PG 生成该文件的程序及命令行（保留完整分析可追溯性）

16.3 SAM Alignment Record：11 个必填字段

每条 alignment 记录由制表符分隔的 11 个必填字段构成：

列号 字段名 示例值 含义
1 QNAME SRR2584863.1 Read 名称
2 FLAG 99 比对属性位标志（见下节）
3 RNAME NC_000913.3 比对至的参考序列名称
4 POS 3482 比对起始位置（1-based）
5 MAPQ 60 比对质量（Mapping Quality）
6 CIGAR 151M CIGAR 字符串（比对操作编码）
7 RNEXT = Mate read 的参考序列名（= 表示相同染色体）
8 PNEXT 3631 Mate read 的起始位置
9 TLEN 300 Template length（insert size 的符号值）
10 SEQ TTCA... Read 序列
11 QUAL CCFF... Read 质量（Phred+33 编码）

16.4 FLAG 字段：位标志解码

FLAG 字段是一个整数，其二进制表示中每一位对应一个属性。对 FLAG = 99（二进制 1100011）进行解码：

位值 含义 FLAG=99 中是否设置
1 (0x1) 该 read 属于 PE 对 ✓
2 (0x2) 所有 reads 均正确 paired ✓
4 (0x4) 该 read 未比对 ✗
8 (0x8) Mate read 未比对 ✗
16 (0x10) 该 read 反向互补比对 ✗
32 (0x20) Mate read 反向互补比对 ✓
64 (0x40) 该 read 为 Read 1 ✓
128 (0x80) 该 read 为 Read 2 ✗

快速解码 FLAG 的实用工具

命令行下可用 samtools flags 99 直接输出各位含义。在线工具 Broad Institute 的 SAM Flag 解码器（picard/explain-flags）提供图形化界面。实际工作中，最需要关注的 FLAG 值：

  • 4：未比对 read（unmapped）
  • 256：次比对（secondary alignment，同一 read 的非最优比对位置）
  • 1024：PCR 或光学重复（duplicate）
  • 2048：补充比对（supplementary，用于嵌合 read 的另一段比对）

16.5 CIGAR 字符串

CIGAR 字符串编码了 read 与参考基因组的对齐操作，是理解比对细节的关键：

操作符 含义 计入 read 长度 计入参考长度
M 匹配或错配（Match/Mismatch） ✓ ✓
I 相对参考的插入（Insertion） ✓ ✗
D 相对参考的缺失（Deletion） ✗ ✓
S 软剪切（Soft Clip，序列保留但不比对） ✓ ✗
H 硬剪切（Hard Clip，序列从 SEQ 中移除） ✗ ✗
N 跳过参考区域（RNA-seq 内含子） ✗ ✓

CIGAR 解读示例：

CIGAR: 5S 30M 2D 80M 35M
含义：
  5S  → 前5 bp 软剪切（可能是低质量末端）
  30M → 30 bp 正常比对
  2D  → 相对参考删除 2 bp（Deletion，参考基因组多出 2 bp）
  80M → 80 bp 正常比对
  35M → 35 bp 正常比对
总 read 长度: 5 + 30 + 80 + 35 = 150 bp（不含 D，因 D 不计入 read）

16.6 常用 Optional 标签

SAM 第 12 列起为可选字段，格式为 TAG:TYPE:VALUE：

标签 含义 典型值
NM:i Edit distance（错配 + Indel 总数） NM:i:2
MD:Z 错配位置编码（用于重建比对细节） MD:Z:10A5T134
AS:i 比对得分（Alignment Score） AS:i:145
XS:i 次优比对得分 XS:i:0
RG:Z Read Group ID（与 @RG ID 对应） RG:Z:SRR2584863

NM（Edit Distance）是变异检测的重要参考：正常高质量比对的 NM 应 ≤ 3；若大量 reads 具有高 NM 值，提示参考基因组选择错误或样本污染。

17.4 MAPQ（Mapping Quality）分布

Mapping Quality 与 Phred 质量评分采用相同的 \(-10 \log_{10}(P)\) 公式，但 \(P\) 表示该 read 比对位置错误的概率。

# Distribution of MAPQ values
samtools view SRR2584863.sorted.bam | awk '{print $5}' | sort -n | uniq -c

典型输出（前几行）：

  315 0
   27 1
    5 3
    8 6
   12 9
 97821 60

MAPQ = 60 是 BWA-MEM 对唯一比对的最大赋值，表示该 read 在参考基因组中只有一个最优比对位置，置信度极高。MAPQ = 0 表示多比对（multimapping），该 read 在参考基因组中存在多个同等最优的比对位置，常见于重复区域。

黄金法则：变异检测中过滤低 MAPQ reads

GATK HaplotypeCaller 默认过滤 MAPQ < 20 的 reads。手动分析时通常使用 samtools view -q 20 来过滤低置信度比对，避免重复区域的假阳性变异。

17.5 课后自测 (Post-Lesson Quiz)

  1. 一个 RNA-seq 实验使用了 75 bp 单端测序，但文库构建的 cDNA 片段长度为 200–500 bp。是否会出现 adapter read-through？请解释原因。

  2. 下载 hg38 参考基因组后发现包含 chr1_alt 等 ALT contig 文件。若将这些 ALT contigs 包含在比对参考中，会对多态性高的区域（如 HLA 位点）的比对结果产生什么影响？

  3. 某 BAM 文件的 samtools flagstat 报告显示 mapped 率仅 68%，properly paired 率仅 52%。请列举至少三种可能的原因。

  4. 解读以下 CIGAR 字符串：12S 88M 3I 47M。请计算：（a）read 总长度；（b）比对至参考基因组的长度；（c）该比对意味着相对参考基因组有怎样的结构？

  5. 为什么 BWA-MEM 对重复序列区域的 reads 赋予低 MAPQ 值（如 MAPQ=0），而不是直接丢弃这些 reads？保留低 MAPQ reads 有何意义？

  6. 在 bwa mem 命令中，若省略 -R 参数（Read Group），BWA 仍可正常完成比对并生成 BAM 文件。但在后续 GATK 变异检测步骤中会发生什么？如何补救？

参考答案 (Reference Answers)

  1. 不会出现 read-through。75 bp read 长度小于 200–500 bp 的 insert size，测序仪在读完 75 bp 后即停止，未能到达对侧 adapter。Read-through 仅在 read 长度 > insert size 时发生。该实验无需担心 adapter 污染，但若 insert size 分布出现 < 75 bp 的片段，则这部分 reads 会发生 read-through。

  2. 包含 ALT contigs 后，原本唯一比对至主染色体的 reads 可能同时比对至 ALT contigs，导致 MAPQ 降低甚至变为 0（多比对）。对于 HLA 等高变区域，这会使变异检测更加复杂；但若使用支持 ALT-aware 比对的工具（如 BWA-MEM2 + bwakit），则可正确处理 ALT contigs，提高该区域的变异检测精度。简单包含 ALT contigs 但不使用 ALT-aware 工具，可能反而降低这些区域的检测质量。

  3. 可能原因：①参考基因组版本与样本不匹配（如用 hg38 比对实际来自小鼠的数据）；②样本存在严重的跨物种污染（如微生物污染）；③ adapter trimming 未完成，大量携带 adapter 序列的 reads 无法比对；④样本实为目标基因组以外的物种（物种鉴定错误）；⑤文库构建失败，reads 为接头二聚体（adapter dimer）或人工序列。

  4. （a）read 总长度 = 12（S）+ 88（M）+ 3（I）+ 47（M）= 150 bp（S 和 I 均计入 read 长度，D 不计）。（b）比对至参考基因组的长度 = 88（M）+ 47（M）= 135 bp（S 和 I 不计入参考长度）。（c）该 read 前 12 bp 未被比对（软剪切，可能为低质量端或 adapter）；随后 88 bp 正常比对；相对参考基因组存在 3 bp 插入（Insertion，read 中多出 3 bp）；最后 47 bp 正常比对。整体提示该 read 横跨一个小型 Insertion 变异位点。

  5. 低 MAPQ reads 保留在 BAM 文件中，而非丢弃，有以下意义：①为分析人员提供数据完整性，可按需过滤；②在特定分析场景（如重复区域的 CNV 拷贝数分析、转座子研究）中，多比对 reads 本身具有分析价值；③过滤阈值因分析目的而异（变异检测通常过滤 MAPQ<20，而覆盖度分析可能选择保留）。直接丢弃会不可逆地损失信息。

  6. 缺少 RG 标签的 BAM 文件在运行 GATK HaplotypeCaller 时会报错：SAM/BAM file does not have a read group in the header，程序终止。补救方法：使用 Picard 工具的 AddOrReplaceReadGroups 命令为 BAM 文件补充 RG 信息，无需重新比对：picard AddOrReplaceReadGroups I=input.bam O=output.bam RGID=1 RGSM=sample RGLB=lib1 RGPL=ILLUMINA RGPU=unit1。

17.6 总结

✓ Adapter read-through 在 insert size < read 长度时发生，必须在比对前去除 ✓ fastp 是目前最高效的预处理工具，自动检测 adapter，内置 QC 报告 ✓ GRCh38（hg38） 是人类 WGS 新项目的标准参考基因组，含 decoy 序列；版本不可混用 ✓ BWT/FM-index 将基因组搜索复杂度从 \(O(G \times L)\) 降至 \(O(L)\)，使快速比对成为可能 ✓ BWA-MEM 采用 Seed-and-Extend 策略，-R 参数写入的 Read Group 是 GATK 的必要条件 ✓ SAM FLAG 和 CIGAR 字符串完整编码了每条 read 的比对属性 ✓ samtools flagstat 的 mapped 率 > 95% 且 properly paired 率 > 90% 是高质量比对的基准
