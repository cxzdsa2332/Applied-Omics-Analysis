12 为什么需要从头组装

12.1 参考基因组的局限性

在第2节中，我们学习了将短读序列（reads）比对至参考基因组的完整流程。这一方法的前提假设是：参考基因组已经存在，且与待研究样本足够相似。

然而，在以下场景中，这一假设会失效：

场景 问题 例子
新物种研究 无任何已知参考序列 深海未知微生物
微生物宏基因组 样本中含数百种未知物种 肠道菌群研究
病原体新变种 参考基因组差异过大 新型冠状病毒变体
高度个性化区域 群体结构变异复杂 HLA 区域、癌症基因组
参考基因组本身的构建 鸡生蛋蛋生鸡 hg38 是如何产生的？

从头组装（De novo Assembly） 正是解决上述问题的核心技术——它仅凭测序读段，不依赖任何先验参考序列，重建目标基因组。

参考基因组也是被组装出来的

第2节中使用的 E. coli K-12 参考基因组（NC_000913.3）本身也经历了从头组装的过程。1997 年，Blattner 等人首次发布 E. coli K-12 完整基因组（Science, 277:1453-1462），使用的正是本节将要介绍的 Overlap-Layout-Consensus 策略。此后经过多轮修订，该参考基因组才达到今天无缝隙（closed）的高质量状态。因此，参考比对和从头组装在研究流程中是相互补充的两种范式。

12.2 从头组装 vs 参考比对

维度 参考比对 从头组装
前提条件 需要参考基因组 无需参考
计算复杂度 较低（线性化查询） 较高（图论优化问题）
适用场景 SNP/Indel 检测、基因表达 新物种、结构变异、泛基因组
结果形式 BAM 比对文件 Contig/Scaffold FASTA
对测序深度要求 20–100X 50–200X（组装更需高深度）
对 read 长度要求 中等（PE150 足够） 越长越好（长读组装质量更高）

13 基因组组装的基本概念

13.1 组装产物的层级

基因组组装不会一次性产生完整的染色体序列，而是由短到长分层次地呈现：

Reads (150 bp)
  ↓ [重叠延伸]
Contigs (连续序列，100 bp ~ 数 Mb，无 gap)
  ↓ [配对末端距离约束]
Scaffolds (骨架，包含已知大小的 gap "NNNNN")
  ↓ [光学图谱/长读/Hi-C]
Chromosomes (完整染色体)

{{Wiki:Contig}}

关键术语：

  • Contig（重叠群）：由 reads 重叠延伸得到的连续序列，内部无未知碱基
  • Scaffold（支架）：将多个 contig 按插入片段距离排序连接，gap 以 N 填充
  • N50：将所有 contig（或 scaffold）按长度从大到小排列，累加长度达到总长度 50% 时所对应的 contig 长度

13.1.1 Scaffold 的构建：从 Contig 到骨架

Contig 是组装图中无歧义路径的直接产物，其终点正好是图中的分叉节点或死端——即序列本身不能再延伸的位置。Scaffold 构建利用配对末端（paired-end）reads 的插入片段大小约束，将彼此间距已知的两条 reads 分别比对到不同 contig 后，推断这两个 contig 的相对方向和间距，从而将它们串联为一个更长的骨架，gap 位置用等量的 N 填充：

Contig A ──────────────>         <─────────────── Contig B
                    |←── insert ~500 bp ──→|
                    ↓
Scaffold: [Contig A]NNNNN[Contig B]   (gap size = insert - 2×read_len)

{{Wiki:Scaffolding_(bioinformatics)}}

一次 scaffold 轮次后仍可能存在多段 scaffold 无法连接的情况。此时可引入更长程的距离信息逐步提升连续性：

距离信息来源 跨越距离 典型工具
Paired-end reads（PE） 300–500 bp SPAdes 内置
Mate-pair reads（MP） 2–20 kb SSPACE
光学图谱（Optical Mapping） 100 kb–1 Mb BioNano Solve
Hi-C 染色质接触图谱 全染色体 3D-DNA、SALSA
长读（PacBio / Nanopore） 10–100 kb Hifiasm、Flye

为什么 gap 用 N 填充而不是跳过？

N 的数量对应估算的 gap 大小，这一信息对下游分析至关重要：基因预测工具需要知道基因间区的真实长度；比较基因组学工具在两物种序列对齐时依赖坐标的准确性。直接拼接（不填充 N）会错误地缩短基因间距，引入系统性偏差。

13.2 N50：组装质量的核心指标

{{Wiki:N50_statistic}}

N50 计算示例：

假设组装结果包含以下 contig（按长度从大到小排列）：

contig1: 8000 bp
contig2: 6000 bp   ← 累计 14000 bp > 总长度 20000 bp 的 50%
contig3: 3000 bp
contig4: 2000 bp
contig5: 1000 bp
总长度：20000 bp

50% × 20000 = 10000 bp。累加到 contig2（14000 bp）时刚刚超过 10000 bp，故 N50 = 6000 bp。

# R function to compute N50
compute_n50 <- function(lengths) {
  lengths_sorted <- sort(lengths, decreasing = TRUE)
  total <- sum(lengths_sorted)
  cumulative <- cumsum(lengths_sorted)
  n50 <- lengths_sorted[which(cumulative >= total * 0.5)[1]]
  return(n50)
}

# Example
lengths <- c(8000, 6000, 3000, 2000, 1000)
cat("N50 =", compute_n50(lengths), "bp\n")
# N50 = 6000 bp

N50 值越大，说明组装越连续（contiguous）。高质量细菌基因组组装的 N50 通常接近其染色体长度（即接近完全封闭），而人类基因组草图 N50 可能仅有数 Mb。

14 两种经典组装算法框架

14.1 Overlap-Layout-Consensus（OLC）

OLC 是最早的系统性组装框架，由 Myers（1995）提出，广泛用于 Sanger 测序时代（read 长 500–1000 bp）和现代长读（PacBio、Nanopore）组装。

{{Wiki:Sequence_assembly}}

{{Wiki:Overlap_graph}}

14.1.1 三阶段原理

设想手中有一段撕碎的纸条，每块碎片保留了若干字符，任务是从这些碎片重建原始句子——这正是 OLC 面对的问题。

阶段一：Overlap（重叠检测）

逐对比较所有 reads，寻找一条 read 的 suffix（后缀）与另一条 read 的 prefix（前缀）高度匹配的片段。重叠关系组织为重叠图（Overlap Graph），其中节点为 reads，有向边表示重叠的方向与长度：

Read 1 suffix:  ...G A T C A
Read 2 prefix:      G A T C A G...
                    ↑↑↑↑↑  overlap = 5 bp

时间复杂度为 \(O(N^2 \times L)\)，对短读数据（数亿条 reads）计算代价巨大，现代实现通过 MinHash 等局部敏感哈希技术将其加速至近似线性。

阶段二：Layout（布局）

依据重叠图推断 reads 之间的相对位置关系，在参考坐标尚不存在的情况下构建内部一致的排列顺序。理想情况下，reads 形成一条无分支的线性路径（环形基因组则为环形路径）。数学上，这等价于在重叠图中寻找哈密顿路径（经过每个节点恰好一次的路径）——这是 NP 完全问题，需要启发式算法近似求解。

阶段三：Consensus（共识序列）

确定布局后，逐位统计覆盖该位置的所有 reads 的碱基投票，取多数碱基作为最终共识序列，同时利用覆盖深度修正测序错误：

Position p: reads 在此位置的碱基为 [A, A, C, A] → 多数表决 → consensus = A

14.1.2 模拟数据完整演示

下面用 Python 对一组模拟的短 reads 完整演示 OLC 三阶段。此处使用模拟数据仅用于说明算法逻辑。

import networkx as nx
import matplotlib.pyplot as plt

# Simulated reads derived from the unknown target sequence "ATCGGATCAGTCG"
reads = [
    "ATCGGATCA",   # positions 0-8
    "TCGGATCAG",   # positions 1-9
    "CGGATCAGT",   # positions 2-10
    "GGATCAGTC",   # positions 3-11
    "GATCAGTCG",   # positions 4-12
]


# ── Stage 1: Overlap ──────────────────────────────────────────────────────────

def suffix_prefix_overlap(read_a, read_b, min_overlap=4):
    """Return the longest suffix-prefix overlap length, or 0 if below threshold."""
    max_len = min(len(read_a), len(read_b))
    for length in range(max_len, min_overlap - 1, -1):
        if read_a.endswith(read_b[:length]):
            return length
    return 0


print("=== Stage 1: Overlap Detection ===")
overlap_matrix = {}
for i, a in enumerate(reads):
    for j, b in enumerate(reads):
        if i != j:
            ov = suffix_prefix_overlap(a, b)
            if ov > 0:
                overlap_matrix[(i, j)] = ov
                print(f"  Read {i+1} -> Read {j+1}: overlap = {ov} bp"
                      f"  ({a[-ov:]} | {b[:ov]})")


# ── Stage 2: Layout (greedy Hamiltonian path) ─────────────────────────────────

def greedy_layout(reads, overlap_matrix):
    """Extend greedily from the read with no incoming edges (leftmost read)."""
    n = len(reads)
    in_degree = {i: 0 for i in range(n)}
    for (i, j) in overlap_matrix:
        in_degree[j] += 1

    start = next(i for i in range(n) if in_degree[i] == 0)
    path, visited, current = [start], {start}, start

    while True:
        candidates = [
            (ov, j) for (i, j), ov in overlap_matrix.items()
            if i == current and j not in visited
        ]
        if not candidates:
            break
        _, nxt = max(candidates)
        path.append(nxt)
        visited.add(nxt)
        current = nxt

    return path


layout = greedy_layout(reads, overlap_matrix)
print("\n=== Stage 2: Layout ===")
print("  Read order:", " -> ".join(f"Read {i+1}" for i in layout))

positions = [0]
for k in range(len(layout) - 1):
    i, j = layout[k], layout[k + 1]
    positions.append(positions[-1] + len(reads[i]) - overlap_matrix[(i, j)])

print("\n  Positional alignment:")
for read_idx, start in zip(layout, positions):
    print(f"    Read {read_idx + 1}: {' ' * start}{reads[read_idx]}")


# ── Stage 3: Consensus ────────────────────────────────────────────────────────

def build_consensus(reads, layout, positions):
    """Majority-vote at each position across all overlapping reads."""
    total_len = max(pos + len(reads[i]) for i, pos in zip(layout, positions))
    columns = [[] for _ in range(total_len)]
    for read_idx, start in zip(layout, positions):
        for offset, base in enumerate(reads[read_idx]):
            columns[start + offset].append(base)
    consensus = "".join(max(set(col), key=col.count) for col in columns)
    return consensus, columns


consensus, columns = build_consensus(reads, layout, positions)

print("\n=== Stage 3: Consensus ===")
for pos, (col, base) in enumerate(zip(columns, consensus)):
    print(f"  Position {pos:2d}: [{', '.join(col)}] -> {base}")

print(f"\n  Assembled consensus: {consensus}")
print(f"  Expected sequence:   ATCGGATCAGTCG")
print(f"  Match: {consensus == 'ATCGGATCAGTCG'}")


def draw_overlap_graph(reads, overlap_matrix):
    print("\n=== Drawing Overlap Graph ===")

    G = nx.DiGraph()
    for i, read in enumerate(reads):
        G.add_node(i)
    for (i, j), ov in overlap_matrix.items():
        G.add_edge(i, j, weight=ov)

    n = len(reads)
    x_gap = 3.5
    pos = {i: (i * x_gap, 0.0) for i in range(n)}

    # Figure height must accommodate upward arc peaks.
    # With rad=0.3 and x_gap=3.5, arc peak ≈ rad * x_gap / 2 ≈ 0.525 data units;
    # adding top/bottom padding keeps every arc and label fully visible.
    fig, ax = plt.subplots(figsize=(15, 5))
    ax.set_title("OLC Assembly: Directed Overlap Graph", fontsize=13, fontweight='bold')

    x_vals = [p[0] for p in pos.values()]
    x_pad, y_pad = 1.8, 1.2
    ax.set_xlim(min(x_vals) - x_pad, max(x_vals) + x_pad)
    ax.set_ylim(-y_pad, y_pad)

    node_size = 2000
    node_labels = {i: f"Read {i+1}\n({reads[i]})" for i in range(n)}

    nx.draw_networkx_nodes(
        G, pos, ax=ax,
        node_color='#add8e6', node_size=node_size,
        edgecolors='#2c6e9e', linewidths=1.8,
    )
    nx.draw_networkx_labels(
        G, pos, labels=node_labels, ax=ax,
        font_size=8, font_family="monospace",
    )

    # Arrows emerge from node edge (not center) via min_source/target_margin.
    # rad=0.3 gives a clear upward arc; arrowhead is large enough to read direction.
    nx.draw_networkx_edges(
        G, pos, ax=ax,
        arrowstyle='-|>',
        arrowsize=25,
        edge_color='steelblue',
        width=2.0,
        connectionstyle='arc3,rad=0.3',
        min_source_margin=36,
        min_target_margin=36,
    )

    # Weight = overlap length in bp (correct: suffix_prefix_overlap returns bp count).
    # label_pos=0.5 places the label at the geometric midpoint of the arc.
    edge_labels = {(i, j): f"{d['weight']} bp" for i, j, d in G.edges(data=True)}
    nx.draw_networkx_edge_labels(
        G, pos, edge_labels=edge_labels, ax=ax,
        font_color='darkred', font_size=9, font_weight='bold',
        label_pos=0.5,
        bbox=dict(boxstyle='round,pad=0.25', fc='white', ec='lightgray', alpha=0.9),
    )

    ax.axis('off')
    plt.tight_layout()
    plt.savefig("olc_overlap_graph.png", dpi=150, bbox_inches='tight')
    print("  Overlap graph saved to: olc_overlap_graph.png")
    plt.show()


# 调用绘图函数
draw_overlap_graph(reads, overlap_matrix)

观察输出中的覆盖深度分布

位置 4–8 被 5 条 reads 覆盖（最大深度），而位置 0 和 12 仅被 1 条 read 覆盖（末端单读）。这一覆盖深度向两端递减的模式是所有线性序列 OLC 组装的普遍特征。真实数据中，末端低覆盖区域的 consensus 置信度较低，通常需要额外验证。

14.1.3 OLC 的固有挑战与长读长的复兴

OLC 在实践中面临的三大难题

  1. 计算复杂度：全对比较时间复杂度为 \(O(N^2 \times L)\)，对数亿条短读不可行。现代实现（Canu、Hifiasm）通过 MinHash 近似哈希将实际复杂度降至 \(O(N \log N)\)。

  2. 重复序列的歧义性：若基因组含有长于 read 的重复序列，重叠图中会出现分叉，导致无法确定重复区域的真实排列。这是短读长 OLC 的根本局限。

  3. 测序错误的干扰：错误会使真实相邻的 reads 重叠不完全匹配，或使不相邻的 reads 因共享低复杂度序列而被错误链接。Nanopore 原始读长错误率约 5–15%，需要在 Overlap 阶段引入误差容忍机制。

OLC 随长读长测序的复兴

OLC 在 Illumina 短读时代（2008–2015）被基于 de Bruijn 图的方法取代，主因是 \(O(N^2)\) 计算代价无法承受。然而，随着 PacBio HiFi（read 长 15–20 kb，准确率 > 99.9%）和 Oxford Nanopore（read 长可达 >100 kb）的普及，read 本身已能跨越大多数重复区域，OLC 重新成为高质量基因组组装的首选框架。目前最先进的组装工具 Hifiasm 即以 OLC 为核心，可将人类基因组组装至几乎染色体级别的连续性。

优势：对长读（> 1 kb）效果优异，天然可跨越中等重复区域。 局限：\(O(N^2)\) 的原始 Overlap 计算对短读数据（> 10 亿条 reads）不可行，需近似算法辅助。

14.2 De Bruijn 图（DBG）

De Bruijn 图方法由 Pevzner 等（2001）引入基因组组装，是目前几乎所有短读组装工具（SPAdes、Velvet、MEGAHIT）的核心算法框架。其关键创新是将 reads 分解为 k-mer，从而将 \(O(n^2)\) 的两两比较问题转化为 \(O(n)\) 的图构建问题。

15 De Bruijn 图算法详解

15.1 从 k-mer 到图：核心思想

k-mer 是长度为 \(k\) 的碱基子串。对于任意一条 read，可以提取 \((L - k + 1)\) 个 k-mer，其中 \(L\) 是 read 长度。

在 De Bruijn 图中： - 节点是 \((k-1)\)-mer - 有向边是 \(k\)-mer（每个 k-mer 连接其左侧 \((k-1)\)-mer 到右侧 \((k-1)\)-mer）

这个转化将基因组组装问题从哈密顿路径问题（NP 完全）转化为欧拉路径问题（多项式时间可解）。

欧拉路径 vs 哈密顿路径

  • 哈密顿路径（Hamilton Path）：经过图中每个节点恰好一次。判断是否存在哈密顿路径是 NP 完全问题，无已知多项式时间算法。
  • 欧拉路径（Euler Path）：经过图中每条边恰好一次。判断欧拉路径的充要条件（Euler 1736）：图中恰好有 0 个或 2 个奇数度节点。可在 \(O(E)\) 时间内用 Hierholzer 算法求解。

OLC 策略需要每个 read 只被访问一次（哈密顿），而 DBG 策略将 reads 分解为 k-mer 后，每个 k-mer 作为图中的一条边，只被访问一次（欧拉）。这一数学转换是 DBG 高效的根本原因。

# ── Step 1: Extract unique k-mers from a list of reads ──────────────────────
get_kmers_from_reads <- function(reads, k) {
  all_kmers <- c()
  for (seq in reads) {
    n <- nchar(seq)
    if (k > n) {
      warning(paste("Sequence too short, skipped:", seq))
      next
    }
    for (i in 1:(n - k + 1)) {
      all_kmers <- c(all_kmers, substr(seq, i, i + k - 1))
    }
  }
  unique(all_kmers)          # deduplicate: each k-mer = one edge in graph
}

# ── Step 2: Build De Bruijn graph as an adjacency list ──────────────────────
build_de_bruijn_graph <- function(kmers) {
  graph <- list()
  for (kmer in kmers) {
    left_node  <- substr(kmer, 1, nchar(kmer) - 1)   # (k-1)-mer prefix
    right_node <- substr(kmer, 2, nchar(kmer))         # (k-1)-mer suffix
    graph[[left_node]] <- c(graph[[left_node]], right_node)
  }
  graph
}

# ── Step 3: Compute in-degree and out-degree for every node ─────────────────
get_degrees <- function(graph) {
  out_deg    <- sapply(graph, length)
  in_deg     <- table(unlist(graph))
  all_nodes  <- unique(c(names(graph), names(in_deg)))
  degrees    <- data.frame(node  = all_nodes,
                           in_d  = 0,
                           out_d = 0,
                           stringsAsFactors = FALSE)
  rownames(degrees) <- all_nodes
  if (length(out_deg) > 0) degrees[names(out_deg), "out_d"] <- as.numeric(out_deg)
  if (length(in_deg)  > 0) degrees[names(in_deg),  "in_d"]  <- as.numeric(in_deg)
  degrees
}

# ── Step 4: Identify Eulerian path start node ───────────────────────────────
# Start node satisfies: out_degree - in_degree == 1
get_start_node <- function(degrees) {
  start_nodes <- degrees[degrees$out_d - degrees$in_d == 1, "node"]
  if (length(start_nodes) == 0) {
    start_nodes <- degrees[degrees$out_d > 0, "node"][1]  # fallback for Eulerian circuit
  }
  start_nodes[1]
}

# ── Step 5: Hierholzer's algorithm for Eulerian path ────────────────────────
find_eulerian_path <- function(graph, start_node) {
  stack <- c(start_node)
  path  <- c()
  while (length(stack) > 0) {
    curr <- stack[length(stack)]
    if (!is.null(graph[[curr]]) && length(graph[[curr]]) > 0) {
      next_node        <- graph[[curr]][1]     # take first available edge
      graph[[curr]]    <- graph[[curr]][-1]    # remove used edge
      stack            <- c(stack, next_node)
    } else {
      path  <- c(path, curr)                   # dead end: add to path
      stack <- stack[-length(stack)]
    }
  }
  rev(path)
}

# ── Step 6: Reconstruct sequence from node path ─────────────────────────────
# Each consecutive pair shares a (k-2)-mer overlap; append only the last base
reconstruct_sequence <- function(path) {
  seq <- path[1]
  for (i in 2:length(path))
    seq <- paste0(seq, substr(path[i], nchar(path[i]), nchar(path[i])))
  seq
}

# ── k-mer extraction (retaining counts for coverage-based filtering) ─────────
get_kmers_from_reads <- function(reads, k) {
  kmer_vec <- character(0)
  for (seq in reads) {
    n <- nchar(seq)
    if (n < k) {
      warning("Sequence too short, skipped: ", seq)
      next
    }
    kmer_vec <- c(kmer_vec, substring(seq, 1:(n - k + 1), k:n))
  }
  table(kmer_vec)   # named frequency table; count = coverage of that edge
}

# ── Build De Bruijn graph as igraph object ───────────────────────────────────
build_de_bruijn_graph <- function(kmer_counts, min_count = 1) {
  kmers <- names(kmer_counts)[kmer_counts >= min_count]
  if (length(kmers) == 0) stop("No k-mers meet the minimum coverage threshold")
  k     <- nchar(kmers[1])
  edges <- data.frame(
    from  = substr(kmers, 1, k - 1),
    to    = substr(kmers, 2, k),
    kmer  = kmers,
    count = as.integer(kmer_counts[kmers]),
    stringsAsFactors = FALSE
  )
  g         <- graph_from_data_frame(edges, directed = TRUE)
  E(g)$label <- edges$kmer
  E(g)$count <- edges$count
  g
}

# ── Graph structure diagnostics ──────────────────────────────────────────────
diagnose_graph <- function(g) {
  cat("-- Graph Diagnostics ---------------------\n")
  cat("  Nodes:", vcount(g), " | Edges:", ecount(g), "\n")
  deg_in  <- degree(g, mode = "in")
  deg_out <- degree(g, mode = "out")
  tips    <- V(g)[deg_in == 0 | deg_out == 0]
  junc    <- V(g)[deg_in > 1  | deg_out > 1]
  cat("  Tips (dead ends):", length(tips), "\n")
  cat("  Junction nodes  :", length(junc),  "\n")
  bubbles <- detect_bubbles(g)
  cat("  Putative bubbles:", length(bubbles), "\n")
  wc <- components(as.undirected(g))
  cat("  Weakly connected components:", wc$no, "\n")
  cat("------------------------------------------\n")
  list(tips = tips, junctions = junc, bubbles = bubbles)
}

# ── Bubble detection ─────────────────────────────────────────────────────────
# A bubble exists when two outgoing branches from the same node reconverge
detect_bubbles <- function(g, max_len = 5) {
  bubbles       <- list()
  branch_nodes  <- V(g)[degree(g, mode = "out") >= 2]
  for (v in branch_nodes) {
    succs <- neighbors(g, v, mode = "out")
    if (length(succs) < 2) next
    for (i in 1:(length(succs) - 1)) {
      for (j in (i + 1):length(succs)) {
        reach_i    <- igraph::ego(g, order = max_len, nodes = succs[i], mode = "out")[[1]]
        reach_j    <- igraph::ego(g, order = max_len, nodes = succs[j], mode = "out")[[1]]
        merge_nodes <- intersect(reach_i, reach_j)
        if (length(merge_nodes) > 0) {
          bubbles <- c(bubbles, list(list(
            source  = V(g)$name[v],
            branch1 = V(g)$name[succs[i]],
            branch2 = V(g)$name[succs[j]],
            merge   = V(g)$name[merge_nodes[1]]
          )))
        }
      }
    }
  }
  bubbles
}

# ── Tip clipping: iteratively remove dead-end chains ────────────────────────
remove_tips <- function(g, max_tip_len = 2) {
  repeat {
    deg_in  <- degree(g, mode = "in")
    deg_out <- degree(g, mode = "out")
    tips    <- V(g)[(deg_in == 0 & deg_out <= max_tip_len) |
                    (deg_out == 0 & deg_in  <= max_tip_len)]
    if (length(tips) == 0) break
    g <- delete_vertices(g, tips)
  }
  g
}

# ── Bubble popping: keep high-coverage path, discard low-coverage branch ─────
pop_bubbles <- function(g, bubbles) {
  edges_to_delete <- integer(0)
  for (b in bubbles) {
    e1 <- E(g)[.from(b$source) & .to(b$branch1)]
    e2 <- E(g)[.from(b$source) & .to(b$branch2)]
    if (length(e1) == 0 || length(e2) == 0) next
    # Remove the lower-coverage branch
    low_edge         <- if (E(g)$count[e1] >= E(g)$count[e2]) e2 else e1
    edges_to_delete  <- c(edges_to_delete, low_edge)
  }
  if (length(edges_to_delete) > 0)
    g <- delete_edges(g, unique(edges_to_delete))
  g
}

# ── Low-coverage edge filtering ──────────────────────────────────────────────
filter_low_coverage <- function(g, min_count = 2) {
  bad_edges <- E(g)[E(g)$count < min_count]
  if (length(bad_edges) > 0) g <- delete_edges(g, bad_edges)
  delete_vertices(g, V(g)[degree(g) == 0])   # remove isolated nodes
}

# ── Hierholzer's Eulerian path (igraph version) ──────────────────────────────
find_eulerian_path <- function(g) {
  adj     <- lapply(V(g)$name, function(v) neighbors(g, v, mode = "out")$name)
  names(adj) <- V(g)$name
  deg_in  <- degree(g, mode = "in")
  deg_out <- degree(g, mode = "out")
  diff    <- deg_out - deg_in
  starts  <- names(diff)[diff == 1]
  start   <- if (length(starts) > 0) starts[1] else names(deg_out)[deg_out > 0][1]
  if (is.na(start)) return(character(0))
  stack <- start
  path  <- character(0)
  while (length(stack) > 0) {
    curr <- tail(stack, 1)
    if (length(adj[[curr]]) > 0) {
      nxt          <- adj[[curr]][1]
      adj[[curr]]  <- adj[[curr]][-1]
      stack        <- c(stack, nxt)
    } else {
      path  <- c(path, curr)
      stack <- stack[-length(stack)]
    }
  }
  rev(path)
}

reconstruct_sequence <- function(path) {
  if (length(path) < 2) return(paste(path, collapse = ""))
  seq <- path[1]
  for (i in 2:length(path))
    seq <- paste0(seq, substr(path[i], nchar(path[i]), nchar(path[i])))
  seq
}

# ── Assembly evaluation ──────────────────────────────────────────────────────
edit_distance <- function(a, b) {
  a <- strsplit(a, "")[[1]]; b <- strsplit(b, "")[[1]]
  m <- length(a);             n <- length(b)
  d <- matrix(0L, m + 1, n + 1)
  d[, 1] <- 0:m;             d[1, ] <- 0:n
  for (i in seq_len(m))
    for (j in seq_len(n))
      d[i+1, j+1] <- if (a[i] == b[j]) d[i, j] else
        1L + min(d[i, j], d[i+1, j], d[i, j+1])
  d[m+1, n+1]
}

evaluate_assembly <- function(contig, ref) {
  len_c   <- nchar(contig);   len_r <- nchar(ref)
  ed      <- edit_distance(contig, ref)
  pct_id  <- round(100 * (1 - ed / max(len_c, len_r)), 2)
  list(length       = len_c,
       ref_length   = len_r,
       edit_dist    = ed,
       pct_identity = pct_id,
       length_ratio = round(100 * len_c / len_r, 2))
}

  cat("\nNo single Eulerian path: assembling per component\n")
  contigs <- character(0)
  comps   <- components(as.undirected(g_clean))
  for (ci in seq_len(comps$no)) {
    vnodes <- names(comps$membership[comps$membership == ci])
    g_sub  <- induced_subgraph(g_clean, vnodes)
    p_sub  <- find_eulerian_path(g_sub)
    if (length(p_sub) > 1)
      contigs <- c(contigs, reconstruct_sequence(p_sub))
  }
}

# Step 6: Evaluation
cat("\n==================================================\n")
cat("  Assembly Evaluation vs. Reference               \n")
cat("==================================================\n")
cat("Reference :", ref_seq, "\n\n")

for (i in seq_along(contigs)) {
  ev <- evaluate_assembly(contigs[i], ref_seq)
  cat(sprintf("Contig %d  : %s\n", i, contigs[i]))
  cat(sprintf("  Length        : %d bp  (reference: %d bp)\n", ev$length, ev$ref_length))
  cat(sprintf("  Length ratio  : %.1f%%\n",  ev$length_ratio))
  cat(sprintf("  Edit distance : %d\n",       ev$edit_dist))
  cat(sprintf("  %% Identity    : %.2f%%\n",  ev$pct_identity))
  verdict <- if (ev$pct_identity >= 100) "PERFECT" else
             if (ev$pct_identity >= 90)  "GOOD (minor errors)" else "POOR"
  cat(sprintf("  Verdict       : %s\n\n",     verdict))
}

SNP 气泡的处理策略

本例中，Read 6（TCGATCGTTC）携带一个 A→T SNP，在图中产生了一个气泡。气泡弹出（bubble popping）策略是：保留覆盖度更高的路径（原始序列，覆盖×2），丢弃低覆盖路径（SNP 路径，覆盖×1）。这实际上是测序深度不足时组装工具丢弃真实变异的机制之一——真实 SNP 在杂合子样本中覆盖度约为总深度的 50%，而测序错误通常覆盖度 < 5%。SPAdes 等工具通过更精细的覆盖度模型区分二者。

重复序列导致 Cleaned Graph 出现闭环：这是正常现象

细心的读者可能会注意到，在经过气泡弹出和尾端修剪之后，Cleaned Graph 中仍然存在一个闭合的环状结构（诊断输出显示 Tips: 0, Junction nodes: 0, Putative bubbles: 0，但图形可视化呈现为环）。这并非代码错误，而是由参考序列本身的重复结构决定的。

本例中参考序列为 ATCGATCGAATCGATCG，其中 ATCGATCG 在首尾各出现一次，形成长度为 8 bp 的重复片段。当 \(k=6\) 时，图的节点为 \((k-1)=5\)-mer；只要重复长度 \(\geq k-1\)，de Bruijn 图就必然在重复区域引入环路——两次经过同一组节点的路径在图中合并为同一条边，从而形成闭环。气泡弹出等清理操作针对的是错误引入的歧义，而非序列本身固有的重复结构，因此无法也不应消除这类环。

此闭环的直接后果是图中不存在唯一的欧拉路径（起点和终点无法同时满足入度等于出度的条件），代码因此进入”逐连通分量分别重建”的分支。在本例的理想条件下（无测序错误、足够覆盖度），最终仍可重建出与参考序列完全一致的 contig；但在真实数据中，重复区域往往导致组装断裂，形成多条较短的 contigs，这正是所有短读组装工具面临的核心挑战。

提高 \(k\) 值可以在一定程度上打断由短重复引起的环，但对于长度远超 \(k\) 的重复（如转座子、rRNA 操纵子），单纯调整 \(k\) 无效；SPAdes 等工具需借助配对末端距离约束或长读跨越重复区域才能将其解析。

17 组装前：k-mer 频谱分析与基因组特征估算

在运行任何组装工具之前，先对测序数据进行 k-mer 频谱分析是标准的工业流程步骤。这一分析以 \(O(N)\) 的时间成本提供三项关键信息：基因组大小估算、测序深度确认、以及杂合度与重复序列比例的预判——这些信息直接影响后续组装参数的选择。

{{Wiki:K-mer}}

17.1 k-mer 频谱的解读原理

对全部 reads 进行 k-mer 计数后，以 k-mer 出现频次为横轴、具有该频次的不同 k-mer 数量为纵轴绘制频谱图。在无误差的理想情况下，单倍体基因组的频谱应呈现单峰分布，峰值位置对应测序深度 \(\lambda\)；而双倍体基因组（含杂合位点）会出现双峰，第一个峰在 \(\lambda/2\) 处（杂合区段，两条单倍型各贡献一半深度），第二个峰在 \(\lambda\) 处（纯合区段）。

k-mer 频谱示意（双倍体，覆盖度 ~30X）：

  k-mer 数量
  ↑
  │        ← 测序错误（极低频，k=1 处高峰）
  │    ╭─╮
  │    │ │  ← 杂合峰（~15X，杂合 SNP 区域）
  │    │ │        ╭──╮
  │    │ │        │  │  ← 纯合峰（~30X，主体基因组）
  │    │ │        │  │
  └────┴─┴────────┴──┴──────→ k-mer 频次
       1  15      30

基因组大小估算公式：

\[\text{Genome size} \approx \frac{\text{Total k-mers} - \text{Error k-mers}}{\lambda}\]

其中 \(\lambda\) 为纯合峰位置，Error k-mers 为频次 = 1 的极低频 k-mer 总数（来源于测序错误）。

18.1 SPAdes 简介

SPAdes（St. Petersburg genome assembler） 是目前最广泛使用的细菌和宏基因组短读组装工具，由俄罗斯圣彼得堡大学开发。其核心创新包括：

  • 多 k-mer 策略：同时使用多个 k 值（如 21, 33, 55）构建 De Bruijn 图，取各自的最优结果合并，弥补单一 k 值的局限
  • BayesHammer 错误校正：在组装前对 reads 进行贝叶斯错误校正，降低图中噪声边的数量
  • 配对末端约束：利用 PE reads 的距离信息辅助解析重复区域和气泡

“misassembly”（组装错误）是最严重的质量问题

N50 偏低仅代表组装不连续，不影响已组装区域的准确性；而 misassembly（错拼）意味着两段本应相距甚远的序列被错误拼接在一起，会导致下游分析（基因预测、变异检测）产生系统性错误。因此，评估组装质量时应优先关注 misassembly 数量，其次才是 N50。

  5. 在气泡弹出（bubble popping）策略中，程序保留覆盖度高的路径，删除覆盖度低的路径。请思考：在对二倍体生物（如人类）进行从头组装时，这一策略会产生什么问题？现代工具如何解决这一问题？

  6. 比较参考基因组比对和从头组装两种策略：对于一个已知参考基因组的物种，在什么情况下仍然需要进行从头组装而不是直接使用参考基因组比对？请给出至少两个具体场景。

参考答案 (Reference Answers)

  1. 覆盖深度 = (6,000,000 × 150) / 5,000,000 = 180X。该深度（180X）远超从头组装的推荐最低深度（50–100X），属于高覆盖测序，足以支持高质量组装。但需注意 180X 不一定优于 60X，过高覆盖有时会引入更多测序错误的累积，现代工具（如 SPAdes）对超高覆盖数据会自动进行降采样。

  2. OLC（哈密顿路径）要求每个读段（节点）恰好被经过一次，是 NP 完全问题，不存在已知多项式时间算法。DBG 将读段分解为 k-mer 后，每个 k-mer 作为图的边，组装问题转化为每条边恰好经过一次的欧拉路径问题。欧拉路径可在 O(E) 时间内用 Hierholzer 算法求解（E 为边数，即 k-mer 数量）。这一转化使得处理数十亿条短读 reads 成为计算可行的任务。

  3. (a) k 过小（如 11）：大量 k-mer 在基因组中出现多次（重复序列），图中产生大量分叉节点，路径无法唯一确定，组装极度碎片化。(b) k 过大（如 101）：每个 k-mer 出现次数接近 1，低深度区域（覆盖波动）导致图断裂，contigs 变短；同时内存占用急剧增加。(c) SPAdes 使用多个 k 值：小 k 值在低覆盖区域保持连续性，大 k 值有效解析重复区域，合并各 k 值的最优区域后得到更完整的组装结果。

  4. N50 = 45,000 bp 相对于 E. coli 4.64 Mb 基因组偏低，说明组装仍较碎片化（理想情况 N50 应接近 1–2 Mb）。总长度 4.61 Mb 接近参考基因组（误差 < 1%），GC 含量正常。最值得关注的问题是 misassemblies = 8，这意味着 8 处大片段序列被错误拼接，可能影响基因预测和基因组结构分析的准确性。建议检查覆盖度是否均匀，考虑补充长读测序数据来解析困难区域。

  5. 对于二倍体生物，同一位点的两个等位基因序列略有不同，在 De Bruijn 图中会产生气泡。单纯按覆盖度保留高覆盖路径（约 50% 深度）、删除低覆盖路径（约 50% 深度），会将一个等位基因丢弃，产生单倍体化（haplotype collapse）的组装结果。现代单倍型分辨组装工具（如 hifiasm、FALCON-Unzip）通过保留两条分叉路径、分别延伸，尝试生成分相（phased）的双倍体组装，即同时输出两套单倍型序列。

  6. 至少两个场景：①研究大规模结构变异（SV）：插入/缺失片段 > 1 kb 的 SV 在比对策略中难以精确重建，从头组装可直接揭示插入的序列内容（如转座子插入）。②癌症基因组学：肿瘤基因组常发生大片段重排、融合基因、新插入序列，这些变异在比对至正常参考基因组时容易遗漏，肿瘤样本从头组装（或局部重组装）可以发现新型序列内容。

19.4 总结

✓ 从头组装 是无参考基因组场景下重建基因组的核心技术，也是参考基因组本身产生的方式 ✓ De Bruijn 图 将 reads 分解为 k-mer，把 NP 完全的哈密顿路径问题转化为多项式时间可解的欧拉路径问题 ✓ 气泡（Bubble） 来源于测序错误或真实 SNP，通过覆盖度比较弹出；尖刺（Tip） 来源于低质量 read 末端，通过迭代截除清理 ✓ N50 是衡量组装连续性的核心指标；misassembly 数量 是衡量组装准确性的最重要指标 ✓ SPAdes 通过多 k 值策略和错误校正，是目前细菌和宏基因组短读组装的行业标准 ✓ QUAST + BUSCO 组合评估提供了有参考和无参考两种场景下的完整质量图像 ✓ 覆盖深度 ≥ 50X 是高质量从头组装的基本要求，低深度（< 20X）会导致严重的图断裂和碎片化
