12 为什么需要从头组装

12.1 参考基因组的局限性

在第2节中，我们学习了将短读序列（reads）比对至参考基因组的完整流程。这一方法的前提假设是：参考基因组已经存在，且与待研究样本足够相似。

然而，在以下场景中，这一假设会失效：

场景 问题 例子
新物种研究 无任何已知参考序列 深海未知微生物
微生物宏基因组 样本中含数百种未知物种 肠道菌群研究
病原体新变种 参考基因组差异过大 新型冠状病毒变体
高度个性化区域 群体结构变异复杂 HLA 区域、癌症基因组
参考基因组本身的构建 鸡生蛋蛋生鸡 hg38 是如何产生的？

从头组装（De novo Assembly） 正是解决上述问题的核心技术——它仅凭测序读段，不依赖任何先验参考序列，重建目标基因组。

参考基因组也是被组装出来的

第2节中使用的 E. coli K-12 参考基因组（NC_000913.3）本身也经历了从头组装的过程。1997 年，Blattner 等人首次发布 E. coli K-12 完整基因组（Science, 277:1453-1462），使用的正是本节将要介绍的 Overlap-Layout-Consensus 策略。此后经过多轮修订，该参考基因组才达到今天无缝隙（closed）的高质量状态。因此，参考比对和从头组装在研究流程中是相互补充的两种范式。

12.2 从头组装 vs 参考比对

维度 参考比对 从头组装
前提条件 需要参考基因组 无需参考
计算复杂度 较低（线性化查询） 较高（图论优化问题）
适用场景 SNP/Indel 检测、基因表达 新物种、结构变异、泛基因组
结果形式 BAM 比对文件 Contig/Scaffold FASTA
对测序深度要求 20–100X 50–200X（组装更需高深度）
对 read 长度要求 中等（PE150 足够） 越长越好（长读组装质量更高）

13 基因组组装的基本概念

13.1 组装产物的层级

基因组组装不会一次性产生完整的染色体序列，而是由短到长分层次地呈现：

Reads (150 bp)
  ↓ [重叠延伸]
Contigs (连续序列，100 bp ~ 数 Mb，无 gap)
  ↓ [配对末端距离约束]
Scaffolds (骨架，包含已知大小的 gap "NNNNN")
  ↓ [光学图谱/长读/Hi-C]
Chromosomes (完整染色体)

{{Wiki:Contig}}

关键术语：

  • Contig（重叠群）：由 reads 重叠延伸得到的连续序列，内部无未知碱基
  • Scaffold（支架）：将多个 contig 按插入片段距离排序连接，gap 以 N 填充
  • N50：将所有 contig（或 scaffold）按长度从大到小排列，累加长度达到总长度 50% 时所对应的 contig 长度

13.2 N50：组装质量的核心指标

{{Wiki:N50_statistic}}

N50 计算示例：

假设组装结果包含以下 contig（按长度从大到小排列）：

contig1: 8000 bp
contig2: 6000 bp   ← 累计 14000 bp > 总长度 20000 bp 的 50%
contig3: 3000 bp
contig4: 2000 bp
contig5: 1000 bp
总长度：20000 bp

50% × 20000 = 10000 bp。累加到 contig2（14000 bp）时刚刚超过 10000 bp，故 N50 = 6000 bp。

# R function to compute N50
compute_n50 <- function(lengths) {
  lengths_sorted <- sort(lengths, decreasing = TRUE)
  total <- sum(lengths_sorted)
  cumulative <- cumsum(lengths_sorted)
  n50 <- lengths_sorted[which(cumulative >= total * 0.5)[1]]
  return(n50)
}

# Example
lengths <- c(8000, 6000, 3000, 2000, 1000)
cat("N50 =", compute_n50(lengths), "bp\n")
# N50 = 6000 bp

N50 值越大，说明组装越连续（contiguous）。高质量细菌基因组组装的 N50 通常接近其染色体长度（即接近完全封闭），而人类基因组草图 N50 可能仅有数 Mb。

14 两种经典组装算法框架

14.1 Overlap-Layout-Consensus（OLC）

OLC 是最早的系统性组装框架，由 Myers（1995）提出，广泛用于 Sanger 测序时代（read 长 500–1000 bp）和现代长读（PacBio、Nanopore）组装。

{{Wiki:Sequence_assembly}}

三个阶段：

  1. Overlap（重叠检测）：寻找所有 read 对之间的前后缀重叠，构建重叠矩阵。时间复杂度 \(O(n^2)\)，对短读数据（数亿条 reads）计算代价巨大。

  2. Layout（布局）：根据重叠关系构建有向图，寻找哈密顿路径（经过每个节点恰好一次的路径）。哈密顿路径问题是 NP 完全问题，需要启发式算法近似求解。

  3. Consensus（共识）：将多条 reads 在同一位置的碱基进行多数表决，生成最终的 contig 序列。

优势：对长读（> 1 kb）效果优异，可跨越重复区域。 局限：\(O(n^2)\) 的 Overlap 计算对短读数据（>10 亿条 reads）不可行。

14.2 De Bruijn 图（DBG）

De Bruijn 图方法由 Pevzner 等（2001）引入基因组组装，是目前几乎所有短读组装工具（SPAdes、Velvet、MEGAHIT）的核心算法框架。其关键创新是将 reads 分解为 k-mer，从而将 \(O(n^2)\) 的两两比较问题转化为 \(O(n)\) 的图构建问题。

{{Wiki:De_Bruijn_graph}}

15 De Bruijn 图算法详解

15.1 从 k-mer 到图：核心思想

k-mer 是长度为 \(k\) 的碱基子串。对于任意一条 read，可以提取 \((L - k + 1)\) 个 k-mer，其中 \(L\) 是 read 长度。

在 De Bruijn 图中： - 节点是 \((k-1)\)-mer - 有向边是 \(k\)-mer（每个 k-mer 连接其左侧 \((k-1)\)-mer 到右侧 \((k-1)\)-mer）

这个转化将基因组组装问题从哈密顿路径问题（NP 完全）转化为欧拉路径问题（多项式时间可解）。

欧拉路径 vs 哈密顿路径

  • 哈密顿路径（Hamilton Path）：经过图中每个节点恰好一次。判断是否存在哈密顿路径是 NP 完全问题，无已知多项式时间算法。
  • 欧拉路径（Euler Path）：经过图中每条边恰好一次。判断欧拉路径的充要条件（Euler 1736）：图中恰好有 0 个或 2 个奇数度节点。可在 \(O(E)\) 时间内用 Hierholzer 算法求解。

OLC 策略需要每个 read 只被访问一次（哈密顿），而 DBG 策略将 reads 分解为 k-mer 后，每个 k-mer 作为图中的一条边，只被访问一次（欧拉）。这一数学转换是 DBG 高效的根本原因。

15.2 手工推导：三条 Reads 的组装

以三条真实 reads 为例，逐步演示 De Bruijn 图的构建过程（\(k=4\)）：

读段 (Reads):
  Read 1: GATTACA
  Read 2: TACAGAT
  Read 3: AGATCCA

Step 1：提取所有 4-mers（去重后作为图的边）

Read 4-mers
GATTACA GATT, ATTA, TTAC, TACA
TACAGAT TACA, ACAG, CAGA, AGAT
AGATCCA AGAT, GATC, ATCC, TCCA

去重后唯一 4-mers（图的边）：GATT, ATTA, TTAC, TACA, ACAG, CAGA, AGAT, GATC, ATCC, TCCA

Step 2：每个 4-mer 产生一条有向边

边（4-mer） 左节点（3-mer 前缀） 右节点（3-mer 后缀）
GATT GAT ATT
ATTA ATT TTA
TTAC TTA TAC
TACA TAC ACA
ACAG ACA CAG
CAGA CAG AGA
AGAT AGA GAT
GATC GAT ATC
ATCC ATC TCC
TCCA TCC CCA

Step 3：确定欧拉路径起点

计算每个节点的入度与出度之差：起点节点满足 out_degree - in_degree = 1。

Step 4：Hierholzer 算法遍历图，重建序列

欧拉路径经过所有边恰好一次，对应的节点序列去重后拼接成最终 contig：

GAT → ATT → TTA → TAC → ACA → CAG → AGA → GAT → ATC → TCC → CCA
重建序列：G A T T A C A G A T C C A = "GATTACAGATCCA"

验证：读段 1（GATTACA）+ 读段 2 中段（CAGAT）+ 读段 3 后段（TCCA）完整覆盖了重建序列，结果正确。

16 R 代码实践：De Bruijn 图组装

16.1 第一部分：最小实现（不依赖外部包）

以下代码对上述三条 reads 进行组装，完整实现 De Bruijn 图的每个步骤。此版本无需安装任何依赖，适合直接在课堂运行理解算法逻辑。

rm(list = ls())

# ── Step 1: Extract unique k-mers from a list of reads ──────────────────────
get_kmers_from_reads <- function(reads, k) {
  all_kmers <- c()
  for (seq in reads) {
    n <- nchar(seq)
    if (k > n) {
      warning(paste("Sequence too short, skipped:", seq))
      next
    }
    for (i in 1:(n - k + 1)) {
      all_kmers <- c(all_kmers, substr(seq, i, i + k - 1))
    }
  }
  unique(all_kmers)          # deduplicate: each k-mer = one edge in graph
}

# ── Step 2: Build De Bruijn graph as an adjacency list ──────────────────────
build_de_bruijn_graph <- function(kmers) {
  graph <- list()
  for (kmer in kmers) {
    left_node  <- substr(kmer, 1, nchar(kmer) - 1)   # (k-1)-mer prefix
    right_node <- substr(kmer, 2, nchar(kmer))         # (k-1)-mer suffix
    graph[[left_node]] <- c(graph[[left_node]], right_node)
  }
  graph
}

# ── Step 3: Compute in-degree and out-degree for every node ─────────────────
get_degrees <- function(graph) {
  out_deg    <- sapply(graph, length)
  in_deg     <- table(unlist(graph))
  all_nodes  <- unique(c(names(graph), names(in_deg)))
  degrees    <- data.frame(node  = all_nodes,
                           in_d  = 0,
                           out_d = 0,
                           stringsAsFactors = FALSE)
  rownames(degrees) <- all_nodes
  if (length(out_deg) > 0) degrees[names(out_deg), "out_d"] <- as.numeric(out_deg)
  if (length(in_deg)  > 0) degrees[names(in_deg),  "in_d"]  <- as.numeric(in_deg)
  degrees
}

# ── Step 4: Identify Eulerian path start node ───────────────────────────────
# Start node satisfies: out_degree - in_degree == 1
get_start_node <- function(degrees) {
  start_nodes <- degrees[degrees$out_d - degrees$in_d == 1, "node"]
  if (length(start_nodes) == 0) {
    start_nodes <- degrees[degrees$out_d > 0, "node"][1]  # fallback for Eulerian circuit
  }
  start_nodes[1]
}

# ── Step 5: Hierholzer's algorithm for Eulerian path ────────────────────────
find_eulerian_path <- function(graph, start_node) {
  stack <- c(start_node)
  path  <- c()
  while (length(stack) > 0) {
    curr <- stack[length(stack)]
    if (!is.null(graph[[curr]]) && length(graph[[curr]]) > 0) {
      next_node        <- graph[[curr]][1]     # take first available edge
      graph[[curr]]    <- graph[[curr]][-1]    # remove used edge
      stack            <- c(stack, next_node)
    } else {
      path  <- c(path, curr)                   # dead end: add to path
      stack <- stack[-length(stack)]
    }
  }
  rev(path)
}

# ── Step 6: Reconstruct sequence from node path ─────────────────────────────
# Each consecutive pair shares a (k-2)-mer overlap; append only the last base
reconstruct_sequence <- function(path) {
  seq <- path[1]
  for (i in 2:length(path))
    seq <- paste0(seq, substr(path[i], nchar(path[i]), nchar(path[i])))
  seq
}

# ════════════════════════════════════════════════════════════════════════════
# Main: assemble 3 reads
# ════════════════════════════════════════════════════════════════════════════
reads <- c("GATTACA",   # Read 1
           "TACAGAT",   # Read 2 (overlaps Read 1 at "TACA")
           "AGATCCA")   # Read 3 (overlaps Read 2 at "AGAT")

k_size <- 4

cat("Input reads:\n")
print(reads)

kmers <- get_kmers_from_reads(reads, k_size)
cat("\nUnique", k_size, "-mers (graph edges):\n")
cat(paste(kmers, collapse = ", "), "\n")

graph       <- build_de_bruijn_graph(kmers)
degrees     <- get_degrees(graph)
start_node  <- get_start_node(degrees)

cat("\nDegree table:\n")
print(degrees[order(degrees$node), ])
cat("\nEulerian path start node:", start_node, "\n")

path          <- find_eulerian_path(graph, start_node)
assembled_seq <- reconstruct_sequence(path)

cat("\nEulerian path (node sequence):\n")
cat(paste(path, collapse = " → "), "\n")
cat("\nAssembled contig:", assembled_seq, "\n")

expected_seq <- "GATTACAGATCCA"
cat("Expected        :", expected_seq, "\n")
cat("Match           :", assembled_seq == expected_seq, "\n")

16.2 第二部分：进阶实现（含气泡/尖刺检测与 igraph 可视化）

真实测序数据面临两类核心问题：

  1. 气泡（Bubble）：测序错误或真实 SNP 导致同一路径出现分叉再汇合，图中形成”气泡”结构
  2. 尖刺/死端（Tip）：低质量 read 末端的错误 k-mer 形成无法延伸的悬空节点链

下图展示了真实 De Bruijn 图中的典型问题：

正常连续路径:  A → B → C → D
                      ↓
气泡结构:      A → B1 → D     (B1 为 SNP 路径，B2 为原始路径)
                   ↘    ↗
                    B2

尖刺结构:      A → B → C → D
                        ↓
                        X → Y   (X-Y 是低质量错误 k-mer 的死端)

以下完整代码演示了含有 SNP（气泡）、噪声读段（尖刺）的复杂场景，并逐步进行图的清理：

if (!requireNamespace("igraph", quietly = TRUE)) install.packages("igraph")
library(igraph)

rm(list = ls())

# ── k-mer extraction (retaining counts for coverage-based filtering) ─────────
get_kmers_from_reads <- function(reads, k) {
  kmer_vec <- character(0)
  for (seq in reads) {
    n <- nchar(seq)
    if (n < k) {
      warning("Sequence too short, skipped: ", seq)
      next
    }
    kmer_vec <- c(kmer_vec, substring(seq, 1:(n - k + 1), k:n))
  }
  table(kmer_vec)   # named frequency table; count = coverage of that edge
}

# ── Build De Bruijn graph as igraph object ───────────────────────────────────
build_de_bruijn_graph <- function(kmer_counts, min_count = 1) {
  kmers <- names(kmer_counts)[kmer_counts >= min_count]
  if (length(kmers) == 0) stop("No k-mers meet the minimum coverage threshold")
  k     <- nchar(kmers[1])
  edges <- data.frame(
    from  = substr(kmers, 1, k - 1),
    to    = substr(kmers, 2, k),
    kmer  = kmers,
    count = as.integer(kmer_counts[kmers]),
    stringsAsFactors = FALSE
  )
  g         <- graph_from_data_frame(edges, directed = TRUE)
  E(g)$label <- edges$kmer
  E(g)$count <- edges$count
  g
}

# ── Graph structure diagnostics ──────────────────────────────────────────────
diagnose_graph <- function(g) {
  cat("-- Graph Diagnostics ---------------------\n")
  cat("  Nodes:", vcount(g), " | Edges:", ecount(g), "\n")
  deg_in  <- degree(g, mode = "in")
  deg_out <- degree(g, mode = "out")
  tips    <- V(g)[deg_in == 0 | deg_out == 0]
  junc    <- V(g)[deg_in > 1  | deg_out > 1]
  cat("  Tips (dead ends):", length(tips), "\n")
  cat("  Junction nodes  :", length(junc),  "\n")
  bubbles <- detect_bubbles(g)
  cat("  Putative bubbles:", length(bubbles), "\n")
  wc <- components(as.undirected(g))
  cat("  Weakly connected components:", wc$no, "\n")
  cat("------------------------------------------\n")
  list(tips = tips, junctions = junc, bubbles = bubbles)
}

# ── Bubble detection ─────────────────────────────────────────────────────────
# A bubble exists when two outgoing branches from the same node reconverge
detect_bubbles <- function(g, max_len = 5) {
  bubbles       <- list()
  branch_nodes  <- V(g)[degree(g, mode = "out") >= 2]
  for (v in branch_nodes) {
    succs <- neighbors(g, v, mode = "out")
    if (length(succs) < 2) next
    for (i in 1:(length(succs) - 1)) {
      for (j in (i + 1):length(succs)) {
        reach_i    <- igraph::ego(g, order = max_len, nodes = succs[i], mode = "out")[[1]]
        reach_j    <- igraph::ego(g, order = max_len, nodes = succs[j], mode = "out")[[1]]
        merge_nodes <- intersect(reach_i, reach_j)
        if (length(merge_nodes) > 0) {
          bubbles <- c(bubbles, list(list(
            source  = V(g)$name[v],
            branch1 = V(g)$name[succs[i]],
            branch2 = V(g)$name[succs[j]],
            merge   = V(g)$name[merge_nodes[1]]
          )))
        }
      }
    }
  }
  bubbles
}

# ── Tip clipping: iteratively remove dead-end chains ────────────────────────
remove_tips <- function(g, max_tip_len = 2) {
  repeat {
    deg_in  <- degree(g, mode = "in")
    deg_out <- degree(g, mode = "out")
    tips    <- V(g)[(deg_in == 0 & deg_out <= max_tip_len) |
                    (deg_out == 0 & deg_in  <= max_tip_len)]
    if (length(tips) == 0) break
    g <- delete_vertices(g, tips)
  }
  g
}

# ── Bubble popping: keep high-coverage path, discard low-coverage branch ─────
pop_bubbles <- function(g, bubbles) {
  edges_to_delete <- integer(0)
  for (b in bubbles) {
    e1 <- E(g)[.from(b$source) & .to(b$branch1)]
    e2 <- E(g)[.from(b$source) & .to(b$branch2)]
    if (length(e1) == 0 || length(e2) == 0) next
    # Remove the lower-coverage branch
    low_edge         <- if (E(g)$count[e1] >= E(g)$count[e2]) e2 else e1
    edges_to_delete  <- c(edges_to_delete, low_edge)
  }
  if (length(edges_to_delete) > 0)
    g <- delete_edges(g, unique(edges_to_delete))
  g
}

# ── Low-coverage edge filtering ──────────────────────────────────────────────
filter_low_coverage <- function(g, min_count = 2) {
  bad_edges <- E(g)[E(g)$count < min_count]
  if (length(bad_edges) > 0) g <- delete_edges(g, bad_edges)
  delete_vertices(g, V(g)[degree(g) == 0])   # remove isolated nodes
}

# ── Hierholzer's Eulerian path (igraph version) ──────────────────────────────
find_eulerian_path <- function(g) {
  adj     <- lapply(V(g)$name, function(v) neighbors(g, v, mode = "out")$name)
  names(adj) <- V(g)$name
  deg_in  <- degree(g, mode = "in")
  deg_out <- degree(g, mode = "out")
  diff    <- deg_out - deg_in
  starts  <- names(diff)[diff == 1]
  start   <- if (length(starts) > 0) starts[1] else names(deg_out)[deg_out > 0][1]
  if (is.na(start)) return(character(0))
  stack <- start
  path  <- character(0)
  while (length(stack) > 0) {
    curr <- tail(stack, 1)
    if (length(adj[[curr]]) > 0) {
      nxt          <- adj[[curr]][1]
      adj[[curr]]  <- adj[[curr]][-1]
      stack        <- c(stack, nxt)
    } else {
      path  <- c(path, curr)
      stack <- stack[-length(stack)]
    }
  }
  rev(path)
}

reconstruct_sequence <- function(path) {
  if (length(path) < 2) return(paste(path, collapse = ""))
  seq <- path[1]
  for (i in 2:length(path))
    seq <- paste0(seq, substr(path[i], nchar(path[i]), nchar(path[i])))
  seq
}

# ── Visualization ────────────────────────────────────────────────────────────
plot_dbg <- function(g, title = "De Bruijn Graph",
                     highlight_nodes = NULL, layout_fn = layout_with_fr) {
  pal     <- c(normal = "#4E79A7", highlight = "#F28E2B", junction = "#E15759")
  is_junc <- (degree(g, mode = "in") > 1 | degree(g, mode = "out") > 1)
  vcols   <- rep(pal["normal"], vcount(g))

  plot(g,
       layout           = layout_fn,

# ── Assembly evaluation ──────────────────────────────────────────────────────
edit_distance <- function(a, b) {
  a <- strsplit(a, "")[[1]]; b <- strsplit(b, "")[[1]]
  m <- length(a);             n <- length(b)
  d <- matrix(0L, m + 1, n + 1)
  d[, 1] <- 0:m;             d[1, ] <- 0:n
  for (i in seq_len(m))
    for (j in seq_len(n))
      d[i+1, j+1] <- if (a[i] == b[j]) d[i, j] else
        1L + min(d[i, j], d[i+1, j], d[i, j+1])
  d[m+1, n+1]
}

evaluate_assembly <- function(contig, ref) {
  len_c   <- nchar(contig);   len_r <- nchar(ref)
  ed      <- edit_distance(contig, ref)
  pct_id  <- round(100 * (1 - ed / max(len_c, len_r)), 2)
  list(length       = len_c,
       ref_length   = len_r,
       edit_dist    = ed,
       pct_identity = pct_id,
       length_ratio = round(100 * len_c / len_r, 2))
}

SNP 气泡的处理策略

本例中，Read 6（TCGATCGTTC）携带一个 A→T SNP，在图中产生了一个气泡。气泡弹出（bubble popping）策略是：保留覆盖度更高的路径（原始序列，覆盖×2），丢弃低覆盖路径（SNP 路径，覆盖×1）。这实际上是测序深度不足时组装工具丢弃真实变异的机制之一——真实 SNP 在杂合子样本中覆盖度约为总深度的 50%，而测序错误通常覆盖度 < 5%。SPAdes 等工具通过更精细的覆盖度模型区分二者。

17 实践：使用 SPAdes 组装真实 E. coli 基因组

17.1 SPAdes 简介

SPAdes（St. Petersburg genome assembler） 是目前最广泛使用的细菌和宏基因组短读组装工具，由俄罗斯圣彼得堡大学开发。其核心创新包括：

  • 多 k-mer 策略：同时使用多个 k 值（如 21, 33, 55）构建 De Bruijn 图，取各自的最优结果合并，弥补单一 k 值的局限
  • BayesHammer 错误校正：在组装前对 reads 进行贝叶斯错误校正，降低图中噪声边的数量
  • 配对末端约束：利用 PE reads 的距离信息辅助解析重复区域和气泡

QUAST 关键指标解读：

指标 50k（低深度） 完整数据（高深度） 含义
# contigs 147 9 组装片段数（越少越好）
Largest contig 92,318 bp 2,283,671 bp 最长单片段
Total length 612,841 bp 4,638,517 bp 组装总长度（应接近参考基因组）
N50 7,824 bp 1,847,234 bp 核心连续性指标
GC (%) 50.79% 50.79% 应与参考基因组一致
Mismatches per 100 kbp 12.3 1.8 组装错误率
# misassemblies 3 0 大片段组装错误数（最严重的指标）

“misassembly”（组装错误）是最严重的质量问题

N50 偏低仅代表组装不连续，不影响已组装区域的准确性；而 misassembly（错拼）意味着两段本应相距甚远的序列被错误拼接在一起，会导致下游分析（基因预测、变异检测）产生系统性错误。因此，评估组装质量时应优先关注 misassembly 数量，其次才是 N50。

当没有参考基因组时，评估组装质量依赖以下指标：

  • C（Complete）= 99.1%：99.1% 的单拷贝直系同源基因被完整检测到
  • S（Single）= 99.1%：均为单拷贝（无重复组装）

18.3 课后自测 (Post-Lesson Quiz)

  1. 已知某基因组大小约 5 Mb，使用 PE150 进行 WGS，共获得 3,000,000 对 reads（即 6,000,000 条 reads）。估算测序覆盖深度，并判断该深度是否足以进行高质量从头组装？

  2. 解释为什么 De Bruijn 图方法将组装问题从哈密顿路径转化为欧拉路径，这一转化的计算复杂度意义是什么？

  3. 在进行 De Bruijn 图组装时，k 值的选择是一个重要参数。请分析：（a）k 值过小（如 k=11）有什么问题？（b）k 值过大（如 k=101）有什么问题？（c）为什么 SPAdes 使用多个 k 值？

  4. 某 E. coli 样本用 100X 深度测序后进行从头组装，得到 QUAST 报告：N50 = 45,000 bp，misassemblies = 8，total length = 4.61 Mb（参考基因组 4.64 Mb），GC = 50.8%。请评价该组装质量，并指出最值得关注的问题。

  5. 在气泡弹出（bubble popping）策略中，程序保留覆盖度高的路径，删除覆盖度低的路径。请思考：在对二倍体生物（如人类）进行从头组装时，这一策略会产生什么问题？现代工具如何解决这一问题？

  6. 比较参考基因组比对和从头组装两种策略：对于一个已知参考基因组的物种，在什么情况下仍然需要进行从头组装而不是直接使用参考基因组比对？请给出至少两个具体场景。

参考答案 (Reference Answers)

  1. 覆盖深度 = (6,000,000 × 150) / 5,000,000 = 180X。该深度（180X）远超从头组装的推荐最低深度（50–100X），属于高覆盖测序，足以支持高质量组装。但需注意 180X 不一定优于 60X，过高覆盖有时会引入更多测序错误的累积，现代工具（如 SPAdes）对超高覆盖数据会自动进行降采样。

  2. OLC（哈密顿路径）要求每个读段（节点）恰好被经过一次，是 NP 完全问题，不存在已知多项式时间算法。DBG 将读段分解为 k-mer 后，每个 k-mer 作为图的边，组装问题转化为每条边恰好经过一次的欧拉路径问题。欧拉路径可在 O(E) 时间内用 Hierholzer 算法求解（E 为边数，即 k-mer 数量）。这一转化使得处理数十亿条短读 reads 成为计算可行的任务。

  3. (a) k 过小（如 11）：大量 k-mer 在基因组中出现多次（重复序列），图中产生大量分叉节点，路径无法唯一确定，组装极度碎片化。(b) k 过大（如 101）：每个 k-mer 出现次数接近 1，低深度区域（覆盖波动）导致图断裂，contigs 变短；同时内存占用急剧增加。(c) SPAdes 使用多个 k 值：小 k 值在低覆盖区域保持连续性，大 k 值有效解析重复区域，合并各 k 值的最优区域后得到更完整的组装结果。

  4. N50 = 45,000 bp 相对于 E. coli 4.64 Mb 基因组偏低，说明组装仍较碎片化（理想情况 N50 应接近 1–2 Mb）。总长度 4.61 Mb 接近参考基因组（误差 < 1%），GC 含量正常。最值得关注的问题是 misassemblies = 8，这意味着 8 处大片段序列被错误拼接，可能影响基因预测和基因组结构分析的准确性。建议检查覆盖度是否均匀，考虑补充长读测序数据来解析困难区域。

  5. 对于二倍体生物，同一位点的两个等位基因序列略有不同，在 De Bruijn 图中会产生气泡。单纯按覆盖度保留高覆盖路径（约 50% 深度）、删除低覆盖路径（约 50% 深度），会将一个等位基因丢弃，产生单倍体化（haplotype collapse）的组装结果。现代单倍型分辨组装工具（如 hifiasm、FALCON-Unzip）通过保留两条分叉路径、分别延伸，尝试生成分相（phased）的双倍体组装，即同时输出两套单倍型序列。

  6. 至少两个场景：①研究大规模结构变异（SV）：插入/缺失片段 > 1 kb 的 SV 在比对策略中难以精确重建，从头组装可直接揭示插入的序列内容（如转座子插入）。②癌症基因组学：肿瘤基因组常发生大片段重排、融合基因、新插入序列，这些变异在比对至正常参考基因组时容易遗漏，肿瘤样本从头组装（或局部重组装）可以发现新型序列内容。

18.4 总结

✓ 从头组装 是无参考基因组场景下重建基因组的核心技术，也是参考基因组本身产生的方式 ✓ De Bruijn 图 将 reads 分解为 k-mer，把 NP 完全的哈密顿路径问题转化为多项式时间可解的欧拉路径问题 ✓ 气泡（Bubble） 来源于测序错误或真实 SNP，通过覆盖度比较弹出；尖刺（Tip） 来源于低质量 read 末端，通过迭代截除清理 ✓ N50 是衡量组装连续性的核心指标；misassembly 数量 是衡量组装准确性的最重要指标 ✓ SPAdes 通过多 k 值策略和错误校正，是目前细菌和宏基因组短读组装的行业标准 ✓ QUAST + BUSCO 组合评估提供了有参考和无参考两种场景下的完整质量图像 ✓ 覆盖深度 ≥ 50X 是高质量从头组装的基本要求，低深度（< 20X）会导致严重的图断裂和碎片化
