24  功能富集分析

Published

June 14, 2026

1 学习目标

完成本专题后，学生应能够：

  1. 描述 Gene Ontology（GO）三个本体的层级结构与生物学含义；
  2. 阐述过表示富集分析（ORA）的超几何检验原理，并解释背景基因集选择的重要性；
  3. 描述 GSEA 的基因集富集得分计算逻辑，解释 NES 与 leading edge 基因的意义；
  4. 区分 ORA 与 GSEA 的适用场景，能根据数据特点做出合理选择；
  5. 使用 clusterProfiler 独立完成 GO ORA、KEGG ORA 与 GSEA 分析，并生成规范的可视化图表；
  6. 识别富集分析结果中的常见陷阱与过度解读风险。

2 从差异基因到生物学意义

差异表达分析的产出是一张基因列表——数百到数千个在处理与对照间有统计显著变化的基因。然而，基因的名字本身（ENSG00000186092、TP53、MYC）并不能直接告诉我们”这些基因的变化意味着什么生物学过程被激活或抑制”。

功能富集分析（Functional Enrichment Analysis）通过将差异基因与人工整理的功能数据库（如 GO、KEGG）比对，将孤立的基因列表转化为有生物学意义的通路和功能模块，是 RNA-seq 分析中将统计结果转化为生物学假设的关键步骤。

3 功能注释数据库

3.1 Gene Ontology（GO）

Gene Ontology 是一个层级化的受控词汇（controlled vocabulary）系统，将基因的功能划分为三个独立的本体（Ontology），如 图 1 所示。

生物学过程（Biological Process, BP）：描述基因参与的生物学事件，从宏观（“免疫应答”）到微观（“受体酪氨酸激酶信号传导”）形成层级结构。这是 RNA-seq 分析中最常用的 GO 本体，也是 clusterProfiler 默认分析的本体类型。

分子功能（Molecular Function, MF）：描述基因产物在分子层面的活性（如”ATP 结合”、“DNA 结合转录因子活性”）。注意：MF 描述的是功能，而非执行该功能的生物学背景。同一基因产物可以同时具有多个 MF 注释（如”激酶活性”与”支架蛋白活性”）。

细胞组分（Cellular Component, CC）：描述基因产物定位的细胞结构（如”线粒体内膜”、“核糖体大亚基”）。CC 注释依赖实验性定位证据，因此注释覆盖度通常低于 BP 和 MF。

GO 采用有向无环图（Directed Acyclic Graph, DAG）结构：每个 GO term 可以有多个亲本节点（parent），而非简单的树形层级。以”线粒体 ATP 合成”这一 GO term 为例，它同时是”ATP 生物合成过程”和”线粒体电子传递”两个亲本的子节点。这种多重继承结构使 GO 能够更准确地表达复杂的生物学关系，但也带来了一个重要的计算问题：祖先传播（ancestral propagation）——当一个基因被注释到某个 GO term 时，它也隐含地注释到该 term 的所有祖先节点。在富集分析中，若不加处理，这会导致父 term 因自动继承子 term 的基因而呈现冗余显著性，产生数百个高度重叠的显著 GO terms，掩盖真正独立的生物学信号。clusterProfiler 的 simplify() 函数通过计算 GO terms 间的语义相似度来解决这一问题（参见本专题后续部分）。

3.2 KEGG 通路数据库

KEGG（Kyoto Encyclopedia of Genes and Genomes）将基因分配到可手工策划的信号通路图（pathway map）中，直观展示基因产物在信号传导、代谢途径中的相互关系。与 GO 的抽象词汇不同，KEGG 通路图可以直接可视化，显示哪些节点（基因）在实验中被激活（上调）或抑制（下调）。

数据库局限性： GO 和 KEGG 的注释覆盖度存在物种偏差——人类、小鼠等模式生物的注释远比非模式生物完善。对于覆盖率低的物种，富集分析结果会呈现系统性偏差，应与直接证据结合解读。

3.3 MSigDB 基因集合库

Molecular Signatures Database（MSigDB）提供了广泛的人工策划基因集，常用于基因集富集分析（Gene Set Enrichment Analysis，GSEA）：

  • H（Hallmark gene sets）：50 个代表性生物学过程的基因集，去冗余、高置信度，适合入门分析；
  • C2（Curated gene sets）：来自已发表文献和通路数据库的实验验证基因集；
  • C5（Ontology gene sets）：GO BP/MF/CC 基因集的系统整理版本。

3.4 Reactome 通路数据库

Reactome 是一个开放获取、经过专家人工策划的通路数据库，覆盖人类及多个模式生物。与 KEGG 相比，Reactome 具有以下特点：

  • 更细粒度的层级：通路可嵌套至反应（Reaction）和参与分子（Participant）层级，生物学意义更精确；
  • 开放获取：完全免费，且提供 Bioconductor 接口（ReactomePA 包）；
  • 与 KEGG 互补：两个数据库在通路覆盖上存在差异，联合使用可提高召回率。

Reactome ORA 与 GSEA 的完整代码示例见本专题「多数据库结果整合与高级可视化」一节。

4 过表示富集分析（Over-Representation Analysis，ORA）

4.1 超几何检验原理

ORA 将差异表达基因（DEGs）与某个功能基因集进行重叠分析，通过超几何检验（hypergeometric test）评估重叠程度是否超出随机预期。简而言之，我们真正想问的是：“我找到的这批差异基因中，属于某个特定功能的基因是不是多得有些反常？”

假设： - \(N\)：背景基因集总数（通常为检测到有表达的所有基因）； - \(K\)：某 GO term 中基因总数； - \(n\)：显著 DEGs 总数； - \(k\)：DEGs 中属于该 GO term 的基因数。

p 值为从 \(N\) 个基因中随机抽取 \(n\) 个，至少有 \(k\) 个属于该 GO term 的概率：

\[p = \sum_{i=k}^{\min(n,K)} \frac{\binom{K}{i}\binom{N-K}{n-i}}{\binom{N}{n}}\]

这等同于从一个含 \(K\) 个”红球”和 \(N-K\) 个”黑球”的瓮中不放回地抽取 \(n\) 个球，至少抽到 \(k\) 个红球的概率。即我们想知道，闭着眼睛抓 \(n\) 个球，居然抓到了 \(k\) 个红球，这是纯粹凭运气（随机发生的），还是因为某种必然的原因（即这个功能确实被激活了）。

背景基因集的选择至关重要。 若将背景集设为”全基因组所有基因”（而非”在该实验中检测到有表达的基因”），则分母 \(N\) 偏大，导致 \(\frac{k}{n}\) 相对 \(\frac{K}{N}\) 的比值（富集倍数）被低估，p 值偏保守。更严重的是，若使用”全基因组”作为背景，但实际实验只能检测到一部分基因，则对每个 GO term 中有多少基因”在实验中可被检测”的假设是错误的。始终使用实验检测到的基因作为背景集。

4.2 手动实现超几何检验

以下代码以模拟数据从头复现超几何检验的完整计算过程，并通过逐步改变参数（背景集大小 \(N\)）直观展示「背景集选择」对 p 值的影响，最后与 R 内置函数 phyper() 进行数值验证。

library(ggplot2)

# ── 参数设置（对应一个典型 GO term）──────────────────────────────────────────
N <- 15000   # 背景基因总数（实验检测到的基因）
K <-   200   # 该 GO term 在背景中的基因数
n <-   500   # 显著 DEGs 总数
k <-    30   # DEGs 中命中该 GO term 的基因数

# ── 手动计算超几何检验 p 值 ─────────────────────────────────────────────────
# P(X >= k)，X ~ Hypergeometric(N, K, n)
# R 的 phyper(q, m, n, k) 计算 P(X <= q)，其中 m=K, n=N-K, k=n
# 因此 P(X >= k) = 1 - P(X <= k-1) = phyper(k-1, K, N-K, n, lower.tail=FALSE)

p_manual <- sum(
  dhyper(k:min(n, K), m = K, n = N - K, k = n)
)
p_phyper <- phyper(k - 1, m = K, n = N - K, k = n, lower.tail = FALSE)

expected_overlap <- n * K / N          # 期望重叠数 = n × K/N
fold_enrichment  <- (k / n) / (K / N) # 富集倍数 = GeneRatio / BgRatio

cat("=== Hypergeometric test (manual) ===\n")
cat(sprintf("  Background N = %d, GO term K = %d, DEGs n = %d, overlap k = %d\n",
            N, K, n, k))
cat(sprintf("  Expected overlap under H0 : %.2f\n", expected_overlap))
cat(sprintf("  Observed overlap          : %d\n", k))
cat(sprintf("  Fold enrichment           : %.2f×\n", fold_enrichment))
cat(sprintf("  p-value (manual dhyper)   : %.2e\n", p_manual))
cat(sprintf("  p-value (phyper verify)   : %.2e\n", p_phyper))
cat(sprintf("  Difference (should be ~0) : %.2e\n", abs(p_manual - p_phyper)))

# ── 展示概率质量函数并标记观测值 ─────────────────────────────────────────────
x_range <- 0:60
pmf_df  <- data.frame(
  x    = x_range,
  prob = dhyper(x_range, m = K, n = N - K, k = n)
)

ggplot(pmf_df, aes(x = x, y = prob)) +
  geom_col(fill = "#4575b4", alpha = 0.7, width = 0.8) +
  geom_vline(xintercept = k, color = "#d73027",
             linewidth = 1.2, linetype = "dashed") +
  geom_vline(xintercept = expected_overlap, color = "#e08214",
             linewidth = 1.0, linetype = "dotted") +
  annotate("text", x = k + 1, y = max(pmf_df$prob) * 0.9,
           label = sprintf("Observed k = %d\np = %.2e", k, p_manual),
           color = "#d73027", hjust = 0, size = 3.2) +
  annotate("text", x = expected_overlap - 1, y = max(pmf_df$prob) * 0.75,
           label = sprintf("Expected\n= %.1f", expected_overlap),
           color = "#e08214", hjust = 1, size = 3.0) +
  labs(
    x     = "Number of DEGs overlapping with GO term (k)",
    y     = "Probability mass",
    title = sprintf("Hypergeometric PMF  (N=%d, K=%d, n=%d)", N, K, n),
    subtitle = sprintf("Shaded area to the right of red line = p-value = %.2e", p_manual)
  ) +
  theme_classic(base_size = 11)

# ── 敏感性分析：背景集大小对 p 值的影响 ──────────────────────────────────────
# 固定 k=30, n=500, K=200，改变 N（模拟"用不同背景集"的场景）
N_vals <- c(5000, 8000, 10000, 15000, 20000, 25000)
sens_df <- data.frame(
  N      = N_vals,
  p      = sapply(N_vals, function(Ni)
    phyper(k - 1, m = K, n = Ni - K, k = n, lower.tail = FALSE)),
  fe     = sapply(N_vals, function(Ni) (k / n) / (K / Ni)),
  label  = ifelse(N_vals == 15000, "Correct\n(detected genes)", "")
)

ggplot(sens_df, aes(x = N, y = -log10(p))) +
  geom_line(color = "#2166ac", linewidth = 1.2) +
  geom_point(size = 3, color = "#2166ac") +
  geom_point(data = sens_df[sens_df$N == 15000, ],
             size = 5, color = "#d73027") +
  geom_text(aes(label = label), vjust = -1.2, size = 3.0, color = "#d73027") +
  scale_x_continuous(labels = scales::comma) +
  labs(
    x     = "Background set size N",
    y     = "-log10(p-value)",
    title = "Sensitivity of ORA p-value to background set size",
    subtitle = sprintf("Fixed: K=%d, n=%d, k=%d. Red dot = correct background.", K, n, k)
  ) +
  theme_classic(base_size = 11)
# Key takeaway: using N=20000 (full genome) instead of N=15000 (detected genes)
# inflates the denominator, reduces the fold enrichment, and gives a more
# conservative (larger) p-value — but the assumption is statistically wrong.

4.3 ORA 的适用场景与局限性

适用场景：有明确的差异基因列表（如 |LFC| > 1，padj < 0.05）；需要快速识别最显著富集的功能类别。

局限性：

  1. 阈值依赖性：结果对差异基因的筛选阈值极为敏感。调整 LFC 阈值从 0.5 到 1.5，富集结果可能截然不同；
  2. 丢失趋势信息：将所有 DEGs 视为等效，忽视了上调和下调的方向性，以及表达变化幅度；
  3. 忽略非显著基因的信息：接近但未达到阈值的基因被完全排除在外。

4.4 解读 ORA 输出指标

clusterProfiler 的 ORA 结果表中有几个容易混淆的列需要注意：

  • GeneRatio：以分数形式表示，分子为该 GO term 中属于 DEG 的基因数（\(k\)），分母为 DEG 总数（\(n\)）。点图（dotplot）的 x 轴即为 GeneRatio 的小数值。GeneRatio 越大，表示 DEG 中命中该 term 的比例越高。
  • BgRatio：同样以分数表示，分子为该 GO term 在背景集中的基因总数（\(K\)），分母为背景集总大小（\(N\)）。
  • 富集倍数（Fold Enrichment）：GeneRatio 除以 BgRatio，是衡量富集强度的效应量指标，但 enrichGO() 默认不输出此列——可手动计算：result$FoldEnrichment <- (result$Count / n) / (result$BgRatio_K / N)。
  • p.adjust 与 qvalue：p.adjust 为 BH 校正后的 FDR，qvalue 为 Storey 方法的 q 值（假阳性发现率的贝叶斯估计）。两者均控制多重检验，通常以 p.adjust < 0.05 作为主要报告标准。

实用建议： 对于大型 DEG 列表（> 2,000 个基因），p.adjust 极小的 GO terms 往往是细胞分裂、代谢等大而泛化的生物学过程，GeneRatio 未必高。应同时报告富集倍数和 leading edge 基因，而非单纯按 p.adjust 排序呈现结果。

5 基因集富集分析（GSEA）

5.1 算法原理

GSEA 不设定差异基因阈值，而是将全部基因按某个相关性指标（通常为 DESeq2 的 Wald stat 或 LFC × -log10(padj) 的乘积）从正到负排序，形成一个有序的基因列表 \(L = [g_1, g_2, \ldots, g_N]\)。

对于每个基因集 \(S\)，GSEA 沿有序列表计算富集得分（Enrichment Score, ES）：

\[ES = \max_{j} \left( \text{running sum at position } j \right)\]

其中 running sum 的规则是： - 遇到基因集 \(S\) 中的基因 → running sum 向上增加（增量与排序指标的绝对值成比例，使用加权版本）； - 遇到不在 \(S\) 中的基因 → running sum 向下减少（均等减量）。

ES 越大，表示基因集 \(S\) 在排序列表的顶端（高度上调基因中）越富集；ES 越负，表示在底端（高度下调基因中）越富集。

标准化富集得分（NES）：由于不同基因集大小不同，需要将 ES 标准化。方法是将真实 ES 除以同大小随机基因集排列置换的平均 |ES|，得到 NES。NES > 1.5 通常视为有意义的正向富集。

置换检验与 p 值计算：GSEA 通过置换（permutation）建立零分布来评估 ES 的显著性。置换策略有两种，选择哪种对结果影响重大：

  • 基因标签置换（gene permutation）：在保持基因排序不变的前提下，随机打乱基因集成员标签，计算各次置换的 ES，以此构建零分布。此方法不依赖样本重复数，适用于样本量少于 7 个的情况，是 fgsea（clusterProfiler 的后端）的默认策略。
  • 样本置换（sample permutation）：随机打乱样本的表型标签，对每次置换重新排序基因、重新计算 ES。此方法保留了基因间相关结构，统计更严格，但要求至少 7 个样本。

置换次数的影响： fgsea 的 eps 参数控制最小可报告 p 值的精度下界。默认 eps = 1e-10 使函数在极小 p 值时切换到精确近似算法，而非单纯增加置换次数。对 padj 极低（如 < 1e-5）的结果应谨慎——它们在统计上可靠，但生物学上仍需独立实验验证。

GSEA 算法流程概览如 图 2 所示。

5.2 Leading Edge 分析

ES 曲线达到最大值（或最小值）的位置，将基因列表分成两部分：最大值之前的基因集 \(S\) 成员称为 leading edge 基因，即驱动富集得分的核心基因子集。这些基因是生物学解读的焦点——它们既在基因集 \(S\) 中，又在排序列表的极端位置，最有可能是生物学上真正相关的效应基因。

黄金法则： GSEA 的 leading edge 基因是实验结果与已知通路知识的交汇点。对 leading edge 基因的深入文献检索，往往是从计算分析迈向生物学发现的最关键一步。

5.3 手动实现 GSEA：ES 计算与基因标签置换检验

以下代码以模拟数据从头复现 GSEA 的核心计算：先手动计算加权富集得分（weighted ES），再通过基因标签置换（gene permutation）构建零分布，最终得到经验 p 值和 NES。与 fgsea 的结果进行数值对比，验证实现的正确性。

library(ggplot2)
library(fgsea)
library(patchwork)

set.seed(2024)

# ── Step 0: 构造模拟排序基因列表 ──────────────────────────────────────────────
# 200 个基因，排序指标模拟 DESeq2 Wald stat（正态分布，轻微偏正）
n_genes  <- 200
gene_ids <- paste0("G", seq_len(n_genes))
stats    <- sort(rnorm(n_genes, mean = 0.3, sd = 1.5), decreasing = TRUE)
names(stats) <- gene_ids

# ── Step 1: 定义基因集 S（刻意让前 30% 的基因集中在排序列表前端）──────────────
# 真实场景：基因集成员聚集在顶端 → ES 应为正且显著
set_size   <- 25
set_top    <- round(set_size * 0.75)   # 75% 成员放在前 60 名（上调区）
set_bottom <- set_size - set_top       # 25% 成员随机分布
gene_set   <- c(
  sample(gene_ids[1:60],   set_top,    replace = FALSE),
  sample(gene_ids[61:200], set_bottom, replace = FALSE)
)
cat("Gene set size:", length(gene_set), "\n")
cat("Members in top-60:", sum(gene_set %in% gene_ids[1:60]), "\n\n")

# ── Step 2: 手动计算加权 ES ──────────────────────────────────────────────────
# 算法（Subramanian et al. 2005 加权版本，p=1）：
#   命中基因（在 S 中）：running sum += |stat_i|^p / sum_hit_weights
#   非命中基因（不在 S）：running sum -= 1 / (N - |S|)
compute_es <- function(ordered_stats, gene_set) {
  n        <- length(ordered_stats)
  in_set   <- names(ordered_stats) %in% gene_set
  # Hit increment: weighted by |stat|
  hit_w    <- abs(ordered_stats[in_set])
  hit_norm <- sum(hit_w)                         # normalizer for hit steps
  miss_norm <- n - length(gene_set)              # normalizer for miss steps

  running_sum <- numeric(n)
  rs          <- 0
  for (i in seq_len(n)) {
    if (in_set[i]) {
      rs <- rs + abs(ordered_stats[i]) / hit_norm
    } else {
      rs <- rs - 1 / miss_norm
    }
    running_sum[i] <- rs
  }
  es <- running_sum[which.max(abs(running_sum))]
  list(es = es, running_sum = running_sum, in_set = in_set)
}

result   <- compute_es(stats, gene_set)
ES_obs   <- result$es
rs_df    <- data.frame(
  position     = seq_along(stats),
  running_sum  = result$running_sum,
  in_set       = result$in_set,
  stat         = stats
)
cat(sprintf("Observed ES = %.4f\n", ES_obs))

# ── Step 3: 基因标签置换（gene permutation）构建零分布 ────────────────────────
# 保持排序指标 stats 不变，随机重新分配基因集成员标签
# 每次置换：从同等大小的随机基因集计算 ES，累积得到零分布
n_perm      <- 2000
ES_perm     <- numeric(n_perm)
for (i in seq_len(n_perm)) {
  perm_set    <- sample(gene_ids, set_size, replace = FALSE)
  ES_perm[i]  <- compute_es(stats, perm_set)$es
}

# Empirical p-value: fraction of permutations with |ES| >= |ES_obs|
p_empirical <- mean(abs(ES_perm) >= abs(ES_obs))

# NES: normalize by mean |ES| of same-sign permutations
pos_perm <- ES_perm[ES_perm >= 0]
NES      <- ES_obs / mean(pos_perm)

cat(sprintf("Permutation p-value (n=%d) : %.4f\n", n_perm, p_empirical))
cat(sprintf("NES                        : %.4f\n\n", NES))

# ── Step 4: 与 fgsea 结果对比 ────────────────────────────────────────────────
gene_sets_list <- list(SimulatedSet = gene_set)
fgsea_res <- fgsea(
  pathways  = gene_sets_list,
  stats     = stats,
  minSize   = 5,
  maxSize   = 500,
  eps       = 0
)
cat("=== fgsea result (reference) ===\n")
print(fgsea_res[, c("pathway", "ES", "NES", "pval", "size")])
cat(sprintf("\n=== Manual result ===\n"))
cat(sprintf("  ES  = %.4f  (fgsea: %.4f)\n", ES_obs, fgsea_res$ES))
cat(sprintf("  NES = %.4f  (fgsea: %.4f)\n", NES,    fgsea_res$NES))
cat(sprintf("  p   = %.4f  (fgsea: %.4f)\n", p_empirical, fgsea_res$pval))

# ── Step 5: 可视化 ────────────────────────────────────────────────────────────
# Panel A: Running enrichment score curve
p_rs <- ggplot(rs_df, aes(x = position, y = running_sum)) +
  geom_line(color = "#2166ac", linewidth = 1.0) +
  geom_hline(yintercept = 0, color = "grey50", linetype = "dashed") +
  geom_hline(yintercept = ES_obs, color = "#d73027",
             linetype = "dotted", linewidth = 0.8) +
  annotate("text", x = which.max(abs(rs_df$running_sum)),
           y = ES_obs * 1.08,
           label = sprintf("ES = %.3f", ES_obs),
           color = "#d73027", size = 3.2) +
  labs(x = "Gene rank (high stat → low stat)",
       y = "Running enrichment score",
       title = "Running ES curve") +
  theme_classic(base_size = 11)

# Panel B: Gene set member rug plot
p_rug <- ggplot(rs_df[rs_df$in_set, ], aes(x = position)) +
  geom_rug(sides = "b", color = "#2166ac", linewidth = 0.6, alpha = 0.9) +
  geom_rug(data = rs_df[!rs_df$in_set, ],
           sides = "b", color = "grey80", linewidth = 0.3, alpha = 0.4) +
  scale_x_continuous(limits = c(1, n_genes)) +
  labs(x = "Gene rank", y = NULL,
       title = "Gene set member positions") +
  theme_classic(base_size = 11) +
  theme(axis.text.y = element_blank(), axis.ticks.y = element_blank())

# Panel C: Null distribution from permutation
perm_df <- data.frame(es_perm = ES_perm)
p_perm <- ggplot(perm_df, aes(x = es_perm)) +
  geom_histogram(bins = 60, fill = "grey70", color = "white") +
  geom_vline(xintercept = ES_obs, color = "#d73027",

(p_rs / p_rug / p_perm) +

上图三个面板对应 GSEA 输出的三个核心层次：

  • 上图（Running ES）：曲线在前半段快速上升（基因集成员集中于排序列表顶端），到达峰值后缓慢下降。ES 即为曲线的最大值，红色虚线标出其位置。
  • 中图（成员位置）：蓝色竖线为基因集成员在排序列表中的位置，灰色为非成员。成员明显聚集于左侧（高 stat 区），与曲线形态吻合。

6.1 理解 GSEA 富集图

gseaplot2() 生成的经典 GSEA 图包含三个面板：

  1. 顶部折线图：Running Enrichment Score (RES) 曲线，纵轴为累积富集得分，达到最大值处即为 ES；
  2. 中间条形图：基因集 \(S\) 中各基因在有序列表中的位置（黑色竖线），聚集在左侧表示上调富集，聚集在右侧表示下调富集；
  3. 底部梯度条：基因排序指标（Wald stat）从正（左）到负（右）的热图，颜色越红表示上调越强，越蓝表示下调越强。

9 结果解读的批判性原则

避免过度解读富集分析结果

以下是富集分析中最常见的误用：

  1. 功能注释覆盖度偏差：GO 对某些功能（如”DNA 修复”）的注释非常详细，对另一些（如”细胞极性建立”）则稀疏。覆盖度高的功能更容易出现在富集结果中，并非一定是最重要的。

  2. 通路重叠假象：许多 GO terms 共享大量基因（如”细胞周期调控”与”DNA 损伤应答”的基因高度重叠）。多个显著 terms 可能被同一组基因驱动，并非独立的生物学信号。使用 simplify() 或 emapplot() 可帮助识别冗余。

  3. 背景集选择偏差：参见 ORA 部分的警告。

  4. 将 p 值大小等同于生物学重要性：富集分析 p 值主要受基因集大小影响（大的基因集更容易显著），而非效应量大小。应结合 Gene Ratio（或 NES）和 leading edge 基因的生物学意义综合判断。

11 课后自测 (Post-Lesson Quiz)

  1. 研究者对一个含 15,000 个背景基因的实验数据进行 GO ORA。某 GO term 在背景中有 200 个基因，在 500 个 DEGs 中有 30 个基因命中。这个富集是否可能显著？请写出需要计算的统计量（无需算出数值）。

  2. 两位研究者分析同一数据集，研究者 A 使用所有检测到的基因（15,000 个）作为背景，研究者 B 使用人类全基因组所有基因（20,000 个）作为背景。谁的 ORA 结果更可靠？为什么？

  3. GSEA 分析显示某通路的 NES = +2.3，padj = 0.001，但该通路的 leading edge 基因只有 8 个，且这 8 个基因恰好都是同一个蛋白质复合体的亚单位。这个结果应如何解读？是否应该在摘要中报告为”该通路被显著激活”？

  4. 你使用 clusterProfiler 的 simplify() 函数（cutoff = 0.7）对 GO BP ORA 结果去冗余，从 120 个显著 terms 减少到 35 个。这个过程的生物学依据是什么？能否会漏掉重要的生物学信息？

  5. 对于一个疾病与健康对照的转录组研究，共识别了 3,000 个 DEGs，但经 ORA 分析后没有发现任何显著富集的通路（所有 padj > 0.1）。请列举至少三种可能的原因，以及对应的诊断方法。

参考答案 (Reference Answers)

  1. 需要计算的统计量是超几何检验的 p 值：\(N = 15000\)，\(K = 200\)，\(n = 500\)，\(k = 30\)。计算公式为 \(p = \sum_{i=30}^{\min(500,200)} \frac{\binom{200}{i}\binom{14800}{500-i}}{\binom{15000}{500}}\)。期望值为 \(n \times K/N = 500 \times 200/15000 \approx 6.67\)，实际观测 30 远超期望，富集倍数约 4.5，很可能显著。但还需经 BH 校正与其他 GO terms 比较后才能确定最终显著性。

  2. 研究者 A（背景 = 实验检测基因，15,000 个）更可靠。使用全基因组 20,000 个基因作为背景，引入了实验无法检测的基因（如在该细胞类型中不表达的基因），这些基因按定义不可能出现在 DEG 列表中，但却虚增了 \(N-K\)（背景中不属于该 GO term 的基因数），导致期望重叠值降低，富集倍数虚高，p 值偏乐观（假阳性增加）。

  3. 虽然统计上显著，但应谨慎解读。NES = +2.3 且 padj 极低，说明该通路的基因整体在排序列表顶端聚集。然而 leading edge 仅 8 个基因且属于同一蛋白复合体，意味着整个通路的富集信号实际上由单一功能模块驱动，而非通路的多个功能节点均受影响。建议在报告中具体说明是”该通路中 XX 复合体相关基因被显著上调”，而非笼统陈述”通路被激活”——后者可能误导读者认为通路的全部功能组分均受影响。

  4. simplify() 通过计算 GO terms 间的语义相似度（基于共享祖先节点的程度），将高度相似（相似度 > 0.7）的 terms 合并，保留最具统计意义的那个。生物学依据是 GO 的层级冗余性：子 term 和父 term 常常共享大部分基因，去冗余能减少视觉上的干扰，突出独立的生物学信号。潜在风险：若两个相似 term 对应不同的细胞类型或亚过程，合并可能丢失这种区分信息。建议在最终结论中补充检查被合并的 terms，确认无重要信息丢失。

  5. 可能原因及诊断方法：（1）DEGs 分散在多个通路中，每个通路的命中基因数低于阈值 → 改用 GSEA（无需阈值）；（2）背景基因集设置不当（如使用了整个基因组）→ 改用实验检测基因作为背景；（3）所用功能数据库对该疾病/组织的注释覆盖度低 → 尝试不同数据库（如 Reactome、WikiPathways）；（4）差异基因筛选阈值过严，遗漏了生物学重要但未达统计阈值的基因 → 放宽 LFC 阈值后重跑 ORA；（5）实验设计问题（如批次效应主导变异）→ 检查 PCA 图，评估主要变异来源。
