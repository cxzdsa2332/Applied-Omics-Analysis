35  Visium 技术原理与上游处理

Published

June 22, 2026

36 专题概述

scRNA-seq 在解离组织时丢失了细胞的空间位置信息——我们知道有哪些细胞类型，却不知道它们在组织中如何排布。空间转录组学（Spatial Transcriptomics）填补了这一空白：它在保留组织切片二维坐标的前提下测量基因表达，回答”基因在哪里表达”的问题。10x Genomics Visium 是目前应用最广的空间转录组平台之一。本专题讲解 Visium 的空间条形码阵列原理、Space Ranger 上游处理流程，以及空间数据的加载与初步可视化，并厘清 Visium 与 scRNA-seq 在数据结构上的关键异同。

学习目标：理解 Visium 空间条形码捕获阵列的结构与 Spot 分辨率的局限；掌握 Space Ranger 上游处理流程与图像配准的作用；理解 Visium 数据格式与空间坐标信息的组织方式；能加载 Visium 数据并完成标志基因的空间表达可视化。

37 Visium 技术原理

37.1 空间条形码捕获阵列

Visium 的核心是一张预先印刷了空间条形码捕获阵列的载玻片。阵列上排列着数千个捕获点（Spot），每个 Spot 直径 55 µm、中心间距 100 µm，呈六角形网格排布；每张标准切片约含 4992 个捕获 Spot。关键在于——同一 Spot 上的所有捕获探针共享一段该 Spot 特有的空间条形码，而不同 Spot 的条形码各不相同。因此条形码与载玻片上的物理坐标一一对应，构成了”分子地址簿”。

{{Wiki:Spatial_transcriptomics}}

实验流程：将组织切片铺在阵列上，固定、染色（苏木精-伊红 H&E）并拍摄高分辨率图像；随后透化处理使细胞内 mRNA 释放并扩散，被最近的 Spot 探针上的 poly-T 捕获；逆转录生成带空间条形码的 cDNA。测序后，依据空间条形码即可把每条读段定位回它在组织上的来源 Spot。

37.2 Spot 是混合信号，而非单细胞

Visium 与 scRNA-seq 最本质的区别在于分辨率。55 µm 直径的 Spot 通常涵盖 1–10 个细胞（取决于组织的细胞密度），因此每个 Spot 的转录谱是其覆盖的多个细胞的混合信号，而非单细胞信号。这一点决定了 Visium 数据分析的诸多特殊性，见 表 1。

“空间域”不等于”细胞类型”： 直接对 Visium 的 Spot 聚类，得到的是转录特征相似且空间毗邻的组织区域，称为空间域（Spatial Domain），如脑的不同皮层、肿瘤的核心区与边缘区。它反映的是组织的功能分区，而非单一细胞类型。要从 Spot 中还原各细胞类型的比例，需要借助 scRNA-seq 参考做细胞解卷积（专题 7.5），这是 Visium 分析区别于 scRNA-seq 的核心环节。

38 Space Ranger 上游处理

38.1 处理流程与图像配准

Space Ranger 是 Visium 数据的官方上游软件，其核心命令 spaceranger count 在 Cell Ranger 的条形码处理、比对、UMI 计数基础上，增加了一个关键环节——把测序数据与组织图像对齐。它读取 H&E 显微图像，借助载玻片四角的基准标记（Fiducial frame）确定阵列在图像中的精确位置，再将每个 Spot 的表达量与其在图像上的像素坐标关联起来。

# Space Ranger: quantify a Visium slide and register it to the H&E image
spaceranger count \
    --id=visium_brain \
    --transcriptome=/path/to/refdata-gex-mm10-2020-A \
    --fastqs=/path/to/visium_fastqs \
    --image=/path/to/tissue_HE_image.tif \
    --slide=V19L01-041 \
    --area=A1 \
    --localcores=8 \
    --localmem=64

--slide 与 --area 指定所用载玻片的型号与捕获区，使 Space Ranger 能加载正确的 Spot 坐标布局。组织自动检测（tissue detection）会区分被组织覆盖的 Spot 与空白背景 Spot，仅保留前者。

图像质量直接影响下游： H&E 图像的对焦、曝光与组织完整性会影响 Space Ranger 的组织检测与 Spot 配准精度。若自动组织检测出错（漏检或误检 Spot），可借助 10x 的 Loupe Browser 手动框选组织区域、调整 Spot 选择，再重新运行。务必在分析前用 Loupe Browser 目视核对配准结果，避免错位的坐标污染全部空间分析。

38.2 输出与数据格式

Space Ranger 的输出在 Cell Ranger 三件套（表达矩阵）之外，额外包含一个 spatial/ 目录，存放空间相关信息：

  • tissue_positions.csv：每个 Spot 的条形码、是否在组织内、阵列行列号、图像像素坐标
  • tissue_hires_image.png / tissue_lowres_image.png：不同分辨率的组织图像
  • scalefactors_json.json：图像像素与 Spot 实际尺寸的换算因子

这一结构使 Visium 数据能被 Seurat 与 Scanpy/Squidpy 直接读取。

39 加载与初步可视化

39.1 加载 Visium 数据

下面使用小鼠脑 Visium 数据（stxBrain，前脑切片）演示加载与可视化。Seurat 的空间对象同时持有表达矩阵与每个 Spot 的图像坐标，使表达可以直接绘制在组织背景上。

运行环境说明： 本节依赖 R 包 Seurat（v5）与 SeuratData。stxBrain 数据集每个切片约 130 MB，含小鼠脑前/后切片。首次使用需 SeuratData::InstallData("stxBrain")。Load10X_Spatial() 也可直接读取 Space Ranger 的输出目录。

library(Seurat)
library(SeuratData)

# Load a Visium mouse brain (anterior) section as a spatial Seurat object
brain <- LoadData("stxBrain", type = "anterior1")

# Spatial QC: in Visium the analog of percent.mt still applies per Spot
brain[["percent.mt"]] <- PercentageFeatureSet(brain, pattern = "^mt-")
brain

39.2 标志基因的空间表达

Visium 分析最直观的输出是把基因表达画在组织切片上。SpatialFeaturePlot() 将每个 Spot 按指定基因的表达量着色，叠加在 H&E 图像上，使空间表达模式一目了然。

#| label: fig-spatial-marker
#| fig-cap: "小鼠脑前切片中经典脑区标志基因的空间表达（stxBrain Visium 数据）。每个 Spot 按基因表达量着色并叠加在组织切片图像上。海马区标志基因 Hpca 与皮层标志基因等呈现出与脑解剖结构高度吻合的空间分布，直观展示了空间转录组保留位置信息的优势。"
library(Seurat)

# Visualize where specific genes are expressed across the tissue section
SpatialFeaturePlot(brain, features = c("Hpca", "Ttr"))
# Total counts also vary spatially -- visualize the per-Spot library size
SpatialFeaturePlot(brain, features = "nCount_Spatial")

黄金法则： 空间数据的最大价值在于把分子信息嵌入形态学背景——任何空间分析的结论都应回到组织图像上核验。一个看似显著的空间表达模式，若与已知的组织解剖结构（皮层分层、海马形态、肿瘤边界）不吻合，往往提示配准错误、组织折叠或技术伪影，而非真实生物学。始终把”分子模式”与”H&E 形态”并置检查，是空间分析可靠性的基本保障。

40 方法总结与衔接说明

本专题介绍了 Visium 空间转录组的技术原理与上游处理：空间条形码阵列如何把表达量与组织坐标绑定、Space Ranger 如何完成图像配准、空间数据如何加载与可视化。理解 Visium 的核心特性——Spot 是多细胞混合信号、聚类得到的是空间域而非细胞类型——是后续分析的前提。专题 6.5 将讲解空间数据的质控、归一化，以及空间转录组特有的核心分析：空间高变基因（SVG）识别与空间聚类。而从 Spot 混合信号中还原细胞类型比例的细胞解卷积，将留待第7章（专题 7.5）展开。

41 课后自测 (Post-Lesson Quiz)

  1. Visium 的空间条形码捕获阵列如何把基因表达量与组织上的物理坐标关联起来？同一 Spot 上的探针条形码有何特点？

  2. Visium 的 Spot 直径为 55 µm，这一尺度对数据的本质意味着什么？为什么说 Spot 是”混合信号而非单细胞信号”？

  3. Space Ranger 相比 Cell Ranger 增加了哪个关键处理环节？基准标记（Fiducial frame）在其中起什么作用？

  4. 什么是”空间域”（Spatial Domain）？为什么直接对 Visium Spot 聚类得到的是空间域而非细胞类型？

  5. Space Ranger 输出的 spatial/ 目录包含哪些关键文件？它们各自的作用是什么？

  6. 为什么在分析前要用 Loupe Browser 目视核对图像配准结果？配准错误会导致什么后果？

  7. SpatialFeaturePlot() 与 scRNA-seq 中的 FeaturePlot() 在可视化逻辑上有何根本区别？

  8. 某研究者在 Visium 脑切片上发现某基因呈现一个清晰的高表达区块，但该区块与 H&E 图像上的任何已知解剖结构都不吻合，且边界笔直。请推测可能的原因，并说明应如何核验。

参考答案 (Reference Answers)

  1. 阵列上每个 Spot 的所有捕获探针共享一段该 Spot 特有的空间条形码，不同 Spot 条形码各不相同，因此条形码与载玻片上的物理坐标一一对应。组织 mRNA 被最近 Spot 的探针捕获并逆转录为带空间条形码的 cDNA，测序后依据条形码即可把读段定位回其来源 Spot 的坐标，从而把表达量与组织位置关联。

  2. 55 µm 通常涵盖 1–10 个细胞（取决于组织密度），因此每个 Spot 测到的是其覆盖的多个细胞的混合转录谱，而非单个细胞。这意味着 Visium 的观测单元是亚组织级而非单细胞级，无法直接通过聚类得到单一细胞类型，需要借助 scRNA-seq 参考做细胞解卷积才能还原细胞类型比例。

  3. Space Ranger 增加了把测序数据与组织 H&E 图像对齐（配准）的环节。基准标记（Fiducial frame，载玻片四角的标记）用于确定阵列在图像中的精确位置，使每个 Spot 的表达量能与其在图像上的像素坐标正确关联，是空间定位准确的前提。

  4. 空间域是转录特征相似且空间毗邻的组织区域（如皮层分层、肿瘤核心/边缘）。直接对 Spot 聚类得到空间域而非细胞类型，是因为每个 Spot 是多细胞混合信号，聚类反映的是组织的功能分区而非单一细胞身份；要得到细胞类型比例须做解卷积。

  5. 关键文件：tissue_positions.csv（每个 Spot 的条形码、是否在组织内、阵列行列号与图像像素坐标）、tissue_hires/lowres_image.png（不同分辨率组织图像）、scalefactors_json.json（图像像素与 Spot 实际尺寸的换算因子）。它们共同提供了把表达矩阵映射回组织图像所需的坐标与比例信息。

  6. 因为图像质量与自动组织检测可能出错（漏检、误检或错位 Spot），而错位的坐标会污染所有下游空间分析。Loupe Browser 可手动核对并框选组织区域、调整 Spot 选择。若配准错误未被发现，所有”空间模式”都建立在错误坐标上，结论不可信。

  7. FeaturePlot() 把基因表达画在 UMAP 等抽象的降维嵌入上，坐标无物理意义；SpatialFeaturePlot() 把表达量画在真实的组织切片图像上、按每个 Spot 的物理坐标着色，坐标对应组织上的实际位置。前者展示表达与细胞状态的关系，后者展示表达的空间分布。

  8. 边界笔直、且不吻合任何解剖结构，强烈提示技术伪影而非生物学信号——可能原因包括：组织折叠/褶皱、捕获阵列局部缺陷、图像配准错位导致某区块 Spot 坐标整体偏移，或透化/捕获效率的空间不均匀。核验方法：在 H&E 图像上叠加该区块、检查是否对应组织折叠或损伤；核对 Space Ranger 配准与 Loupe Browser 中该区域的 Spot 是否错位；查看该区块的总 UMI、线粒体比例等 QC 指标是否异常。确认为伪影后应在质控中剔除相关 Spot。
