Published

21.1 学习目标

完成本专题后，学生应能够：

  1. 区分 poly-A 富集与 rRNA 去除两种 RNA-seq 文库策略的适用场景与覆盖度差异；
  2. 解释链特异性文库的 dUTP 构建原理，并使用 infer_experiment.py 确认链方向；
  3. 解读 FastQC 报告中的六个核心质控模块，识别 RNA-seq 特有的异常模式；
  4. 编写 fastp 修剪命令并正确设置 Adapter 和质量过滤参数；
  5. 使用 MultiQC 生成多样本质控汇总报告并识别批次异常样本。

21.2 从 Microarray 到 RNA-seq：范式转变

在 RNA-seq 出现之前，转录组研究的主流工具是微阵列（Microarray）。微阵列依赖于已知序列的探针杂交，只能检测预先设计好的转录本，动态范围有限（约 2 个数量级），且在低表达基因和高同源性基因上信噪比差。2008 年，多个研究组几乎同时发表了使用高通量测序直接测量 RNA 分子的方法，即 RNA-seq。这一范式转变带来了质的飞跃：RNA-seq 不依赖先验序列知识，能够发现新的转录本、检测可变剪接事件，动态范围扩展至 4–5 个数量级，并且理论上可以达到单碱基分辨率。

历史注记： RNA-seq 的早期代表性论文包括 Mortazavi et al.（2008，Nature Methods）对小鼠肌肉组织的 mRNA 测序，以及 Wang et al.（2008，Nature）对人类转录组可变剪接的系统分析。这两篇论文奠定了 RNA-seq 分析的基本框架，至今仍是该领域引用最高的文献之一。

尽管技术进步迅速，RNA-seq 的核心分析流程在过去十余年间已趋于成熟和标准化。理解这一流程的每个步骤背后的原理，不仅有助于正确执行分析，更重要的是能在遇到异常数据时准确定位问题来源。

21.3 Illumina 测序化学：边合成边测序

理解 RNA-seq 数据质量问题的根源，必须先了解 Illumina 测序仪的工作原理。Illumina 采用的核心技术称为边合成边测序（Sequencing by Synthesis, SBS），其本质是在 DNA 聚合反应过程中，通过检测每次掺入的荧光标记碱基来实现序列读取。

21.3.1 流动槽与簇扩增

测序在玻璃芯片（Flow Cell）上进行。Flow Cell 表面密布寡核苷酸引物，文库片段通过接头序列与之杂交后，经过桥式扩增（Bridge Amplification）在原位形成约 1000 个拷贝的簇（Cluster）。每个簇来自同一 DNA 分子，确保信号足够强以被光学系统检测。

21.3.2 测序循环与质量分数的产生

每个测序循环（Cycle）分为三步：（1）加入四种携带可逆终止基团和不同荧光染料的 dNTP，聚合酶将其中与模板互补的一种掺入到正在延伸的链上；（2）光学系统扫描 Flow Cell，记录每个簇的荧光信号，判断掺入的碱基种类；（3）化学切除荧光基团和终止基团，恢复 3’-OH，进入下一循环。

碱基的质量分数反映了测序仪对该次碱基判断的置信程度。随着循环数增加，簇内各拷贝的相位同步性逐渐下降（相位漂移，phasing），导致信号混叠，质量分数自然下降——这正是 FastQC 中 read 末端质量降低的根本原因（Phred 分数的数学定义参见本专题后续章节）。

图1.1：Illumina SBS 原理。四种 dNTP 各携带不同荧光染料与可逆终止基团，聚合酶掺入后由摄像系统记录荧光颜色，完成碱基判读。来源：Wikipedia, Illumina dye sequencing。

双端测序（Paired-End Sequencing）的原理： 完成第一轮测序（Read 1）后，测序仪对原始链去杂交，以另一端的引物重新扩增簇，从片段的另一端开始第二轮测序（Read 2）。双端数据能够提供片段大小信息，对可变剪接检测和结构变异分析尤为重要。

21.4 RNA-seq 文库构建：两条技术路线

21.4.1 poly-A 富集（mRNA-seq）

poly-A 富集文库利用 oligo-dT 磁珠捕获含有 poly-A 尾的成熟 mRNA。由于真核细胞总 RNA 中约 80% 是核糖体 RNA（rRNA），直接建库会导致测序资源被 rRNA 大量占用。poly-A 富集有效去除 rRNA，使 mRNA 分子富集至总 reads 的 90% 以上。

优势：操作简便，成本低，信噪比高。

局限性：

  • 只能捕获含 poly-A 尾的 RNA，遗漏组蛋白 mRNA（无 poly-A 尾）和大多数非编码 RNA；
  • oligo-dT 对 mRNA 3’ 端的结合亲和力高于 5’ 端，导致转录本的 3’ 端覆盖度偏高，称为3’ 端偏差（3’ bias）；
  • RNA 质量要求高：降解的 RNA（RIN < 7）会加剧 3’ 端偏差，导致 5’ 端几乎无覆盖。

21.4.2 rRNA 去除（Total RNA-seq / Ribo-Zero）

rRNA 去除文库通过与 rRNA 互补的探针捕获并去除 rRNA，保留其余全部 RNA 分子，包括 pre-mRNA、lncRNA、circRNA、组蛋白 mRNA 等。

优势：覆盖全转录组，适合研究非编码 RNA；对 RNA 质量要求相对宽松；覆盖度更均匀。

局限性：成本较高；低表达 mRNA 的检出灵敏度略低于 poly-A 富集方法（因为非 mRNA 分子占用了部分测序资源）。

选择建议： 若研究目标明确为蛋白质编码基因的差异表达，且 RNA 质量良好（RIN ≥ 8），优先选择 poly-A 富集；若需要研究非编码 RNA、RNA 降解样本，或希望覆盖更完整的转录组，选择 rRNA 去除方案。本章使用的 airway 数据集（Himes et al. 2014，SRR1039508–SRR1039517）为 poly-A 富集文库。

21.4.3 链特异性文库

常规 RNA-seq 文库在逆转录后会丢失链方向信息——测序 read 无法判断来自正义链（sense strand）还是反义链（antisense strand）。对于有大量正反链重叠基因或需要区分正反义转录的研究，链方向信息至关重要。

目前最主流的链特异性文库构建方法是 dUTP 法（TruSeq Stranded 采用此方案），流程如下：

  1. mRNA 随机打断；
  2. 逆转录合成第一链 cDNA（方向为 3’→5’，与 mRNA 反向互补）；
  3. 以第一链为模板，合成含 dUTP 而非 dTTP 的第二链 cDNA；
  4. 用尿嘧啶-DNA 糖基化酶（UDG）降解含 dUTP 的第二链；
  5. 剩余第一链加接头，测序。

由于测序的 read 来自第一链（反义链），read 2（R2）与原始 mRNA 方向相同，read 1（R1）与 mRNA 反向互补，即所谓的 RF 链方向（Illumina TruSeq Stranded 标准）。

图1.2：RNA-seq 实验全流程。来源：Wikipedia, RNA-Seq。

21.6 FASTQ 文件格式详解

在进行任何质控或比对分析前，必须理解 FASTQ 文件的基本结构。FASTQ 是 RNA-seq 原始数据的通用存储格式，每条 read 由严格的四行组成：

@SRR1039508.1 1/1
GATCGGAAGAGCACACGTCTGAACTCCAGTCACATCACGATCTCGTATGCCGTCTTCTGCTTGAAAAAAAAAA
+
JJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJ
  • 第 1 行：以 @ 开头的 read 标识符，包含样本 ID、read 编号及测序仪信息；
  • 第 2 行：碱基序列（A/T/C/G/N），长度即 read 长度；
  • 第 3 行：固定为 +，可重复第 1 行标识符（现代数据通常省略）；
  • 第 4 行：与第 2 行等长的质量字符串，每个字符对应一个碱基的 Phred 质量分数。

21.6.1 Phred+33 编码

质量字符串采用 Phred+33 编码（Sanger 编码，自 Illumina 1.8+ 起为通用标准）：

\[Q_{\text{ASCII}} = Q + 33\]

例如，字符 J 的 ASCII 码为 74，对应 \(Q = 74 - 33 = 41\)，即错误率约为 \(10^{-4.1} \approx 0.008\%\)。字符 !（ASCII 33）对应 \(Q = 0\)，代表完全不可信的碱基（Phred 分数的完整数学推导参见本专题”RNA 完整性与样本质量评级”章节）。

快速估算技巧： 在 FASTQ 质量字符串中，I 对应 Q40（准确率 99.99%），5 对应 Q20（准确率 99%），+ 对应 Q10（准确率 90%）。凡出现大量 ! 或 " 字符的 read，几乎可判定为低质量序列。

21.6.2 压缩与文件大小

原始 FASTQ 文件极大：airway 数据集单个样本约 3–5 GB（未压缩）。实际存储和传输均使用 gzip 压缩（.fastq.gz），压缩比约 4:1，使文件缩减至约 1 GB。所有主流工具（FastQC、fastp、STAR、HISAT2）均可直接处理 .fastq.gz 格式，无需解压。

21.7 RNA 完整性与样本质量评级

在送测之前，RNA 样本的完整性（Integrity）是决定数据质量的关键先决条件。降解的 RNA 在 poly-A 富集建库时会产生严重的 3’ 端偏差，甚至导致整个实验失败。

21.7.1 RNA 完整性评分（RIN）

RNA 完整性数字（RNA Integrity Number, RIN）是由 Agilent Bioanalyzer 或 Fragment Analyzer 根据 28S/18S rRNA 峰比值与降解特征自动计算的 1–10 评分：

RIN 值 完整性描述 适用场景
9–10 完整，无降解 所有建库方法
7–8 轻度降解 poly-A 富集可接受
5–6 中度降解 推荐 rRNA 去除；总 RNA 分析
< 5 严重降解 仅适合 3’ 端定向建库或放弃

RIN 的局限性： RIN 基于 rRNA 完整性推断 mRNA 完整性，但两者降解速率不同。植物样本（叶绿体 rRNA 比例高）和 FFPE（福尔马林固定石蜡包埋）样本的 RIN 解读需要额外谨慎。FFPE 样本通常 RIN < 3，但用 DV200（> 200 nt 片段的比例）评估更适合。

21.7.2 Phred 质量分数的数学基础

Illumina 测序仪为每个碱基分配一个 Phred 质量分数 \(Q\)，定义为：

\[Q = -10 \log_{10} P_{\text{error}}\]

其中 \(P_{\text{error}}\) 是该碱基被错误识别的概率。常用对应关系：

Q 值 错误率 碱基准确率
Q10 10% 90%
Q20 1% 99%
Q30 0.1% 99.9%
Q40 0.01% 99.99%

在 FASTQ 文件中，每个碱基的 Q 值用单个 ASCII 字符编码（Phred+33 编码）：实际字符 = chr(Q + 33)。例如，I 对应 ASCII 73，Q = 73 - 33 = 40。

21.8 原始数据质量控制

21.8.1 FastQC：单样本质量评估

FastQC 是 RNA-seq 质控的标准起点，对 FASTQ 文件执行 10 余项统计检测。以下是 RNA-seq 数据最关键的六个模块：

1. Per Base Sequence Quality

展示每个循环（cycle）位置上碱基质量分数的箱形图分布。RNA-seq 与 DNA-seq 的正常模式相同：read 末端（3’ 端）质量通常低于 5’ 端，这是信号强度随测序循环递减的结果。

  • 绿色区域（Q ≥ 28）：合格；
  • 黄色区域（20 ≤ Q < 28）：边界；
  • 红色区域（Q < 20）：不合格，提示需要截短。

2. Per Tile Sequence Quality（仅 Illumina）

显示流动槽（Flow Cell）上各 Tile 位置的质量。异常的 Tile 图案（蓝色代表高质量，橙/红代表低质量）提示测序过程中的物理问题（气泡、污染、激光问题），通常无法通过过滤解决。

3. Per Sequence GC Content

正常 RNA-seq 数据应呈现与物种基因组 GC 含量一致的单峰正态分布。RNA-seq 特有的异常模式：

  • 双峰或宽分布：提示 rRNA 污染（rRNA 的 GC 含量与 mRNA 显著不同）或样本混合；
  • 窄高峰偏移：提示 Adapter 二聚体（Adapter dimer）污染，通常位于 GC 约 50% 的位置。

4. Sequence Duplication Levels

显示各重复度级别的 reads 比例。RNA-seq 数据天然存在较高的重复率，因为高表达基因会产生大量相同序列的 cDNA，这与 DNA 测序中重复暗示 PCR 扩增偏差的情形不同。因此，RNA-seq 不建议对重复 reads 进行常规去除（除非有 UMI 标记）。FastQC 的 Duplicate Levels 模块对 RNA-seq 通常显示 WARN 或 FAIL，可忽略。

5. Overrepresented Sequences

列出出现频率超过 0.1% 的序列及其可能来源。需要关注的异常：

  • 未被去除的 Illumina Universal Adapter 序列（AGATCGGAAGAGC...）；
  • poly-A 或 poly-T 序列（提示 Read-through 发生）；
  • 特定基因的 3’ 端序列（严重 3’ 偏差时高表达基因的 3’ 末端大量重复）。

6. Adapter Content

直接检测各循环位置上 Adapter 序列的比例。若在 read 末端 20–30 bp 处出现 Adapter 信号，说明 Insert Size 小于 read 长度，发生了 Read-through，需要进行 Adapter 修剪。

21.8.2 测序深度与读长：实验设计关键参数

在分析数据之前，需要了解实验设计本身的两个核心参数——测序深度（Sequencing Depth）和读长（Read Length），因为它们直接决定了数据的分析能力上限，也影响质控指标的解读方式。

测序深度指每个样本的 reads 总数。不同研究目标对深度的需求差异显著：

研究目标 推荐深度（双端） 说明
基因差异表达（高表达基因为主） 20–30 M reads DESeq2/edgeR 标准分析
低表达基因与 lncRNA 检测 50–80 M reads 提高低丰度转录本的检测灵敏度
可变剪接事件分析 100–200 M reads 覆盖度均匀性要求高
单细胞 RNA-seq（每细胞） 5,000–20,000 reads 远低于 bulk，但细胞数多

读长影响比对精度与可变剪接检测能力。75 bp 读长足以用于基因水平定量；150 bp 读长在覆盖转录本时信息量更大，对外显子跨越连接（exon-exon junction）的检测更准确，是目前 bulk RNA-seq 的主流选择。airway 数据集（Himes et al. 2014）采用 75 bp 单端或双端设计，其读长限制意味着某些长外显子的可变剪接分析精度会低于 150 bp 数据。

深度饱和度检验： 实际项目中建议在大规模测序前进行测序深度饱和度分析（Rarefaction Analysis）：对同一样本以不同 reads 数进行下采样，观察检出基因数与差异基因数随深度的增长曲线，找到边际效益递减的拐点，即为最优测序深度。

21.8.3 fastp：Adapter 修剪与质量过滤

fastp 是目前速度最快、功能最全面的 FASTQ 预处理工具，集 Adapter 检测、质量过滤、低复杂度过滤于一体，并输出内置 HTML 质控报告。

# fastp for paired-end RNA-seq — using 1M-read teaching subset
# Input files are located in the local bio_example directory
DATA=/Users/angdong/Documents/bio_example/bulkRNA

fastp \
  --in1 ${DATA}/SRR1039508_1_1M.fastq.gz \
  --in2 ${DATA}/SRR1039508_2_1M.fastq.gz \
  --out1 SRR1039508_1_trimmed.fastq.gz \
  --out2 SRR1039508_2_trimmed.fastq.gz \
  --json SRR1039508_fastp.json \
  --html SRR1039508_fastp.html \
  --detect_adapter_for_pe \
  --trim_poly_x \
  --qualified_quality_phred 20 \
  --length_required 36 \
  --thread 8

# Key parameters explained:
# --detect_adapter_for_pe  : auto-detect adapter sequences for PE data
# --trim_poly_x            : trim poly-A/T/G/C tails (RNA-seq specific)
# --qualified_quality_phred: minimum base quality threshold (Q20)
# --length_required        : discard reads shorter than 36 bp after trimming

参数选择逻辑：

  • --detect_adapter_for_pe：自动检测时 fastp 会从数据中推断接头序列，适合不确定接头类型的情况；若已知接头（如 Illumina TruSeq：AGATCGGAAGAGCACACGTCTGAAC），可用 --adapter_sequence 明确指定，速度更快且更可靠；
  • --trim_poly_x：去除 read 末端的 poly-A/T 尾，这在 RNA-seq 中尤为重要；
  • --qualified_quality_phred 20：去除质量低于 Q20 的碱基，对应错误率 1%；
  • --length_required 36：修剪后过短的 reads 对比对没有贡献，反而增加多比对（multimapping）风险。

黄金法则：先质控，后比对。 始终在 fastp 修剪后再次运行 FastQC，确认质控效果。FastQC-fastp-FastQC 的三步流程是 RNA-seq 质控的标准操作规程（SOP）。

21.9 链方向的生物信息学确认

在开始比对前，必须确认文库的链特异性方向，因为后续定量步骤（featureCounts 的 -s 参数、Salmon 的 --libType 参数）需要明确指定。错误的链方向设置会导致约 50% 的 reads 被丢弃或错误计数。

概念辨析：链方向确认 ≠ 正式比对。 链方向确认是正式比对之前的一步预检工作，目的是确定文库的链方向参数，以供后续工具调用。其本质是对少量 reads（通常 1M 条）做一次临时比对，获得 BAM 文件供 infer_experiment.py 检测，检测完即可丢弃该 BAM。正式的全样本比对（用于计数与定量）在专题 1.2 中进行，规模完全不同。将链方向确认与正式比对混为同一步骤，会在概念上造成混乱。

推荐工具为 RSeQC 的 infer_experiment.py。该工具从比对后的 BAM 文件中随机抽取若干 reads，与参考注释中基因的转录方向比对，统计 read 方向与基因注释一致和不一致的比例，从而推断文库的链特异性。实践中只需对一个代表性样本完成此流程即可；若批次差异较大，可各抽一个样本分别确认。以下以 airway 数据集（SRR1039508，人类气道平滑肌细胞）为例演示。

21.9.4 链方向判读规则

infer_experiment.py 输出特征 链方向 featureCounts Salmon libType
两组比例均约 0.47–0.50 非链特异性（unstranded） -s 0 IU / U

21.11 批次效应的早期检测

批次效应（Batch Effect）是 RNA-seq 实验中最常见的技术干扰来源之一，指由于样本在不同时间、不同操作人员、不同试剂批次或不同测序机器上处理，引入的与生物学处理无关的系统性变异。批次效应如果未被识别和校正，会产生大量假阳性差异基因，严重时可完全掩盖真实的生物学信号。

历史案例： 2010 年 Nature Methods 上发表了一篇影响深远的文章，展示了两篇顶级期刊（Nature、Science）的 TCGA 癌症基因组数据中存在显著的批次效应，部分”发现”的分子亚型实际上是不同测序批次的人工产物。这一事件推动了批次效应校正成为 RNA-seq 分析的标准步骤。

21.11.1 批次效应的来源

  • 文库制备批次：不同日期制备的文库，试剂状态和操作细节差异；
  • 测序批次：不同 Flow Cell、不同 Lane 之间的系统误差；
  • RNA 提取批次：提取试剂盒批次、RIN 差异；
  • 实验操作人员：不同人员的操作差异；
  • 样本保存方式：新鲜组织 vs 冻存 vs FFPE。

21.11.2 实验设计：从源头规避批次效应

批次效应最有效的应对策略是在实验设计阶段预防，而非事后校正。事后校正始终有信息损失；而设计良好的实验即使存在批次效应，也能通过统计模型准确估计处理效应。

核心原则：避免处理组与批次完全混淆（Complete Confounding）

最常见的错误是将所有处理组样本在同一批次制库/测序，所有对照组样本在另一批次。这种设计使批次效应与处理效应完全重叠，任何统计方法都无法区分两者。

❌ 错误设计（完全混淆）          ✓ 正确设计（平衡分组）

批次 1：treated_1  treated_2     批次 1：treated_1  untreated_1
批次 2：untreated_1 untreated_2  批次 2：treated_2  untreated_2
         ↑ 无法区分处理与批次               ↑ 批次效应可在模型中估计

具体操作建议

  1. 随机化上样顺序：不要将同一处理组的所有样本连续上机，应按随机顺序排列，将批次效应均匀分散到各处理组；
  2. 平衡区组设计（Blocked Design）：若不可避免多批次（如样本量大、跨周期采集），确保每个批次内各处理组的样本数相等，并在统计模型中将批次纳入设计矩阵（~ batch + condition）；
  3. 包含内部对照（Spike-in）：在每批文库中加入相同的 ERCC spike-in RNA，可定量批次间的系统偏差；
  4. 记录所有批次元数据：建立完整的样本元数据表，记录 RNA 提取日期、文库制备日期、操作员、试剂批号、测序 Lane 等，这些信息在下游分析中不可或缺。

黄金法则：将批次信息编码进实验设计，而非寄希望于事后校正。 Himes et al.（2014，即本教材使用的 airway 数据集）采用了配对细胞系设计——每个细胞系（N61311、N052611、N080611、N061011）各提供一对 untreated/treated 样本，细胞系差异可作为区组因子在模型中控制，是教科书级别的平衡设计范例。

21.11.3 PCA 检测批次效应

主成分分析（PCA）是检测批次效应最直观的方法。在 PCA 图中，用不同颜色标记处理组和批次：若 PC1/PC2 的分离方向与处理组一致，说明生物学信号主导；若分离方向与批次一致，说明批次效应超过了生物学效应。

以下代码使用 airway Bioconductor 包中的真实数据——即本教材所用 airway 数据集的 8 个双端样本（SRR1039508–SRR1039521，人类气道平滑肌细胞，dexamethasone 处理）：

library(airway)
library(DESeq2)
library(ggplot2)

# Load real data: Himes et al. 2014, 8 paired-end samples
data(airway)

# Build DESeqDataSet; cell line as batch covariate
dds <- DESeqDataSet(airway, design = ~ cell + dex)
dds <- dds[rowSums(counts(dds) >= 10) >= 3, ]   # filter low counts

# VST normalization (blind=TRUE: ignore design for QC purpose)
vsd <- vst(dds, blind = TRUE)

# Annotate batch: group 4 cell lines into 2 batches for illustration
colData(vsd)$batch <- factor(
  ifelse(colData(vsd)$cell %in% c("N61311", "N052611"),
         "batch1", "batch2")
)

# PCA: color by treatment, shape by batch
pca_data <- plotPCA(vsd, intgroup = c("dex", "batch"), returnData = TRUE)
pv <- round(100 * attr(pca_data, "percentVar"))

ggplot(pca_data, aes(x = PC1, y = PC2, color = dex, shape = batch)) +
  geom_point(size = 5) +
  xlab(paste0("PC1: ", pv[1], "% variance")) +
  ylab(paste0("PC2: ", pv[2], "% variance")) +
  scale_color_manual(values = c("untrt" = "#4575b4", "trt" = "#d73027"),
                     labels = c("untrt" = "Untreated", "trt" = "Dex-treated")) +
  labs(title = "PCA of airway dataset (VST normalized)",
       subtitle = "Color = treatment; Shape = batch (grouped cell lines)",
       color = "Treatment", shape = "Batch") +
  theme_classic(base_size = 13)

真实输出解读：PC1（42% 方差）将 untreated（负方向）与 treated（正方向）完全分离，说明地塞米松处理是最主要的变异来源，生物学信号主导数据结构。PC2（26% 方差）反映细胞系差异——这正是 Himes et al. 采用配对细胞系设计的价值所在：细胞系效应被捕捉在 PC2，而非混入 PC1 干扰处理效应的估计。

21.11.4 RLE 图：检测系统性偏移

相对对数表达量图（Relative Log Expression, RLE）是另一种有效的批次效应检测工具。对每个样本，计算每个基因的表达量与所有样本中该基因中位数的差值（对数尺度），绘制箱形图。正常情况下所有样本的中位线应接近 0；若某些样本整体偏移，说明存在系统性偏差。

library(matrixStats)
library(tidyr)

# RLE from raw (unnormalized) counts — reveals pre-normalization biases
counts_mat <- counts(dds)
rle_mat <- log2(counts_mat + 1) - rowMedians(log2(counts_mat + 1))

# Reshape for ggplot
rle_df <- as.data.frame(rle_mat)

21.12 实践：airway 数据集质控流程

以下代码演示对 airway 数据集（Himes et al. 2014）全部 10 个样本的完整质控流程，使用本地 1M-read 教学子集（PE 样本每端约 50 MB，SE 样本约 50 MB）。PE 与 SE 样本的 fastp 参数不同，需分别处理。

# === Environment setup ===
conda activate rnaseq-env  # requires: fastqc fastp multiqc

DATA=/Users/angdong/Documents/bio_example/bulkRNA
mkdir -p data/trimmed data/qc/fastqc/pre data/qc/fastqc/post data/qc/fastp

# === Paired-end samples ===
PE_SAMPLES=(SRR1039508 SRR1039510 SRR1039512 SRR1039514 SRR1039516)

# === Single-end samples ===
SE_SAMPLES=(SRR1039509 SRR1039511 SRR1039513 SRR1039515 SRR1039517)

# === Step 1: Pre-trimming QC (all 10 samples) ===
# PE: both R1 and R2
for SRR in "${PE_SAMPLES[@]}"; do
  fastqc ${DATA}/${SRR}_1_1M.fastq.gz ${DATA}/${SRR}_2_1M.fastq.gz \
    --outdir data/qc/fastqc/pre --threads 4
done
# SE: single file (no _1/_2 suffix in subset naming)
for SRR in "${SE_SAMPLES[@]}"; do
  fastqc ${DATA}/${SRR}_1M.fastq.gz \
    --outdir data/qc/fastqc/pre --threads 4
done

# === Step 2: Adapter trimming — PE samples ===
for SRR in "${PE_SAMPLES[@]}"; do
  fastp \
    --in1  ${DATA}/${SRR}_1_1M.fastq.gz \
    --in2  ${DATA}/${SRR}_2_1M.fastq.gz \
    --out1 data/trimmed/${SRR}_1.fastq.gz \
    --out2 data/trimmed/${SRR}_2.fastq.gz \
    --json data/qc/fastp/${SRR}_fastp.json \
    --html data/qc/fastp/${SRR}_fastp.html \
    --detect_adapter_for_pe \
    --trim_poly_x \
    --qualified_quality_phred 20 \
    --length_required 36 \
    --thread 8
done

# === Step 2: Adapter trimming — SE samples ===
for SRR in "${SE_SAMPLES[@]}"; do
  fastp \
    --in1  ${DATA}/${SRR}_1M.fastq.gz \
    --out1 data/trimmed/${SRR}.fastq.gz \
    --json data/qc/fastp/${SRR}_fastp.json \
    --html data/qc/fastp/${SRR}_fastp.html \
    --trim_poly_x \
    --qualified_quality_phred 20 \
    --length_required 36 \
    --thread 8
done

# === Step 3: Post-trimming QC ===
fastqc data/trimmed/*.fastq.gz \
  --outdir data/qc/fastqc/post --threads 8

# === Step 4: MultiQC summary report ===
multiqc data/qc/ \
  --outdir data/qc/multiqc \
  --filename airway_qc_report

关于子集文件的生成方式： 教学用 1M-read 子集由以下命令从完整数据截取，完整数据可从 ENA 下载（SRR1039508–SRR1039517，约 2 GB/样本）：

# Generate 1M-read subset from full FASTQ (4M lines = 1M reads × 4 lines/read)
gunzip -c SRR1039508_1.fastq.gz | head -4000000 | gzip > SRR1039508_1_1M.fastq.gz

# Stream-download and subset without saving full file (~1 GB saved per sample)
wget -q -O - ftp://ftp.sra.ebi.ac.uk/vol1/fastq/SRR103/006/SRR1039516/SRR1039516_1.fastq.gz | \
  gunzip -c | head -4000000 | gzip > SRR1039516_1_1M.fastq.gz

完成后，打开 data/qc/multiqc/airway_qc_report.html，结合下一节的解读指南进行逐项分析。

21.13.2 逐项解读

通过率（95.1–98.7%）

全部样本过滤通过率均超过 95%，表明原始数据质量良好，低质量碱基和 N 碱基比例极低。PE 样本（95.1–97.7%）通过率略低于 SE 样本（96.9–98.7%），原因在于 PE 数据需要 R1 和 R2 同时满足质量要求，单端失败即丢弃整对 reads。SRR1039514 通过率最低（95.1%），但仍在可接受范围内。

Q30 碱基率（94.5–97.8%）

所有样本修剪后 Q30 率均超过 94%。PE 样本整体高于 SE 样本（PE 均值 96.4% vs SE 均值 96.0%），这与 PE 数据通常来自更高上样浓度的文库有关。SRR1039513 的 Q30 率相对最低（94.5%），提示该样本测序质量略弱，但不影响后续分析。

GC 含量（48.6–50.0%）

全部 10 个样本 GC 含量高度一致，集中在 48.6–50.0%，符合人类转录组的预期分布（~49%）。无样本出现 GC 双峰或显著偏移，排除 rRNA 污染和样本混淆的可能性。

重复率（PE: 0.9–1.6%，SE: 16.6–17.8%）

PE 与 SE 样本的重复率存在显著差异，但这是正常现象：PE 重复需要 R1 和 R2 均完全相同，概率远低于 SE 的单端匹配。SE 样本的 ~17% 重复率在 63 bp 读长的 poly-A 富集 RNA-seq 中属于正常水平——高表达基因的 3’ 端片段大量重复是生物学现象而非 PCR 偏差（参见本专题 FastQC 节）。

接头残留（≤0.5%）

fastp 修剪后，所有样本接头含量均降至 0.5% 以下，修剪效果良好。部分 SE 样本在 multiqc_general_stats 中接头列显示为”—“，这是因为 fastp 对 SE 数据使用不同的接头检测策略，当接头比例极低时不写入该字段。

21.13.3 FastQC 模块状态异常

Per Tile Sequence Quality FAIL（SRR1039513、SRR1039515、SRR1039516）

上述三个样本在修剪前后的 FastQC Per Tile Sequence Quality 模块均显示 FAIL，提示这些样本在测序时特定 Flow Cell Tile 上出现了局部质量下降（可能由气泡、污染或激光问题引起）。由于全局 Q30 率（94.5–95.4%）仍在合理范围内，且模块失败仅涉及少数 Tile，不足以影响后续比对和定量分析。若这一问题在正式分析中出现，可通过 RSeQC 的 read_distribution.py 进一步排查比对模式是否异常。

Overrepresented Sequences WARN（SRR1039508、SRR1039510、SRR1039514 修剪前）

修剪前部分样本出现 Overrepresented Sequences 警告，主要来源为 Illumina TruSeq 接头序列（AGATCGGAAGAGC...）。fastp 修剪后，所有样本的该警告均消失，确认为接头污染而非生物学来源的过表达序列。

21.13.4 整体评估

airway 1M 教学子集的整体数据质量优良，所有 10 个样本均通过关键质控指标（通过率 >95%，Q30 >94%，GC 分布一致）。Per Tile FAIL 是该批数据的唯一值得关注的问题，但不影响后续分析流程。该数据集可直接进入比对步骤（专题 1.2）。

  7. 研究团队计划对 100 个人类 PBMC 样本进行 bulk RNA-seq，目标是检测差异表达基因并初步筛查 lncRNA 的表达变化。请根据本专题所学内容，说明应选择哪种文库构建方案，并给出测序深度和读长的推荐值及理由。

参考答案 (Reference Answers)

  1. poly-A 富集文库仅捕获含 poly-A 尾的成熟 mRNA，无法检测组蛋白 mRNA、lncRNA、circRNA 等非编码 RNA，适合研究蛋白质编码基因差异表达；rRNA 去除文库保留全部非 rRNA 分子，适合研究 lncRNA、circRNA 等非编码 RNA，以及使用降解 RNA 的样本（如 FFPE 组织）。

  2. 该样本为 RF-stranded（read 1 与 mRNA 反向互补，read 2 与 mRNA 同向），即 Illumina TruSeq Stranded 标准方向。featureCounts 应使用 -s 2（RF 方向）。

  3. RNA-seq 中高表达基因会产生大量具有相同序列的 cDNA 片段，这是生物学现象而非 PCR 扩增偏差。FastQC 无法区分生物学重复与 PCR 重复，因此对 RNA-seq 数据的重复率警告通常可以忽略。只有在有 UMI 标记的文库中，才能真正区分并去除 PCR 重复。

  4. poly-A 富集步骤有时会捕获 poly-A 本身延伸到 read 中（特别是 Insert Size 较短时），这些 poly-A 序列在基因组比对时无法有效映射，会显著降低比对率并产生软截断（soft-clipping）。修剪 poly-A 尾可提高比对率 1–5%，特别是在短 Insert Size 的样本中效果更明显。

  5. 可能原因：（1）这 2 个样本存在 rRNA 或 DNA 污染（GC 双峰）；（2）文库构建失败导致 Adapter 二聚体过多；（3）物种或组织来源不同（如样本混淆）；（4）RNA 降解严重导致短片段偏多。排查步骤：检查 FastQC 的 Overrepresented Sequences 模块确认污染类型；查看 RSeQC read_distribution 结果中各基因组区域的 reads 比例；若 rRNA 比例高，考虑重新建库。

  6. + 的 ASCII 码为 43，Phred 质量分数 Q = 43 − 33 = 10，对应碱基错误率 10%。由于 Q10 < Q20（fastp 阈值），该碱基将被标记为低质量；若此位置位于 read 末端且连续低质量碱基达到滑动窗口阈值，fastp 将从此处截短 read。

  7. 应选择 rRNA 去除方案（如 Ribo-Zero 或 RiboMinus），因为研究目标包括 lncRNA，而 poly-A 富集无法捕获大多数 lncRNA。测序深度推荐 50–80 M reads/样本（lncRNA 通常低表达，需要足够深度保证检出灵敏度）；读长推荐 150 bp 双端（PE150），以提高比对准确性和可变剪接检测能力。100 个样本建议分批次制备文库，并将批次信息记录在样本元数据中以便后续批次效应校正。
