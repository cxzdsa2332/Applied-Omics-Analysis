19  精细定位、功能注释与因果推断

20 专题 3.6：精细定位、功能注释与因果推断

20.1 学习目标

完成本专题学习后，学生应能够：

  1. 阐释 GWAS 精细定位（fine-mapping）的必要性及其与 LD 裁剪的本质区别；
  2. 理解贝叶斯精细定位方法（SuSiE、FINEMAP）的核心统计框架，解释可信集（Credible Set）的定义与生物学意义；
  3. 描述候选位点功能注释的标准流程：变异分类（ANNOVAR/VEP）、eQTL 查询（GTEx）、共定位分析（coloc）与表观基因组注释（ENCODE/Roadmap）；
  4. 描述孟德尔随机化（Mendelian Randomization，MR）的因果推断框架与三大工具变量假设，比较 IVW、MR-Egger、加权中位数方法的适用条件；
  5. 区分多基因风险评分（PRS）与单位点 GWAS 的应用场景，比较 C+T 与 LDpred2 方法；
  6. 解释多群体 GWAS 的科学意义与多群体精细定位的 LD 互补原理；
  7. 概述稀有变异聚合检验（SKAT、SKAT-O）的动机与统计策略；
  8. 阐释功能作图（Functional Mapping）的核心思想——将生物数学模型嵌入 QTL 似然函数，以动态曲线参数的基因型间差异量化遗传效应，理解 Logistic 均值模型与 AR(1) 协方差结构的数学意义及其相对于逐时间点 GWAS 的检验力优势。

20.2 精细定位的必要性

GWAS 中的显著信号通常对应一个 LD 区块，该区块内可能含有数十至数百个相互高度连锁的 SNP，其中只有少数（甚至仅一个）是真正的致因变异（causal variant）。精细定位（fine-mapping）的目标是在统计层面缩小因果候选变异的范围，为后续功能实验提供优先靶标。

精细定位 vs. LD 裁剪

LD 裁剪（clumping）识别的是统计独立的信号位点，目的是消除冗余；精细定位识别的是在已知信号区域内最可能为因果变异的集合，目的是缩小候选范围。二者互补，缺一不可。

{{Wiki:Linkage_disequilibrium}}

20.3 贝叶斯精细定位

20.3.1 统计框架

贝叶斯精细定位方法将每个 SNP 视为候选因果变异，计算其后验因果概率（Posterior Inclusion Probability，PIP）：

\[ \text{PIP}_j = P(\text{SNP } j \text{ 为因果变异} \mid \text{数据}) \]

早期方法（如 FINEMAP）假设候选区域内因果变异数量服从先验分布，通过马尔可夫链蒙特卡罗（MCMC）或 shotgun stochastic search 枚举可能的因果变异组合。

SuSiE（Sum of Single Effects） 方法采用变分贝叶斯（variational Bayes）推断，将多因果变异问题分解为多个”单效应”成分之和：

\[ \mathbf{y} = \sum_{l=1}^{L} \mathbf{X} \mathbf{b}_l + \boldsymbol{\varepsilon} \]

其中每个 \(\mathbf{b}_l\) 仅有一个非零元素（单效应假设），\(L\) 为预设的最大因果变异数。SuSiE 通过迭代贝叶斯岭回归（iterative Bayesian stepwise selection，IBSS）算法求解各成分的后验分布。

20.3.2 可信集（Credible Set）

对于每个效应成分 \(l\)，SuSiE 输出一个可信集（Credible Set）：包含所有 SNP 中，累积后验概率达到指定覆盖率（通常 95%）的最小变异子集。即：

\[ \sum_{j \in \mathcal{CS}_l} \text{PIP}_{lj} \geq 0.95 \]

可信集越小，精细定位的分辨率越高。若某个可信集仅含 1–2 个 SNP，则这些变异是高置信度的因果候选变异。

20.3.3 SuSiE R 实现

library(susieR)
library(tidyverse)

# 准备输入数据
# z: z-score 向量（GWAS 统计量）
# R: LD 矩阵（候选区域内所有 SNP 的 r 相关矩阵）
# n: 样本量

# 示例：从 GWAS summary stats 和参考 LD 矩阵运行 SuSiE
# （实际应用中 LD 矩阵由参考面板如 1000 Genomes 计算）
set.seed(42)
n_snp <- 200
n_samp <- 50000

# Simulate z-scores with one causal variant at position 100
causal_pos <- 100
z_scores <- rnorm(n_snp, mean = 0, sd = 1)
z_scores[causal_pos] <- 8.0  # strong signal at causal variant

# Simulate LD matrix (block structure)
R <- diag(n_snp)
for (i in 1:n_snp) {
  for (j in i:n_snp) {
    r_ij <- exp(-0.02 * abs(i - j))
    R[i, j] <- R[j, i] <- r_ij
  }
}

# Run SuSiE with summary statistics
fit_susie <- susie_rss(
  z    = z_scores,
  R    = R,
  n    = n_samp,
  L    = 5,          # max number of causal variants
  coverage = 0.95
)

# Extract results
cat("Number of credible sets:", length(fit_susie$sets$cs), "\n")

# PIP for each SNP
pip_df <- data.frame(
  snp_idx = 1:n_snp,
  pip     = fit_susie$pip
)

# Visualize PIP
ggplot(pip_df, aes(x = snp_idx, y = pip)) +
  geom_col(fill = "steelblue", width = 0.8) +
  geom_hline(yintercept = 0.9, color = "red", linetype = "dashed") +
  labs(x = "SNP Index",
       y = "Posterior Inclusion Probability (PIP)",
       title = "SuSiE Fine-mapping Result") +
  theme_classic()

# Print credible set SNPs
for (l in seq_along(fit_susie$sets$cs)) {
  cat(sprintf("Credible set %d: SNPs %s (coverage = %.3f)\n",
              l,
              paste(fit_susie$sets$cs[[l]], collapse = ", "),
              fit_susie$sets$coverage[l]))
}

SuSiE 的实际应用注意事项

精细定位的质量高度依赖于 LD 矩阵的准确性。应优先使用与 GWAS 样本来源相同的群体作为参考面板计算 LD；若使用公共参考面板（如 1000 Genomes EUR），需确认与研究样本的祖源高度匹配。样本量不足时，SuSiE 可信集可能包含大量 SNP，精细定位分辨率有限。

20.4 候选位点的功能注释

精细定位缩小了因果变异的统计候选范围，但统计概率本身无法说明变异通过何种分子机制影响表型。功能注释（functional annotation）是将统计候选变异与基因组功能信息整合、提出生物学假说的关键步骤。GWAS 发现的显著位点中约 90% 位于非编码区，其机制多涉及顺式调控（cis-regulation），需借助多层次基因组注释加以解析。

{{Wiki:Regulatory_element}}

20.4.1 变异功能分类：ANNOVAR 与 VEP

ANNOVAR（Annotate Variation）和 VEP（Variant Effect Predictor，Ensembl）是最常用的变异功能注释工具，可对每个候选 SNP 及其 LD 伙伴变异（\(r^2 > 0.8\)）进行多数据库联合注释，输出功能分类：

注释类型 含义 功能解读重要性
非同义变异（nonsynonymous） 导致氨基酸替换 高——直接改变蛋白序列
剪接位点变异（splice site） 影响 pre-mRNA 剪接 高——可能改变蛋白结构
5’/3’ UTR 变异 调控翻译效率或 mRNA 稳定性 中——需实验验证
内含子变异 落在内含子内 低—中（若重叠调控元件则重要）
基因间区变异（intergenic） 基因体外 需结合表观基因组数据判断 
# Annotate candidate SNPs with ANNOVAR (gene-based + filter-based)
perl annotate_variation.pl \
  -geneanno \
  -dbtype refGene \
  -buildver hg38 \
  candidate_snps.avinput \
  humandb/

# Also run region-based annotation for regulatory elements
perl annotate_variation.pl \
  -regionanno \
  -dbtype phastConsElements46way \
  -buildver hg38 \
  candidate_snps.avinput \
  humandb/

20.4.2 eQTL 整合：GTEx 数据库

表达数量性状位点（eQTL） 是与基因表达水平统计关联的遗传变异。若 GWAS 候选 SNP 同时也是某基因在相关组织中的 cis-eQTL，则该基因上升为优先候选致因基因。GTEx v8 提供了 54 种人类组织的 cis-eQTL 图谱，是功能注释的核心参考资源。

library(tidyverse)

# Load GTEx whole blood eQTL (significant pairs)
eqtl <- read_tsv(
  "GTEx_Analysis_v8_eQTL/Whole_Blood.v8.signif_variant_gene_pairs.txt.gz"
)

# Parse variant position from GTEx variant ID (chr_pos_ref_alt_build)
eqtl <- eqtl %>%
  mutate(
    CHR = as.integer(sub("chr(\\d+)_.*", "\\1", variant_id)),
    BP  = as.integer(sub("chr\\d+_(\\d+)_.*", "\\1", variant_id))
  )

# Query eQTLs in candidate region (e.g., chr6:32.4–33.4 Mb)
hits_in_region <- eqtl %>%
  filter(CHR == 6, BP >= 32.4e6, BP <= 33.4e6) %>%
  arrange(pval_nominal)

cat("eQTL genes overlapping candidate region:\n")
print(unique(hits_in_region$gene_id))

选择 eQTL 组织的原则

应优先查询与研究表型最相关的组织或细胞类型。例如，精神疾病 GWAS 优先查询脑组织 eQTL；炎性肠病 GWAS 优先查询结肠和小肠 eQTL；血液性状 GWAS 优先查询全血和特定免疫细胞亚型 eQTL（可进一步查询 eQTLGen 数据库的血液 cis-eQTL 汇总结果）。组织特异性 eQTL 往往比跨组织 eQTL 更具生物学意义，但查询范围应根据已知病理机制合理设定。

20.4.3 共定位分析（Colocalization）

eQTL 与 GWAS 信号的区域重叠不足以说明共享因果变异——两个信号可能来自同一 LD 区块内两个紧密连锁但不同的变异。共定位分析（colocalization） 通过贝叶斯框架量化两个性状的关联信号共享同一因果变异的后验概率。coloc 包计算五种假设的后验概率（PPH0–PPH4）：

假设 含义
H0 两性状均无信号
H1 仅 GWAS 有信号
H2 仅 eQTL 有信号
H3 两者有信号，但各由不同变异驱动
H4 两者共享同一因果变异

PPH4 > 0.8 是共定位的常用阈值，意味着有超过 80% 的后验概率支持 GWAS 与 eQTL 信号由同一遗传变异驱动，为该基因提供了强烈的致因基因证据。

H3（各由独立变异驱动）不代表”无关”——该情形提示同一 LD 区块内存在两个独立调控元件，分别影响基因表达和疾病风险，同样具有重要的生物学意义。SuSiE-coloc（coloc::coloc.susie()）在各数据集上先运行 SuSiE 精细定位识别多信号，再对每对信号分别做共定位，避免多信号区域中信号混叠导致的误判，是现代分析的推荐流程。

library(coloc)

# Prepare GWAS dataset for candidate region
dataset_gwas <- list(
  beta    = region_gwas$beta,
  varbeta = region_gwas$se^2,
  N       = 50000,
  type    = "quant",
  snp     = region_gwas$SNP
)

# Prepare eQTL dataset for overlapping gene
dataset_eqtl <- list(
  beta    = overlapping_eqtl$slope,
  varbeta = overlapping_eqtl$slope_se^2,
  N       = 670,              # GTEx v8 whole blood sample size
  type    = "quant",
  snp     = overlapping_eqtl$variant_id
)

result_coloc <- coloc.abf(dataset1 = dataset_gwas,
                           dataset2 = dataset_eqtl)

cat("PP.H4 (shared causal variant):", result_coloc$summary["PP.H4.abf"], "\n")
cat("PP.H3 (independent variants):", result_coloc$summary["PP.H3.abf"], "\n")

20.4.4 表观基因组注释：增强子与染色质可及性

对于位于非编码区的候选变异，需结合染色质状态数据判断其是否位于活跃调控元件上。核心数据来源：

  • ENCODE 和 Roadmap Epigenomics：H3K27ac（活跃增强子/启动子标志）、H3K4me1（增强子标志）、H3K4me3（活跃启动子标志）ChIP-seq 峰图；
  • ATAC-seq / DNase-seq：开放染色质区域，直接反映转录因子结合的可及位置；
  • Hi-C / promoter-capture Hi-C：识别增强子与基因启动子之间的远程染色质环，帮助连接基因间区变异与其调控靶基因。
# Intersect candidate SNPs with H3K27ac peaks (active regulatory elements)
bedtools intersect \
  -a candidate_snps_with_ld_proxies.bed \
  -b h3k27ac_peaks_relevant_tissue.bed \
  -wa -wb \
  > snps_in_active_enhancers.txt

# Count overlapping SNPs
echo "SNPs in active enhancers: $(wc -l < snps_in_active_enhancers.txt)"

组织特异性富集分析将所有全基因组显著位点的 LD 代理变异集合与多组织表观基因组数据做系统性富集检验，确认该性状的 GWAS 信号是否在特定细胞类型的调控元件上富集（如精神分裂症信号在神经元 H3K27ac 峰上的高度富集），为后续功能研究指定最相关的组织/细胞类型。常用工具为 LDSC-SEG 和 GARFIELD。

20.4.5 候选位点解析的整合框架

GWAS 全局结果
      │
      ▼
  LD clumping → 独立显著位点列表
      │
      ▼
  候选区域定义 (lead SNP ± 500 kb)
      │
      ├──► 精细定位 (SuSiE / FINEMAP)
      │         └── 可信集 + PIP → 高置信度候选变异
      │
      ├──► 变异功能注释 (ANNOVAR / VEP)
      │         └── 编码变异？非编码？剪接位点？
      │
      ├──► eQTL 查询 (GTEx / eQTLGen)
      │         └── 候选效应基因
      │
      ├──► 共定位分析 (coloc / SuSiE-coloc)
      │         └── PPH4 > 0.8 → 强共定位证据
      │
      ├──► 染色质可及性 / 组蛋白修饰 (ENCODE / Roadmap)
      │         └── 候选调控元件 (enhancer, promoter)
      │
      └──► 整合结论：候选致因基因与致因变异

黄金法则：从统计显著到生物学假说

GWAS 的统计命中仅是起点，而非终点。候选基因的确立需要多组学证据的汇聚：精细定位（统计）→ 功能注释（基因组）→ eQTL/共定位（分子）→ 实验验证（细胞/动物）。任何单一证据都不足以建立因果结论。

20.5 多群体 GWAS 与跨祖源分析

{{Wiki:Population_stratification}}

20.5.1 单群体主导的历史局限

过去二十年间，已发表 GWAS 中超过 80% 的样本来自欧洲祖源人群。这一不均衡带来多重局限：基于欧洲人群构建的 PRS 在其他群体中预测性能显著下降；某些在特定群体中频率较高的致病变异可能被忽略；跨群体效应量比较受阻。UK Biobank 非白人子集、BBJ（日本生物样本库）、All of Us 研究计划等近年来显著改善了这一状况。

20.5.2 多群体分析策略

多群体 GWAS 主要有两种实施模式：

  1. 分群体分析后元分析：各祖源群体内独立运行 GWAS，对汇总统计量做跨群体元分析。METAL 的 IVW 模式（逆方差加权）为首选，自动对齐效应等位基因方向；对于 A/T 和 C/G 回文 SNP（palindromic SNPs），需依赖等位基因频率辅助链方向判断，或直接排除以避免方向反转错误。

  2. 联合分析（mega-analysis）：合并原始个体水平数据，统一模型中使用多祖源 PCA 或稀疏多祖源 GRM 校正群体结构。统计效率更高，但对数据共享协议要求严格。

多群体精细定位利用不同祖源群体 LD 结构的互补性缩小可信集。欧洲人群中 \(r^2 = 0.95\) 的两个 SNP 在东亚人群中可能 \(r^2 = 0.20\)，东亚样本的数据即可区分二者，将 PIP 集中在真正的因果变异上。SuSiE 多群体扩展版和 PAINTOR 均支持多群体联合精细定位，其多数据集似然为：

\[P(\text{all data} \mid \mathcal{C}) = \prod_{a=1}^{A} P(\text{data}_a \mid \mathcal{C}, \mathbf{R}_a)\]

20.5.3 跨群体异质性与健康公平性

Cochran’s Q 检验和 \(I^2\) 统计量量化各群体间效应量的异质性。高异质性（\(I^2 > 75\%\)）提示存在基因-环境交互或群体特异性 LD 代理效率差异，此时应报告随机效应元分析结果并通过亚组森林图展示各群体方向一致性。

PRS 跨群体迁移性差的根本原因在于 LD 代理关系的群体特异性：训练群体中的 lead SNP 通过 LD 代理因果变异，当 LD 结构不同时代理效率下降，效应量估计偏差。PRS-CSx 等多群体方法在贝叶斯框架下为每个群体各自估计收缩效应量，再线性加权组合，显著提升非欧洲群体预测性能，是推动基因组医学公平性的重要技术方向。

祖源描述的规范用语

应使用”欧洲祖源（European ancestry）“、”东亚祖源（East Asian ancestry）“等基于遗传相似性的描述，而非社会文化种族标签。遗传祖源是多维主成分空间中的连续分布，而非离散分类。研究报告应明确说明祖源分组的定义方法（通常基于与 1000 Genomes 参考超群的 PCA 比对），以保证可重复性。

20.6 孟德尔随机化（Mendelian Randomization）

20.6.1 因果推断的挑战

观察性流行病学研究中，暴露因素（如 BMI、LDL 胆固醇、炎症因子）与疾病的关联可能受到混杂因素的干扰，或受到反向因果（reverse causation）的影响，难以直接推断因果关系。

孟德尔随机化（Mendelian Randomization，MR） 借助遗传变异作为暴露因素的工具变量（instrumental variable，IV），利用遗传变异随机分配（类似随机对照试验）的特性推断因果效应。

{{Wiki:Mendelian_randomization}}

20.6.2 三个核心假设

有效工具变量必须满足三个假设：

  1. 相关性（Relevance）：遗传变异与暴露因素显著关联（通常要求 F 统计量 > 10）；
  2. 独立性（Independence）：遗传变异与混杂因素无关（随机化假设，由减数分裂的随机性保证）；
  3. 排他性（Exclusion restriction）：遗传变异仅通过暴露因素影响结局，不存在水平多效性（horizontal pleiotropy）。
遗传变异 (G) ──────────► 暴露 (X) ──────────► 结局 (Y)
     │                                              ▲
     │               混杂因素 (U)                   │
     │─────────────────────────────────────────────┘
                    (假设此路径不存在)

水平多效性是 MR 的主要威胁

若工具变量 SNP 除通过暴露路径外，还通过其他生物学路径影响结局（水平多效性），则 MR 估计有偏。MR-Egger 回归通过允许非零截距来检测和校正平均多效性，但其估计效率低，且假设多效性与 IV 强度不相关（InSIDE 假设）。加权中位数法和众数法在不同多效性模式下均提供稳健估计，与 IVW 结果一致性越高，因果推断越可信。

20.7 多基因风险评分（PRS）

20.7.1 基本原理

多基因风险评分（Polygenic Risk Score，PRS） 将个体基因组中所有（或筛选后的）GWAS 关联 SNP 的效应叠加，得到一个量化该个体遗传风险的单一分数：

\[ \text{PRS}_i = \sum_{j=1}^{M} \hat{\beta}_j \cdot G_{ij} \]

其中 \(\hat{\beta}_j\) 为第 \(j\) 个 SNP 在 GWAS 训练集中估计的效应量，\(G_{ij}\) 为个体 \(i\) 在该 SNP 的基因型编码（0/1/2）。

PRS 捕捉的是多个位点的累积遗传效应，而非单个位点的大效应，适用于复杂多基因遗传结构的表型（如身高、BMI、心血管疾病）。

20.7.2 LD 裁剪 + p 值阈值（C+T）方法

最简单的 PRS 构建方法为 C+T（Clumping + Thresholding）：

# Step 1: LD clumping（同前，去除 LD 冗余）
plink \
  --bfile target_genotypes \
  --clump base_gwas.assoc \
  --clump-p1 1.0 \
  --clump-r2 0.1 \
  --clump-kb 250 \
  --clump-field P \
  --out clumped_snps

# Step 2: 提取不同 p 值阈值下的 SNP 集合
for PTHRESH in 5e-8 1e-5 1e-3 0.01 0.05 0.1 0.2 0.5 1.0; do
  plink \
    --bfile target_genotypes \
    --score base_gwas.assoc 3 4 7 header \
    --q-score-range p_thresholds.txt p_values.txt \
    --extract clumped_snps.clumped \
    --out prs_pt_${PTHRESH}
done
# 选择最优 p 值阈值（在独立验证集中最大化 R² 或 AUC）
library(tidyverse)

thresholds <- c(5e-8, 1e-5, 1e-3, 0.01, 0.05, 0.1, 0.2, 0.5, 1.0)

# 读取各阈值下的 PRS 并在验证集中评估
results <- map_df(thresholds, function(pt) {
  prs <- read_table(sprintf("prs_pt_%.0e.profile", pt),
                    col_types = cols()) %>%
    select(IID, SCORE)

  r2  <- summary(fit)$r.squared

})

LDpred2 与 PRS-CS：贝叶斯 PRS 方法

C+T 方法简单但次优，因为它丢弃了阈值以下大量有信息的小效应位点。LDpred2 和 PRS-CS 通过贝叶斯收缩方法在全基因组范围内同时估计所有 SNP 的后验效应量，通常比 C+T 方法有更高的预测准确性，尤其在多基因性（polygenicity）高的性状上。

20.8 稀有变异聚合检验

20.8.1 稀有变异分析的动机

标准 GWAS 主要针对常见变异（MAF > 1%），对稀有变异（MAF < 1%）的统计检验力极低：在有限样本量下，稀有变异在任何基因型组别中仅有少数个体携带，单 SNP 检验几乎无法达到显著。

然而，稀有变异（尤其是功能性稀有变异，如 Loss-of-Function 变异）往往具有较大的效应量，其在基因水平的聚合分析可显著提升检验力。

20.8.2 SKAT 与 SKAT-O

序列核关联检验（Sequence Kernel Association Test，SKAT） 将同一基因（或基因组窗口）内所有稀有变异的效应量视为服从先验分布（均值为零）的随机变量，通过核函数（kernel function）将多个稀有变异的基因型信息聚合为单一统计量：

\[ Q_{\text{SKAT}} = (\mathbf{y} - \hat{\boldsymbol{\mu}})^\top \mathbf{K} (\mathbf{y} - \hat{\boldsymbol{\mu}}) \]

其中 \(\mathbf{K} = \mathbf{G} \mathbf{W} \mathbf{G}^\top\) 为加权核矩阵，\(\mathbf{G}\) 为基因型矩阵，\(\mathbf{W}\) 为基于 MAF 的权重矩阵（稀有变异权重更高）。

SKAT-O（Optimal SKAT） 通过线性组合负担检验（burden test）和 SKAT 统计量，自适应地在”方向一致的稀有变异效应”和”效应方向混合的稀有变异”两种场景之间取得最优检验力：

\[ Q_\rho = (1-\rho) \cdot Q_{\text{SKAT}} + \rho \cdot Q_{\text{burden}}, \quad \rho \in [0, 1] \]

20.9 动态性状的 GWAS：功能作图方法

{{Wiki:Quantitative_trait_locus}}

20.9.1 纵向表型的映射挑战

标准 GWAS 以单时间点表型（如某次测量的身高、血压值）为分析对象，将其视为固定标量与基因型关联。然而，许多生物学性状天然具有时间动态性（time-varying nature）：生长速率、药物代谢曲线、疾病进展轨迹、季节性表达波动……这些性状的生物学核心不在于某一时刻的表型值，而在于整条动态轨迹的遗传控制机制。对这类性状逐时间点独立做 GWAS 有两大缺陷：其一，大量低效的重复检验造成严重的多重检验负担；其二，完全丢失了时间点之间的生物学依赖结构，等于抛弃了最具信息量的动态信息。

功能作图（Functional Mapping，FM） 正是为解决上述问题而提出的统计框架。其核心思想是：将生物数学模型（描述性状动态规律的参数方程）直接嵌入 QTL/GWAS 关联分析的似然函数，用数量性状的发育参数而非原始表型值作为遗传效应的载体。这一框架由邬荣领（Rongling Wu）、马长兴（Changxing Ma）和 George Casella 于 2002 年在 Genetics 上正式提出，随后在 Nature Reviews Genetics（2006）上系统综述，发展为功能基因组学中动态性状遗传解析的基础理论之一。

20.9.2 功能作图的数学框架

设 \(n\) 个个体在 \(T\) 个时间点 \(t_1, t_2, \ldots, t_T\) 上各测量一次表型。个体 \(i\) 的观测向量为 \(\mathbf{y}_i = (y_{i1}, y_{i2}, \ldots, y_{iT})^\top\)。在 QTL/SNP 关联分析框架下，将个体按基因型 \(g \in \{AA, Aa, aa\}\)（加性模型下简化为 0/1/2）分组，每组的期望轨迹由参数化曲线描述：

\[E[y_{it} \mid g_i = g] = f(t;\, \boldsymbol{\theta}_g)\]

其中 \(\boldsymbol{\theta}_g\) 是第 \(g\) 组基因型的曲线参数向量（如 Logistic 模型的渐近值 \(a\)、增长率 \(r\)、拐点 \(t_0\)），不同基因型之间曲线参数的差异即为 QTL 效应。

均值模型——三参数 Logistic 生长曲线：

\[f(t;\, a, r, t_0) = \frac{a}{1 + e^{-r(t - t_0)}}\]

  • \(a\)：渐近值（最终体量上限）；
  • \(r\)：增长率（曲线陡峭程度）；
  • \(t_0\)：拐点时刻（增长速率最大处）。

在加性模型下，两个纯合基因型（\(g = 0\) 和 \(g = 2\)）的参数向量 \(\boldsymbol{\theta}_0 = (a_0, r_0, t_{0,0})\) 与 \(\boldsymbol{\theta}_2 = (a_2, r_2, t_{0,2})\) 直接反映等位基因替换对生长轨迹各维度的加性效应。

协方差结构——结构化时间相关：

纵向测量之间存在时间自相关，不满足独立性假设。功能作图采用AR(1) 自回归结构对残差协方差建模：

\[\Sigma_{jk} = \sigma^2 \rho^{|t_j - t_k|}\]

其中 \(\sigma^2\) 为残差方差，\(\rho\) 为相邻时间点间的自相关系数（\(0 < \rho < 1\)）。AR(1) 结构的闭合形式逆矩阵和行列式使得计算复杂度从 \(O(T^3)\) 降至 \(O(T)\)，是功能作图计算可行的关键技巧。

混合模型似然函数：

\[\ln L(\boldsymbol{\Theta}) = \sum_{i=1}^{n} \ln \left[\sum_{g=0}^{2} p_g \cdot \phi(\mathbf{y}_i;\, \boldsymbol{\mu}_g,\, \boldsymbol{\Sigma})\right]\]

其中 \(p_g\) 为各基因型频率（由 Hardy-Weinberg 平衡和等位基因频率确定），\(\phi(\cdot;\, \boldsymbol{\mu}_g, \boldsymbol{\Sigma})\) 为多元正态密度函数，均值向量 \(\boldsymbol{\mu}_g = (f(t_1;\boldsymbol{\theta}_g), \ldots, f(t_T;\boldsymbol{\theta}_g))^\top\)。通过 EM 算法迭代估计 \(\boldsymbol{\Theta} = \{\boldsymbol{\theta}_0, \boldsymbol{\theta}_2, \sigma^2, \rho\}\)，计算似然比统计量（LRT）检验 \(H_0: \boldsymbol{\theta}_0 = \boldsymbol{\theta}_2\)（无 QTL 效应）。

参数的生物学可解释性是功能作图的核心优势

传统多时间点 GWAS 报告的是”在时间点 \(t_k\) 处，等位基因 A 将表型平均提升 \(\beta_k\) 个单位”——这些 \(\beta_k\) 是统计描述，缺乏生物学含义的整合。功能作图报告的是”等位基因 A 将渐近体量 \(a\) 提升 \(\Delta a\)，同时不影响生长速率 \(r\)“——每个参数对应一个具有生物学意义的发育维度，检验效力也因参数聚合了整条曲线信息而显著高于逐点检验。

20.9.3 正交 Legendre 多项式：非参数化扩展

当研究者不确定数据适合哪种生物数学曲线时，可改用正交 Legendre 多项式（Legendre Orthogonal Polynomials，LOP） 作为均值函数的非参数化基底：

\[f(t;\, \boldsymbol{\phi}) = \sum_{k=0}^{K} \phi_k P_k(\tilde{t})\]

其中 \(P_k(\tilde{t})\) 为 \(k\) 阶 Legendre 多项式（\(\tilde{t} \in [-1, 1]\) 为归一化时间），\(\phi_k\) 为各基底的展开系数。阶数 \(K\) 通过 AIC/BIC 模型选择确定（通常 \(K = 3\)–\(5\) 即可捕捉主要动态特征）。不同基因型的 LOP 系数向量 \(\boldsymbol{\phi}_g\) 之差即为遗传效应的非参数估计。Legendre 多项式基底因正交性使设计矩阵数值稳定，且可通过 \(R\) 的 poly() 函数直接生成。

20.9.4 R 模拟实现

以下代码从零开始模拟一个功能作图场景：100 个个体在 10 个时间点上的纵向生长数据，含一个影响 Logistic 增长曲线参数的 QTL，演示模型拟合与基因型轨迹差异可视化。

# Functional mapping simulation and fitting
# Simulates longitudinal growth data with a QTL affecting curve parameters
set.seed(2024)
library(ggplot2)

# --- 1. Parameters ---
n        <- 150          # sample size
t_obs    <- seq(1, 10)   # 10 time points
T        <- length(t_obs)
maf      <- 0.35         # minor allele frequency of QTL

# Logistic growth: f(t; a, r, t0) = a / (1 + exp(-r*(t - t0)))
logistic_growth <- function(t, a, r, t0) a / (1 + exp(-r * (t - t0)))

# Genotype-specific curve parameters (additive model)
# Genotype AA (g=0): slower, lower asymptote
# Genotype Aa (g=1): intermediate (midpoint)
# Genotype aa (g=2): faster, higher asymptote
theta <- list(
  "0" = c(a = 8.0, r = 0.5, t0 = 5.0),   # AA
  "1" = c(a = 9.5, r = 0.7, t0 = 4.5),   # Aa
  "2" = c(a = 11.0, r = 0.9, t0 = 4.0)   # aa
)

# --- 2. Simulate genotypes under HWE ---
p    <- maf
q    <- 1 - p
freq <- c(q^2, 2*p*q, p^2)   # P(AA), P(Aa), P(aa)
geno <- sample(0:2, n, replace = TRUE, prob = freq)

# --- 3. AR(1) covariance matrix ---
rho   <- 0.75
sigma <- 1.2
ar1_cov <- function(T, sigma, rho) {
  Sigma <- matrix(0, T, T)
  for (i in seq_len(T))
    for (j in seq_len(T))
      Sigma[i, j] <- sigma^2 * rho^abs(i - j)
  Sigma
}
Sigma <- ar1_cov(T, sigma, rho)
L     <- t(chol(Sigma))   # Cholesky for simulation

# --- 4. Simulate phenotype trajectories ---
sim_data <- lapply(seq_len(n), function(i) {
  g  <- geno[i]
  mu <- logistic_growth(t_obs,

# --- 6. Fit functional mapping model via numerical optimization ---
# Log-likelihood under logistic mean + AR(1) covariance
neg_loglik_fm <- function(params, dat, t_obs, T) {
  # params: a0, r0, t0_0, a2, r2, t0_2, sigma, rho
  a0   <- params[1]; r0  <- params[2]; tt0_0 <- params[3]
  a2   <- params[4]; r2  <- params[5]; tt0_2 <- params[6]
  sig  <- exp(params[7])   # log-parameterized for positivity
  rho_ <- plogis(params[8]) * 0.999 + 0.001   # [0.001, 1)

  # AR(1) covariance and its inverse
  Sigma <- ar1_cov(T, sig, rho_)
  tryCatch({
    Sigma_inv <- solve(Sigma)
    log_det   <- determinant(Sigma, logarithm = TRUE)$modulus
  }, error = function(e) return(1e9))

  maf_est <- mean(dat$genotype) / 2
  freq_g  <- c((1-maf_est)^2, 2*maf_est*(1-maf_est), maf_est^2)

  # Mean curves for each genotype
  mu_list <- list(
    logistic_growth(t_obs, a0, r0, tt0_0),
    logistic_growth(t_obs, (a0+a2)/2, (r0+r2)/2, (tt0_0+tt0_2)/2),  # Aa midpoint
    logistic_growth(t_obs, a2, r2, tt0_2)
  )

  # Mixture log-likelihood (sum over individuals)
  ll <- 0
  for (i in unique(dat$id)) {
    y_i  <- dat$y[dat$id == i]
    g_i  <- dat$genotype[dat$id == i][1]
    comp <- numeric(3)
    for (g in 0:2) {
      resid <- y_i - mu_list[[g + 1]]
      comp[g+1] <- freq_g[g+1] *
        exp(-0.5 * (t(resid) %*% Sigma_inv %*% resid +
                    log_det + T * log(2 * pi)))
    }
    ll <- ll + log(sum(comp) + 1e-300)
  }
  -ll   # return negative log-likelihood for minimization
}

# Initial values near true parameters
init_params <- c(a0 = 8.5, r0 = 0.55, tt0_0 = 4.8,
                 a2 = 10.5, r2 = 0.85, tt0_2 = 4.2,
                 log_sigma = log(1.0), logit_rho = qlogis(0.7))

cat("Fitting functional mapping model (this may take ~30 seconds)...\n")
fit <- optim(init_params, neg_loglik_fm,
             dat = dat, t_obs = t_obs, T = T,
             method = "Nelder-Mead",
             control = list(maxit = 2000, reltol = 1e-6))

# --- 7. LRT: test QTL effect (H0: theta_0 == theta_2) ---
# H0: fit single curve for all genotypes
neg_loglik_null <- function(params, dat, t_obs, T) {
  a  <- params[1]; r  <- params[2]; t0 <- params[3]
  sig <- exp(params[4]); rho_ <- plogis(params[5]) * 0.999 + 0.001
  Sigma     <- ar1_cov(T, sig, rho_)
  Sigma_inv <- solve(Sigma)
  log_det   <- determinant(Sigma, logarithm = TRUE)$modulus
  mu <- logistic_growth(t_obs, a, r, t0)
  ll <- 0
  for (i in unique(dat$id)) {
    y_i   <- dat$y[dat$id == i]
    resid <- y_i - mu
    ll    <- ll - 0.5 * (t(resid) %*% Sigma_inv %*% resid +
                          log_det + T * log(2*pi))
  }
  -ll
}

20.9.6 功能作图的延伸与局限

功能作图框架在过去二十年中延伸至多个方向，包括：反应规范映射（reaction norm mapping）——以环境梯度（温度、光照）替换时间轴，映射基因型-环境交互效应；网络功能作图——以系统发育树或代谢网络约束为先验，映射发育网络的遗传控制；高维纵向 GWAS——结合 LASSO 或 Elastic Net 正则化在全基因组范围内扩展功能作图。

然而，功能作图的实践也存在若干值得关注的局限，其中一项根本性约束来自其历史设计背景：Wu 等（2002）的原始框架是为受控实验交配群体（如回交 BC、F2、重组自交系 RIL）量身设计的。在这类群体中，各基因型频率由孟德尔分离比例精确确定（无需从群体等位基因频率估算），且实验设计本身消除了群体分层——所有个体来自同一受控杂交，亲缘关系均一可知，因此既无群体结构混杂（Q），也无隐性亲缘关系干扰（K）。

当功能作图被移植至自然群体或人类 GWAS 场景时，这一假设前提不再成立。自然群体中个体间存在不同程度的亲缘关系与祖源分层，若不加以校正，纵向表型中嵌入的家系内相关或亚群体表型差异会被错误地归因于 SNP 效应，产生系统性假阳性。解决方案是将功能作图的似然框架与混合线性模型的 Q+K 校正相结合：将主成分（Q 矩阵）作为固定效应协变量纳入均值方程，将全基因组亲缘关系矩阵（K 矩阵，即 GRM）引入残差协方差结构，以消除群体分层和隐性亲缘的双重混杂。这一整合尚未在主流功能作图软件中得到充分实现，是将该方法推广至人类纵向队列 GWAS 的重要方法论挑战。

此外，功能作图还面临其他实践局限：其一，对生物数学模型的选择具有主观性，错误的曲线模型会导致参数偏差；其二，AR(1) 协方差假设未必适合所有纵向数据（如非等间隔时间点或缺失数据较多时）；其三，计算开销远高于标准 GWAS，在基因组规模（数百万 SNP）下需要并行计算或两阶段筛选策略。对于现代大规模 GWAS 数据，通常先用标准单点 GWAS 初步筛选候选区域，再对候选 SNP 子集运行功能作图以获取动态效应的精细描述。

功能作图的实际应用场景

功能作图特别适合以下研究设计：（1）动植物育种——作物产量的生长曲线、体重增长轨迹的 QTL 定位；（2）药物基因组学——不同基因型对药物的时序反应曲线（药效动力学 PD 轨迹）；（3）纵向队列研究——老龄化研究中认知能力下降曲线、BMI 随年龄变化轨迹的遗传架构；（4）生理节律研究——昼夜节律、季节性表型波动的遗传基础。

参考答案 (Reference Answers)

  1. LD 裁剪（clumping）的目的是在全基因组范围内识别统计独立的信号位点，通过将 LD 区块内相关 SNP 合并为一个代表性位点来消除冗余，输出独立命中列表。精细定位（fine-mapping）的目的是在已知信号区域内，基于贝叶斯框架估计每个 SNP 为因果变异的后验概率，缩小因果候选变异的集合。在分析流程中，LD 裁剪先识别独立信号区域，再对每个区域进行精细定位，以获取高置信度的因果候选变异。

  2. 95% 可信集定义为：从 PIP 最高的 SNP 开始累加，直到累积 PIP 达到 0.95 为止的最小 SNP 集合。若该集合仅含一个 SNP，表明在当前数据和 LD 结构下，有超过 95% 的后验概率支持该变异为唯一因果变异，精细定位分辨率极高，该 SNP 是高优先级功能实验靶标。

  3. 三个假设为：（1）相关性：IV（遗传变异）与暴露因素显著关联；（2）独立性：IV 与混杂因素无关（由孟德尔随机化遗传原理保证）；（3）排他性：IV 仅通过暴露影响结局，不存在其他路径。水平多效性违背的是排他性假设。常用敏感性分析方法包括：MR-Egger 回归（检测平均多效性，截距不为零提示多效性）、加权中位数法（在 50% 以下 IV 有效时仍稳健）、加权众数法和 MR-PRESSO（检测并校正异常 IV）。

  4. C+T 方法优点是简单透明、计算快速，缺点是阈值以下小效应 SNP 被完全丢弃，且对 LD 校正不充分。LDpred2 优点是通过贝叶斯收缩利用全基因组 SNP 信息，在高多基因性性状上预测准确性更高，缺点是计算复杂度高、依赖参考 LD 矩阵质量。PRS 预测性能通常用连续表型的 Nagelkerke R²（或 Pearson \(r^2\)）以及二元表型的 AUC（曲线下面积）评估，均应在独立验证集中评估以避免过拟合。

  5. 稀有变异（MAF < 1%）在群体中携带者极少，单 SNP 关联检验在有限样本中的等位基因组数量太小，导致统计量方差极大，几乎无法在全基因组显著性阈值下达到显著。SKAT 将同一基因内所有稀有变异的基因型信息通过核矩阵聚合为单一统计量，相当于在基因水平上汇聚多个稀有变异的微小信号，从而实现检验力的提升，且允许各变异效应方向不同（与 burden test 的区别）。

  6. 分析流程：（1）从 LDL 胆固醇 GWAS（如 CARDIoGRAMplusC4D 或 UK Biobank）提取显著 SNP 作为工具变量，进行 LD 裁剪保留独立 IV；（2）在心肌梗死 GWAS 中提取对应 SNP 的效应量；（3）使用 TwoSampleMR 运行 IVW（主估计）、加权中位数法和 MR-Egger（敏感性）；（4）通过 MR-Egger 截距检验和 Cochran Q 统计量评估多效性和异质性。主要局限性：PCSK9 区域 SNP 已知仅通过 LDL 水平影响心血管风险，但若纳入其他区域的 LDL 相关 SNP，需检查是否存在通过 HDL、甘油三酯等其他路径的水平多效性；GWAS 样本重叠（winner’s curse 效应）也可能导致 IV 效应高估。

  7. （1）在候选区域（lead SNP ± 500 kb）内运行 SuSiE-RSS，获得每个 SNP 的 PIP 值和 95% 可信集，将 PIP > 0.1 的 SNP 列为优先候选。（2）在 GTEx 中查询可信集内 SNP 是否为候选基因（尤其是区域内或邻近基因）在相关组织中的 cis-eQTL；若存在 eQTL 重叠，运行 coloc 分析评估 PPH4 是否 > 0.8。（3）利用 ENCODE 或 Roadmap 的 H3K27ac、ATAC-seq 峰数据，通过 bedtools intersect 检查可信集 SNP 是否位于相关细胞类型的活跃增强子或启动子区域；进一步通过 Hi-C 数据确认该调控元件与哪个基因启动子存在染色质互作环，建立”SNP→调控元件→靶基因”的调控假说。

  8. 欧洲人群 lead SNP 通常不是因果变异本身，而是通过 LD 代理因果变异的标记物。当目标群体（非欧洲）的 LD 结构不同时，同一 lead SNP 与因果变异的 \(r^2\) 降低，代理效率下降，导致效应量估计偏向零，PRS 预测性能随之下降。多群体精细定位的原理：设两群体的 LD 矩阵分别为 \(\mathbf{R}_1\)（欧洲）和 \(\mathbf{R}_2\)（东亚），多群体似然为各群体似然之积 \(P(\text{data}_1 | \mathcal{C}, \mathbf{R}_1) \times P(\text{data}_2 | \mathcal{C}, \mathbf{R}_2)\)；若欧洲群体无法区分 SNP A 和 B（\(r^2 = 0.95\)），但东亚群体中 A 与 B 的 \(r^2 = 0.10\)，则东亚数据可有效区分二者，将 PIP 集中于真正的因果变异，实现可信集缩小。

  9. H3 代表该区域同时存在两个独立的遗传效应：一个驱动 GWAS 关联信号，另一个驱动 eQTL 信号，二者的因果变异不同。这意味着在同一 LD 区块内存在两个独立的调控元件，各自影响不同的生物学过程，生物学上同样有深度探索的意义。H4 则代表同一因果变异同时驱动疾病关联和基因表达变化，是”该基因通过表达量改变影响疾病”这一假说的最直接统计证据。当 PPH3 = 0.70 而 PPH4 = 0.25 时，数据更支持两个独立信号共存（H3）；此时不应宣称 eQTL 基因是疾病的致因基因，而应进一步检查该区域是否确实有两个独立关联信号（通过条件分析验证），并考虑（1）增大样本量以提升 coloc 分辨率；（2）整合精细定位结果（可信集是否重叠）辅助判断；（3）探索是否存在另一基因的 eQTL 与 GWAS 信号共定位（PPH4 更高）。
