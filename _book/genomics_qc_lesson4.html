12  变异检测算法与 VCF 通用格式

Published

May 16, 2026

在上一专题中，我们完成了基因组的从头组装，并通过 QUAST 和 BUSCO 评估了组装质量。至此，我们手中已经有了一份连续、完整的基因组序列。然而，一份”组装完成”的基因组并不意味着分析的终点——对大多数研究而言，真正的科学问题往往藏在个体之间或样本之间的序列差异之中：这株 E. coli 与参考菌株相比，究竟在哪些位点发生了突变？这些突变是单碱基替换（SNP）、小片段插入缺失（Indel），还是更大规模的结构变异？

回答这些问题，需要经历从”比对”到”检测”再到”注释”的完整流程（参见第2章）。其中，变异检测（Variant Calling）是承上启下的核心环节：它以比对后的 BAM（Binary Alignment Map，二进制比对文件）为输入，以标准化的 VCF 文件为输出，将序列层面的信号转化为可供下游分析使用的变异表。

从组装到变异检测的逻辑链条

基因组分析的典型流程是：原始 reads → 质量控制 → 参考基因组比对（或从头组装）→ 变异检测 → 功能注释。本专题聚焦于变异检测这一环节，重点讲解其统计原理和 VCF 格式规范，为后续使用 GATK（Genome Analysis Toolkit，基因组分析工具包）进行实战分析奠定基础（参见专题 2.5）。

13 变异类型与分类体系

13.1 基因组变异的四大类型

基因组变异按规模和机制可分为四大类，每类的检测方法和生物学影响存在本质差异：

{{Wiki:Single-nucleotide_polymorphism}}

类型 英文名称 规模 典型例子 主要检测工具
SNP Single Nucleotide Polymorphism 1 bp A → G GATK HaplotypeCaller
Indel Insertion/Deletion 1–50 bp 插入 3 bp / 缺失 7 bp GATK HaplotypeCaller
SV Structural Variant > 50 bp 倒位、易位、大缺失 Manta、DELLY、Lumpy
CNV Copy Number Variant kb–Mb 基因组片段扩增/缺失 CNVnator、GATK gCNV

本节重点关注 SNP 和 Indel，这是二代短读测序最擅长检测的类型。

SNP 与 SNV 的区别

SNP（Single Nucleotide Polymorphism） 特指在群体中频率 ≥ 1% 的单核苷酸变异，强调其多态性；SNV（Single Nucleotide Variant） 则泛指任何单碱基替换，包括频率极低的体细胞突变。肿瘤基因组学中习惯用 SNV，群体遗传学和 GWAS 中习惯用 SNP。两者在 VCF 文件中格式相同，仅概念范畴不同。

13.2 碱基替换类型与 Ti/Tv 比值

单碱基替换按化学结构分为两类，这一分类是评估变异集质量的重要依据：

{{Wiki:Transition_(genetics)}}

类型 英文 方向 化学结构
转换（Transition） Ti A ↔︎ G、C ↔︎ T 嘌呤↔︎嘌呤 或 嘧啶↔︎嘧啶
颠换（Transversion） Tv A/G ↔︎ C/T 交叉 嘌呤↔︎嘧啶

由于 DNA 聚合酶容错机制对化学结构相似的碱基替换（转换）更宽松，真实基因组中转换发生概率约为颠换的 2 倍。这一规律产生了重要的 QC 指标：

\[\text{Ti/Tv ratio} = \frac{\text{转换 SNP 数}}{\text{颠换 SNP 数}}\]

数据类型 期望 Ti/Tv 偏低说明 偏高说明
全基因组 WGS（Whole Genome Sequencing） 2.0–2.1 假阳性偏多（错误倾向颠换） 过度过滤颠换
全外显子组 WES 2.8–3.0 过滤不足 选择压力偏差
CpG 位点富集区 > 3.5 — 正常（CpG 甲基化偏好 C→T 转换）

Ti/Tv 比值是变异集整体质量的”体温计”——偏离正常范围意味着过滤策略需要调整，而不是某个变异的问题。

# R: 计算 Ti/Tv 比值（从 VCF 提取的 SNP 数据框）
compute_titv <- function(snp_df) {
  # snp_df 需含 ref 和 alt 列（单字符）
  transitions  <- c("A>G", "G>A", "C>T", "T>C")
  transversions <- c("A>C", "A>T", "G>C", "G>T",
                     "C>A", "C>G", "T>A", "T>G")

  change <- paste0(snp_df$ref, ">", snp_df$alt)
  ti_count <- sum(change %in% transitions)
  tv_count <- sum(change %in% transversions)

  cat(sprintf("Transitions   : %d\n", ti_count))
  cat(sprintf("Transversions : %d\n", tv_count))
  cat(sprintf("Ti/Tv ratio   : %.3f\n", ti_count / tv_count))
  cat(sprintf("Status        : %s\n",
              ifelse(ti_count / tv_count >= 1.8 & ti_count / tv_count <= 3.5,
                     "PASS (expected range)", "WARNING: check filtering")))
}

# 示例
snps <- data.frame(
  ref = c("A","C","T","G","A","C"),
  alt = c("G","T","C","A","C","A")
)
compute_titv(snps)

14 变异检测原理

14.1 基本概念：等位体与基因型

变异检测的核心是：在给定的比对数据中，判断某个位置是否存在真实的生物学变异，而不是测序或比对错误。

在深入算法之前，需要明确几个基本概念。等位体（Allele）是指基因组某一特定位点上出现的具体碱基序列。在二倍体生物中，每个位点有两个等位体，分别来自父方和母方染色体。参考等位体（Reference Allele，REF）是参考基因组在该位点记录的序列；替代等位体（Alternate Allele，ALT）则是样本中观察到的与参考不同的序列。等位体频率（Allele Frequency，AF）描述的是某一等位体在所有观测碱基中所占的比例，是区分杂合变异（AF ≈ 0.5）与纯合变异（AF ≈ 1.0）的核心指标。

基因型与等位体的关系

基因型（Genotype）是对一个样本在某位点上两个等位体组合的描述，在 VCF 文件中用数字编码表示：0 代表参考等位体，1 代表第一个替代等位体，2 代表第二个替代等位体，以此类推。0/1 表示杂合（一个参考、一个替代），1/1 表示纯合替代，0/0 表示纯合参考。

14.2 频率法（Frequency-Based）

最简单的方法：直接统计比对到某位点的各碱基数量，计算替代等位体的观测频率。

\[\text{AF} = \frac{\text{替代等位体数}}{\text{总等位体数}} \times 100\%\]

优点：快速、直观。缺点：不考虑测序质量信息，单个低质量 read 即可造成假阳性。

位置: chr1:1000
参考: A
比对情况:
  Read 1: A (Q30)
  Read 2: A (Q30)
  Read 3: G (Q15)  <- 可能是测序错误
  Read 4: A (Q30)

AF = 1/(1+3) = 25%，但 Read3 质量低，不可信

14.3 贝叶斯推断与 GATK HaplotypeCaller

{{Wiki:Bayesian_inference}}

贝叶斯推断的核心思想是：将”已有知识（先验）“与”观测数据（似然）“结合，得到更新后的”后验概率”。在变异检测场景中，先验是该位点在群体中存在变异的背景概率，似然是在假设某种基因型成立的情况下，观测到当前 reads 数据的概率；后验则是综合两者后，该变异真实存在的概率。

\[P(\text{变异} | \text{数据}) \propto P(\text{数据} | \text{变异}) \times P(\text{变异})\]

优点：整合质量评分、等位体频率、群体背景等多维信息。缺点：计算复杂，参数设置对结果影响较大。

GATK HaplotypeCaller 是目前最广泛使用的贝叶斯变异检测工具，由 Broad Institute 开发，专为人类及模式生物胚系变异检测设计。与简单频率法不同，HaplotypeCaller 首先在活跃区域（Active Region）——即 reads 与参考基因组存在显著差异的局部区段——内重新进行 de novo 组装，避免因初始比对偏差引入错误，再对组装得到的单倍型进行似然评分。其核心步骤如下：

  1. 活跃区域识别：扫描比对文件，标记碱基不一致率超过阈值的区域（默认窗口 200 bp）
  2. 局部重组装 (Local Reassembly)：在活跃区域内用 De Bruijn 图方法重新组装 reads，得到所有可能的单倍型序列
  3. 等位体假设检验：将每条 read 与候选单倍型配对，对每个可能的基因型（0/0, 0/1, 1/1）计算 pair-HMM 似然
  4. 贝叶斯后验：结合群体等位频率先验（来自 dbSNP（Single Nucleotide Polymorphism Database，NCBI 维护的已知变异数据库）等数据库），得出最可能的基因型
  5. 变异置信度输出：将后验概率转换为 PHRED 质量评分（QUAL）写入 VCF

PHRED 质量评分尺度

PHRED 评分将错误概率转换为对数尺度，便于处理极小概率值：\(Q = -10\log_{10}(P_{\text{error}})\)。Q30 对应错误概率 0.1%，Q50 对应 0.001%。GATK 输出的 QUAL 字段即为该变异为假阳性的 PHRED 概率——QUAL=30 意味着该变异有 99.9% 的置信度是真实变异。

以下 R 代码以单个位点为例，模拟贝叶斯基因型推断的完整计算过程：给定 20× 覆盖深度、观测到 10 条 ALT reads，分别计算三种基因型（REF 纯合 0/0、杂合 0/1、ALT 纯合 1/1）的似然、先验与后验，最终输出最可能的基因型及其 PHRED 置信度。

# ── 贝叶斯基因型推断：单位点模拟 ────────────────────────────────────────────

# 观测数据
total_depth <- 20       # 该位点总覆盖深度
alt_count   <- 10       # 观测到的 ALT reads 数
ref_count   <- total_depth - alt_count   # REF reads 数

# 每条 read 的测序错误率（对应 Q30 碱基质量）
base_error  <- 0.001

# 三种基因型在理想情况下观测到 ALT 碱基的期望概率
# 0/0（REF 纯合）：ALT reads 仅来自测序错误
# 0/1（杂合）    ：ALT reads 来自真实变异（期望 50%）
# 1/1（ALT 纯合）：所有 reads 应为 ALT，错误率导致极少量 REF
p_alt_given_genotype <- c(
  "0/0" = base_error,
  "0/1" = 0.5,
  "1/1" = 1 - base_error
)

# 似然：P(数据 | 基因型) = 二项分布概率
likelihood <- dbinom(alt_count, size = total_depth,
                     prob = p_alt_given_genotype)
names(likelihood) <- names(p_alt_given_genotype)

# 先验：P(基因型) —— 来自群体等位频率
# 假设该位点人群 ALT 等位频率 (AF) = 0.3（来自 dbSNP）
AF <- 0.3
prior <- c(
  "0/0" = (1 - AF)^2,           # Hardy-Weinberg 定律
  "0/1" = 2 * AF * (1 - AF),
  "1/1" = AF^2
)

# 后验（未归一化）：P(基因型 | 数据) ∝ 似然 × 先验
posterior_unnorm <- likelihood * prior
posterior <- posterior_unnorm / sum(posterior_unnorm)

# 输出结果
cat("=== 单位点贝叶斯基因型推断 ===\n")
cat(sprintf("观测：total_depth=%d, alt_count=%d, ref_count=%d\n",
            total_depth, alt_count, ref_count))
cat("\n--- 似然 P(data | genotype) ---\n")
print(format(likelihood, scientific = TRUE, digits = 4))
cat("\n--- 先验 P(genotype) [AF=0.3, Hardy-Weinberg] ---\n")
print(round(prior, 4))
cat("\n--- 后验 P(genotype | data) ---\n")
print(round(posterior, 6))

best_gt <- names(which.max(posterior))
# 截断后验至 [0, 1 - 1e-15]，避免 log10(0) = -Inf
qual    <- -10 * log10(1 - pmin(posterior[best_gt], 1 - 1e-15))
# GATK 内部将 QUAL 截断至 10000（对应后验 > 0.9999999）
qual    <- min(qual, 10000)
cat(sprintf("\n最可能基因型：%s（后验 = %.4f，PHRED QUAL ≈ %.1f）\n",
            best_gt, posterior[best_gt], qual))

运行后的参考输出如下：

=== 单位点贝叶斯基因型推断 ===
观测：total_depth=20, alt_count=10, ref_count=10

--- 似然 P(data | genotype) ---
        0/0         0/1         1/1
1.024e-60   1.765e-01   1.024e-60

--- 先验 P(genotype) [AF=0.3, Hardy-Weinberg] ---
  0/0   0/1   1/1
0.4900 0.4200 0.0900

--- 后验 P(genotype | data) ---
       0/0        0/1        1/1
0.000000   1.000000   0.000000

最可能基因型：0/1（后验 = 1.0000，PHRED QUAL ≈ 10000.0）

模拟结果的解读

在 20× 深度、ALT reads 占比恰好 50% 的情况下，杂合基因型（0/1）的似然远大于其余两种基因型（相差约 60 个数量级），先验虽对三种基因型均有权重，但完全无法抵消似然的优势，后验几乎全部集中于 0/1。QUAL 输出 10000 是因为浮点精度下后验等于 1.0，1 - 1.0 = 0，log10(0) 为负无穷；代码以 pmin(..., 1 - 1e-15) 截断后验，再将结果上限限定为 10000，与 GATK 的内部处理一致。若将 alt_count 改为 2（低频变异），可观察到先验的权重如何影响基因型判断——这正是贝叶斯框架相对频率法的核心优势：在低深度或低频等位体场景下，群体背景先验能有效压制噪声。

15 VCF 格式规范

{{Wiki:Variant_Call_Format}}

15.1 VCF 文件结构

VCF（Variant Call Format）是由国际千人基因组计划（1000 Genomes Project）制定的变异信息存储标准，目前维护于 samtools/hts-specs 规范库。VCF 文件以纯文本形式存储，由三部分构成：以 ## 开头的元信息行（描述文件版本、参考基因组、INFO/FORMAT 字段定义等）；以单个 # 开头的表头行（定义固定列名及样本名称）；以及每行代表一个变异位点的数据行。

##fileformat=VCFv4.2
##fileDate=20240101
##source=GATK4.0
##reference=reference.fasta
##INFO=<ID=DP,Number=1,Type=Integer,Description="Total depth">
##INFO=<ID=AF,Number=A,Type=Float,Description="Allele frequency">
##FORMAT=<ID=GT,Number=1,Type=String,Description="Genotype">
##FORMAT=<ID=DP,Number=1,Type=Integer,Description="Read depth">
##FORMAT=<ID=GQ,Number=1,Type=Integer,Description="Genotype quality">
#CHROM  POS     ID      REF     ALT     QUAL    FILTER  INFO    FORMAT  sample1 sample2
chr1    10000   .       A       G       50      PASS    DP=100;AF=0.5   GT:DP:GQ        0/1:50:40       0/0:50:50
chr1    10500   rs123   C       T,A     100     PASS    DP=120;AF=0.4,0.1       GT:DP:GQ        0/1:60:45       1/1:60:50

15.2 固定列说明

列号 列名 说明
1 CHROM 染色体名称
2 POS 1-based 位置
3 ID 变异 ID（通常来自 dbSNP，无则用 .）
4 REF 参考等位体（通常是最长的）
5 ALT 替代等位体（可多个，逗号分隔）
6 QUAL 变异的 PHRED 质量评分 (Phred Scale)
7 FILTER 过滤标志（PASS 或失败原因）
8 INFO 注释信息（键值对）
9+ FORMAT 样本数据格式
10+ 样本 每个样本的基因型和相关字段

16 GVCF 格式简介

在标准 VCF 文件中，只有检测到变异的位点才会被记录；参考一致的区域会被直接跳过。这在单样本分析中没有问题，但当需要多样本联合基因型定型（Joint Genotyping）时，就会产生歧义：某个样本在某位置没有变异记录，究竟是”该位置测序深度足够、确认为参考纯合”，还是”该位置根本没有被测到”？两者的生物学含义截然不同，但在普通 VCF 中无法区分。

GVCF（Genomic VCF）正是为解决这一问题而设计的 VCF 扩展格式。GVCF 对基因组每一个位置都写出记录：对于检测到变异的位点，格式与普通 VCF 相同；对于与参考一致的连续区域，则用一条参考匹配块（Reference Block）记录，以 <NON_REF> 作为占位 ALT 等位体，并用 END 字段标注该块的终止位置，从而压缩表示整段参考一致区域。

这种设计使得多样本分析时，工具（如 GATK GenotypeGVCFs）可以在群体层面对每个位点做出正确的联合推断——即便某个样本在该位置没有变异，其参考深度信息也被完整保留。

##gVCFVersion=v1.4
#CHROM  POS     ID      REF     ALT          QUAL  FILTER  INFO         FORMAT    sample1
chr1    10000   .       A       <NON_REF>    .     .       END=10010    GT:DP     0/0:50
chr1    10011   .       C       T            50    PASS    DP=100       GT:DP:GQ  0/1:50:40
chr1    10012   .       T       <NON_REF>    .     .       END=10100    GT:DP     0/0:48

上例中：chr1:10000–10010 为参考匹配块（深度 50X，单条记录压缩表示）；chr1:10011 为一个杂合 SNP；chr1:10012–10100 又回到参考匹配块。这种格式在样本数量较多时每个 GVCF 文件比普通 VCF 大约大 2–5 倍，但因此换来了联合分析所必需的完整信息。

17 变异过滤策略

17.1 质量指标与过滤阈值

变异过滤的目标是在精度（Precision）和召回率（Recall）之间取得合理平衡。过滤过严会漏掉真实变异（假阴性增多），过滤过松则保留大量假阳性。以下三类指标构成过滤决策的核心依据。

QUAL 评分：

\[\text{QUAL} = -10 \log_{10}(P_{\text{error}})\]

QUAL 范围 含义 建议
0–10 极低置信 丢弃
10–30 低置信 谨慎
30–50 中等置信 基本过滤
50+ 高置信 推荐使用

深度过滤：

条件 问题 过滤阈值
深度过低 (< 10X) 统计功率不足 DP >= 10
深度过高 (> 300X) 可能是重复或污染 DP <= 300
平衡度差 暗示短 read 或 contamination AD_ratio >= 0.25

位置效应过滤：

指标 阈值 解释
ReadPosRankSum > -2.0 变异位置偏向 read 末端 → 假阳性
MQRankSum > -2.0 替代等位体的 MAPQ 偏低
FS (Fisher Strand) < 60 变异偏向某一条链（测序 bias）
QD (Quality/Depth) >= 2.0 确保质量与深度相匹配

17.2 硬过滤（Hard Filtering）

硬过滤对每个变异位点逐条套用预设阈值，满足条件则标注通过（PASS），否则标注失败原因。GATK VariantFiltration 实现了此逻辑：它不删除变异记录，而是在 FILTER 列写入失败标签，保留原始数据的完整性。具体过滤命令及 GIAB 精度评估见实践部分步骤四。

硬过滤的核心局限在于阈值固定：不同测序深度、不同物种、不同测序平台下，同一阈值的精度-召回率表现差异显著。当样本量充足时，GATK 推荐使用更精细的 VQSR 方法自适应地学习过滤边界。

17.3 VQSR：变异质量分数重校正

{{Wiki:Gaussian_mixture_model}}

VQSR（Variant Quality Score Recalibration） 是 GATK 针对人类中大型 WGS 队列（通常 ≥ 30 个样本）的推荐过滤策略。其核心思路是：用已知高质量变异数据集训练高斯混合模型，学习真实变异在多维质量特征空间中的分布，再为每个待过滤变异打出重校正后的置信分（VQSLOD），最终按分位线软过滤，而非预设固定阈值。

以下 R 代码通过模拟数据演示 VQSR 的统计核心：高斯混合模型如何在多维特征空间中区分真实变异与测序错误，并生成连续的置信度评分（VQSLOD）用于 tranche 过滤。

library(mclust)
library(ggplot2)

# ── 1. Simulate variant features (QD and MQ for two classes) ─────────────────
set.seed(42)
n_true  <- 1000
n_false <- 500

data <- data.frame(
  QD     = c(rnorm(n_true,  mean = 28, sd = 4),  rnorm(n_false, mean = 12, sd = 6)),
  MQ     = c(rnorm(n_true,  mean = 60, sd = 2),  rnorm(n_false, mean = 40, sd = 10)),
  actual = factor(c(rep("True_Variant", n_true), rep("False_Artifact", n_false)))
)

# Raw feature space: the two classes overlap — fixed hard cutoffs fail here
ggplot(data, aes(x = MQ, y = QD, color = actual)) +
  geom_point(alpha = 0.4, size = 0.8) +
  theme_minimal() +
  labs(title = "Raw variant features (before VQSR)",
       subtitle = "Simple hard cutoffs cannot cleanly separate the two classes")

# ── 2. Fit Gaussian Mixture Model (unsupervised — labels not used) ────────────
features  <- data[, c("QD", "MQ")]
gmm_model <- Mclust(features, G = 2)

# ── 3. Compute pseudo-VQSLOD = log10( P(true cluster) / P(artifact cluster) ) ─
# Identify the component with higher mean MQ as the "true variant" cluster
true_component <- which.max(gmm_model$parameters$mean["MQ", ])
P_true  <- gmm_model$z[,  true_component]
P_false <- gmm_model$z[, -true_component]

data$VQSLOD <- log10((P_true + 1e-10) / (P_false + 1e-10))

# VQSLOD landscape: high-quality variants score blue, artifacts score red
ggplot(data, aes(x = MQ, y = QD, color = VQSLOD)) +
  geom_point(alpha = 0.6, size = 0.8) +
  scale_color_gradient2(low = "red", mid = "white", high = "steelblue", midpoint = 0) +
  theme_minimal() +
  labs(title = "VQSLOD scores from GMM (pseudo-VQSR)",
       subtitle = "Continuous score replaces fixed threshold; blue = high confidence")

# ── 4. Simulate 99% sensitivity tranche ──────────────────────────────────────
# Find the VQSLOD cutoff that retains 99% of true variants (1% false-negative rate)
threshold_99 <- quantile(data$VQSLOD[data$actual == "True_Variant"], 0.01)
cat(sprintf("99%% sensitivity tranche threshold: VQSLOD >= %.3f\n", threshold_99))

data$filter <- ifelse(data$VQSLOD >= threshold_99, "PASS", "FILTERED")

# Score distribution with the tranche cutoff line
ggplot(data, aes(x = VQSLOD, fill = actual)) +
  geom_histogram(binwidth = 0.2, alpha = 0.6, position = "identity") +
  geom_vline(xintercept = threshold_99, linetype = "dashed") +
  annotate("text", x = threshold_99 + 0.1, y = Inf, vjust = 2,
           label = "99% tranche", hjust = 0, size = 3) +
  theme_minimal() +
  labs(title = "VQSLOD distribution and 99% sensitivity tranche",
       x = "VQSLOD", y = "Count")

# Confusion matrix and precision / recall
cm <- table(Actual = data$actual, Filter = data$filter)
print(cm)
tp <- cm["True_Variant",   "PASS"]
fp <- cm["False_Artifact", "PASS"]
fn <- cm["True_Variant",   "FILTERED"]
cat(sprintf("Precision : %.1f%%\n", tp / (tp + fp) * 100))
cat(sprintf("Recall    : %.1f%%\n", tp / (tp + fn) * 100))

从模拟结果理解 VQSR 的设计哲学

上述代码演示了三个关键洞察：其一，GMM 以无监督方式学习特征分布——真实 GATK VQSR 同样不依赖人工标注，而是从已知变异数据库的统计特征中自动推断类别边界；其二，VQSLOD 是连续评分而非二值判断，允许研究者在精度与召回率之间灵活权衡；其三，“99% sensitivity tranche”意味着接受约 1% 的真实变异损失，以换取大幅压缩假阳性——不同阈值对应不同的 Precision / Recall 曲线，最优点取决于下游分析目标。

训练所需的参考变异资源均来自 GATK Resource Bundle（hg38 版本）：

资源集 用途 known training truth prior

1000G Omni 2.5 SNP 训练集 false true false 12.0

先验权重（prior）的含义

--resource 参数中的 prior 值（如 prior=15.0）表示该数据集中变异为真阳性的 log-odds 先验，数值越高表示越可信。HapMap（prior=15）是最高质量的已知变异集；dbSNP 中包含大量低频和错误条目，因此 prior 仅为 2.0，只用作已知变异标注而非训练。

VQSR 的样本量要求

SNP 模型至少需要约 30 个 WGS 样本才有足够的变异位点训练高斯混合模型。全外显子组测序（WES）因覆盖区域有限、变异数量不足，通常也使用硬过滤而非 VQSR。单样本数据集同样属于此类情况，应使用硬过滤方案；VQSR 代码段（见步骤四）仅作队列场景下的概念演示。

18.1.1 整合脚本（假设数据已就位）

以下脚本将步骤二至步骤四的所有分析环节整合为一个可直接执行的流水线，适合在数据下载完成后一键运行。脚本中所有 wget/samtools view 等数据获取命令均已省略；如需重现完整下载过程，请参照步骤一各子步骤。

#!/usr/bin/env bash
# GATK Best Practices — HG002 chr21 analysis pipeline
# Assumes: all input files already present (see Step 1 for download)
# Runtime: ~25–40 min on a single core with 8 GB RAM

set -euo pipefail

# ── Environment variables ────────────────────────────────────────────────────
REF=reference/hg38_chr21.fasta
DBSNP=reference/dbsnp138_chr21.vcf.gz
SAMPLE=HG002_chr21
INPUT_BAM=hg002_chr21/${SAMPLE}_rg.bam    # pre-aligned BAM with Read Groups
TRUTH_VCF=giab/HG002_chr21_truthset.vcf.gz
CONFIDENT_BED=giab/HG002_chr21_confident.bed
REF_SDF=reference/hg38_chr21.sdf          # rtg-format SDF index (see Step 1A3)

echo "=== Pipeline start: $(date) ==="

# ── Step 2A: Mark PCR duplicates ─────────────────────────────────────────────
# Flags duplicate reads (FLAG 0x400) without removing them.
# Prevents inflated allele depth at over-amplified loci.
echo "[2A] MarkDuplicates..."
gatk MarkDuplicates \
    -I  ${INPUT_BAM} \
    -O  hg002_chr21/${SAMPLE}_dedup.bam \
    -M  logs/${SAMPLE}_dup_metrics.txt
samtools index hg002_chr21/${SAMPLE}_dedup.bam

# ── Step 2B: Base Quality Score Recalibration (BQSR) ─────────────────────────
# Phase 1: build a recalibration model from observed vs reported base qualities.
#   --known-sites: dbSNP positions are excluded from error-rate estimation;
#   mismatches at known SNP sites reflect true polymorphism, not sequencing error.
echo "[2B] BaseRecalibrator..."
gatk BaseRecalibrator \
    -I  hg002_chr21/${SAMPLE}_dedup.bam \
    -R  ${REF} \
    --known-sites ${DBSNP} \
    -O  logs/${SAMPLE}_recal.table

# Phase 2: rewrite quality scores in the BAM using the recalibration model.
echo "[2B] ApplyBQSR..."
gatk ApplyBQSR \
    -I  hg002_chr21/${SAMPLE}_dedup.bam \
    -R  ${REF} \
    --bqsr-recal-file logs/${SAMPLE}_recal.table \
    -O  hg002_chr21/${SAMPLE}_recal.bam
samtools index hg002_chr21/${SAMPLE}_recal.bam

# ── Step 3: Variant calling with HaplotypeCaller ──────────────────────────────
# -ERC GVCF: emit reference confidence records, enabling joint genotyping later.
# -L chr21:  restrict to chr21 to limit runtime.
# --dbsnp:   annotates known variants with rsIDs in the INFO/ID column.
echo "[3] HaplotypeCaller..."
gatk HaplotypeCaller \
    -I  hg002_chr21/${SAMPLE}_recal.bam \
    -R  ${REF} \
    --dbsnp ${DBSNP} \
    -L  chr21 \
    -ERC GVCF \
    -O  hg002_chr21/${SAMPLE}.g.vcf.gz

# Convert GVCF → VCF (genotype the single sample)
echo "[3] GenotypeGVCFs..."
gatk GenotypeGVCFs \
    -R  ${REF} \
    -V  hg002_chr21/${SAMPLE}.g.vcf.gz \
    --dbsnp ${DBSNP} \
    -O  hg002_chr21/variants_raw.vcf.gz

echo "[3] Raw variant counts:"
bcftools stats hg002_chr21/variants_raw.vcf.gz | grep "^SN"

# ── Step 4: Variant filtration ────────────────────────────────────────────────
# Separate SNPs and INDELs before applying type-specific filter thresholds.

echo "[4] SelectVariants: SNPs..."
gatk SelectVariants \
    -V  hg002_chr21/variants_raw.vcf.gz \
    --select-type-to-include SNP \
    -O  hg002_chr21/snps_raw.vcf.gz

echo "[4] SelectVariants: INDELs..."
gatk SelectVariants \
    -V  hg002_chr21/variants_raw.vcf.gz \
    --select-type-to-include INDEL \
    -O  hg002_chr21/indels_raw.vcf.gz

# Hard-filter SNPs (GATK recommended thresholds for human WGS single-sample)
#   QD  < 2.0  : quality normalised by depth; guards against depth-inflated QUAL
#   FS  > 60.0 : Fisher strand bias; alt allele enriched on one strand → artifact
#   MQ  < 40.0 : mean mapping quality; low MQ implies multi-mapping in repeats
#   MQRankSum < -12.5 : ALT reads systematically lower MAPQ than REF reads
#   ReadPosRankSum < -8.0 : ALT reads cluster at read ends (higher error rate)
#   SOR > 3.0  : strand odds ratio; supplements FS at high-depth sites
echo "[4] VariantFiltration: SNPs..."
gatk VariantFiltration \
    -R  ${REF} \
    -V  hg002_chr21/snps_raw.vcf.gz \
    --filter-expression "QD < 2.0"              --filter-name "QD2" \
    --filter-expression "FS > 60.0"             --filter-name "FS60" \
    --filter-expression "MQ < 40.0"             --filter-name "MQ40" \
    --filter-expression "MQRankSum < -12.5"     --filter-name "MQRankSum-12.5" \
    --filter-expression "ReadPosRankSum < -8.0" --filter-name "ReadPosRankSum-8" \
    --filter-expression "SOR > 3.0"             --filter-name "SOR3" \
    -O  hg002_chr21/snps_filtered.vcf.gz

# Hard-filter INDELs — thresholds are wider than for SNPs because insertions and
# deletions inherently produce greater strand asymmetry and read-end enrichment.
#   FS  > 200.0 : INDEL strand bias threshold
#   ReadPosRankSum < -20.0 : INDELs tolerate higher read-end enrichment
#   SOR > 10.0  : correspondingly relaxed strand-ratio threshold
echo "[4] VariantFiltration: INDELs..."
gatk VariantFiltration \
    -R  ${REF} \
    -V  hg002_chr21/indels_raw.vcf.gz \
    --filter-expression "QD < 2.0"               --filter-name "QD2" \
    --filter-expression "FS > 200.0"             --filter-name "FS200" \
    --filter-expression "ReadPosRankSum < -20.0" --filter-name "ReadPosRankSum-20" \
    --filter-expression "SOR > 10.0"             --filter-name "SOR10" \
    -O  hg002_chr21/indels_filtered.vcf.gz

# Merge filtered SNPs and INDELs back into a single coordinate-sorted VCF
echo "[4] MergeVcfs..."
gatk MergeVcfs \
    -I hg002_chr21/snps_filtered.vcf.gz \
    -I hg002_chr21/indels_filtered.vcf.gz \
    -O hg002_chr21/variants_merged.vcf.gz

# Extract only FILTER=PASS variants as the final call set
echo "[4] SelectVariants (PASS only)..."
gatk SelectVariants \
    -V  hg002_chr21/variants_merged.vcf.gz \
    --exclude-filtered \
    -O  hg002_chr21/variants_final.vcf.gz

echo "[4] Final variant counts (post-filter):"
bcftools stats hg002_chr21/variants_final.vcf.gz | grep "^SN"

# ── Step 5: Evaluate against GIAB truth set ───────────────────────────────────
# --bed-regions restricts evaluation to GIAB high-confidence regions,
# excluding centromeres, tandem repeats, and other low-mappability intervals
# where even the truth set has uncertain calls.
echo "[5] rtg vcfeval: benchmarking against GIAB truth set..."
rtg vcfeval \
    -b ${TRUTH_VCF} \
    -c hg002_chr21/variants_final.vcf.gz \
    --bed-regions ${CONFIDENT_BED} \
    -t ${REF_SDF} \
    -o eval_hardfilter/

echo ""
echo "=== Evaluation summary ==="
cat eval_hardfilter/summary.txt

echo "=== Pipeline complete: $(date) ==="

关键设计决策说明

  • GVCF 模式而非直接调用 VCF：-ERC GVCF 输出包含参考置信区间，允许后续将多个样本合并进行联合基因型定型（Joint Genotyping），避免”丢失”低频变异时的批次效应。单样本也走 GVCF → GenotypeGVCFs 路径是 GATK Best Practices 的标准做法。
  • SNP 与 INDEL 分开过滤：两类变异的技术特性（链偏倚分布、read 末端富集度）差异显著，若使用相同阈值，要么对 INDEL 过于严格（假阴性增加），要么对 SNP 过于宽松（假阳性增加）。
  • GIAB 高置信区间限定评估范围：--bed-regions 的使用至关重要——着丝粒、端粒和高度重复区域的变异即使在真实样本中也难以评估，将其排除可以得到更准确的 Precision/Recall 估算。

18.3 步骤二：预处理（MarkDuplicates + BQSR）

由于子步骤 B 截取的 BAM 来自 GIAB 官方已比对文件（Novoalign 比对至 GRCh38），无需重新执行 BWA 比对，直接从排序 BAM 开始进行预处理。

REF=reference/hg38_chr21.fasta
SAMPLE=HG002_chr21
# 输入为子步骤 B4 补充了 Read Group 的 BAM
INPUT_BAM=hg002_chr21/HG002_chr21_rg.bam

# ── 1. MarkDuplicates：标记 PCR 扩增重复 ────────────────────────────────────
# 建库 PCR 扩增会产生完全相同的 read copies，若不标记将被 HaplotypeCaller
# 重复计数，虚增等位体深度，导致假阳性。此步骤不删除 reads，仅在 FLAG
# 字段设置 0x400 标记，GATK 后续步骤会自动忽略这些 reads。
# -M: 输出重复率统计报告（包含文库复杂度估算）
gatk MarkDuplicates \
    -I ${INPUT_BAM} \
    -O hg002_chr21/${SAMPLE}_dedup.bam \
    -M logs/${SAMPLE}_dup_metrics.txt
samtools index hg002_chr21/${SAMPLE}_dedup.bam

# ── 2. BQSR 第一步：下载已知变异数据库 ──────────────────────────────────────
# BQSR 的核心原理：测序仪报告的碱基质量分数（Phred Q值）存在系统性偏差
#（如循环数越大错误率越高，但质量分数并未相应下调）。BQSR 通过统计
# BAM 中碱基的"实际错误率"来校正这一偏差，但前提是：已知真实变异位点
# 的碱基差异不能被误算为测序错误。因此需要 dbSNP 作为排除列表。
#
# 来源：Broad Institute GATK Resource Bundle FTP（匿名访问，无需账号）
# 注：Google Cloud Storage 同名文件已对匿名访问关闭（403），使用以下 FTP
wget -P reference/ \
    ftp://gsapubftp-anonymous@ftp.broadinstitute.org/bundle/hg38/dbsnp_146.hg38.vcf.gz
wget -P reference/ \
    ftp://gsapubftp-anonymous@ftp.broadinstitute.org/bundle/hg38/dbsnp_146.hg38.vcf.gz.tbi

# 仅截取 chr21 子集以加速后续步骤（基于 TBI 索引，秒级完成）
bcftools view -r chr21 reference/dbsnp_146.hg38.vcf.gz \
    -O z -o reference/dbsnp138_chr21.vcf.gz
tabix -p vcf reference/dbsnp138_chr21.vcf.gz

# ── 2. BQSR 第二步：BaseRecalibrator 建立校正模型 ───────────────────────────
# 扫描 BAM 中每个碱基，统计"报告质量 vs 实际错误率"的偏差，
# 按（read group、链方向、循环位置、前一碱基上下文）四个协变量分层建模。
# --known-sites: 标注已知变异位点，这些位置的碱基差异不计入错误
# 输出 .table 文件记录校正系数，不修改 BAM 本身
gatk BaseRecalibrator \
    -I hg002_chr21/${SAMPLE}_dedup.bam \
    -R ${REF} \
    --known-sites reference/dbsnp138_chr21.vcf.gz \
    -O logs/${SAMPLE}_recal.table

# ── 2. BQSR 第三步：ApplyBQSR 将校正系数写回 BAM ────────────────────────────
# 读取上一步的 .table，对 BAM 中每个碱基的质量分数进行实际修正，
# 输出新的 BAM 文件。此后 HaplotypeCaller 使用的质量分数已经过校正。
gatk ApplyBQSR \
    -I hg002_chr21/${SAMPLE}_dedup.bam \
    -R ${REF} \
    --bqsr-recal-file logs/${SAMPLE}_recal.table \
    -O hg002_chr21/${SAMPLE}_recal.bam
samtools index hg002_chr21/${SAMPLE}_recal.bam

为什么在人类基因组上 BQSR 有效，而细菌基因组上不做？

BQSR 需要一个”已知真实变异”的排除列表（dbSNP）：在这些位点观测到的碱基差异被认为是真实多态性而非测序错误，不参与错误率模型训练。人类 hg38 有完善的 dbSNP（1.5 亿+ 条目），可为 BQSR 提供充足的排除位点。E. coli 没有等效的公共已知变异库，若强行运行 BQSR，训练数据将同时包含真实变异和测序错误，导致模型失真。

18.4 步骤三：HaplotypeCaller 变异检测

REF=reference/hg38_chr21.fasta
SAMPLE=HG002_chr21

# ── 1. HaplotypeCaller：局部重组装 + 贝叶斯基因型定型 ───────────────────────
# 扫描 BAM，在碱基不一致率超过阈值的活跃区域（active region）内
# 用 De Bruijn 图重新组装 reads，得到候选单倍型；再用 pair-HMM
# 计算每条 read 与各单倍型的似然，最终输出贝叶斯后验最高的基因型。
# -ERC GVCF: 对所有位点（含参考一致区域）输出记录，供后续联合分析
# -L chr21:  限定分析范围，避免处理其他染色体（本教学数据集仅含 chr21）
gatk HaplotypeCaller \
    -R ${REF} \
    -I hg002_chr21/${SAMPLE}_recal.bam \
    -O hg002_chr21/${SAMPLE}.g.vcf.gz \
    -ERC GVCF \
    -L chr21

# ── 2. GenotypeGVCFs：单样本 GVCF → 标准 VCF ───────────────────────────────
# 对 GVCF 中每个位点进行最终基因型定型（genotyping），将 <NON_REF>
# 参考匹配块展开并输出只含变异位点的标准 VCF。
# 单样本场景此步为必须；多样本场景需先用 GenomicsDBImport 合并再运行此步。
# -L chr21: 与 HaplotypeCaller 保持一致，限定输出范围
gatk GenotypeGVCFs \
    -R ${REF} \
    -V hg002_chr21/${SAMPLE}.g.vcf.gz \
    -O hg002_chr21/variants_raw.vcf.gz \
    -L chr21

# ── 3. SelectVariants：将 SNP 与 Indel 分离 ─────────────────────────────────
# GATK 推荐对 SNP 和 Indel 分别应用不同的过滤阈值（下一步骤），
# 因此先拆分为两个文件，避免不同类型变异的阈值互相干扰。
gatk SelectVariants -R ${REF} \
    -V hg002_chr21/variants_raw.vcf.gz --select-type-to-include SNP \
    -O hg002_chr21/snps_raw.vcf.gz

gatk SelectVariants -R ${REF} \
    -V hg002_chr21/variants_raw.vcf.gz --select-type-to-include INDEL \
    -O hg002_chr21/indels_raw.vcf.gz

# 查看原始变异统计（过滤前基线，用于后续对比过滤效果）
echo "Raw variant counts:"
bcftools stats hg002_chr21/variants_raw.vcf.gz | grep "^SN"

对于 HG002 chr21，预期检出约 85,000–95,000 个原始 SNP 和 10,000–15,000 个原始 Indel（其中约 20–30% 为假阳性，将被步骤四的过滤去除）。

18.5 步骤四：变异过滤

步骤三生成的 snps_raw.vcf.gz 和 indels_raw.vcf.gz 包含原始检出变异，需通过过滤去除假阳性。本节提供两套方案：硬过滤适用于本教学数据集（单样本）；VQSR 演示适用于 ≥ 30 个样本的大型队列。

18.5.1 硬过滤（本节教学数据集）

REF=reference/hg38_chr21.fasta

# ── SNP 硬过滤（GATK 官方推荐阈值，适用于人类 WGS 单样本）────────────────────
# VariantFiltration 不删除记录，而是在 FILTER 列写入失败标签，保留原始数据。
# 各阈值含义：
#   QD  < 2.0  : 质量/深度比过低，深度虚高时 QUAL 虚高，QD 可归一化
#   FS  > 60.0 : Fisher 链偏倚过大，变异仅出现在某一条链（正/负链），暗示测序 artifact
#   MQ  < 40.0 : 平均比对质量过低，reads 可能来自重复区域的多重映射
#   MQRankSum < -12.5 : ALT reads 的 MAPQ 系统性低于 REF reads，暗示比对偏差
#   ReadPosRankSum < -8.0 : ALT reads 集中出现在 read 末端，末端错误率更高
#   SOR > 3.0  : 链比值偏倚，补充 FS 对高深度位点的检测能力
gatk VariantFiltration \
    -R ${REF} \
    -V hg002_chr21/snps_raw.vcf.gz \
    --filter-expression "QD < 2.0"              --filter-name "QD2" \
    --filter-expression "FS > 60.0"             --filter-name "FS60" \
    --filter-expression "MQ < 40.0"             --filter-name "MQ40" \
    --filter-expression "MQRankSum < -12.5"     --filter-name "MQRankSum-12.5" \
    --filter-expression "ReadPosRankSum < -8.0" --filter-name "ReadPosRankSum-8" \
    --filter-expression "SOR > 3.0"             --filter-name "SOR3" \
    -O hg002_chr21/snps_filtered.vcf.gz

# ── Indel 硬过滤（阈值与 SNP 不同：Indel 的 FS/ReadPosRankSum 天然更分散）──
#   FS  > 200.0 : Indel 链偏倚阈值更宽松，因插入缺失本身会产生更大的链不均衡
#   ReadPosRankSum < -20.0 : 同理，Indel 在 read 末端的富集程度更高
#   SOR > 10.0  : Indel 的链比值阈值相应放宽
gatk VariantFiltration \
    -R ${REF} \
    -V hg002_chr21/indels_raw.vcf.gz \
    --filter-expression "QD < 2.0"               --filter-name "QD2" \
    --filter-expression "FS > 200.0"             --filter-name "FS200" \
    --filter-expression "ReadPosRankSum < -20.0" --filter-name "ReadPosRankSum-20" \
    --filter-expression "SOR > 10.0"             --filter-name "SOR10" \
    -O hg002_chr21/indels_filtered.vcf.gz

# ── MergeVcfs：将过滤后的 SNP 与 Indel 合并回单一 VCF ──────────────────────
# 维持坐标排序，后续 SelectVariants 和 rtg vcfeval 均要求输入有序 VCF
gatk MergeVcfs \
    -I hg002_chr21/snps_filtered.vcf.gz \
    -I hg002_chr21/indels_filtered.vcf.gz \
    -O hg002_chr21/variants_merged.vcf.gz

# ── SelectVariants：提取 FILTER=PASS 的变异作为最终结果 ─────────────────────
# --exclude-filtered 排除所有 FILTER 列不为 PASS 的记录
gatk SelectVariants \
    -V hg002_chr21/variants_merged.vcf.gz \
    --exclude-filtered \
    -O hg002_chr21/variants_final.vcf.gz

# 查看过滤后变异统计（与过滤前对比，评估假阳性去除比例）
bcftools stats hg002_chr21/variants_final.vcf.gz | grep "^SN"

# ── rtg vcfeval：与 GIAB 真阳性集比对，计算 Precision / Recall / F1 ──────────
# -b: benchmark（真阳性 VCF）  -c: call set（本次检测结果）
# --bed-regions: 限定评估范围至 GIAB 高置信区间，排除低质量区域干扰
# -t: SDF 格式参考基因组索引（由子步骤 A3 的 rtg format 生成）
# 输出目录 eval_hardfilter/ 含 summary.txt（含 F1 最优阈值）和 ROC 曲线数据
rtg vcfeval \
    -b giab/HG002_chr21_truthset.vcf.gz \
    -c hg002_chr21/variants_final.vcf.gz \
    --bed-regions giab/HG002_chr21_confident.bed \
    -t reference/hg38_chr21.sdf \
    -o eval_hardfilter/

用 GIAB 真阳性集量化过滤效果

HG002 的 GIAB 真阳性变异集已对 chr21 全部位点做出”高置信”标注，可直接用 rtg vcfeval 或 hap.py 计算过滤前后的 Precision / Recall / F1，量化硬过滤阈值对假阳性和假阴性的具体影响——这是评估过滤策略最严格的方式。须通过 --regions giab/HG002_chr21_confident.bed 参数限定评估范围。

18.6 步骤五：Python 解析与质量评估

步骤四产生的 variants.vcf 是经过过滤的最终变异集。本步骤使用 Python 对其进行统计分析，验证过滤效果并可视化质量分布。

18.6.1 VCF 解析与过滤

import vcf
from collections import defaultdict

class VCFAnalyzer:
    def __init__(self, vcf_file):
        self.vcf_reader = vcf.Reader(filename=vcf_file)
        self.variants = []

    def parse_vcf(self):
        for record in self.vcf_reader:
            self.variants.append(record)

    def quality_distribution(self):
        quals = [v.QUAL for v in self.variants if v.QUAL is not None]

        print("\n=== 质量评分分布 ===")
        print(f"总变异数: {len(self.variants)}")
        print(f"平均 QUAL: {sum(quals) / len(quals):.2f}")
        print(f"最小 QUAL: {min(quals):.2f}")
        print(f"最大 QUAL: {max(quals):.2f}")
        print(f"QUAL >= 30: {len([q for q in quals if q >= 30])} ({len([q for q in quals if q >= 30]) / len(quals) * 100:.1f}%)")
        print(f"QUAL >= 50: {len([q for q in quals if q >= 50])} ({len([q for q in quals if q >= 50]) / len(quals) * 100:.1f}%)")

    def variant_type_distribution(self):
        snp_count = 0
        indel_count = 0
        other_count = 0

        for v in self.variants:
            ref_len = len(v.REF)
            alt_len = len(v.ALT[0]) if v.ALT else 0

            if ref_len == alt_len and ref_len == 1:
                snp_count += 1
            elif ref_len != alt_len:
                indel_count += 1
            else:
                other_count += 1

        print("\n=== 变异类型分布 ===")
        print(f"SNP: {snp_count} ({snp_count / len(self.variants) * 100:.1f}%)")
        print(f"Indel: {indel_count} ({indel_count / len(self.variants) * 100:.1f}%)")
        print(f"其他: {other_count}")

    def depth_analysis(self):
        depths = []

        for v in self.variants:
            if 'DP' in v.INFO:
                dp = v.INFO['DP']
                if isinstance(dp, list):
                    depths.append(dp[0])
                else:
                    depths.append(dp)

        if depths:
            print("\n=== 覆盖深度分析 ===")
            print(f"平均深度: {sum(depths) / len(depths):.2f}X")
            print(f"中位数深度: {sorted(depths)[len(depths)//2]:.2f}X")
            print(f"最小深度: {min(depths)}X")
            print(f"最大深度: {max(depths)}X")

    def filter_variants(self, qual_min=50, dp_min=10, dp_max=300):
        filtered = []

        for v in self.variants:
            if v.QUAL is None or v.QUAL < qual_min:
                continue

            if 'DP' in v.INFO:
                dp = v.INFO['DP']
                if isinstance(dp, list):
                    dp = dp[0]
                if dp < dp_min or dp > dp_max:
                    continue

            if 'QD' in v.INFO:
                qd = v.INFO['QD']
                if isinstance(qd, list):
                    qd = qd[0]
                if qd < 2.0:
                    continue

            filtered.append(v)

        print(f"\n=== 过滤结果 ===")
        print(f"原始变异: {len(self.variants)}")
        print(f"过滤后: {len(filtered)} ({len(filtered)/len(self.variants)*100:.1f}%)")
        print(f"已过滤: {len(self.variants) - len(filtered)}")

        return filtered

    def genotype_analysis(self):
        gt_counts = defaultdict(int)

        for v in self.variants:
            for sample in v.samples:
                gt = sample['GT']
                gt_counts[gt] += 1

        print("\n=== 基因型分布 ===")
        for gt, count in sorted(gt_counts.items()):
            print(f"  {gt}: {count}")


if __name__ == "__main__":
    analyzer = VCFAnalyzer("variants.vcf")
    analyzer.parse_vcf()
    analyzer.quality_distribution()
    analyzer.variant_type_distribution()
    analyzer.depth_analysis()
    analyzer.genotype_analysis()
    filtered = analyzer.filter_variants(qual_min=50, dp_min=10, dp_max=300)

18.6.2 基因型质量手工计算

def calculate_genotype_quality(pl_values):
    """
    PL (Phred-Likelihood) 是各基因型的负对数似然
    GQ = 第二好基因型 - 最好基因型的 PL 值

    例: PL = [0, 30, 100]
        最好: 0 (0/0)
        第二好: 30 (0/1)
        GQ = 30 - 0 = 30
    """
    if len(pl_values) < 2:
        return 0

    sorted_pl = sorted(pl_values)
    gq = sorted_pl[1] - sorted_pl[0]
    return gq


pl_example = [0, 30, 100]
gq = calculate_genotype_quality(pl_example)
print(f"PL 值: {pl_example}")
print(f"GQ = {gq}")
# 输出: GQ = 30
# 含义: 第二好基因型（0/1）比最好基因型（0/0）低 30 PHRED，
#       即该位点为杂合的置信度为 99.9%（Q30 = 0.1% 错误率）

参考答案 (Reference Answers)

  1. HaplotypeCaller 使用贝叶斯模型，在活跃区域（active region）内进行局部 de novo 重组装，再对所有可能的单倍型组合计算似然，避免了比对偏差（alignment bias）对低频变异的压制。贝叶斯先验还引入了群体等位频率信息，使低频真实变异（如 AF=1%）不因频率低而被误判为错误。

  2. chr1 位置 1000，参考等位体 A，替代等位体 G。QUAL=100 表示变异置信度极高（错误概率 10⁻¹⁰）；FILTER=PASS 表示通过所有质量过滤；DP=100 为合理深度；AF=0.05 表示替代等位体频率 5%，属低频变异。样本基因型 GT=0/1（杂合），GQ=40 表示基因型置信度 Q40（99.99%）。综合判断：QUAL 高、深度合理、GQ 充足，该变异可信，但 AF=5% 在二倍体样本中偏低（杂合预期约 50%），需进一步确认是体细胞变异还是胚系嵌合。

  3. 1000X 远超正常范围（WGS 通常 10–300X）。三种成因：(1) 重复序列多倍映射——同一基因组区域的序列过于相似，来自不同位置的 reads 错误堆积在此处；应检查该区域是否为已知重复序列，并考虑过滤 MAPQ < 20 的低质量比对；(2) 样本混合或污染——多个样本混入导致深度翻倍，应重新检查建库和测序流程；(3) CNV（拷贝数变异）——该区域在基因组中真实存在多拷贝，此时高深度是真实生物信号，不应简单过滤，而应专门分析 CNV。

  4. (a) 正常 WGS Ti/Tv 为 2.0–2.1，1.3 远低于正常值，不在可接受范围内。(b) 最可能原因：过滤阈值不足导致大量假阳性 SNP 残留，而假阳性 SNP 倾向于颠换（因测序错误随机分布在8种碱基替换中，颠换概率高于转换）；也可能是比对质量差导致大量错误比对位点被错误检出。(c) 应收紧过滤参数（提高 QUAL 阈值、增加 QD 要求），重新统计 Ti/Tv 验证改善效果；若问题持续，检查参考基因组版本和比对参数。

  5. 核心区别：硬过滤使用预设的固定阈值（如 QUAL ≥ 50）逐条判断变异；VQSR 使用多维高斯混合模型，学习已知真实变异的质量特征分布，为每个变异打出连续的概率评分，再按分位线过滤。VQSR 更精细但需要足够数据训练模型。5 个 WGS 样本应选择硬过滤，原因：VQSR 的 SNP 模型至少需要 30 个样本才有足够的变异位点提供训练统计功效，样本量不足会导致模型过拟合或无法收敛。

  6. GVCF 与普通 VCF 的核心区别：GVCF 记录所有基因组位置的基因型信息（包括与参考一致的区域，用 <NON_REF> 占位符和 END 字段表示参考匹配块），而普通 VCF 只记录检测到变异的位置。GVCF 必须使用的场景：多样本联合基因型定型（joint genotyping）——在 GenotypeGVCFs 步骤中，工具需要知道每个样本在每个位置的”参考置信度”才能正确计算群体基因型，避免将某样本的”无数据”误判为”参考纯合”。

18.8 总结

  • 变异类型按规模分为 SNP、Indel、SV、CNV 四类；Ti/Tv 比值是 SNP 集合整体质量的关键 QC 指标
  • 贝叶斯模型（GATK HaplotypeCaller）通过局部重组装和先验信息，显著优于简单频率法
  • VCF 格式是变异信息的通用存储标准；GVCF 扩展了 VCF 以支持多样本联合分析
  • 过滤策略需综合 QUAL、DP、QD、链偏倚等多指标；单样本用硬过滤，大型队列用 VQSR
  • GIAB 真阳性集提供了量化过滤效果的黄金标准，可精确计算 Precision / Recall / F1
