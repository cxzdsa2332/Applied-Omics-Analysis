20  Bulk 转录组分析基础：质量控制、定量与差异表达

Published

June 22, 2026

21 整体概述

Bulk RNA-seq 是转录组学研究的基石技术，自 2008 年前后逐步取代微阵列（Microarray），成为差异表达研究的主流范式。其核心分析问题可以概括为：在给定实验条件下，哪些基因的转录水平发生了统计显著的变化，变化幅度有多大，以及这些变化在生物学功能层面意味着什么。要回答这些问题，分析者需要依次完成三个阶段的工作：原始数据的质量控制与预处理、基因表达量的精确定量、以及在统计模型框架下的差异表达检验与功能解读。本章以贯穿全章的 airway 数据集（Himes et al. 2014，人类气道平滑肌细胞，4 个细胞系、dexamethasone 处理与对照的 2×4 配对设计）为实践主线，系统讲解从原始 FASTQ 文件到功能富集报告的完整标准流程。在方法论层面，本章着重阐明每一步骤的统计原理与参数选择逻辑——从 Phred 质量分数的概率解释、负二项分布对过离散的建模，到超几何检验与 GSEA 置换检验的手动推演——以便读者在使用工具的同时真正理解工具背后的假设与局限，为第5章高级转录组分析奠定方法论基础。

学习目标：掌握 RNA-seq 文库特性与数据质量评估标准；理解 splice-aware 比对算法原理及两种主流定量策略的差异；能独立运行 DESeq2/edgeR/limma-voom 差异表达分析并正确解读结果；掌握 ORA 与 GSEA 功能富集分析的原理与实践，识别结果解读中的常见误区。

23 模块 A：数据质控与表达量定量

模块目标： 理解 RNA-seq 文库特性对数据质量的影响，掌握 splice-aware 比对的算法原理，能在比对后计数与伪比对两种定量策略之间做出合理选择。

23.1 专题 4.1：测序原理、文库特性与原始数据质量控制

RNA-seq 数据的质量特征与 DNA 测序数据存在系统性差异，这些差异根植于文库构建方式的不同。poly-A 富集文库优先捕获转录本 3’ 端，rRNA 去除（RiboZero）文库覆盖分布更均匀但可能残留非编码 RNA 信号，而链特异性文库（Strand-specific library）通过 dUTP 策略记录 RNA 链的方向信息，是现代 RNA-seq 实验的标配。本专题系统讲解上述文库类型的特性，介绍 Phred 质量分数的概率解释与 FASTQ 格式编码规则，并讲解如何通过 FastQC 和 MultiQC 识别文库特有的质控异常信号、利用 fastp 完成 Adapter 修剪与质量过滤。本专题还重点讨论批次效应的成因与早期检测方法——PCA 图与 RLE 图是判断批次效应是否主导样本间变异的核心工具，应在差异分析之前完成诊断。

核心概念：

  • Illumina 测序化学：流动槽与簇扩增（Cluster amplification）；边合成边测序（SBS）的荧光信号产生机制
  • Phred 质量分数：\(Q = -10 \log_{10}(P_e)\)；Phred+33 编码（FASTQ 格式）；Q30 作为质控基准
  • RNA 完整性：RNA 完整性评分（RIN, RNA Integrity Number）；RIN 与 3’/5’ 覆盖度偏差的关系
  • 文库构建两条路线：poly-A 富集（mRNA-seq）vs rRNA 去除（Total RNA-seq）；各自的覆盖度偏差模式
  • 链特异性文库：TruSeq Stranded 的 dUTP 策略；链方向的生物信息学确认（infer_experiment.py / RSeQC）
  • FastQC 关键模块解读：Per base sequence quality、Per sequence GC content、Sequence Duplication Levels、Overrepresented sequences
  • RNA-seq 特有的质控指标：rRNA 污染比例、3’ 端偏差（read_distribution.py）、外显子 / 内含子 / 基因间区比例
  • fastp 参数选择：--detect_adapter_for_pe（自动检测）vs 手动指定；--trim_poly_x 处理 poly-A read-through
  • MultiQC 汇总报告：多样本质控状态的快速纵览
  • 批次效应早期检测：PCA 图判断主要变异来源；RLE（Relative Log Expression）图识别系统性偏移；实验设计层面的规避策略

实践内容：对 airway 数据集（SRR1039508–SRR1039521，1M-read 教学子集）运行 FastQC → fastp 修剪 → FastQC 复检 → MultiQC 汇总，确认链特异性方向，生成贯穿全章的样本元数据表；运行 PCA 与 RLE 图诊断批次效应。

23.2 专题 4.2：Splice-Aware 比对与表达量定量

RNA-seq 比对面临 DNA 比对所没有的根本挑战：成熟 mRNA 已切除内含子，一个跨越剪接位点的读段（junction read）在基因组上对应两段不连续区间，普通比对工具无法正确处理。splice-aware 比对算法（STAR、HISAT2）通过两步策略解决这一问题：先将读段映射至外显子区域，再利用已知或新发现的剪接位点正确拼接跨内含子部分。在定量层面，本专题深入对比两种主流策略：featureCounts 依赖比对结果逐外显子计数，而 Salmon 的伪比对（quasi-mapping）直接将读段与转录本序列匹配，无需全基因组比对，速度更快且内置偏差校正模型。本专题还介绍 SAM/BAM 格式的核心字段解析与比对质量诊断方法，以及如何通过与 airway 官方计数矩阵的相关性分析验证定量结果的可靠性。

{{Wiki:RNA-Seq}}

核心概念：

  • Splice-aware 比对的必要性：junction read 的挑战；Read-through 问题
  • STAR 算法：最大可映射前缀（Maximal Mappable Prefix, MMP）种子查找；两遍比对（2-pass mode）提高新型剪接位点检出率；--outSAMtype BAM SortedByCoordinate 直接输出排序 BAM
  • HISAT2 的层次图 FM 索引（Hierarchical Graph FM-index）：全局基因组索引 + 局部剪接图索引；适用于内存受限场景
  • GTF 注释文件的作用：外显子坐标为 featureCounts 提供计数边界；转录本序列用于 Salmon 索引构建
  • SAM/BAM 格式：FLAG 字段（比对方向、配对状态）；CIGAR 字符串解析（M/N/S 含义）；samtools 常用操作
  • featureCounts 参数：-s（链特异性方向，0/1/2）；-p（双端模式）；多比对读段处理策略；Summary 文件解读
  • Salmon 工作流：salmon index（转录本索引）→ salmon quant（伪比对定量）；--gcBias 与 --seqBias 偏差校正
  • TPM vs FPKM vs 原始计数（raw counts）：各自适用场景；DESeq2 必须使用原始计数的统计原因
  • tximport：将 Salmon 转录本级别定量汇总至基因级别，传递推断不确定性（inferential replicates）
  • 定量质量诊断：未分配 reads 的成因分类；基因检出数量的合理范围；与参考计数矩阵的相关性验证

实践内容：STAR 双端比对 airway 数据集（1M-read 子集）至 GRCh38；featureCounts 链特异性计数（-s 2）；Salmon 伪比对定量；tximport 汇总；诊断 featureCounts Summary 中未分配比例；散点图对比两种定量方法与 airway 官方矩阵的相关性（Pearson r > 0.99 为合格）。

24 模块 B：差异表达分析与功能富集

模块目标： 掌握负二项分布框架下的差异表达统计检验原理，理解多重检验校正的必要性，能独立运行 DESeq2/edgeR/limma-voom 并正确解读结果；掌握 ORA 与 GSEA 两类富集分析的方法论差异与适用场景。

24.1 专题 4.3：差异表达分析

差异表达分析的统计问题本质是：在有限重复数（通常 3–5 个）条件下，如何可靠估计每个基因的表达变异，并从数以万计的基因中识别出真实的表达差异，同时将假阳性率控制在可接受水平。本专题从泊松分布的适用边界出发，系统推导负二项分布模型对生物学过离散的建模方式，并深入讲解 DESeq2 的三步核心流程——几何均值大小因子估计、Empirical Bayes 离散度缩减、NB-GLM Wald 检验——以及离散度参数如何通过 Fisher 信息矩阵影响系数的标准误差，进而决定检验效力。本专题同时覆盖 edgeR 拟似然框架（QLF）与 limma-voom 线性模型方法，提供三种主流工具的方法论对比与共识基因集策略；在工程层面，系统讲解批次效应的两种处理路径：在设计矩阵中建模（推荐）与 ComBat/removeBatchEffect 可视化校正。

{{Wiki:Negative_binomial_distribution}}

核心概念：

  • 泊松分布的适用边界：技术重复 vs 生物学重复；过离散（Overdispersion）的三类来源
  • 负二项分布：方差结构 \(\text{Var}(K_{ij}) = \mu_{ij} + \alpha_i \mu_{ij}^2\)；基因特异性离散参数 \(\alpha_i\) 的生物学含义
  • DESeq2 第一步：几何均值大小因子估计；中位数对极端表达基因的鲁棒性
  • DESeq2 第二步：离散度估计三步流程（MLE → 趋势曲线拟合 → MAP 贝叶斯缩减）；plotDispEsts() 诊断图解读
  • DESeq2 第三步：NB-GLM Wald 检验；\(\tilde{\alpha}_i\) 固定后的系数估计；\(\text{SE}(\hat{\beta}_{i1})\) 与 \(\alpha_i\)、\(\mu_{ij}\) 的定量关系
  • LFC 缩减（apeglm）：自适应 Laplace 先验；低计数基因 LFC 向零收缩；MA 图”左侧收敛”特征
  • edgeR QLF 检验：TMM 归一化；estimateDisp() BCV 估计；quasi-likelihood 方差压缩；BCV 图解读
  • limma-voom：log-CPM 转换；LOWESS 均值-方差趋势拟合；观测值级别精度权重；eBayes(trend=TRUE, robust=TRUE)
  • 三方法比较：DESeq2 vs edgeR vs limma-voom 在 airway 数据集的 DEG 交集（Venn/UpSet）；共识 DEG 策略
  • 批次效应建模：设计矩阵包含批次协变量 ~ batch + condition（推荐）
  • 批次效应可视化校正：ComBat()（参数/非参数）与 limma::removeBatchEffect()；仅用于可视化，不得作为 DEA 输入；kBET 定量评估
  • 多重检验校正：Benjamini-Hochberg 过程的数学推导；FDR 的正确解读（padj < 0.05 ≠ 单基因 97% 真实概率）
  • 结果可视化：MA 图（apeglm 收缩效果诊断）；火山图（基础版 / 标注 HGNC symbol 增强版 / 特定基因高亮变体）；VST-PCA 图；样本间距离热图

实践内容：以 airway 完整 8 样本数据集（~ cell + dex 区组设计）运行 DESeq2 标准流程；手动重现大小因子估计与 BH 校正，对比 DESeq2 内置结果；apeglm 缩减前后 LFC 对比；edgeR QLF 与 limma-voom 并行分析；三方法 DEG 交集 Venn 图；生成带 HGNC symbol 标注的增强版火山图。

24.2 专题 4.4：功能富集分析

差异表达基因列表仅回答了”哪些基因变了”，而”这些变化在生物学上意味着什么”需要通过功能富集分析来回答。本专题先介绍 GO、KEGG、Reactome、MSigDB 四类主流功能数据库的覆盖范围与层级结构，再从超几何检验的概率质量函数出发手动推演 ORA 计算过程，并通过背景集敏感性分析直观揭示错误背景集选择的统计后果。GSEA 部分同样提供从零实现：以模拟排序列表手动计算加权富集得分（ES），并通过 2000 次基因标签置换（gene permutation）构建经验零分布，与 fgsea 结果对比验证，使读者真正理解 NES 标准化的含义与 leading edge 基因的来源。在工具应用层面，本专题以 airway 数据集运行完整的 clusterProfiler 流程，并通过 UpSet 图比较 GO/KEGG/Reactome 三库结果的跨数据库重叠，以 pathview 将 LFC 数据叠加到 KEGG 通路图，以 pheatmap 展示 Top DEG 表达谱。

{{Wiki:Gene_ontology}}

核心概念：

  • 功能数据库体系：GO 三本体（BP/MF/CC）及有向无环图（DAG）结构；祖先传播（ancestral propagation）问题与 simplify() 语义去冗余；KEGG 通路图的可视化特性；Reactome 的细粒度层级结构；MSigDB Hallmark / C2 / C5 基因集
  • ORA 统计原理：超几何检验 \(p = \sum_{i=k}^{\min(n,K)} \frac{\binom{K}{i}\binom{N-K}{n-i}}{\binom{N}{n}}\)；GeneRatio、BgRatio、富集倍数（Fold Enrichment）的计算与解读；背景基因集必须使用实验检测基因
  • GSEA 算法：基因排序指标的选择（Wald stat 优于 LFC）；加权 ES 计算（命中步增量 ∝ |stat|，未命中步均等减量）；标准化富集得分 NES；leading edge 基因的定义与生物学意义
  • 置换策略：基因标签置换（gene permutation，适用于 n < 7）vs 样本置换（sample permutation，统计更严格）；fgsea 的 eps 参数与精确近似算法
  • clusterProfiler 工具链：enrichGO()、enrichKEGG()、enrichPathway()、gseGO()、gseKEGG()、gsePathway()；ID 转换（Ensembl → Entrez）与 bitr() 的多重映射处理
  • 富集结果可视化：dotplot（GeneRatio + padj）；emapplot（GO term 语义相似度网络）；cnetplot（基因-通路关联网络，LFC 着色）；ridgeplot（GSEA 基因排序分布）；pathview（KEGG 通路 LFC 叠加）
  • 跨数据库整合：UpSet 图比较 GO / KEGG / Reactome 共同检出基因；ORA 与 GSEA 共识通路作为高置信度报告对象
  • 结果批判性解读：功能注释覆盖度偏差；通路重叠假象；p 值大小与生物学重要性的非对应关系；leading edge 基因数量对结论可靠性的影响

实践内容：Ensembl ID → Entrez ID 转换；clusterProfiler GO BP / GO MF / KEGG ORA；GO BP GSEA 与 KEGG GSEA；手动实现超几何检验并验证背景集敏感性；手动实现加权 ES + 2000 次置换检验并与 fgsea 对比；UpSet 图比较三库重叠；pathview 可视化最显著 KEGG 通路；Top DEG pheatmap；生成汇总 CSV 文件。
