32  10x Genomics 技术原理与上游处理

Published

June 22, 2026

33 专题概述

单细胞 RNA-seq（scRNA-seq）的核心技术挑战在于：如何在数以千计的细胞中，为每个细胞的 mRNA 分子打上独一无二的”地址”，以便测序完成后将读段准确归属到正确的细胞。本专题讲解 10x Genomics Chromium 平台如何用液滴微流控（droplet microfluidics）解决这一问题，以及上游软件 Cell Ranger 与开源替代 STARsolo 如何将原始测序读段转化为可供下游分析的细胞×基因表达矩阵。理解上游处理的每一步——条形码解码、UMI 去重、空液滴过滤——是正确解读下游结果、排查异常的前提。

学习目标：理解 10x Chromium 液滴微流控的技术原理与 GEM、Cell Barcode、UMI 三者的关系；掌握 Cell Ranger 上游处理的核心步骤；理解条形码纠错与 UMI 去重的逻辑；区分 EmptyDrops 与膝点法两种空液滴过滤策略；能解读 Cell Ranger Web Summary 的关键质量指标。

34 10x Genomics 技术原理

34.1 液滴微流控与 GEM

10x Genomics Chromium 平台的核心是把单个细胞与一个携带特异条形码的凝胶珠（gel bead）共同包裹进一滴油中，形成所谓的 GEM（Gel bead-in-EMulsion，凝胶珠乳液）。在微流控芯片上，细胞悬液、凝胶珠和逆转录试剂分别从不同通道流入，与油相在交汇处剪切形成数万个纳升级油滴；通过控制细胞浓度，使绝大多数油滴最多包裹一个细胞。

{{Wiki:Microfluidics}}

每个凝胶珠表面携带数十万条寡核苷酸捕获探针，它们共享同一段细胞条形码（Cell Barcode）——这是该 GEM 的”地址”，标记 mRNA 来自哪个细胞。每条探针还带有一段不同的分子条形码（Unique Molecular Identifier, UMI），用于标记单个 mRNA 分子，以及一段 poly-T 序列用于捕获带 poly-A 尾的 mRNA。油滴内完成逆转录后，所有 cDNA 都携带了”细胞地址 + 分子标签”的双重信息。

34.2 三个标签的分工

理解 scRNA-seq 数据，关键在于厘清三个标签的不同作用，见 表 1。

为什么需要 UMI： 文库制备包含 PCR 扩增，同一条原始 mRNA 会被复制成许多拷贝。若直接计数测序读段，高扩增效率的转录本会被严重高估。UMI 在逆转录时就标记了每个原始分子，因此测序后只需统计”不同 UMI 的数量”而非”读段数量”，即可还原真实的分子数，消除 PCR 扩增偏差。这是 scRNA-seq 定量准确性的基石。

35 Cell Ranger 上游处理流程

35.1 从 BCL 到表达矩阵

Cell Ranger 是 10x 官方的上游分析软件，把测序仪输出的原始数据转化为表达矩阵，核心分为两步：

  1. cellranger mkfastq：将测序仪的 BCL 原始文件按样本索引拆分为标准 FASTQ 文件。
  2. cellranger count：对单个样本的 FASTQ 执行条形码处理、比对、UMI 计数与细胞识别，输出表达矩阵。
# Step 1: demultiplex raw BCL into per-sample FASTQ files
cellranger mkfastq \
    --run=/path/to/bcl_run_folder \
    --csv=/path/to/sample_sheet.csv \
    --output-dir=/path/to/fastq_output

# Step 2: align, deduplicate UMIs, and call cells for one sample (e.g. PBMC 3k)
cellranger count \
    --id=pbmc3k \
    --transcriptome=/path/to/refdata-gex-GRCh38-2020-A \
    --fastqs=/path/to/fastq_output \
    --sample=pbmc3k \
    --localcores=8 \
    --localmem=64

cellranger count 内部依次完成：条形码处理与纠错 → STAR 比对到参考转录组 → UMI 去重与计数 → 空液滴过滤识别真实细胞。

35.2 条形码纠错

测序存在错误率，细胞条形码也可能被读错。Cell Ranger 持有一份合法条形码的白名单（whitelist）。对每个读到的条形码：若它精确匹配白名单，则直接接受；若它与某个白名单条形码仅差一个碱基，则根据该碱基的测序质量值与该候选条形码的丰度，按概率将其”纠正”到最可能的白名单条形码上。这一机制挽救了大量因单碱基测序错误而本会被丢弃的读段。

35.3 UMI 去重

比对到同一基因、且携带相同细胞条形码与相同 UMI 的多条读段，被认为来自同一原始 mRNA 分子的 PCR 拷贝，应合并计为一次。Cell Ranger 采用基于有向网络（directional network）的去重策略：考虑到 UMI 本身也可能有测序错误，它将仅差一个碱基、且丰度呈悬殊关系的 UMI 合并，从而避免把”测错的 UMI”误计为新分子。

--include-introns 参数： 默认情况下 Cell Ranger 只统计比对到外显子的读段。对于单核 RNA 测序（snRNA-seq），细胞核内富含未剪接的前体 mRNA（含内含子），此时应开启 --include-introns（新版本默认开启）以纳入内含子读段，否则会严重低估基因表达。判断依据是样本类型：完整细胞用默认设置，细胞核样本需纳入内含子。

35.4 空液滴过滤：区分细胞与背景

并非每个 GEM 都包裹了真实细胞——大量油滴只含游离 RNA（来自破裂细胞的环境 RNA，称为 ambient RNA）。区分”真实细胞条形码”与”空液滴条形码”是上游处理最关键也最易出错的一步。

两种主流策略：

  • 膝点法（Knee point）：将所有条形码按其总 UMI 数从高到低排序作图（barcode rank plot），真实细胞通常 UMI 数高、聚集在曲线左侧的”平台”，空液滴 UMI 数低、聚集在右侧；曲线急剧下降的”膝点/拐点”即为分界阈值。简单直观，但对细胞 RNA 含量差异大的样本不稳健。
  • EmptyDrops：不依赖单一阈值，而是先用极低 UMI 的条形码估计 ambient RNA 的表达谱，再对每个条形码统计检验其表达谱是否显著偏离 ambient 背景。能挽救 UMI 较低但表达谱明确的真实小细胞（如静息淋巴细胞），是当前推荐方法。

{{Wiki:Knee_of_a_curve}}

Cell Ranger 现代版本结合了两种思想：先用膝点确定一批高置信细胞，再用 EmptyDrops 式检验从其余条形码中挽救低 UMI 的真实细胞。

36 STARsolo：开源替代方案

STARsolo 是 STAR 比对器内置的 scRNA-seq 定量模块，是 Cell Ranger 的开源、高效替代，输出与 Cell Ranger 高度兼容的表达矩阵。其参数与 Cell Ranger 的处理逻辑一一对应。

运行环境说明： 下面的命令需要预先用 STAR 构建好参考基因组索引。--soloCBwhitelist 需指向对应化学版本的 10x 条形码白名单文件（10x 官方随 Cell Ranger 提供，或可单独下载）。--soloType CB_UMI_Simple 适用于标准 3’ 文库。

# STARsolo: open-source alternative producing Cell Ranger-compatible output
STAR --runMode alignReads \
    --genomeDir /path/to/STAR_index_GRCh38 \
    --readFilesIn R2.fastq.gz R1.fastq.gz \
    --readFilesCommand zcat \
    --soloType CB_UMI_Simple \
    --soloCBwhitelist /path/to/3M-february-2018.txt \
    --soloCBstart 1 --soloCBlen 16 \
    --soloUMIstart 17 --soloUMIlen 12 \
    --soloFeatures Gene \
    --runThreadN 8

读段顺序易错点： 在 10x 文库中，R1 携带细胞条形码与 UMI，R2 携带 cDNA 测序序列。STARsolo 的 --readFilesIn 要求先 cDNA（R2）后 barcode（R1），即 R2.fastq.gz R1.fastq.gz，顺序与直觉相反。写错会导致比对率几乎为零，是初学者最常见的错误之一。

37 上游输出与质量解读

37.1 输出文件结构

Cell Ranger 与 STARsolo 均输出稀疏矩阵格式的三件套文件，构成下游分析的统一入口：

  • barcodes.tsv.gz：每个真实细胞的条形码列表（矩阵的列）
  • features.tsv.gz：检测到的基因列表（矩阵的行）
  • matrix.mtx.gz：稀疏格式的表达计数矩阵（Matrix Market 格式，仅存储非零值）
library(Seurat)

# Load the 10x sparse matrix (three-file directory) into a Seurat object
pbmc_data <- Read10X(data.dir = "/path/to/pbmc3k/outs/filtered_feature_bc_matrix")
pbmc <- CreateSeuratObject(counts = pbmc_data, project = "pbmc3k",
                           min.cells = 3, min.features = 200)

# Quick sanity check: cells (columns) x genes (rows)
pbmc

37.2 Web Summary 关键指标

Cell Ranger 生成一份 HTML 格式的 Web Summary 报告，是判断实验成败的第一手依据。表 2 列出最需关注的指标。

Fraction Reads in Cells 偏低的诊断： 该指标若显著偏低（如 < 60%），通常提示样本中游离 ambient RNA 过多，常见原因是细胞解离时损伤严重、活力低。这是一个早期预警信号——它意味着大量信号来自背景而非细胞，后续即使过滤也难以完全挽救，应在湿实验层面改进细胞制备。

38 方法总结与衔接说明

本专题梳理了从液滴微流控的物理原理到表达矩阵的完整上游链条：细胞条形码定位、UMI 去重计数、空液滴过滤识别真实细胞。理解这些步骤，才能正确解读 Web Summary 指标、在数据异常时定位问题根源。但上游输出的”真实细胞”仍混杂着低质量细胞、双细胞与高背景细胞——专题 6.2 将讲解如何在细胞层面进行系统的质量控制与预处理，为降维聚类准备干净的数据。

39 课后自测 (Post-Lesson Quiz)

  1. 在 10x Chromium 平台中，Cell Barcode 与 UMI 各起什么作用？为什么二者都不可或缺？

  2. 什么是 GEM？请描述单个细胞从进入微流控芯片到生成带双标签 cDNA 的过程。

  3. 为什么 scRNA-seq 定量要统计”不同 UMI 的数量”而非”测序读段的数量”？若省略 UMI 去重会导致什么偏差？

  4. Cell Ranger 的条形码纠错机制如何工作？它在什么情况下会把一个条形码”纠正”到白名单序列上？

  5. 膝点法（Knee point）与 EmptyDrops 在空液滴过滤上有何区别？为什么 EmptyDrops 能挽救膝点法可能丢弃的真实小细胞？

  6. --include-introns 参数在什么样本类型上必须开启？为什么对单核 RNA 测序（snRNA-seq）尤为重要？

  7. 在用 STARsolo 时，--readFilesIn 中 R1 与 R2 的顺序为何与直觉相反？写错会导致什么后果？

  8. 某 Cell Ranger Web Summary 报告显示 Fraction Reads in Cells 仅为 45%，Estimated Number of Cells 远高于上样预期。请推测可能的实验问题，并说明这对下游分析意味着什么。

参考答案 (Reference Answers)

  1. Cell Barcode 标记 mRNA 来自哪个细胞（同一 GEM 内所有探针共享同一条形码），用于把读段归属到正确的细胞；UMI 标记单个 mRNA 原始分子，用于去除 PCR 扩增产生的重复。前者解决”哪个细胞”，后者解决”多少个真实分子”，缺一则要么无法归属细胞、要么无法消除扩增偏差。

  2. GEM（Gel bead-in-EMulsion，凝胶珠乳液）是包裹了单个细胞、一个携带特异条形码的凝胶珠和逆转录试剂的油滴。过程：细胞悬液、凝胶珠、试剂在微流控芯片交汇处被油相剪切成数万个纳升油滴，控制浓度使多数油滴最多含一个细胞；油滴内细胞裂解，mRNA 被凝胶珠探针的 poly-T 捕获，逆转录生成带 Cell Barcode + UMI 双标签的 cDNA。

  3. 因为文库制备含 PCR 扩增，同一条原始 mRNA 会被复制成许多拷贝，直接计读段会高估高扩增效率的转录本。UMI 在逆转录时即标记每个原始分子，统计不同 UMI 的数量即可还原真实分子数。省略 UMI 去重会引入 PCR 扩增偏差，使定量被扩增效率扭曲而非反映真实表达量。

  4. Cell Ranger 持有合法条形码白名单。读到的条形码若精确匹配白名单则直接接受；若与某白名单条形码仅差一个碱基，则结合该碱基的测序质量值与候选条形码的丰度，按概率将其纠正到最可能的白名单条形码上。这挽救了大量因单碱基测序错误本会被丢弃的读段。

  5. 膝点法将条形码按总 UMI 降序作图，以曲线急剧下降的拐点为单一阈值区分细胞与空液滴，简单但对细胞 RNA 含量差异大的样本不稳健；EmptyDrops 先用极低 UMI 条形码估计 ambient RNA 表达谱，再检验每个条形码是否显著偏离该背景。EmptyDrops 能挽救 UMI 低但表达谱明确偏离背景的真实小细胞（如静息淋巴细胞），而膝点法会因其 UMI 低于阈值而误删。

  6. 在细胞核样本（snRNA-seq）上必须开启。因为细胞核内富含未剪接的前体 mRNA（含内含子），默认只计外显子读段会严重低估基因表达；开启 --include-introns 纳入内含子读段才能正确定量。完整细胞样本则用默认设置即可。

  7. 因为 10x 文库中 R1 携带 Cell Barcode 与 UMI、R2 携带 cDNA 测序序列，而 STARsolo 的 --readFilesIn 要求先提供 cDNA（R2）、后提供 barcode（R1），即 R2 R1。写反会使比对器用条形码序列去比对基因组，导致比对率几乎为零。

  8. Fraction Reads in Cells 仅 45% 说明大量读段来自细胞外游离 ambient RNA，结合细胞数远超预期，提示空液滴/低质量条形码被误判为细胞——很可能是细胞解离损伤严重、活力低、破裂细胞释放大量游离 RNA。对下游意味着：背景信号污染严重，即使过滤也难完全挽救，细胞类型边界会模糊，应优先在湿实验层面改进细胞制备而非仅靠计算补救。
