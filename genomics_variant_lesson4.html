18  GWAS 结果可视化、多重检验与候选位点解析

18.1 学习目标

完成本专题学习后，学生应能够：

  1. 绘制并规范解读曼哈顿图（Manhattan plot）和 QQ 图（Quantile-Quantile plot），正确识别信号峰、背景噪声和系统性偏差的视觉特征；
  2. 理解基因组膨胀因子 \(\lambda_{GC}\) 的定义、来源与局限，并使用 LD Score 回归（LDSC）区分群体分层偏差与真实多基因性信号；
  3. 掌握 Bonferroni 校正、FDR（BH 程序）与置换检验三种多重检验校正方法的原理与适用场景，理解 FWER 与 FDR 的概念区别；
  4. 根据非中心参数（NCP）公式估算 GWAS 统计检验力，理解样本量、效应量、MAF 与显著性阈值的交互影响；
  5. 使用 PLINK 运行完整的逻辑回归和线性回归关联分析，解读输出结果，并利用 GCTA-GREML 估计 SNP 遗传力；
  6. 利用 LD 裁剪（clumping）和逐步条件分析识别独立关联信号，并利用局部关联图（LocusZoom 风格）展示候选区域内的 LD 结构与信号形态。

18.2 GWAS 可视化概述

一项典型的 GWAS 针对数百万个 SNP 分别计算关联统计量，最终产生等量的 p 值、效应量和标准误。面对如此规模的统计结果，可视化不仅是展示工具，更是质量控制和科学发现的必要环节。GWAS 结果的可视化目标体现在三个层次：在全基因组层次，需要快速定位统计显著位点在染色体上的分布格局，识别可重复的强信号；在统计诊断层次，需要评估检验统计量的整体分布是否符合零假设预期，判断群体分层等系统性偏差是否已被充分控制；在候选区域层次，需要精细呈现显著位点周围的 LD 结构，为候选基因识别和精细定位提供视觉基础。

三类核心图形各有侧重，在 GWAS 报告中缺一不可：

图形类型 目的 核心信息 典型使用场景
曼哈顿图 全局信号定位 各染色体上显著 SNP 位置与 \(-\log_{10}(p)\) 论文主图，展示发现位点
QQ 图 模型诊断 观测 p 值与理论均匀分布的偏离程度 与曼哈顿图并排，质控证据
局部关联图 候选区域精细分析 LD 结构、候选 SNP 与周边变异的关系 补充材料，每个显著位点一图

从数据流动的角度来看，GWAS 可视化遵循”从粗到细”的分析逻辑：首先通过 QQ 图确认统计模型无系统性偏差，再通过曼哈顿图识别全基因组显著位点，最后对每个候选位点生成局部关联图以深入解析信号结构。这一顺序不仅是展示逻辑，也是分析流程的自然顺序——跳过 QQ 图诊断直接报告曼哈顿图是对统计严谨性的放弃，因为系统性偏差会产生假阳性”信号峰”，在曼哈顿图中与真实信号外观无异。

18.3 曼哈顿图

18.3.1 图形结构与规范解读

{{Wiki:Manhattan_plot}}

曼哈顿图（Manhattan plot）是 GWAS 结果的标准可视化形式，其名称来源于纽约曼哈顿天际线的外形相似性——显著位点如高楼般突出于背景噪声之上。x 轴按染色体顺序排列所有 SNP 的基因组位置，y 轴显示 \(-\log_{10}(p)\)。通常叠加两条水平参考线：

  • 显著性阈值线（实线）：\(-\log_{10}(5 \times 10^{-8}) \approx 7.3\)，对应 Bonferroni 校正后的全基因组显著性；
  • 建议性阈值线（虚线）：\(-\log_{10}(1 \times 10^{-5}) \approx 5.0\)，用于标记值得关注但未达严格显著的候选位点。

显著性峰（signal peaks） 是关注的核心。超过 \(5 \times 10^{-8}\) 阈值的位点被视为基因组显著（genome-wide significant）。一个典型的真实信号通常不是单个孤立的点，而是一组相邻 SNP 共同超过阈值——这是因为芯片对 LD 区块内多个 SNP 均进行了分型，每个 SNP 获得的信号强度正比于其与因果变异的 \(r^2\)，由此形成峰顶最显著、两侧随 LD 衰减平滑下降的”山峰”轮廓，峰宽反映 LD 区块的物理范围。若仅有单个孤立 SNP 超过阈值而邻近位点均无信号，则应怀疑技术假象（如基因型判定错误或染色体结构变异）。

背景分布应贴近 \(y = 0\) 基线（即 \(-\log_{10}(p) \approx 0\) 对应 \(p \approx 1\)）。若整体背景升高，而非仅有少数峰突出，这是 \(\lambda_{GC}\) 膨胀的直观表现，提示群体分层或亲缘关系未被充分控制（参见 3.5.3 节）。

超过显著性阈值的 SNP 簇通常对应一个 LD 区块内的多个相关变异，而非多个独立位点，需要通过条件分析或 LD 裁剪（clumping）识别独立信号（参见 3.5.7 节）。

18.3.2 R 实现（ggplot2）

library(tidyverse)

# 读取 GWAS 结果（以 MLM 输出格式为例）
gwas <- read_tsv("output/gwas_mlm_result.assoc.txt") %>%
  filter(!is.na(p_wald)) %>%
  rename(CHR = chr, BP = ps, P = p_wald)

# 计算每个染色体的累积坐标
gwas_plot <- gwas %>%
  group_by(CHR) %>%
  summarize(chr_len = max(BP), .groups = "drop") %>%
  mutate(tot = cumsum(chr_len) - chr_len) %>%
  left_join(gwas, by = "CHR") %>%
  mutate(BPcum = BP + tot) %>%
  arrange(CHR, BP)

# 染色体中心坐标（用于 x 轴标签）
axis_df <- gwas_plot %>%
  group_by(CHR) %>%
  summarize(center = (max(BPcum) + min(BPcum)) / 2, .groups = "drop")

# 绘制曼哈顿图
ggplot(gwas_plot, aes(x = BPcum, y = -log10(P))) +
  geom_point(aes(color = as.factor(CHR %% 2)),
             alpha = 0.75, size = 0.8) +
  scale_color_manual(values = c("steelblue", "grey60")) +
  geom_hline(yintercept = -log10(5e-8),
             color = "red", linetype = "dashed", linewidth = 0.6) +
  geom_hline(yintercept = -log10(1e-5),
             color = "orange", linetype = "dotted", linewidth = 0.5) +
  scale_x_continuous(
    breaks = axis_df$center,
    labels = axis_df$CHR,
    expand = c(0.02, 0)
  ) +
  labs(x = "Chromosome",
       y = expression(-log[10](p)),
       title = "Manhattan Plot: GWAS Results") +
  theme_classic() +
  theme(legend.position = "none",
        axis.text.x = element_text(size = 7))

18.4 QQ 图与基因组膨胀因子

18.4.1 QQ 图的系统解读

QQ 图（Quantile-Quantile Plot）是评估 GWAS 统计假设是否成立的关键诊断工具。其横轴为期望 \(-\log_{10}(p)\)（假设所有 SNP 均为零假设下的均匀分布），纵轴为观测值。理想 QQ 图的形态：绝大多数点（约 99% 以上）沿 \(y = x\) 对角线分布，仅在图的右上角（高显著性区域）出现偏离，形成”钩形（hook）“上翘，代表真实关联信号。全局膨胀（global inflation）：曲线从低 \(-\log_{10}(p)\) 区域便开始系统性上偏对角线，\(\lambda_{GC}\) 明显大于 1，主要原因有三：群体分层校正不足、样本间存在未被考虑的亲缘关系，或极强的多基因性。全局通缩（deflation）：曲线整体低于对角线，提示样本量过小（检验力不足）或模型过度校正。

# QQ 图（基于真实 GWAS 结果）
observed <- sort(-log10(gwas$P), decreasing = TRUE)
expected <- -log10(ppoints(length(observed)))

qq_df <- data.frame(expected = expected, observed = observed)

ggplot(qq_df, aes(x = expected, y = observed)) +
  geom_point(size = 0.5, alpha = 0.5, color = "steelblue") +
  geom_abline(slope = 1, intercept = 0,
              color = "red", linetype = "dashed") +
  labs(x = expression("Expected" ~ -log[10](italic(p))),
       y = expression("Observed" ~ -log[10](italic(p))),
       title = "QQ Plot") +
  theme_classic()

# Genomic inflation factor
lambda_gc <- median(qchisq(gwas$P, df = 1, lower.tail = FALSE),
                    na.rm = TRUE) /
             qchisq(0.5, df = 1, lower.tail = FALSE)
cat(sprintf("lambda_GC = %.4f\n", lambda_gc))

18.4.2 基因组膨胀因子（Genomic Inflation Factor）

基因组膨胀因子（Genomic Inflation Factor，\(\lambda_{GC}\)） 是 GWAS 中评估全基因组范围内系统性统计偏差的核心诊断指标，最早由 Devlin 和 Roeder（1999）在基因组控制（Genomic Control，GC）框架下提出。其定义为：

\[ \lambda_{GC} = \frac{\text{median}(\chi^2_{1,\,\text{observed}})}{F_{\chi^2_1}^{-1}(0.5)} = \frac{\text{median}(\chi^2_{1,\,\text{observed}})}{0.4549} \]

其中分母 \(F_{\chi^2_1}^{-1}(0.5) \approx 0.4549\) 为自由度为 1 的 \(\chi^2\) 分布的理论中位数。在零假设成立且样本相互独立的情况下，\(\lambda_{GC}\) 的期望值为 1.0。选用中位数而非均值是刻意的：中位数对右尾极端值（即真实关联信号）不敏感，能更稳健地反映背景分布的系统偏差。

18.4.2.1 膨胀的来源

导致 \(\lambda_{GC} > 1\) 的原因可分为以下三类：

（1）群体分层（Population Stratification）：若病例组和对照组在祖先构成上存在差异，则任何与群体起源相关的 SNP 都会产生虚增的 \(\chi^2\) 值。群体分层导致的膨胀均匀作用于全基因组，QQ 图曲线从低 \(-\log_{10}(p)\) 区域即开始整体高于对角线。

（2）隐匿亲缘关系（Cryptic Relatedness）：若研究样本中存在未被记录的亲缘关系，相关个体在基因型和表型上均产生系统性相似，违背样本独立性假设，同样导致全局 \(\lambda_{GC}\) 升高。混合线性模型通过将基因组亲缘关系矩阵（GRM）纳入随机效应，可同时校正分层与隐匿亲缘关系。

（3）真实的多基因性（Polygenicity）：对于高度多基因性的复杂性状（如身高、BMI、精神分裂症），成千上万个 SNP 各携带微弱但真实的效应，导致全基因组 \(\chi^2\) 统计量整体性升高。在超大型队列（如 UK Biobank，\(N \approx 50\) 万）中，\(\lambda_{GC}\) 可能达到 1.5 甚至更高，完全源于多基因性而非统计假象。

18.4.2.2 \(\lambda_{GC}\) 的尺度依赖性

\(\lambda_{GC}\) 的数值与样本量高度相关。理论上，\(\lambda_{GC}\) 随样本量 \(N\) 近似线性增长：

\[ \lambda_{GC} \approx 1 + \frac{N \cdot h^2_{\text{SNP}}}{M} \]

为便于跨研究比较，可将 \(\lambda_{GC}\) 标准化至 \(N = 1000\) 的等效值：

\[ \lambda_{1000} = 1 + (\lambda_{GC} - 1) \times \frac{1000}{N_{\text{median}}} \]

在不同样本量的研究之间直接比较 \(\lambda_{GC}\) 是无意义的，应改用 \(\lambda_{1000}\) 评估残余系统偏差。

18.4.2.3 基因组控制校正及其局限

当确认 \(\lambda_{GC}\) 升高源于系统偏差（而非多基因性）时，可通过基因组控制对所有 \(\chi^2\) 统计量进行缩放校正：

\[ \chi^2_{\text{corrected}} = \frac{\chi^2_{\text{observed}}}{\lambda_{GC}} \]

然而，GC 校正存在两项根本性局限：第一，对全基因组所有位点施加相同的缩放系数，无法区分真实信号与噪声，会同时压缩真实关联信号和假阳性；第二，无法区分膨胀来源——若膨胀主要来自多基因性而非分层，校正将丢失真实信号。

切勿在多基因性导致膨胀时使用 GC 校正

若 LD Score 回归（LDSC）分析显示截距接近 1.0，说明膨胀主要来源于多基因信号而非系统偏差，此时对统计量施加 \(\lambda_{GC}\) 校正将不当压缩真实信号，增加假阴性。正确做法是保留原始 p 值，并在报告中注明膨胀来源。

18.4.2.4 LD Score 回归：区分膨胀来源

LD Score 回归（LD Score Regression，LDSC） 是当前区分群体分层与多基因性导致的 \(\lambda_{GC}\) 升高的标准方法。其核心原理为：若膨胀来自真实的多基因信号，则位点的 \(\chi^2\) 统计量期望值应与该位点的 LD Score（\(\ell_j = \sum_k r^2_{jk}\)）成正比；若膨胀来自群体分层（与 LD 结构无关的全局偏差），则截距项应升高：

\[ E[\chi^2_j] = \frac{Nh^2_{\text{SNP}}}{M} \ell_j + Na + 1 \]

截距接近 1 意味着无显著分层，\(\lambda_{GC}\) 的膨胀完全可以归因于多基因性；截距显著大于 1 则提示需要进一步的分层校正。

0.95 – 1.05 无系统偏差（正常） 无需处理

> 1.10（小样本，\(N < 10{,}000\)） 群体分层或亲缘关系 增加 PCA 主成分数或使用 MLM
> 1.10（大样本，\(N > 100{,}000\)） 多基因性为主 运行 LDSC 判断来源，不建议 GC 校正
< 0.95 过度校正或样本量不足 检查 QC 流程，避免去除过多样本

18.4.3 模拟实验：PCA 校正前后的 QQ 图对比

以下代码模拟一个含有两个亚群的 GWAS 数据集，分别展示未经群体结构校正和纳入前两个主成分校正后的 QQ 图。模拟采用现实中较为常见的不平衡招募设计：两亚群的等位基因频率差异约为 0.15（对应中等强度 \(F_{ST}\)），病例中 75% 来自亚群1，对照中 75% 来自亚群2。这一设计贴近多中心招募或多民族队列中常见的祖先-表型混淆情形，足以产生视觉上清晰可辨的 QQ 图膨胀（\(\lambda_{GC} \approx 1.5\)–\(2.5\)），同时保留了真实信号在校正后的可见性。

第一步：构建含群体结构的基因型矩阵

两个亚群（各 400 人）在每个 SNP 上具有不同的等位基因频率。maf1 为亚群1的 MAF，maf2 通过在 maf1 基础上叠加均值为 0、标准差为 0.15 的正态扰动获得，使两群间的平均 MAF 差约为 0.15，对应中等强度的群体分化（\(F_{ST}\)）。每个个体的基因型在加性模型下独立地服从 \(\text{Binomial}(2, \text{maf})\)，分别为 0、1、2 个效应等位基因拷贝数。

set.seed(2024)
library(ggplot2)

n         <- n_per_pop * 2

             nrow = n_per_pop, ncol = M)

             nrow = n_per_pop, ncol = M)

第二步：模拟存在群体分层的病例/对照表型

病例和对照在两个亚群间的分配并不均等：亚群1中 75% 为病例（300 人），亚群2中 75% 为对照（300 人）。这种不平衡使”病例”在一定程度上等价于”亚群1成员”，从而将亚群间的任何等位基因频率差异都转化为虚假的关联信号。在现实中，这类情形常见于多中心 GWAS——各中心的病例/对照比例不同，且各中心招募的人群在祖先构成上有所差异。

随后通过阈值模型（liability threshold model） 在分层表型基础上叠加少量真实遗传信号：计算 20 个因果 SNP 对个体遗传负担（liability）的贡献，将遗传负担最高的 25% 个体的病例/对照标签翻转，使真实信号以中等强度混入分层噪声中。

# 75% of cases come from pop1, 75% of controls from pop2

n_case_p2 <- n_per_pop - round(n_per_pop * 0.75)
n_ctrl_p2 <- n_per_pop - n_case_p2

y <- c(rep(1L, n_case_p1), rep(0L, n_ctrl_p1),
       rep(1L, n_case_p2), rep(0L, n_ctrl_p2))

# Liability threshold model: overlay weak true signals on top of stratification
causal_idx   <- sample(M, n_causal)
causal_betas <- rnorm(n_causal, sd = 0.5)
liability    <- as.numeric(G[, causal_idx] %*% causal_betas + rnorm(n, sd = 1))
flip_mask    <- liability > quantile(liability, 0.75)
y[flip_mask] <- 1L - y[flip_mask]   # flip top-25% liability individuals

第三步：单 SNP 关联检验函数

gwas_pval() 对单个 SNP 拟合 logistic 回归，返回基因型系数的 Wald 检验双侧 p 值。当传入 covariates 参数（即主成分矩阵）时，模型变为 \(\text{logit}(p_i) = \beta_0 + \beta_g g_i + \boldsymbol{\gamma}^\top \mathbf{PC}_i\)，PCA 校正即通过这一协变量调整实现。glm.fit() 直接操作设计矩阵，避免公式解析开销，在逐 SNP 循环中效率更高。标准误通过费雪信息矩阵（\(\mathbf{X}^\top \mathbf{W} \mathbf{X}\)，\(\mathbf{W}\) 为 IRLS 权重对角矩阵）的对角元素求得。

gwas_pval <- function(g, phenotype, covariates = NULL) {
  if (var(g) == 0) return(1.0)   # skip monomorphic SNPs
  X   <- if (is.null(covariates)) cbind(1, g) else cbind(1, g, covariates)
  fit <- glm.fit(X, phenotype, family = binomial())
  # SE from observed Fisher information: diag( (X'WX)^{-1} )[2]
  coef_g <- fit$coefficients[2]
  se_g   <- sqrt(diag(solve(crossprod(X * sqrt(fit$weights))))[2])
  2 * pnorm(abs(coef_g / se_g), lower.tail = FALSE)
}

第四步：提取主成分用于分层校正

对基因型矩阵进行列标准化（零均值、单位方差），再进行奇异值分解（SVD），取左奇异向量的前两列作为个体的主成分得分。这与 EIGENSTRAT 的核心步骤等价（参见专题 3.4 PCA 校正原理小节）。svd(..., nu = 2, nv = 0) 仅计算前两个左奇异向量，避免对 \(5000 \times 5000\) 右奇异矩阵的无效计算。

18.5 多重检验校正

18.5.1 多重比较问题

GWAS 对数百万个 SNP 同时进行独立检验。若以 \(\alpha = 0.05\) 为单次检验显著性阈值，则仅凭随机机会，预期假阳性数量为：

\[ E[\text{false positives}] = 0.05 \times M \]

当 \(M = 10^6\) 时，期望假阳性数量高达 50,000 个，远超任何真实信号。解决多重比较问题有两条主线：其一是严格控制家族性 I 类错误率（Family-Wise Error Rate，FWER），确保整个实验中犯至少一次假阳性的概率不超过 \(\alpha\)；其二是控制错误发现率（False Discovery Rate，FDR），允许一定比例的假阳性，以换取更高的统计检验力。

18.5.2 Bonferroni 校正与 \(5 \times 10^{-8}\) 阈值的由来

Bonferroni 校正将全局显著性水平 \(\alpha\) 均摊至每次检验：

\[ \alpha_{\text{adjusted}} = \frac{\alpha}{M} \]

若以 \(\alpha = 0.05\) 且有效独立检验数 \(M_{\text{eff}} \approx 10^6\)（人类基因组约 100 万个独立 LD 区块），则：

\[ \alpha_{\text{Bonferroni}} = \frac{0.05}{10^6} = 5 \times 10^{-8} \]

这正是人类 GWAS 中广泛采用的显著性阈值的来源。该值并非来自严格的理论推导，而是基于约 100 万个独立 SNP 的经验估计，由 Risch 和 Merikangas 于 1996 年提出，随后被反复验证具有合理的假阳性控制效果。Bonferroni 校正的核心假设是所有检验相互独立。在 GWAS 中，相邻 SNP 之间由于 LD 而相关，这意味着 Bonferroni 校正理论上过于保守。更精确的有效独立 SNP 数 \(M_{\text{eff}}\) 可以通过 Galwey 方法或 Li-Ji 方法估计，也可通过置换检验直接从数据的 LD 结构生成经验阈值。

Bonferroni 阈值不可普适于所有物种

\(5 \times 10^{-8}\) 仅适用于人类 GWAS。对于其他物种，应根据实际的有效独立 SNP 数（可通过 LD 结构估算）重新计算阈值。例如，Arabidopsis thaliana 或 E. coli 的 LD 结构与人类差异悬殊，直接套用该阈值将导致过于保守或过于宽松的结论。

18.5.3 错误发现率（FDR）与 Benjamini-Hochberg 程序

错误发现率（False Discovery Rate，FDR）控制的是所有显著结果中假阳性的比例，而 Bonferroni 控制的是犯至少一次 I 类错误的概率。

Benjamini-Hochberg（BH）程序：设 \(p_{(1)} \leq p_{(2)} \leq \cdots \leq p_{(M)}\) 为 \(M\) 个检验的排序 p 值，令 FDR 阈值为 \(q\)，则：

\[ \text{拒绝所有满足 } p_{(i)} \leq \frac{i}{M} \cdot q \text{ 的零假设} \]

BH 程序的一个关键假设是检验之间独立或正相关，对于 GWAS 中由 LD 导致的正相关结构，BH 程序是有效的。在大规模探索性研究（如 eQTL 分析、差异表达分析）中，FDR 控制比 Bonferroni 更为常用。

以下代码以一个小规模示例（20 次检验）展示 BH 程序的逐步执行逻辑，并与 R 内置的 p.adjust() 函数结果进行对照验证：

# ── BH procedure: step-by-step illustration ──────────────────────────────────
set.seed(42)
M <- 20          # total number of tests
q <- 0.05        # target FDR level

# Simulate p-values: 15 null (uniform) + 5 true signals (small p)
p_vals <- c(runif(15), runif(5, min = 0, max = 0.01))

# Step 1: rank p-values in ascending order
rank_order <- order(p_vals)
p_sorted   <- p_vals[rank_order]

# Step 2: compute BH critical values  i/M * q  for each rank i
bh_critical <- (seq_len(M) / M) * q

# Step 3: find the largest rank k such that p_(k) <= k/M * q
k_max <- max(c(0, which(p_sorted <= bh_critical)))

# Step 4: reject all hypotheses with rank <= k_max
rejected_original <- rank_order[seq_len(k_max)]

cat(sprintf("Largest qualifying rank k: %d\n", k_max))
cat(sprintf("Hypotheses rejected (original indices): %s\n",
            paste(sort(rejected_original), collapse = ", ")))

# Verify against R built-in p.adjust
q_vals    <- p.adjust(p_vals, method = "BH")
n_bh_base <- sum(q_vals < q)
cat(sprintf("p.adjust(BH) significant count: %d\n", n_bh_base))
cat(sprintf("Manual BH significant count:    %d\n", length(rejected_original)))

# Visualise the BH procedure
library(ggplot2)
df_bh <- data.frame(
  rank     = seq_len(M),
  p_sorted = p_sorted,
  bh_crit  = bh_critical,
  rejected = seq_len(M) <= k_max
)

ggplot(df_bh, aes(x = rank)) +
  geom_line(aes(y = bh_critical), color = "steelblue",
            linetype = "dashed", linewidth = 0.8) +
  geom_point(aes(y = p_sorted, color = rejected), size = 2.5) +
  scale_color_manual(values = c("FALSE" = "grey50", "TRUE" = "firebrick"),
                     labels = c("Not rejected", "Rejected (BH)")) +
  annotate("text", x = M * 0.6, y = max(bh_critical) * 0.6,
           label = "BH critical line: i/M × q", color = "steelblue", size = 3.5) +
  labs(x = "Rank (i)", y = "p-value",
       title = "Benjamini-Hochberg Procedure", color = NULL) +
  theme_classic() +
  theme(legend.position = "top")

上图中，蓝色虚线为 BH 临界线 \(i/M \times q\)；红色点为被拒绝的零假设，灰色点为未拒绝。BH 程序在第 \(k\) 个排序位置”截断”，拒绝所有排名在 \(k\) 及以前的假设。这一”一刀切”特性确保了 q 值的单调性，也是 p.adjust() 内部实现中从右向左取累积最小值（cummin）的原因。

p.adjust() 与手动 BH 的等价性

R 的 p.adjust(p, method = "BH") 返回的是经过单调调整的 q 值，以 q_i < 0.05 筛选等价于手动 BH 程序以 FDR \(= 0.05\) 执行拒绝。两者在数值上完全一致，实际分析中直接使用 p.adjust() 即可；手动推导的目的在于理解其统计逻辑。

18.5.4 置换检验

置换检验（Permutation test） 通过随机打乱表型标签来生成经验零分布，是估计全基因组显著性阈值的最直接方法。其完整步骤如下：

  1. 记录原始统计量：对真实数据运行完整的 GWAS，记录所有 \(M\) 个 SNP 的关联统计量（如卡方值或 Wald 统计量），并以最大统计量 \(T_{\text{obs}} = \max_j T_j\) 作为初始参考。
  2. 表型置换：随机打乱 \(n\) 个样本的表型标签（保持基因型矩阵不变），在置换数据上重新运行完整的 GWAS，记录该次置换的最大统计量 \(T_b^*\)。
  3. 重复：重复步骤 2 共 \(B\) 次（通常 \(B \geq 1,000\)；若需要高精度的极端尾部分位数，可增至 \(B = 10,000\)），得到置换最大统计量的经验分布 \(\{T_1^*, T_2^*, \ldots, T_B^*\}\)。
  4. 确定阈值：以 \(B\) 次置换中最大统计量的 \(1 - \alpha\) 分位数（\(\alpha = 0.05\)）作为显著性阈值，即满足该阈值的位点在全基因组 FWER \(\leq \alpha\) 的意义下显著。

置换检验的核心优势在于：它不假设 SNP 检验相互独立，而是直接从数据的 LD 结构生成经验零分布，因此既不像 Bonferroni 那样过度保守（Bonferroni 假设完全独立，会高估有效检验数），也不会遗漏 LD 结构带来的依赖关系。对于 LD 结构异常（如存在大量高 LD SNP）或芯片设计非均匀的数据集，置换检验生成的阈值比通用的 \(5 \times 10^{-8}\) 更适合该数据。

然而，置换检验在实际应用中面临两大局限：其一，计算代价极高——对于 \(n = 5,000\) 个样本、\(M = 500,000\) 个 SNP 的 GWAS，每次完整置换的计算量与原始 GWAS 相当，\(B = 1,000\) 次置换意味着 1,000 倍的计算开销，通常需要分布式计算集群；其二，群体分层下的置换失效问题——简单随机置换打乱了样本-表型配对关系，但若数据中存在群体分层，打乱后的数据因群体结构与表型失去对应关系，导致零分布被”人为净化”，实际上低估了未校正检验统计量的真实零分布（等于用比较干净的数据估计比较脏的真实数据的阈值）。解决方法是残差置换（residual permutation）：先对表型对群体主成分（PCA）进行回归，对残差进行置换，再还原为置换后的表型，从而保留群体结构信息。

以下 R 代码给出一个小规模演示（仅用于教学理解，非真实 GWAS 规模）：

# Small-scale permutation test demonstration
set.seed(42)
n_indiv <- 200   # samples
n_snp   <- 1000  # SNPs (GWAS-scale would be millions)
B       <- 500   # permutation replicates

# Simulate genotypes and phenotype (no true associations)
geno  <- matrix(rbinom(n_indiv * n_snp, 2, 0.3), nrow = n_indiv)
pheno <- rnorm(n_indiv)

# Original GWAS: chi-square test for each SNP (simplified)
chisq_obs <- apply(geno, 2, function(g) {
  if (var(g) < 1e-8) return(0)
  cor(g, pheno)^2 * (n_indiv - 2) / (1 - cor(g, pheno)^2)
})
T_obs <- max(chisq_obs)

# Permutation loop
T_perm <- numeric(B)
for (b in seq_len(B)) {
  pheno_perm <- sample(pheno)
  chisq_b    <- apply(geno, 2, function(g) {
    if (var(g) < 1e-8) return(0)
    cor(g, pheno_perm)^2 * (n_indiv - 2) / (1 - cor(g, pheno_perm)^2)
  })
  T_perm[b] <- max(chisq_b)
}

# Empirical threshold at alpha = 0.05
threshold_perm <- quantile(T_perm, 0.95)
# Equivalent p-value threshold
p_threshold_perm <- pchisq(threshold_perm, df = 1, lower.tail = FALSE)

cat(sprintf("Permutation threshold (chi-sq): %.2f\n", threshold_perm))
cat(sprintf("Equivalent p-value: %.3e\n", p_threshold_perm))
cat(sprintf("Bonferroni p-value: %.3e\n", 0.05 / n_snp))
方法 控制目标 适用场景 检验力
Bonferroni 家族性 I 类错误率（FWER） 少量独立假设；后续验证实验代价极高 保守，检验力低
FDR (BH) 错误发现率 大规模探索性分析；允许一定比例假阳性 较高
置换检验 经验分布下的 FWER LD 结构复杂或非标准数据 计算代价高

18.6 统计检验力与样本量估计

统计检验力（statistical power）是在真实效应存在时，正确拒绝零假设的概率。GWAS 中检验力受以下因素共同决定：样本量 \(n\)、效应量 \(\beta\)（或 OR）、次等位基因频率（MAF）以及显著性阈值 \(\alpha\)。

在加性线性模型下，非中心参数（Non-Centrality Parameter，NCP）近似为：

\[ \text{NCP} = \frac{n \cdot \beta^2 \cdot 2p(1-p)}{\sigma^2} \]

其中 \(p\) 为 MAF，\(\sigma^2\) 为表型残差方差，对应 \(\chi^2\) 分布的非中心参数。

18.6.1 二元表型（病例-对照）的检验力近似

上述 NCP 公式适用于连续表型。对于病例-对照设计，表型为 0/1 二值变量，logistic 回归的 Wald 检验在大样本下仍渐近服从 \(\chi^2_1\)，其非中心参数可由以下公式近似：

\[ \text{NCP}_\text{cc} \approx \frac{n \cdot \beta^2 \cdot 2p(1-p) \cdot K(1-K)}{\pi(1-\pi)} \]

其中 \(n = n_\text{case} + n_\text{control}\) 为总样本量，\(K\) 为人群患病率，\(\pi = n_\text{case} / n\) 为样本中病例比例，\(\beta\) 为加性对数OR。当样本比例 \(\pi\) 偏离患病率 \(K\) 时（过采样病例），检验力显著依赖于二者的比值：以 \(K = 0.01\)、\(\pi = 0.5\)（1:1 设计）为例，\(K(1-K)/[\pi(1-\pi)] \approx 0.04\)，因此病例-对照 GWAS 在相同总样本量下的检验力远低于对应的连续表型研究，需要更大样本量弥补这一损失。

在实践中，当 \(K\) 未知或变动时，更稳健的方式是将统计检验力换算至责任尺度（liability scale） 后再比较，可使用 GPC（Genetic Power Calculator）或 pwr.2p.test 等工具直接输入样本量、OR 和 MAF 求解病例-对照检验力。

18.6.2 检验力对 MAF 的依赖

上式揭示了一个重要结论：低频变异在相同样本量下的检验力显著低于常见变异。这是因为 \(2p(1-p)\) 在 \(p = 0.5\) 时最大，当 \(p = 0.05\) 时该项值约为 0.095，而 \(p = 0.3\) 时为 0.42，相差约 4.4 倍。对于极低频变异（MAF < 0.01），常规 SNP 芯片往往不予收录，且标准 GWAS 框架几乎没有检验力，这是推动外显子测序和全基因组测序以及专为低频变异设计的聚合检验（burden test）和 SKAT 检验发展的核心动力。

18.6.3 GWAS 的规模效应

GWAS 历史上最引人注目的趋势之一是对样本量的持续追求。以 BMI 为例，2007 年第一个 GWAS 在约 17,000 个样本中发现了 FTO 基因区域；到 2015 年，样本量扩大至 34 万，发现了 97 个独立位点；到 2022 年，UK Biobank 和全球 GWAS 联合已将样本量推向百万级，已知独立 BMI 相关位点超过 900 个。这一趋势深刻说明：GWAS 的统计检验力在样本量接近线性增长时，能够以超线性的速度发现新位点，因为越来越多具有微小效应量的位点越过了显著性阈值。

以下代码基于加性线性模型的 NCP 公式，计算不同样本量与效应量组合下的统计检验力，并绘制检验力曲线（MAF 固定为 0.2，\(\alpha = 5 \times 10^{-8}\)）。虚线标注 80% 的常用检验力目标，可直观读取每条曲线达到该目标所需的最低样本量。

# GWAS power calculation under additive linear model
gwas_power <- function(n, beta, maf, sigma2 = 1.0, alpha = 5e-8) {
  ncp          <- n * beta^2 * 2 * maf * (1 - maf) / sigma2
  critical_val <- qchisq(1 - alpha, df = 1)
  pchisq(critical_val, df = 1, ncp = ncp, lower.tail = FALSE)
}

  }

18.6.4 元分析对统计检验力的倍增效应

当单个研究的样本量不足以检出小效应量位点时，多个 GWAS 研究的结果可以通过元分析（meta-analysis） 合并，以获得更大的有效样本量和更高的统计检验力。在进入数学推导之前，先建立基本直觉：若两个研究都给出对同一真实效应量 \(\beta\) 的无偏估计 \(\hat\beta_1, \hat\beta_2\)，应如何合并？最朴素的做法是取算术平均，但这隐含假设两个研究的精度相同；当样本量差异较大时，精度更高的估计反而会被精度低的估计”稀释”。统计学上的最优策略是按方差的倒数加权——精度越高的研究权重越大。可以证明，当各估计独立且无偏时，这种加权使合并估计的方差达到理论最小值，这正是逆方差加权法的核心思想。

将这一直觉推广到检验力分析，则有：在固定效应加性线性模型下，元分析的非中心参数（NCP）等于各研究 NCP 之和：

\[\text{NCP}_\text{meta} = \sum_{k=1}^{K} \text{NCP}_k = \frac{\left(\sum_{k=1}^{K} n_k\right) \cdot \beta^2 \cdot 2p(1-p)}{\sigma^2}\]

这意味着元分析的理论检验力等价于单个样本量为 \(\sum_k n_k\) 的研究——只要各研究在表型定义、基因型数据质量和协变量校正上保持一致。正因如此，当代大型 GWAS 合作联盟（如用于研究精神疾病的 PGC、用于研究身高的 GIANT、用于研究血脂的 GLGC）往往通过汇总数十个独立队列的汇总统计量实现几十万乃至百万级别的有效样本量，而无需共享原始个体水平数据。

然而，元分析的检验力增益并非总能达到理论上限。若各研究的效应量存在异质性（heterogeneity），则应使用随机效应元分析（random-effects meta-analysis）。在随机效应模型下，研究间方差 \(\tau^2\)（DerSimonian-Laird 估计）使合并权重向更保守方向收缩，导致实际检验力低于固定效应模型的理论值。\(I^2\) 统计量（异质性指数，取值 0–100%）量化了效应量变异中可归因于研究间真实差异（而非抽样误差）的比例：\(I^2 < 25\%\) 为低异质性，\(25\%–75\%\) 为中度异质性，\(> 75\%\) 为高异质性。高异质性下的元分析结果需结合亚组分析和敏感性分析审慎解读，不宜仅看合并 p 值。

GWAS 元分析的标准软件为 METAL（密歇根大学开发）和 PLINK 的 --meta-analysis 功能。METAL 支持两种合并模式：

  • Z 分数加权法（Z-score based）：以样本量的平方根为权重合并每个研究的方向性 Z 分数，无需各研究报告统一的效应等位基因，对效应等位基因编码方式不敏感；
  • 逆方差加权法（Inverse Variance Weighted，IVW）：以各研究效应量标准误的倒数为权重合并效应量估计，要求各研究报告相同参考等位基因的效应方向，统计效率更高，是多中心 GWAS 的首选方法。

什么时候用元分析而不是合并原始数据？

原始数据合并（mega-analysis）在理论上统计效率更高，因为它可以在统一模型下估计协变量效应，避免研究间模型不一致导致的偏差。然而，mega-analysis 面临数据共享协议、受试者隐私保护和数据存储成本的现实限制。元分析使用汇总统计量（每个 SNP 仅需效应量和标准误两列数字），既能大幅扩展有效样本量，又规避了原始数据共享障碍，是现实中多中心 GWAS 合作的标准策略。GIANT 身高联盟和精神疾病基因组学联盟（PGC）均是元分析策略取得重大成功的典型案例。

以下 R 代码模拟两个独立 GWAS 研究的汇总统计量，并演示 IVW 元分析的手动实现逻辑。代码以”先设定真值、再模拟观测、最后合并”的顺序组织，便于教学过程中分步演示：

# Simulated two-study GWAS meta-analysis (IVW method)
# Goal: combine two independent studies' summary statistics to recover
#       statistical power equivalent to a single n1 + n2 study.
set.seed(2024)

# --- Step 1: define the ground truth we want to detect ---
# In a real GWAS the true effect is unknown; here we fix it so that
# we can later check whether meta-analysis recovers it correctly.
true_beta <- 0.08      # additive effect per risk allele (SD units)
maf       <- 0.25      # minor allele frequency of the test SNP

# --- Step 2: compute each study's expected standard error ---
# Under an additive linear model with standardized phenotype (sigma^2 = 1),
# Var(beta_hat) ~ 1 / (n * 2p(1-p)), so SE = 1 / sqrt(n * 2 * MAF * (1-MAF)).
# Larger n  -> smaller SE -> higher precision -> larger meta-weight.
n1 <- 5000; n2 <- 8000
se1 <- 1 / sqrt(n1 * 2 * maf * (1 - maf))   # smaller study, larger SE
se2 <- 1 / sqrt(n2 * 2 * maf * (1 - maf))   # larger  study, smaller SE

# --- Step 3: simulate one observed beta per study ---
# Each cohort's reported beta_hat equals the truth plus a Gaussian
# sampling error drawn from N(0, SE_k). This mimics what each study
# would write into its summary-statistics file.
beta1 <- true_beta + rnorm(1, 0, se1)
beta2 <- true_beta + rnorm(1, 0, se2)

# --- Step 4: IVW meta-analysis ---
# Weight each study by 1 / SE^2 so that more precise studies dominate.
# Under the fixed-effect assumption (one shared true beta, no between-
# study heterogeneity), this weighting yields the minimum-variance
# unbiased combined estimator.
w1 <- 1 / se1^2
w2 <- 1 / se2^2
beta_meta <- (w1 * beta1 + w2 * beta2) / (w1 + w2)   # weighted average
se_meta   <- sqrt(1 / (w1 + w2))                     # pooled SE
z_meta    <- beta_meta / se_meta                     # Wald Z statistic
p_meta    <- 2 * pnorm(abs(z_meta), lower.tail = FALSE)   # two-sided p

# --- Step 5: report per-study and pooled results ---
cat(sprintf("Study 1: beta = %.4f, SE = %.4f, Z = %.2f\n",
            beta1, se1, beta1 / se1))
cat(sprintf("Study 2: beta = %.4f, SE = %.4f, Z = %.2f\n",
            beta2, se2, beta2 / se2))
cat(sprintf("Meta-analysis (IVW): beta = %.4f, SE = %.4f, Z = %.2f, p = %.2e\n",
            beta_meta, se_meta, z_meta, p_meta))

# Effective sample size implied by the meta-analytic SE.
# Inverting SE^2 = 1 / (N_eff * 2p(1-p)) gives N_eff = 1 / (SE^2 * 2p(1-p)).
# A successful fixed-effect meta-analysis should produce N_eff ~ n1 + n2.
cat(sprintf("Effective N (meta): %.0f vs actual N1+N2: %d\n",
            1 / se_meta^2 / (2 * maf * (1 - maf)), n1 + n2))

运行这段代码可观察到三个关键现象，对应元分析的三条核心结论：(1) 合并估计 \(\hat\beta_\text{meta}\) 比任一单独研究的估计更接近真值 \(0.08\)，因为大数定律使加权平均的随机误差缩小；(2) 合并 Z 统计量显著高于单研究 Z 值，对应统计检验力的提升；(3) 由 \(\text{SE}_\text{meta}\) 反推的有效样本量近似等于 \(n_1 + n_2 = 13{,}000\)，验证了”固定效应元分析等价于单个 13,000 样本研究”的理论结论。课堂演示时，可让学生改变 n1、n2、true_beta 或 maf 的取值，体会样本量、效应量与等位基因频率如何共同决定合并 p 值与有效样本量的关系。

18.7.2 SNP 遗传力估计（GCTA-GREML）

单 SNP 关联分析识别的是显著性达到全基因组阈值的位点，但这些位点通常只能解释性状总遗传方差的一小部分。遗传力（heritability，\(h^2\)） 表示在总表型方差中可归因于遗传变异的比例，是理解遗传结构的基础指标。SNP 遗传力（SNP heritability，\(h^2_{\text{SNP}}\)） 则进一步定义为可被芯片上所有常见 SNP 共同解释的表型方差比例，通常低于总遗传力（即存在”遗传力缺失”，missing heritability）。

GCTA-GREML（Genomic-relatedness-based restricted maximum likelihood）是估计 SNP 遗传力的金标准方法。其核心思想是：利用全基因组 SNP 构建遗传关系矩阵（Genomic Relationship Matrix，GRM），将表型方差分解为遗传成分和环境成分，通过限制性最大似然（REML） 估计各方差成分。GRM 的第 \((i,j)\) 元素定义为两个个体在标准化基因型上的内积均值：

\[A_{ij} = \frac{1}{M} \sum_{k=1}^{M} \frac{(x_{ik} - 2p_k)(x_{jk} - 2p_k)}{2p_k(1-p_k)}\]

其中 \(x_{ik}\) 为个体 \(i\) 在 SNP \(k\) 的等位基因计数（0/1/2），\(p_k\) 为 SNP \(k\) 的等位基因频率，\(M\) 为总 SNP 数。GRM 的对角元素 \(A_{ii}\) 反映个体自身的”近交系数估计”（无近交时约为 1），非对角元素 \(A_{ij}\) 度量两个个体的遗传相似性（无亲缘关系的随机个体期望值约为 0）。

在线性混合模型框架下，表型向量 \(\mathbf{y}\) 的方差结构为：

\[\text{Var}(\mathbf{y}) = h^2_{\text{SNP}} \cdot \mathbf{A}\sigma_y^2 + (1 - h^2_{\text{SNP}}) \cdot \mathbf{I}\sigma_y^2\]

REML 优化对数限制似然函数（marginalizing over fixed effects），估计 \(h^2_{\text{SNP}}\) 使上述方差结构与观测表型协方差矩阵最匹配。

# Step 1: Construct Genomic Relationship Matrix (GRM)
gcta64 \
  --bfile qc_step4 \
  --make-grm \
  --out grm_all_snps \
  --thread-num 8

# Step 2: Filter out closely related individuals (GRM off-diagonal > 0.05)
gcta64 \
  --grm grm_all_snps \
  --grm-cutoff 0.05 \
  --make-grm \
  --out grm_unrelated \
  --thread-num 8

# Step 3: REML estimation of SNP heritability
gcta64 \
  --grm grm_unrelated \
  --pheno pheno.txt \
  --qcovar continuous_covariates.txt \
  --covar discrete_covariates.txt \
  --reml \
  --out heritability_result \
  --thread-num 8

GCTA 的 .hsq 输出文件包含遗传方差（\(V(G)\)）和残差方差（\(V(e)\)）的 REML 估计及标准误，以及 \(h^2_{\text{SNP}} = V(G) / V(P)\) 的点估计和置信区间。解读时需注意：（1）GREML 对样本量敏感，通常需要 \(N > 3,000\) 才能获得精确估计；（2）祖源不均匀的混合样本可能导致 GRM 非正定或 REML 收敛问题；（3）二元性状的 \(h^2_{\text{SNP}}\) 在观测尺度（observed scale）上受患病率影响，需换算至责任尺度（liability scale）以获得可比较的遗传力估计。

LDSC：无需个体数据的遗传力估计

当没有原始个体水平数据时，LD Score 回归（LDSC） 可直接利用 GWAS 汇总统计量估计 SNP 遗传力。其原理是在 \(\chi^2\) 统计量与该 SNP 的 LD Score（即与周围 SNP 的 \(r^2\) 之和）之间建立回归关系：\(E[\chi_j^2] = N h^2_{\text{SNP}} l_j / M + \text{intercept}\)，斜率项反映多基因遗传信号，截距项反映分层偏差。LDSC 的计算仅需数分钟，输出的截距同时用于诊断 \(\lambda_{GC}\) 的来源（参见 3.5.3 节）。

18.8 独立信号的识别：LD 裁剪与条件分析

18.8.1 LD 裁剪（Clumping）

由于 LD 的存在，显著性峰往往包含多个互相连锁的 SNP，而非单一的独立信号。LD 裁剪以最显著的 SNP（index SNP）为核心，将其一定窗口内 \(r^2\) 超过阈值的关联 SNP 合并为同一位点，输出每个独立信号的代表 SNP。

LD 裁剪（clumping）与 LD 修剪（pruning）的区别

LD 修剪（--indep-pairwise，参见专题 3.2 QC 章节）以滑动窗口逐对移除高 LD 的 SNP，结果集中保留低 LD 标记，用于 PCA、亲缘关系矩阵等需要弱相关标记的计算。LD 裁剪则以 p 值为权重，优先保留关联证据最强的 SNP，专门用于识别独立关联信号，两者目的和参数体系完全不同，不可混用。

# PLINK LD clumping
plink \
  --bfile qc_step4 \
  --clump gwas_logistic.assoc.logistic \
  --clump-p1 5e-8 \
  --clump-p2 1e-5 \
  --clump-r2 0.1 \
  --clump-kb 1000 \
  --clump-field P \
  --out gwas_clumped

各参数含义如下表所示：

参数 作用 典型值 说明
--clump-p1 index SNP 的 p 值阈值 5e-8 仅以达到该阈值的 SNP 作为潜在 index SNP
--clump-p2 候选成员 SNP 的 p 值上限 1e-5 仅考虑 p < p2 的 SNP 作为 index SNP 的 LD 伴随体
--clump-r2 LD 阈值 0.1 \(r^2 >\) 该值的 SNP 与 index SNP 归为同一簇
--clump-kb 物理窗口半径（kb） 1000 仅在 index SNP 上下游 1 Mb 内搜索候选成员

输出文件 gwas_clumped.clumped 每行对应一个独立信号，核心列为：INDEX_SNP（代表 SNP）、TOTAL（该簇内 SNP 总数）、NSIG（其中 p < p1 的 SNP 数）以及 SP2（成员 SNP 列表）。独立信号数量即为输出行数，可直接统计：

# Count independent loci after clumping
clumped <- read_table("gwas_clumped.clumped", col_types = cols())
cat(sprintf("Independent significant loci: %d\n", nrow(clumped)))

# Retrieve index SNPs for downstream analysis
index_snps <- clumped$SNP

18.8.2 逐步条件分析

当局部关联图（见下节）或生物学先验提示同一区域可能存在多个独立信号时，应进行逐步条件分析（stepwise conditional analysis）：将已识别的 index SNP 基因型作为额外协变量纳入模型，检验该区域内是否仍有残余关联。

PLINK 通过 --condition 或 --condition-list 直接支持条件关联分析：

# Conditional analysis: condition on index SNP rs1234567
plink \
  --bfile qc_step4 \
  --logistic \
  --covar covariates.txt \
  --condition rs1234567 \
  --hide-covar \
  --out gwas_conditional_step1

# If a second independent signal is found (e.g., rs7654321),
# condition on both index SNPs simultaneously
plink \
  --bfile qc_step4 \
  --logistic \
  --covar covariates.txt \
  --condition-list two_index_snps.txt \
  --hide-covar \
  --out gwas_conditional_step2

判断是否存在第二个独立信号的准则：在条件模型中，若某 SNP 仍满足 \(p < 5 \times 10^{-8}\) 且与第一个 index SNP 的 \(r^2 < 0.1\)，则视为独立的第二信号，将其纳入下一轮条件模型，直至区域内无新显著位点。多数区域在 2–3 轮后收敛；若发现 5 个以上独立信号，则需核实该区域是否存在复杂的结构变异（如拷贝数变异或倒位多态性）。

条件分析的替代方案：近似贝叶斯因子法

当样本量足够大（\(N > 10{,}000\)）时，基于 Bayes Factor 的方法（如 FINEMAP 的多因果变异模式）能在单次分析中同时推断区域内因果变异的数量和位置，无需逐步迭代。具体原理与实现参见专题 3.6 精细定位章节。

后续分析深入：精细定位与功能解析

本节介绍了如何通过 LD 裁剪和逐步条件分析识别独立关联信号。对于如何在候选区域内进一步缩小因果变异范围（精细定位，包括 SuSiE 与 FINEMAP）、如何整合 eQTL 与表观基因组数据解析致因机制，以及如何开展孟德尔随机化因果推断和多群体比较分析，参见专题 3.6。

18.9.1 局部关联图的深度解读

局部关联图蕴含的信息远超单纯的 p 值展示，有经验的研究者会从中读取以下几类信息：

信号轮廓（signal shape） 反映 LD 结构的质量。理想的真实信号表现为以 lead SNP（紫色菱形）为核心、颜色由红（高 \(r^2\)）向蓝（低 \(r^2\)）平滑过渡的单峰结构，峰宽与当地 LD 区块范围一致。若出现双峰（两个颜色不相关的 \(r^2\) 模式），则提示该区域可能存在两个独立信号，需要进行条件分析（参见 3.5.7 节）。若背景 SNP 的 \(r^2\) 颜色呈现棋盘格花纹，则可能是局部 LD 块状结构（block structure）的体现。

基因结构叠加是确认候选效应基因的第一步。显著信号的物理位置相对于周围基因的分布（exonic、intronic、intergenic）直接影响功能注释的优先策略：落在外显子上的编码变异可直接预测氨基酸变化；落在非编码区的变异需要结合表观基因组数据（H3K27ac、ATAC-seq）判断调控功能；完全位于基因间区的信号则需查询远端增强子或非编码 RNA 数据库。

重组率（recombination rate） 曲线（通常叠加在图底部，单位为 cM/Mb）标志着 LD 区块的边界。高重组率位点（峰值）通常对应 LD 断点（recombination hotspot），用于判断候选区域的精确边界——若 lead SNP 恰好位于 LD 断点附近，则信号边界清晰，精细定位候选集较小；若位于长 LD 区块中央，则候选集更大。

局部关联图与精细定位图的互补关系

局部关联图以统计证据（\(-\log_{10} p\)）和 LD 结构为核心，适合初步评估信号形态和候选基因范围，是 GWAS 发表的标准配图。精细定位图（如 SuSiE 输出的 PIP 图）则将 y 轴换为每个 SNP 的后验纳入概率，直接展示因果变异候选的贝叶斯置信度，是精细定位的输出图形。两者分别对应不同分析阶段：先通过局部关联图确认信号形态，再通过精细定位量化因果证据（参见专题 3.6）。

参考答案 (Reference Answers)

  1. 同一 LD 区块内的 SNP 彼此高度相关（高 \(r^2\)），因此若某一致因变异（或其紧密连锁的标记物）与表型真实关联，该区块内所有与其连锁的 SNP 也会显示出类似的关联信号，形成”信号柱”。LD 裁剪以 p 值最小的 SNP（index SNP）为核心，将其特定窗口内 \(r^2\) 超过阈值（如 0.1）的所有 SNP 归并为同一位点，仅保留 index SNP 代表该独立信号，从而在全基因组范围内统计独立命中数量。

  2. 仅尾部上翘说明统计模型整体校准良好（零假设下的位点符合均匀分布），少数位点产生了真实的关联信号，这是理想的 GWAS 结果模式。全体点整体偏离对角线向上（inflation）则提示存在系统性偏差，最常见原因是群体分层未充分控制、隐性亲缘关系（cryptic relatedness）或基因型质量问题，应计算 \(\lambda_{GC}\) 并采用 MLM 或增加 PCA 主成分进行校正。

  3. \(\lambda_{GC} = 1.8\) 表明 GWAS \(\chi^2\) 统计量的中位数是理论期望的 1.8 倍，如按传统解读则提示严重的系统性偏差。然而，LD Score 回归截距为 1.02（接近 1），说明全局膨胀几乎全部来源于真实的多基因性（polygenic signal），而非统计假象：在 50 万样本的超大型队列中，成千上万个效应极小的真实位点都略微超过零假设，使中位 \(\chi^2\) 整体升高，截距接近 1 则意味着无分层偏差。此时不应对统计量做基因组控制（GC）校正，因为那样会损失真实信号。正确做法是保留原始 p 值，用 LDSC 截距向读者说明膨胀来源于多基因性。

  4. 全局显著性水平 \(\alpha = 0.05\)，有效独立检验数 \(M_{\text{eff}} \approx 10^6\)，则 Bonferroni 校正后每次检验的阈值为 \(\alpha / M_{\text{eff}} = 0.05 / 10^6 = 5 \times 10^{-8}\)。该值不适用于其他物种：例如，拟南芥（Arabidopsis thaliana）基因组较小且 LD 衰减较快，有效独立 SNP 数与人类差异显著，应根据该物种实际 LD 结构重新估算有效检验数并计算相应阈值。

  5. Bonferroni 校正控制的是家族性 I 类错误率（FWER），即在所有检验中犯至少一次假阳性的概率不超过 \(\alpha\)。FDR（BH 程序）控制的是错误发现率，即所有显著结果中假阳性所占的期望比例不超过 \(q\)。Bonferroni 更适合检验数量少、每次假阳性代价极高的场景（如临床诊断基因检测）；FDR 更适合大规模探索性分析（如转录组差异表达分析），在允许一定比例假阳性的前提下保留更多真实信号。

  6. 利用 NCP 公式 \(\text{NCP} = n \cdot \beta^2 \cdot 2p(1-p) / \sigma^2\)，代入 \(\beta = 0.12\)、\(p = 0.15\)、\(\sigma^2 = 1\)，需要迭代求解使得检验力达到 80% 的 \(n\)。R 代码验证：调用 min_n_for_power(beta = 0.12, maf = 0.15) 函数（见 3.5.5 节），预期结果约为 60,000–90,000 样本（具体取决于迭代精度）。

  7. 群体分层混杂的统计机制：若两个亚群在某 SNP 的等位基因频率上存在差异，同时在表型均值上也存在差异，则该 SNP 的等位基因与表型在混合样本中表现出统计相关，即使在各亚群内部二者完全不相关。这是因为群体标签既是 SNP 频率的预测因子，也是表型的预测因子，形成了混杂路径。PCA 校正通过将基因组主成分作为固定效应协变量纳入回归模型，将群体标签引起的表型差异”吸收”为协变量效应，从而使 SNP 系数的估计只反映亚群内部的关联，消除跨群体的混杂。

  8. LD 修剪（pruning）以滑动窗口逐对去除超过 \(r^2\) 阈值的 SNP，保留一组低 LD 的近似独立标记集，不考虑关联 p 值；用于 QC 阶段的 PCA 计算和亲缘关系矩阵构建。LD 裁剪（clumping）则以 p 值为权重，优先保留关联证据最强的 SNP 作为 index SNP，并将其窗口内高 LD 的伴随 SNP 归并至同一簇；用于关联分析结束后的独立信号计数。两者目的不同，参数体系也不同（pruning 使用 --indep-pairwise，clumping 使用 --clump），不可替换使用。

  9. 连续表型 NCP = \(10000 \times 0.15^2 \times 2 \times 0.2 \times 0.8 / 1 = 72\)。病例-对照 NCP = \(10000 \times 0.15^2 \times 2 \times 0.2 \times 0.8 \times 0.05 \times 0.95 / (0.5 \times 0.5) \approx 10.26\)。连续表型 NCP 约为病例-对照的 7 倍，前者检验力远高于后者。其根本原因是在 \(K = 0.05\) 的稀有疾病中，\(K(1-K) = 0.0475\) 远小于 \(\pi(1-\pi) = 0.25\)（1:1 设计），二值表型的信息量损失严重。这是为何大型生物银行（UK Biobank）对慢性病以连续表型代理指标（如血压、BMI）替代二值病例/对照标签进行 GWAS 时往往能发现更多位点的直接原因。
