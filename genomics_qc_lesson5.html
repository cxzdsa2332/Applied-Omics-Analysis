14 变异功能注释

14.1 从坐标到生物学意义

变异的可视化检测只是第一步，真正的价值在于理解其生物学后果。

14.1.1 基因组注释资源

{{Wiki:Gene_annotation}}

变异注释依赖于：

  1. 基因组注释文件 (GTF/GFF)
    • 包含所有已知基因、转录本、外显子、内含子的坐标
    • 常用数据库：Ensembl, RefSeq, GENCODE
  2. 蛋白质序列 (FASTA)
    • 用于判断同义/错义突变
  3. 保守性评分 (Conservation)
    • PhyloP, GERP：评估碱基的进化保守性
    • 保守位点突变更可能造成功能损伤
  4. 注释数据库
    • ClinVar：临床相关变异
    • gnomAD：全球人群频率
    • dbSNP：已知 SNP
    • COSMIC：肿瘤体细胞变异

14.1.2 变异的功能效应分类

效应类型 说明 重要性
同义突变 (Synonymous) 改变 DNA 不改变蛋白质（如 GCT → GCC，都编码 Ala） ✓ 通常低
错义突变 (Missense) 改变蛋白质序列（如 ATG (Met) → GTG (Val)） ✓✓✓ 高
无义突变 (Nonsense) 产生终止密码子，蛋白质截断（如 TGG (Trp) → TAG (Stop)） ✓✓✓ 非常高
移码 (Frameshift) Indel 非 3 倍数，导致阅读框改变 ✓✓✓ 非常高
剪接位点 (Splice Site) 影响剪接信号（GT-AG），改变转录本结构 ✓✓ 高
启动子/增强子 (Regulatory) 影响基因表达调控 ✓✓ 中等
内含子/2kb 外 (Intron/Intergenic) 不直接编码，功能未知 ✓ 低

14.1.3 注释工具对比

{{Wiki:Gene_prediction}}

工具 算法 特点 性能
SnpEff 基因组坐标比对 快速、全面的效应预测 推荐用于常规注释
VEP (Variant Effect Predictor) Ensembl API 功能全面、注释数据丰富 最全面但较慢
ANNOVAR 多库整合 轻量化、支持多物种 适合快速批量注释
CADD 机器学习 损伤性评分 预测变异致病性

14.2 GATK Best Practices 补充：BQSR 与群体层面 QC

本专题与专题 2.4 共享同一套 GATK 流程框架（FASTQ → BWA → MarkDuplicates → HaplotypeCaller → VariantFiltration，参见专题 2.4 各步骤代码），但有两个关键环节在上一专题中未能深入展开：碱基质量分数重校正（BQSR）和群体层面的变异质控（HWE 检验与 MAF 过滤）。这两步在人类 WGS 队列分析中是标准必做流程，理解其原理对后续 GWAS 章节至关重要。

{{Wiki:GATK}}

14.2.1 BQSR：碱基质量分数重校正

测序仪报告的碱基质量分数（Phred Q 值）往往存在系统性偏差：特定的读段位置、特定的碱基上下文（如 GCC 三联体）、或特定的碱基化学类型（如 G 碱基）会出现质量分持续高估或低估的现象。这种偏差不是随机误差，而是由测序化学反应机制决定的系统性偏差（systematic bias），会导致 HaplotypeCaller 在计算似然时使用的质量权重失真，进而影响变异检测的准确性。

{{Wiki:Base_quality_score_recalibration}}

BQSR 的核心思路是利用已知变异数据库作为”地面真相”：在已知变异位点处观测到的碱基差异归因于真实多态性，排除后剩余的差异才视为测序错误，据此统计每种上下文下的真实错误率，建立一张从”原始 Q 值 + 上下文特征”到”校正 Q 值”的映射表。

BQSR 考虑的协变量：

协变量 典型偏差模式
读段位置 (ReadPosition) read 末端错误率通常更高，Q 值被高估
碱基上下文 (Context) GGC 等特定三联体化学结构不稳定，错误率偏高
循环数 (Cycle) 测序循环越多，错误累积，但 Q 值不一定下调
读段方向 (Strand) 正反向读段化学特性不完全对称 
# 以 E. coli SRR2584863 数据为例（人类数据将 --known-sites 替换为 dbsnp.vcf）
# 注：细菌基因组 BQSR 通常不做（无已知变异库），此处以概念演示为主

# Step 1: 统计错误模式，生成重校正表
gatk BaseRecalibrator \
    -I ecoli_rg.bam \
    -R reference/ecoli_K12.fasta \
    --known-sites variants.vcf \       # 用已检测到的高置信变异作为已知位点
    -O recal_data.table

# Step 2: 可视化校正前后的质量分布（可选）
gatk AnalyzeCovariates \
    -bqsr recal_data.table \
    -plots bqsr_report.pdf

# Step 3: 应用校正，生成新 BAM
gatk ApplyBQSR \
    -I ecoli_rg.bam \
    -R reference/ecoli_K12.fasta \
    --bqsr-recal-file recal_data.table \
    -O ecoli_recalibrated.bam

细菌基因组的 BQSR 取舍

BQSR 在人类 WGS 中效果显著（可将 SNP 检测精度提升约 1–3%），但对细菌基因组收益有限：一方面，细菌无已知变异参考库，只能用本次检测结果”自举”，形成循环依赖；另一方面，细菌基因组变异率极低，BQSR 训练样本量不足。实践中，对模式生物细菌（如 E. coli）通常跳过 BQSR 步骤，直接使用原始质量分进行变异检测。

14.2.2 Hardy-Weinberg 平衡检验与 MAF 过滤

通过技术过滤（QUAL、DP、FS 等）后，还需对变异进行生物学层面的质量检验。这一步骤在群体遗传学分析和 GWAS 中尤为关键，但也适用于队列测序的常规质控。

Hardy-Weinberg 平衡（HWE）是群体遗传学的基本定律：在理想种群中，等位基因频率保持稳定，基因型频率满足：

\[P(\text{AA}) = p^2, \quad P(\text{Aa}) = 2pq, \quad P(\text{aa}) = q^2\]

{{Wiki:Hardy–Weinberg_principle}}

一个位点若显著偏离 HWE（通常以 \(p < 10^{-6}\) 为阈值），往往提示以下技术问题而非真实生物学信号：

HWE 偏离方向 最可能的原因
杂合子过少（heterozygote deficit） 基因分型错误、等位基因丢失（allele dropout）、近亲交配
杂合子过多（heterozygote excess） 样本混合、体细胞嵌合、比对错误

MAF（Minor Allele Frequency，次等位基因频率） 过滤则用于排除统计功效不足的极低频变异：

# 使用 PLINK 进行 HWE 检验和 MAF 过滤（常用于 GWAS 前处理）
plink \
  --vcf filtered.vcf.gz \
  --hwe 1e-6 \          # 移除 HWE p-value < 1e-6 的位点
  --maf 0.01 \          # 移除 MAF < 1% 的位点（极低频变异）
  --geno 0.05 \         # 移除缺失率 > 5% 的位点
  --mind 0.10 \         # 移除缺失率 > 10% 的个体
  --make-bed \
  --out filtered_qc

# 查看过滤统计
plink --bfile filtered_qc --freq --out allele_freq
# R: 手工计算 HWE p 值（卡方检验）
hwe_test <- function(n_hom_ref, n_het, n_hom_alt) {
  n_total <- n_hom_ref + n_het + n_hom_alt
  p <- (2 * n_hom_ref + n_het) / (2 * n_total)   # ref allele freq
  q <- 1 - p

  expected_hom_ref <- n_total * p^2
  expected_het     <- n_total * 2 * p * q
  expected_hom_alt <- n_total * q^2

  chi2 <- (n_hom_ref - expected_hom_ref)^2 / expected_hom_ref +
          (n_het     - expected_het)^2     / expected_het +
          (n_hom_alt - expected_hom_alt)^2 / expected_hom_alt

  p_value <- pchisq(chi2, df = 1, lower.tail = FALSE)

  cat(sprintf("Observed  : AA=%d  Aa=%d  aa=%d\n",
              n_hom_ref, n_het, n_hom_alt))
  cat(sprintf("Expected  : AA=%.1f  Aa=%.1f  aa=%.1f\n",
              expected_hom_ref, expected_het, expected_hom_alt))
  cat(sprintf("Chi2 = %.4f  p-value = %.2e  %s\n",
              chi2, p_value,
              ifelse(p_value < 1e-6, "FAIL HWE", "PASS HWE")))
}

# 示例：正常位点
cat("--- Normal locus ---\n")
hwe_test(n_hom_ref = 400, n_het = 450, n_hom_alt = 150)

# 示例：杂合子严重缺失（疑似基因分型错误）
cat("\n--- Suspected genotyping error ---\n")
hwe_test(n_hom_ref = 600, n_het = 50, n_hom_alt = 350)

MAF 阈值的选择

MAF < 1% 的变异通常称为低频变异（low-frequency variant），MAF < 0.1% 称为罕见变异（rare variant）。常规 GWAS 一般过滤 MAF < 1%–5%，保留常见变异（common variant）进行关联分析；若专门研究罕见变异对疾病的聚合效应，则需专用方法（如 SKAT、BURDEN test），不宜用简单 MAF 过滤。

14.3 变异注释与优先级评估

14.3.1 SnpEff 实战注释

SnpEff 内置了数千个物种的基因组数据库，可直接按物种标识符调用。对于本节使用的 E. coli K-12 MG1655 参考基因组，内置数据库标识符为 Escherichia_coli_k_12。

# 1. 查询 SnpEff 内置数据库（确认 E. coli 版本）
snpEff databases | grep -i "coli"

# 2. 对 E. coli variants.vcf 进行基础功能注释
snpEff \
    -stats ecoli_snpeff_report.html \
    -csvStats ecoli_snpeff_stats.csv \
    Escherichia_coli_k_12 \
    variants.vcf > annotated.vcf

# 3. 统计各效应类型的变异数量
grep -v "^#" annotated.vcf | \
    grep -oP "(?<=ANN=)[^;]+" | \
    cut -d'|' -f2 | sort | uniq -c | sort -rn

# 4. 提取高影响（HIGH impact）变异——优先关注无义突变和移码突变
grep -v "^#" annotated.vcf | \
    awk -F'\t' '{
        if ($8 ~ /IMPACT=HIGH/ || $8 ~ /HIGH/) print $0
    }' > high_impact_variants.vcf

echo "High impact variants: $(grep -v "^#" high_impact_variants.vcf | wc -l)"

对于人类 WGS 数据（如 GIAB HG002 基准集），SnpEff 结合 VEP 能提供更丰富的注释，包括 SIFT/PolyPhen 预测评分和 ClinVar 临床分类：

# VEP 注释（人类 hg38，需提前下载 cache）
vep \
    --input_file variants_hg38.vcf \
    --output_file annotated_hg38.vcf \
    --assembly GRCh38 \
    --cache --offline \
    --sift b \          # SIFT 预测：氨基酸替换对蛋白功能的影响
    --polyphen b \      # PolyPhen-2：错义突变致病性预测
    --regulatory \      # 注释调控区域（启动子、增强子）
    --variant_class \   # 标注变异类型（SNV/indel/structural）
    --fork 4            # 并行线程数

14.3.2 注释字段解读

注释后的 VCF 会添加 ANN 字段：

##ANN=<ID=ANN,Number=.,Type=String,Description="Annotation">

ANN=T|missense_variant|MODERATE|TP53|ENSG00000141510|transcript|ENST00000269305|
protein_altering|2/7|c.715C>T|p.Arg239Trp|818/2584|79%|...
字段 含义
T 替代等位体
missense_variant 效应类型（同义/错义/无义等）
MODERATE 影响程度（HIGH/MODERATE/LOW）
TP53 基因名称
ENSG… 基因 ID
c.715C>T 核酸变化 (cDNA)
p.Arg239Trp 蛋白质变化（单字母：RW，三字母：Arg→Trp）
79% 蛋白质位置百分比

14.3.3 致病性评估

ACMG 指南 将变异分为 5 级：

  1. 病原性 (Pathogenic) - 明确导致疾病
  2. 可能病原性 (Likely Pathogenic) - 很可能导致疾病
  3. 意义不明 (VUS, Variant of Uncertain Significance) - 无法判断
  4. 可能良性 (Likely Benign) - 很可能无害
  5. 良性 (Benign) - 无致病性

评估标准： - 群体频率（gnomAD, 1000 Genomes） - 在致病数据库中的注释（ClinVar） - 计算预测（CADD, SIFT, PolyPhen） - 功能研究证据 - 家族聚集性

14.4 IGV 交互式可视化

Integrative Genomics Viewer (IGV) 用于直观查看比对和变异。

14.4.1 IGV 核心功能

  1. 加载参考基因组和注释
    • 选择 hg38/hg19
    • 导入 GTF 基因注释
  2. 导入 BAM 文件
    • 自动显示 read 覆盖度
    • 颜色编码：
      • 绿色：正向比对
      • 红色：反向比对
      • 灰色：低 MAPQ（质量差）
      • 彩色错配：红(A) 蓝(C) 绿(G) 黄(T)，与参考不同时显示
  3. 导入 VCF 文件
    • 标记变异位置
    • 显示基因型和注释
  4. 肉眼检查技巧
    • 覆盖度均匀 → 无明显偏差
    • 错配位置一致 → 可能真实变异
    • 错配散布不规则 → 可能测序或比对错误
    • 配对 reads 距离正确 → 有效比对

14.5 实践：VCF 注释与致病性评估

14.5.1 Python 整合注释和过滤

import vcf
import math

class VariantPrioritizer:
    def __init__(self, annotated_vcf):
        """Initialize with annotated VCF (from SnpEff/VEP)"""
        self.vcf = vcf.Reader(filename=annotated_vcf)
        self.variants = []

    def parse_snpeff_annotation(self, ann_field):
        """Parse SnpEff ANN field"""
        if not ann_field:
            return None

        # SnpEff ANN format
        annotations = []
        for ann in ann_field:
            parts = ann.split('|')
            if len(parts) >= 15:
                annotations.append({
                    'alt': parts[0],
                    'effect': parts[1],
                    'impact': parts[2],
                    'gene_name': parts[3],
                    'gene_id': parts[4],
                    'feature_type': parts[5],
                    'hgvs_c': parts[9],
                    'hgvs_p': parts[10]
                })
        return annotations

    def classify_impact(self, impact_str):
        """Convert IMPACT string to numeric score"""
        impact_scores = {
            'HIGH': 4,
            'MODERATE': 3,
            'LOW': 2,
            'MODIFIER': 1
        }
        return impact_scores.get(impact_str, 0)

    def get_population_frequency(self, info_dict):
        """Extract population frequency (AF from gnomAD or other sources)"""
        for key in ['AF', 'gnomAD_AF']:
            if key in info_dict:
                af = info_dict[key]
                if isinstance(af, list):
                    af = af[0]
                return float(af) if af else None
        return None

    def prioritize_variants(self):
        """Prioritize variants by pathogenicity"""
        results = []

        for record in self.vcf:
            variant_info = {
                'chrom': record.CHROM,
                'pos': record.POS,
                'ref': record.REF,
                'alt': record.ALT[0] if record.ALT else None,
                'qual': record.QUAL
            }

            # Parse annotations
            if 'ANN' in record.INFO:
                anns = self.parse_snpeff_annotation(record.INFO['ANN'])
                if anns:
                    ann = anns[0]  # Use most severe
                    variant_info['effect'] = ann['effect']
                    variant_info['impact'] = ann['impact']
                    variant_info['gene'] = ann['gene_name']
                    variant_info['hgvs_p'] = ann['hgvs_p']
                    variant_info['impact_score'] = self.classify_impact(ann['impact'])

            # Get population frequency
            variant_info['af'] = self.get_population_frequency(record.INFO)

            # Calculate priority score
            variant_info['priority_score'] = self._calculate_priority(variant_info)

            results.append(variant_info)

        # Sort by priority score (descending)
        results.sort(key=lambda x: x.get('priority_score', 0), reverse=True)
        return results

    def _calculate_priority(self, variant):
        """Calculate variant priority score (0-100)"""
        score = 0

        # Impact score (max 60)
        impact_score = variant.get('impact_score', 0)
        score += impact_score * 15

        # Quality score (max 20)
        qual = variant.get('qual', 0)
        if qual:
            score += min(20, qual / 5)

        # Population frequency (max 20, rare variants score higher)
        af = variant.get('af')
        if af:
            if af < 0.01:
                score += 20  # Very rare
            elif af < 0.05:
                score += 10  # Rare
            else:
                score += 0   # Common
        else:
            score += 15  # Unknown frequency

        return min(100, score)

    def report_top_variants(self, top_n=10):
        """Print top N prioritized variants"""
        variants = self.prioritize_variants()

        print("\n" + "="*80)
        print("变异优先级排序（致病性从高到低）")
        print("="*80)

        for i, var in enumerate(variants[:top_n], 1):
            print(f"\n{i}. {var['chrom']}:{var['pos']} {var['ref']}->{var['alt']}")
            print(f"   基因: {var.get('gene', 'Unknown')}")
            print(f"   效应: {var.get('effect', 'Unknown')}")
            print(f"   影响: {var.get('impact', 'Unknown')}")
            print(f"   蛋白变化: {var.get('hgvs_p', 'Unknown')}")
            print(f"   群体频率: {var.get('af', 'Unknown')}")
            print(f"   质量评分: {var.get('qual', 'N/A')}")
            print(f"   优先级分数: {var.get('priority_score', 0):.1f}/100")

# 使用示例
if __name__ == "__main__":
    prioritizer = VariantPrioritizer("annotated.vcf")
    prioritizer.report_top_variants(top_n=10)

15 从个体变异到群体研究：迈向 GWAS

经过前四节专题的完整训练，读者已掌握从原始测序数据到高质量注释变异集的全套流程。本节作为本章的收尾，从个体变异检测视角转向群体遗传学视角，为下一章 GWAS（全基因组关联分析）的学习奠定概念基础。

15.1 等位基因频率谱：从样本到群体

当我们将同一套流程应用于数百乃至数千个个体时，最终产生的不再是单个 VCF，而是一个记录全体样本在数百万变异位点上基因型的大型矩阵。对这个矩阵进行统计汇总，得到每个位点的等位基因频率，形成等位基因频率谱（Allele Frequency Spectrum，AFS）。

{{Wiki:Allele_frequency_spectrum}}

AFS 的形状蕴含丰富的进化信息：大量极低频变异（稀有突变）和少量高频变异（经选择或漂变固定）共同构成典型的 L 形分布。人群规模扩张、瓶颈效应、自然选择都会改变 AFS 的形状，使其成为推断人群历史的重要工具。

# R: 绘制等位基因频率谱（模拟数据）

15.2 群体分层：GWAS 的最大混杂因素

不同地理来源的个体在基因组上存在系统性频率差异（群体分层，Population Stratification）。若一项 GWAS 研究中病例组和对照组的祖先来源不均衡，则祖先来源本身就会成为混杂因素，产生大量假阳性关联信号——这与目标疾病毫无关联。

{{Wiki:Population_stratification}}

检测和校正群体分层的标准方法是主成分分析（PCA）：对全基因组 SNP 数据进行 PCA，前若干个主成分会捕获样本的祖先来源信息，在 GWAS 的线性/逻辑回归模型中将这些主成分作为协变量纳入，即可在统计上校正祖先背景的影响。

# R: 使用 SNP 矩阵进行简化 PCA 演示（实际中用 PLINK/flashPCA）
# 假设 geno_matrix: 行=个体，列=SNP，值=0/1/2（等位基因计数）

library(ggplot2)

15.3 连锁不平衡：基因组的”区块结构”

基因组上相邻的 SNP 并非彼此独立——由于染色体重组频率有限，邻近位点的等位基因倾向于以固定组合（单倍型，Haplotype）共同遗传。这种非随机关联被称为连锁不平衡（Linkage Disequilibrium，LD）。

{{Wiki:Linkage_disequilibrium}}

LD 通常用相关系数 \(r^2\)（或 D’ 统计量）衡量：

\[r^2 = \frac{(p_{AB} - p_A p_B)^2}{p_A(1-p_A) \cdot p_B(1-p_B)}\]

\(r^2 = 1\) 表示两个 SNP 完全连锁（知道一个就能推断另一个）；\(r^2 = 0\) 表示完全独立。

LD 对 GWAS 有两个关键含义：

  1. 标签 SNP（Tagging SNP）：因为 LD 的存在，基因组中并不需要对每一个 SNP 都进行基因分型。只要基因分型了区块内的”标签 SNP”，就能通过 LD 关系插补（imputation）出区块内其他 SNP 的基因型，大幅降低成本。HapMap 和 1000 Genomes 项目正是基于这一思路建立的基因型参考面板。

  2. 关联信号的精细定位（Fine-mapping）：GWAS 发现的”命中位点”往往是该 LD 区块内 \(r^2\) 最高的标签 SNP，而真正的因果变异（causal variant）可能是与之高度连锁的另一个位点。从 GWAS 命中到因果变异的定位，是后 GWAS 时代的核心挑战之一。

LD 区块结构示意：

染色体片段：  |═══════════════════════════════════|
LD 区块：     [Block A ████]  [Block B ████████]  [Block C ██]
标签 SNP：        ↑                  ↑                ↑
因果变异可能：    ← 在 Block A 任意位置，r² ≈ 1 ─→
插补覆盖：        所有未基因分型的 SNP 可通过参考面板推断
# R: 计算两个 SNP 之间的 r²（连锁不平衡系数）
# 假设已知 200 个个体在两个位点的等位基因计数（0/1/2 编码）
compute_r2 <- function(snp_a, snp_b) {
  # 中心化等位基因计数
  a <- snp_a - mean(snp_a)
  b <- snp_b - mean(snp_b)
  r2 <- (sum(a * b) / length(a))^2 / (var(snp_a) * var(snp_b))
  cat(sprintf("r² = %.4f  (%s)\n", r2,
              ifelse(r2 > 0.8, "Strong LD", ifelse(r2 > 0.3, "Moderate LD", "Weak LD"))))
  return(r2)
}

set.seed(1)
n <- 500
freq_a <- 0.4
# 模拟强连锁的两个 SNP（r² ≈ 0.9）
hap_freq <- c(AA = 0.35, Aa = 0.07, aA = 0.05, aa = 0.53)
haplotypes <- sample(names(hap_freq), n, replace = TRUE, prob = hap_freq)
snp1 <- ifelse(substr(haplotypes, 1, 1) == "A", 2, 0)
snp2 <- ifelse(substr(haplotypes, 2, 2) == "A", 2, 0)
compute_r2(snp1, snp2)

15.4 群体分化：Fst 统计量

当两个亚群在漫长演化或地理隔离中积累了不同的等位基因频率，可以用固定指数（Fixation Index，Fst）量化它们之间的分化程度。Fst 的直觉含义是”亚群内部的遗传多样性比总群体少多少”：

\[F_{ST} = \frac{H_T - H_S}{H_T}\]

其中 \(H_T\) 是合并群体的杂合度，\(H_S\) 是各亚群杂合度的加权平均。Fst = 0 表示两群体完全一致，Fst = 1 表示完全固定为不同等位基因。

{{Wiki:Fixation_index}}

Fst 范围 群体分化程度 典型例子
0 – 0.05 微弱分化 同一大陆内毗邻人群
0.05 – 0.15 中等分化 欧洲 vs. 东亚人群（全基因组平均 Fst ≈ 0.12）
0.15 – 0.25 显著分化 非洲 vs. 非非洲人群（Fst ≈ 0.16）
> 0.25 高度分化 人类 vs. 黑猩猩在特定区域

Fst 的选择信号应用

在中性进化下，绝大多数位点的 Fst 接近全基因组均值。若某个位点的 Fst 显著高于均值（分布尾端），则提示该位点在不同群体中经历了方向性自然选择（positive selection），可能具有适应性功能——这是群体遗传学中检测选择压力的经典方法（Fst outlier analysis）。

15.5 遗传漂变与有效群体大小

与自然选择相对的另一个驱动等位基因频率变化的力量是遗传漂变（Genetic Drift）：在有限大小的种群中，随机采样误差会导致等位基因频率的随机波动，极端情况下某些等位基因会完全消失（丢失）或固定（fixation）。

{{Wiki:Genetic_drift}}

遗传漂变的强度与有效群体大小（Effective Population Size，\(N_e\)）成反比：\(N_e\) 越小，漂变越强，等位基因频率变化越剧烈。\(N_e\) 并非实际个体数，而是一个统计概念，反映了”在遗传上等效于多大的理想随机交配群体”。

现代人类的全球平均 \(N_e\) 约为 10,000–20,000，远小于实际人口数量（约 80 亿），这一历史瓶颈在 AFS 中留下了明显印记——非非洲人群经历出非洲迁徙时的强烈瓶颈，AFS 中低频变异比例明显低于留在非洲的群体。

# R: 模拟遗传漂变对等位基因频率的影响
simulate_drift <- function(N_e, n_gen, init_freq = 0.3, n_replicates = 5) {
  freqs <- matrix(NA, nrow = n_gen + 1, ncol = n_replicates)
  freqs[1, ] <- init_freq

  for (rep in seq_len(n_replicates)) {
    freq <- init_freq
    for (gen in seq_len(n_gen)) {
      freq <- rbinom(1, 2 * N_e, freq) / (2 * N_e)
      freqs[gen + 1, rep] <- freq
      if (freq == 0 || freq == 1) {
        freqs[(gen + 2):(n_gen + 1), rep] <- freq
        break
      }
    }
  }
  return(freqs)
}

par(mfrow = c(1, 2))

# 小群体（Ne=50）：强漂变，频率剧烈波动
f_small <- simulate_drift(N_e = 50, n_gen = 200)
matplot(f_small, type = "l", lty = 1, col = rainbow(5),
        ylim = c(0, 1), xlab = "Generation", ylab = "Allele Frequency",
        main = "Genetic Drift  (Ne = 50)")
abline(h = c(0, 1), lty = 2, col = "gray")

# 大群体（Ne=5000）：弱漂变，频率相对稳定
f_large <- simulate_drift(N_e = 5000, n_gen = 200)
matplot(f_large, type = "l", lty = 1, col = rainbow(5),
        ylim = c(0, 1), xlab = "Generation", ylab = "Allele Frequency",
        main = "Genetic Drift  (Ne = 5000)")

15.6 基因型插补：从芯片到全基因组

全基因组关联研究的一个实践难题是覆盖度与成本的权衡：直接对所有人进行全基因组测序成本极高，而 SNP 芯片虽然经济，但仅覆盖约 50 万个预选位点，遗漏了基因组中绝大多数变异。

基因型插补（Genotype Imputation）利用 LD 区块结构弥补这一缺口。其流程如下：

  1. 参考面板构建：使用已完成全基因组深度测序的大型参考人群（如 1000 Genomes Project、HRC 参考面板、gnomAD），建立高密度单倍型库
  2. 单倍型相位推断（Phasing）：对芯片基因型数据推断每个染色体上的等位基因排列顺序（SHAPEIT、EAGLE 等工具）
  3. 插补：将样本的已知单倍型与参考面板比对，利用隐马尔可夫模型在 LD 区块内”填入”未测定 SNP 的最可能基因型（IMPUTE2、Minimac4、Beagle 等工具）
插补流程示意：

参考面板（已测序）：
  SNP1  SNP2  SNP3  SNP4  SNP5  SNP6  SNP7  SNP8
   A     C     G     T     A     A     C     G     ← Haplotype ref1
   G     C     A     T     G     A     C     G     ← Haplotype ref2
   A     T     G     T     A     G     C     T     ← Haplotype ref3

芯片样本（仅检测标签 SNP）：
   A     -     -     T     -     -     C     -     ← 待插补样本

插补结果（推断填充）：
   A    [C]   [G]    T    [A]   [A]    C    [G]    ← 插补后完整基因型
        r²>0.9  r²>0.9      r²>0.8  r²>0.9

插补质量用 \(r^2\) info score 衡量：info ≥ 0.8 表示插补可信度高，通常只保留 info ≥ 0.3 的位点进行 GWAS 分析。

  1. 全球 2% 的频率属于常见多态性（common variant），ACMG 指南的 BS1 标准（频率 > 预期患病率）本应降低致病性评级；但东亚人群中仅 0.05% 的频率提示该变异在高风险群体中仍属罕见，与疾病群体特异性一致。因此不能仅以全球频率否定致病性，应分层查看各祖先人群的频率，结合 ClinVar 的具体临床表型描述、分离分析（家族研究）和功能实验证据综合判断。ACMG 框架要求多证据集成，单一指标不可决定最终分级。

  2. BQSR 的核心假设是：已知变异位点处观测到的碱基差异是真实变异，而非测序错误；未列入数据库的差异则作为潜在错误用于训练模型。没有数据库时，可采用以下策略：(1) 先用较宽松的参数进行初步变异检测，获得”高置信变异集”作为临时替代数据库（bootstrapping approach）；(2) 对低覆盖区域不进行 BQSR，直接使用原始质量分；(3) 使用整体均值校正代替位点特异性校正。上述方法均有局限，非模式生物分析中 BQSR 的收益通常较人类基因组小。

  3. (a) 对应 INFO 字段中 FS（Fisher Strand bias）值异常偏高（通常 > 60 时过滤），以及 SOR（Strand Odds Ratio）偏高。(b) 技术成因：(i) 边合成边测序（SBS）化学反应在特定模板链上产生系统性错误，尤其在 GC 富集或发夹结构区域；(ii) 文库制备时的链特异性偏好；(iii) PCR 扩增偏向某一链。(c) 应当过滤——单链支持的变异是典型假阳性特征，真实变异应在两条链上均匀分布，Fisher Strand 检验正是为此设计的质量控制指标。

A 等位基因频率：p = (2×720 + 50) / (2×1000) = 1490/2000 = 0.745；q = 0.255。期望基因型：AA = 1000×0.745² = 555.0，Aa = 1000×2×0.745×0.255 = 380.0，aa = 1000×0.255² = 65.0。卡方统计量 = (720-555)²/555 + (50-380)²/380 + (230-65)²/65 ≈ 49.0 + 286.1 + 418.8 = 753.9，p 值远小于 10⁻⁶，未通过 HWE 检验。观察模式为杂合子严重缺失（50 << 380），最可能的技术原因是等位基因丢失（allele dropout）或基因分型错误（如芯片探针设计不良，杂合信号被判读为纯合）。

  5. 连锁不平衡意味着基因组被划分为若干 LD 区块，区块内相邻 SNP 的等位基因高度相关（r² ≈ 1）。基因型插补的原理：利用高密度参考面板（如 1000 Genomes、HRC，包含数千万 SNP）与芯片检测到的约 50 万个标签 SNP，通过隐马尔可夫模型在局部单倍型结构中”填入”未直接测定的 SNP。由于人类基因组的常见变异约有 95% 集中在约 5 万个 LD 区块中，且每个区块可由少量标签 SNP 有效代表，50 万个标签 SNP 即可通过插补扩展覆盖约 1000 万个常见变异。

  6. (a) 病例-对照之间存在系统性祖先差异（群体分层），导致本应中性的基因组区域也出现频率差异，在 GWAS 关联分析中产生大量假阳性信号，与目标疾病毫无关联。(b) 对全基因组 SNP 数据做 PCA：若前 2 个主成分的散点图中病例（东亚）和对照（欧洲）形成两个明显分离的云团，即确认存在群体分层。同时可绘制 Q-Q 图，若基因组膨胀因子（λ）远大于 1（如 λ > 1.1），也提示系统性混杂。(c) 校正方法：在逻辑回归关联模型中将前 10 个 PC 作为协变量纳入（logistic(phenotype ~ SNP + PC1 + ... + PC10)），可在统计上消除祖先背景的混杂效应；理想情况下应在研究设计阶段确保病例对照具有相同的祖先组成。

  7. 合并群体等位基因频率：\(\bar{p}\) = (500×0.8 + 500×0.3) / 1000 = 0.55。总群体杂合度：\(H_T = 2 \times 0.55 \times 0.45 = 0.495\)。各亚群杂合度：\(H_1 = 2×0.8×0.2 = 0.32\)，\(H_2 = 2×0.3×0.7 = 0.42\)，\(H_S = (0.32+0.42)/2 = 0.37\)。Fst = (0.495 - 0.37) / 0.495 ≈ 0.253，属于高度分化。含义：若将两个亚群纳入同一 GWAS 研究而不校正群体结构，该位点将产生强烈的假阳性关联信号，必须通过 PCA 协变量或分层分析加以校正。

  8. 岛屿种群 AFS 特征：(1) 低频变异（MAF < 5%）比例明显低于大陆种群，因为瓶颈效应使许多低频等位基因在创始者群体中丢失；(2) 某些中高频变异的频率因随机漂变被抬高；(3) 整体多样性（杂合度 π）低于大陆种群；(4) LD 区块更长，因为历史上的低 \(N_e\) 减少了重组的机会。漂变影响：尽管现在有 5000 人，历史瓶颈（Ne≈20）留下的遗传印记持续存在，瓶颈期的强烈漂变对现代基因组多样性格局影响不可忽略。有效群体大小 \(N_e\) 反映历史平均值，而非当前个体数，因此该种群的 \(N_e\) 仍显著低于稳定的大陆种群。
