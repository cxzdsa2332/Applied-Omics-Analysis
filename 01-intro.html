2  生物信息学入门课程总览

Published

April 12, 2026

2.1 课程宗旨

本课程专为零计算背景的生物学学生设计，旨在搭建连接”湿实验”与”干计算”的完整桥梁。课程从分子生物学底层逻辑出发，逐步深入高通量数据的产生、处理和分析，系统地引入 Linux 命令行、R/Python 编程、Git 版本控制和 AI 辅助开发，最终掌握现代生物信息学的完整工作流。

学习目标：掌握从生物样本到数据分析的全链路技能，具备独立进行转录组、基因组等组学分析的能力。

课程设计哲学：问题驱动、递进式学习，每一课都以”为什么需要这个”开启，在实际问题中引入新工具。

2.2 📚 五部曲课程架构

Lesson 1: WHY?        → Lesson 2: DATA?      → Lesson 3: TOOLS?
(理论基础)              (技术原理)             (工程工具)
    ↓                     ↓                      ↓
分子生物学原理      高通量测序原理         Linux+命令行+项目管理
中心法则+组学思维   从样本到FASTQ         数据处理自动化
                                              ↓
              ← Lesson 5: FORMAT?  ← Lesson 4: CODE?
              (数据标准)             (编程与工程)
                ↓                       ↓
         数据库与文件格式      R/Python+Git+VS Code+AI辅助
         国际数据规范          全栈工作流

2.3 📖 Lesson 1: 分子生物学基础与高通量概念启蒙

  • 核心目标：统一学科语言，理解细胞内大分子的物理化学本质，以及”高通量”究竟在测量什么。
  • 主要内容：
    • 📌 分子生物学中心法则（Central Dogma）：DNA 复制、转录与翻译的信息流向
    • 📌 基因表达的多层次调控：转录、翻译后修饰、表观遗传等
    • 📌 从”低通量”到”高通量”的范式转变：传统凝胶电泳 vs 现代阵列/微流控技术
    • 📌 什么是”组学”（Omics）？系统性测量如何重塑生物学认知
    • 📌 多组学整合的力量：为什么基因序列无法直接预测表型
  • 学习成果：
    • ✅ 理解生命的核心规律：信息流、调控机制、系统性观点
    • ✅ 认识”高通量”的本质与挑战
    • ✅ 为什么需要生物信息学

2.4 📖 Lesson 2: 高通量测序原理与生物数据生命周期

  • 核心目标：了解测序仪器的底层逻辑，掌握数据从生物样本到数字矩阵的转化过程。
  • 主要内容：
    • 📌 从生物样本到纯净 DNA：样本制备、DNA 提取、电泳分离
    • 📌 三代测序技术原理：Sanger（一代）→ Illumina（二代）→ PacBio/Nanopore（三代）
    • 📌 Phred 质量分数与 FASTQ 格式：测序错误的量化与存储
    • 📌 数据生命周期全景：样本→测序→原始数据→质控→处理→分析→发表
  • 代码示例：
    • Python 脚本解码 Phred 质量分数（5 分钟动手体验）
  • 学习成果：
    • ✅ 理解测序数据的由来与质量评估
    • ✅ 认识 FASTQ 格式的含义
    • ✅ 关键问题：有了 FASTQ 数据，我们该怎么处理？→ 导向 Lesson 3

2.5 📖 Lesson 3: 拥抱命令行——Linux 操作系统与工程工具

  • 核心目标：掌握在 HPC 服务器上高效处理大规模数据的工程能力，建立规范的项目管理习惯。
  • 主要内容：
    • 📌 为什么生物信息学离不开 Linux？（算力、稳定性、开源生态）
    • 📌 Linux 文件系统、权限管理、基本命令（ls, cd, mkdir 等）
    • 📌 文本处理工具：grep, cut, sort, awk, sed（生信数据操作的瑞士军刀）
    • 📌 管道与重定向：Unix 哲学的精髓（| 与 > >> 的组合威力）
    • 📌 远程连接：SSH 密钥认证与文件传输（rsync）
    • 📌 软件环理：Conda/Mamba 与虚拟环境隔离
    • 📌 高性能计算：Slurm 作业调度系统
    • 📌 项目管理初步：标准目录结构、环境配置文件、日志管理
  • 实战内容（占比 60%）：
    • 创建项目目录结构并配置 .gitignore
    • 编写 environment.yml 并创建虚拟环境
    • 用 grep/awk 处理模拟 FASTQ 数据
    • 在 Slurm 集群上提交批处理任务
  • 学习成果：
    • ✅ 能在任何 Linux/HPC 环境中独立工作
    • ✅ 能用 bash 脚本自动化数据处理
    • ✅ 关键问题：数据处理好了，但分析需要编程，怎么办？→ 导向 Lesson 4

2.6 📖 Lesson 4: 生物信息学全栈工作流——R/Python + Git + VS Code + AI 辅助

  • 核心目标：构建现代化、可复现、团队协作的开发环境，掌握 AI 辅助编程和文档撰写。
  • 主要内容：
    • 📌 Conda 深入：依赖隔离、虚拟环境管理、环境导出与复现
    • 📌 编程环境：VS Code 配置、Python 与 R 的集成、Jupyter Notebook 最佳实践
    • 📌 全栈工作流（新增重点）：
      • 云端 GitHub 库（远程）↔︎ 本地克隆（开发）
      • VS Code 编辑代码 + 内置 Git 管理
      • Conda 环境激活与 Python/R 语言服务器
      • 提交、推送、团队协作
    • 📌 AI 辅助开发（新增）：
      • 用 Claude/GitHub Copilot 加速代码编写
      • AI 辅助撰写 README 和论文 Methods
      • 提示词工程示例
    • 📌 R vs Python 选择：何时用 R（统计、出图）、何时用 Python（机器学习、流水线）
  • 实战内容（占比 50%）：
    • Conda 创建隔离的 R+Python 混合环境
    • VS Code 中配置 Git、Python、R 扩展
    • 完整的 Git 工作流：clone → edit → commit → push
    • 编写代码块，让 AI 扩展和优化
    • 用 AI 从参数脚本生成方法学文字
  • 学习成果：
    • ✅ 能用 R 进行统计分析（DESeq2、ggplot2）
    • ✅ 能用 Python 进行数据处理和机器学习
    • ✅ 能在 GitHub 上协作和发布分析代码
    • ✅ 能利用 AI 加速编码和文档写作
    • ✅ 关键问题：代码和工具准备好了，但数据从何而来？→ 导向 Lesson 5

2.7 📖 Lesson 5: 探索生物大数据——数据库、文件格式与生态整合

  • 核心目标：了解全球生物数据的流通体系，掌握标准化文件格式，能优雅地集成公共数据进行分析。

  • 主要内容：

    • 📌 全球三大数据库：NCBI、EBI、DDBJ 的分工与 INSDC 同步机制
    • 📌 核心文件格式详解：
      • FASTA / FASTQ：序列与质量
      • SAM / BAM：比对结果
      • BED / GFF / GTF：基因注释
      • VCF：变异信息
      • TSV / CSV：定量数据
    • 📌 格式陷阱：0-based vs 1-based、链方向编码、坐标系差异
    • 📌 工具生态：Biopython（Python）、Bioconductor（R）生态简介

  • 实战内容（占比 30%）：

    • 从 NCBI/EBI 下载示例数据
    • 用命令行查看和解析各种文件格式
    • 用 Python/R 读取标准格式文件

  • 测验/作业：FASTA、FASTQ 格式识别题

  • 学习成果：

    • ✅ 理解生物数据的国际标准
    • ✅ 能快速诊断未知文件格式
    • ✅ 能用 Python/R 优雅地处理标准格式

2.8 💡 课程特色与学习建议

2.8.1 为什么这个顺序？

✅ 问题驱动：每一课都回答”为什么需要这个” ✅ 递进式：从理论→技术→工具→编程→数据 ✅ 学以致用：大量代码和实操，不是纯讲座 ✅ 现代化：融合 Git、VS Code、AI 辅助等最新工作流

2.8.2 学习建议

  1. 不跳章：虽然可以独立学 Lesson 3，但强烈建议按顺序学
  2. 边学边做：代码块前加注释”你现在可以尝试运行这段代码”
  3. 遇到问题：先搜索、查文档，再找老师或 ChatGPT
  4. 定期回顾：每个 Lesson 之后，看一遍”本章小结”，强化知识链条

2.8.3 预先准备

  • 电脑（Windows / Mac / Linux）
  • 网络连接（下载软件和数据）
  • 一个文本编辑器（VS Code 推荐）
  • 耐心与好奇心 🎓
